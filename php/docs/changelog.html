Every release, every change — documented in one place.

This log covers all notable changes to Outpost CMS. The format follows Keep a Changelog.

4.11.2 — 2026-03-24

Fixed

  • Compass auto-populate broken — Dropdown and checkbox filters were sending the raw data-source value (e.g. folder:categories) to the API instead of the stripped facet name (categories). Fixed in compass-client.js — auto-populate now uses the correct facet parameter.

4.11.1 — 2026-03-24

Added

  • Environment indicator — Top bar now shows a color-coded pill auto-detected from hostname: Local (blue) for localhost, Staging (yellow) for staging/dev subdomains, Production (green) for live sites. Zero configuration required.

4.11.0 — 2026-03-24

Added

  • RSS Feed Generator — Auto-generated RSS 2.0 feeds for collections. Site-wide feed at /feed.xml (also /feed, /rss, /rss.xml) and per-collection feeds at /{collection}/feed. Per-collection feed_enabled toggle in collection settings (default: off). Auto-discovery <link rel="alternate"> tags injected in <head> via cms_seo() / <outpost-seo>. Handles excerpt generation, XML escaping, and missing fields gracefully.
  • Site Search — Full-text site-wide search across all pages and collection items. New <outpost-search> template tag compiles to a search widget with debounced input, AJAX results, and URL state sync (?q= parameter). Public API at search/site with rate limiting (30/min/IP). Search index with incremental updates on publish. Results ranked by title relevance with highlighted excerpts. Search queries logged to analytics. Vanilla JS client with minimal CSS.
  • Sitemap Enhancement — Per-collection sitemap_enabled toggle to exclude collections from sitemap.xml. Added <changefreq> and <priority> hints (homepage: daily/1.0, pages: weekly/0.8, items: monthly/0.6). Auto-generated /robots.txt with sitemap reference and admin path blocking. Draft and member-only/paid-only pages properly excluded.

Security

  • Search excerpt XSS prevention — HTML-escapes excerpt text before injecting <mark> highlight tags
  • Feed open redirect protection — Collection slug sanitized to alphanumeric/hyphens only

4.10.0 — 2026-03-23

Added

  • URL Redirects — Full redirect management from Tools → Redirects. Three match types: exact path, wildcard (/old/* → /new/*), and regex (~^/post/(\d+)$ with capture group substitution). Supports 301, 302, and 307 status codes. CSV bulk import, URL test tool, hit counter with last-hit timestamps, and per-redirect enable/disable toggle.
  • Open redirect prevention — Redirect targets are validated at execution time. Only relative paths and same-domain absolute URLs are allowed; external domains and protocol-relative URLs are blocked.

Changed

  • Sidebar reorganization — Admin sidebar restructured into accordion groups: Content, Site, Members, Build, and Tools. Redirects, Shield, Boost, Review Links, Releases, and Workflows appear under Tools. Groups are collapsible with state persisted to localStorage.

4.9.0 — 2026-03-23

Added

  • Boost Performance Suite — Comprehensive performance optimization with admin settings panel at Settings → Boost. Page caching with configurable TTL and path exclusions, browser cache headers (Cache-Control, ETag, Expires, 304 Not Modified) for static assets, GZIP compression, HTML minification, automatic lazy loading for images and iframes, database optimization, cache preloading, and a real-time performance dashboard. Developer Mode toggle disables all caching and optimization with one click.
  • Shield Security Suite — Comprehensive security hardening at Settings → Shield. Login lockout protection (configurable attempts and duration with auto-IP-blocking), WAF-lite firewall (blocks SQL injection, XSS, path traversal, PHP injection, null byte attacks in block or log-only mode), IP blocklist (manual and automatic with expiring bans), file integrity monitoring (MD5 hash verification of core PHP files), security headers, live traffic logging (last 1,000 requests), security event log with threat indicators, and optional email notifications.

4.8.4 — 2026-03-23

Changed

  • Compass v2 — Data-attribute architecture — Rebuilt Compass to use data-compass attributes on native HTML elements instead of wrapper <outpost-compass> elements. Developers write their own HTML and add data attributes. No generated wrapper divs, no forced classes, no locked-in layout. The <outpost-compass> tag syntax still works for backward compatibility.
  • Compass client JS rewrite — New controller-per-collection architecture. Elements discovered by [data-compass] and grouped by data-collection. Supports instant filtering and submit-button mode. Auto-populates empty dropdowns, checkboxes, and radio containers from the API.
  • Compass CSS minimal — Reduced from 362 lines to ~95 lines. Only styles auto-generated content (checkbox labels, pager buttons, A–Z buttons, selection pills, loading state). Theme-owned elements are never styled.

4.6.0 — 2026-03-20

Added

  • Lodge — Member-Owned Content Portal — Members can create, edit, and delete their own collection items from the front end. Enable Lodge on any collection via Admin → Collections → Lodge tab. Includes a full API (8 endpoints), template tags (<outpost-lodge-dashboard>, <outpost-lodge-items>, <outpost-lodge-form>), configurable routing with custom slugs, starter templates, and an approval workflow with admin review queue.
  • Lodge API — 8 REST endpoints under member-api.php?action=lodge/*: dashboard, list items, create item, update item, delete item, get profile, update profile, and file upload. All requests are ownership-scoped — members can only access their own items.
  • Lodge config options — Per-collection Lodge settings: allow_create, allow_edit, allow_delete, require_approval, max_items_per_member, editable_fields (whitelist), readonly_fields, and required_tiers for tier-gated access.
  • Lodge template tags — Three new custom elements for theme templates: <outpost-lodge-dashboard> (member overview with item counts and recent activity), <outpost-lodge-items> (filterable list of the member’s items), and <outpost-lodge-form> (create/edit form with field whitelisting).
  • Lodge starter templates — Ready-to-use HTML templates for dashboard, item list, and item form pages, installable from the Lodge setup UI.
  • Approval workflow — When require_approval is enabled, new items and edits are held in a pending status. Admins review and approve/reject from the admin panel. Webhook events fire on submission and approval.
  • Tier gating — Lodge access can be restricted to specific member tiers (free, paid, or custom tiers). Members without the required tier see a gated message.
  • Feature Toggles — 14 sidebar features can be individually toggled on/off via Admin → Settings → Features. Disabled features are hidden from the sidebar and their API endpoints return 403.
  • Sidebar Accordion Groups — The admin sidebar is now organized into 5 collapsible groups: Content, Design, Connect, Members, and System. Group state persists across sessions.
  • Member tier column — New tier column on the members table for granular access control beyond free/paid roles.
  • Member meta column — New meta JSON column on the members table for storing arbitrary member metadata (bio, social links, preferences).
  • Item ownership — New owner_member_id column on collection items linking items to the member who created them.

Changed

  • Sidebar reorganization — All sidebar items grouped into accordion sections with persistent expand/collapse state.

Security

  • Rate limiting — Lodge API endpoints are rate-limited per member session to prevent abuse.
  • Atomic transactions — Lodge create/update/delete operations use database transactions to prevent partial writes.
  • MIME validation — Lodge file uploads validate MIME type against an allowlist before accepting.
  • Slug sanitization — Member-submitted slugs are sanitized to prevent path traversal and injection.
  • XSS protection — All member-submitted content is sanitized before storage and escaped on output.
  • Field whitelisting — Only fields listed in editable_fields can be written via the Lodge API. Unlisted fields are silently dropped.

4.5.0 — 2026-03-20

Added

  • Smart Forge AI — When an AI API key is configured (Claude, OpenAI, or Gemini via Ranger settings), Smart Forge uses AI instead of the PHP scanner for dramatically better HTML annotation. AI understands semantic context, names fields intelligently (e.g. hero_heading instead of heading_1), and detects section boundaries more accurately.
  • API: forge/ai-scan — POST endpoint that sends raw HTML to the configured AI provider with a detailed annotation prompt, returns fully annotated HTML with data-outpost attributes and section comments.
  • API: forge/ai-status — GET endpoint that returns whether AI is available and which provider is configured.
  • Automatic fallback — If no AI key is configured, Smart Forge falls back to the existing PHP scanner seamlessly. The button label changes between "Smart Forge AI" and "Smart Forge" based on availability.
  • Liquid-to-v2 conversion — Smart Forge AI can convert legacy Liquid {{ }} / {% %} templates to v2 data-outpost attribute syntax in a single pass.

Changed

  • Save system overhaul — Single unified save path for all content types. Cmd+S / Ctrl+S keyboard shortcut works everywhere. beforeunload warns on unsaved changes. Eliminates duplicate save logic and race conditions.
  • Global fields save correctly — Global fields now persist reliably through the unified save path, fixing intermittent data loss when saving pages with global field edits.

Fixed

  • 25 audit fixes — Comprehensive quality pass across the entire admin SPA and PHP backend covering accessibility, edge cases, and UI consistency.
  • Block settings injection fix — Block settings CSS custom properties now inject correctly when multiple blocks define settings on the same page.
  • JWT auth + CORS — Fixed token validation edge cases and CORS preflight handling for cross-origin mobile app requests.

4.4.0 — 2026-03-20

Added

  • JWT Bearer Token Auth — Stateless HS256 JWT authentication for mobile apps and headless clients. Pure PHP implementation in jwt.php with no external libraries. Per-installation secret derived from database path.
  • Member Token API — Four new endpoints: POST member-api.php?action=token (exchange credentials for JWT), POST token/register (register + JWT), POST token/refresh (refresh expiring token), GET token/me (profile via bearer).
  • CORS Configuration — New api_cors_origins setting controls cross-origin access. Supports * (wide open), comma-separated origins, or unset (localhost only). Applied to api.php, member-api.php, and content-api.php.
  • Bearer Auth on Member API — All authenticated member-api.php endpoints now accept Authorization: Bearer <jwt> alongside session cookies. CSRF skipped for token auth.

3.0.0 — 2026-03-19

Major release. Outpost 3.0 introduces Ranger, an AI-powered assistant built into the admin panel.

Added

  • Ranger AI Assistant — AI-powered assistant in the admin sidebar with 35 tools covering every CMS operation: content management, theme building, file operations, media uploads, channels, forms, users, members, webhooks, backups, database queries, template debugging, email config, navigation, settings, and frontend actions.
  • Multi-provider AI — 3 AI providers supported: Claude (Anthropic), OpenAI (GPT-4o/GPT-4), and Google (Gemini). Configure at Settings > Integrations with your own API key.
  • Streaming responses — Real-time SSE streaming with live tool execution display. Tool calls render as compact cards with running/success/error states.
  • Conversation history — Persistent conversations with auto-generated titles, searchable history panel, rename, and delete.
  • Token usage tracking — Per-message input/output/cache token counts with cost display in USD.
  • Prompt caching (Claude) — System prompt and tool definitions use ephemeral caching. Cached rounds cost up to 90% less in input tokens.
  • Dynamic tool loading — Intent classifier sends only relevant tools per message (3–13 tools instead of all 35) to minimize token usage.
  • Screenshot paste — Paste images directly into chat for visual context (Claude provider).
  • Output style — Configurable: concise, detailed, casual, technical, or custom free-text instructions.
  • Role-scoped tools — Editors get content tools, developers get code tools, admins get everything. Enforced server-side.
  • Forge Playground reset — One-click reset button on Themes page for themes with .forge-snapshot backup directories.

Changed

  • Ranger UI: Production polish — Mobile responsive (full-width overlay below 768px with slide-up animation), smart auto-scroll, full-width stop button bar, textarea auto-grow with character count, refined typography, compact tool call cards, focus-visible outlines, and aria-live chat region.
  • Ranger: Token cost optimization — Compact system prompt (~60% smaller), conversation trimming (first 2 + last 16 messages), tool result truncation.

Fixed

  • Template engine: nested conditionals in loops — {% if %}...{% else %}...{% endif %} inside {% for %} loops no longer breaks compilation.
  • Form Builder: dark mode — Replaced all hardcoded light-mode colors with CSS custom properties across 9 form-related components.

Security

  • SSRF protection — Ranger outbound HTTP requests validate against private IP ranges and internal hostnames.
  • Path traversal prevention — File operation tools enforce theme directory boundaries.
  • Encrypted API keys — Provider API keys stored using AES-256-GCM encryption at rest.
