GraphQL API

A full GraphQL API for Outpost CMS. Read queries are public (no auth). Write mutations require a Bearer API key. Schema is auto-generated from your CMS database. No external dependencies.

Overview

The GraphQL API is a single endpoint that accepts both queries and mutations. It supports field selection, aliases, variables, fragments, and full introspection (works with GraphiQL, GraphQL Playground, and Apollo Studio).

POST https://example.com/outpost/graphql.php
Content-Type: application/json

{"query": "{ collections { slug name itemCount } }"}

Rate limiting: 120 requests per 60 seconds per IP address.

Interactive playground Open /outpost/graphql-playground.html in your browser to explore the API with GraphiQL. For mutations, add #key=op_xxx to the URL to set your API key.

Authentication

OperationAuth required?Details
Queries (read)NoAll query fields are fully public, identical to the REST Content API.
Mutations (write)YesRequires Authorization: Bearer <api_key> header. Generate API keys in Settings > API Keys.

Unauthenticated mutation attempts return:

{"errors": [{"message": "Authentication required. Provide Authorization: Bearer <api_key> header."}]}

Quick start

Fetch all collections

curl -X POST https://yoursite.com/outpost/graphql.php \
  -H "Content-Type: application/json" \
  -d '{"query": "{ collections { slug name itemCount } }"}'

Fetch items with pagination

curl -X POST https://yoursite.com/outpost/graphql.php \
  -H "Content-Type: application/json" \
  -d '{"query": "{ items(collection: \"blog\", limit: 10) { data { slug title fields } total } }"}'

Fetch a single page

curl -X POST https://yoursite.com/outpost/graphql.php \
  -H "Content-Type: application/json" \
  -d '{"query": "{ page(path: \"/\") { title fields } }"}'

Using variables

curl -X POST https://yoursite.com/outpost/graphql.php \
  -H "Content-Type: application/json" \
  -d '{
    "query": "query GetItem($col: String!, $slug: String!) { item(collection: $col, slug: $slug) { title url fields } }",
    "variables": {"col": "blog", "slug": "my-post"}
  }'

Using aliases

{
  blogs: items(collection: "blog", limit: 5) { total data { title } }
  events: items(collection: "event", limit: 5) { total data { title } }
}

Query types

collections

List all collections.

{ collections { id slug name singularName urlPattern sortField sortDirection itemsPerPage fields { name type label required } itemCount } }

collection(slug: String!)

Get a single collection by slug. Returns null if not found.

{ collection(slug: "blog") { name fields { name type } itemCount } }

items(collection: String!, limit: Int, offset: Int, orderBy: String, order: String, folder: String, label: String)

Paginated list of published items in a collection. Filter by folder + label.

{ items(collection: "blog", limit: 10, offset: 0, orderBy: "published_at", order: "DESC") {
  data { id slug title url status createdAt updatedAt publishedAt fields labels }
  total limit offset
} }
ArgumentTypeDefaultDescription
collectionString!-Collection slug (required)
limitInt10Max items (1-100)
offsetInt0Skip N items
orderByStringCollection defaultcreated_at, updated_at, published_at, slug, sort_order
orderStringCollection defaultASC or DESC
folderString-Folder slug for filtering
labelString-Label slug within the folder

item(collection: String!, slug: String!)

Get a single published item by collection + slug.

{ item(collection: "blog", slug: "my-post") { title url fields labels } }

pages

List all public pages.

{ pages { id path title metaTitle metaDescription fields } }

page(path: String!)

Get a single page by path. Use / for the homepage.

{ page(path: "/about") { title fields } }

globals

Get all global field values as a JSON object.

{ globals }

menus

List all menus (summary only).

{ menus { slug name itemCount } }

menu(slug: String!)

Get a single menu with its full item tree.

{ menu(slug: "main") { name items { label url target children { label url } } } }

media(limit: Int, offset: Int)

Paginated list of uploaded media files.

{ media(limit: 20) { data { id filename path mimeType width height altText } total } }

folders(collection: String)

List folders, optionally filtered by collection slug.

{ folders(collection: "blog") { slug name type collectionSlug labelCount } }

labels(folder: String!)

List labels in a folder.

{ labels(folder: "categories") { id slug name description parentId itemCount data } }

schema

Full site schema: collections, pages, globals, folders.

{ schema { collections { slug name fields { name type } } pages { path title } globals { name type } folders { slug name } } }

Mutation types

All mutations require Authorization: Bearer <api_key>.

createItem

Create a new collection item.

mutation {
  createItem(
    collection: "blog"
    slug: "my-new-post"
    status: "published"
    data: {title: "My New Post", body: "<p>Hello world</p>"}
  ) {
    id slug title url status publishedAt
  }
}

updateItem

Partial update — only provided fields are changed. Data is merged with existing data.

mutation {
  updateItem(id: 42, data: {title: "Updated Title"}, status: "published") {
    id slug title status updatedAt
  }
}

deleteItem

mutation { deleteItem(id: 42) }

updatePage

Update a page's meta fields and/or content fields.

mutation {
  updatePage(id: 1, fields: {hero_title: "Welcome!"}, metaTitle: "Home") {
    id path title fields
  }
}

updateGlobals

Update global field values.

mutation { updateGlobals(fields: {site_name: "My Site", footer_text: "2024"}) }

deleteMedia

mutation { deleteMedia(id: 7) }

assignLabels / removeLabels

mutation { assignLabels(itemId: 42, labelIds: [1, 2, 3]) }
mutation { removeLabels(itemId: 42, labelIds: [3]) }

createCollection

mutation {
  createCollection(
    name: "Products"
    slug: "product"
    singularName: "Product"
    urlPattern: "/product/{slug}"
    schema: {fields: [{name: "title", type: "text"}, {name: "price", type: "number"}]}
  ) {
    id slug name fields { name type }
  }
}

updateCollection

mutation {
  updateCollection(id: 5, name: "Updated Name", urlPattern: "/shop/{slug}") {
    id slug name urlPattern
  }
}

createFolder / createLabel

mutation {
  createFolder(collectionId: 1, name: "Categories", slug: "categories", type: "flat") {
    slug name type
  }
}

mutation {
  createLabel(folderId: 1, name: "Tutorials", slug: "tutorials") {
    id slug name itemCount
  }
}

JavaScript example

// Helper function
async function graphql(query, variables = {}, apiKey = null) {
  const headers = { 'Content-Type': 'application/json' };
  if (apiKey) headers['Authorization'] = `Bearer ${apiKey}`;

  const res = await fetch('/outpost/graphql.php', {
    method: 'POST',
    headers,
    body: JSON.stringify({ query, variables }),
  });

  const json = await res.json();
  if (json.errors) throw new Error(json.errors[0].message);
  return json.data;
}

// Read (no auth)
const { collections } = await graphql(`{
  collections { slug name itemCount }
}`);

// Write (API key required)
const { createItem } = await graphql(`
  mutation CreatePost($slug: String!, $data: JSON!) {
    createItem(collection: "blog", slug: $slug, status: "published", data: $data) {
      id slug url
    }
  }
`, { slug: 'hello-world', data: { title: 'Hello World', body: '<p>First post!</p>' } }, 'op_your_api_key');

Response format

Standard GraphQL response envelope:

// Success
{"data": {"collections": [...]}}

// Error
{"errors": [{"message": "Collection 'xyz' not found"}]}

Differences from the REST Content API

REST Content APIGraphQL API
Endpointapi.php?action=content/...graphql.php
MethodGET onlyPOST (GET for simple queries)
Field selectionReturns all fields alwaysOnly returns requested fields
Multiple resourcesOne endpoint per requestMultiple in a single request
Write operationsNot supportedFull CRUD via mutations
Auth for readsNoneNone
Auth for writesN/ABearer API key
IntrospectionN/AFull __schema support

Rate limiting

120 requests per 60 seconds per IP. Exceeding returns 429 Too Many Requests.

CORS

Fully open: Access-Control-Allow-Origin: *. Supports Content-Type and Authorization headers.
