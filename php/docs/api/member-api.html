Member API

A dedicated API for front-end member authentication flows — registration, login, password reset, and profile management. Uses a separate session from the Admin API. Suitable for embedding login and signup forms directly in your theme pages.

Overview

The Member API lives at member-api.php, not api.php. It manages front-end member accounts (users with the free_member or paid_member role). It is entirely separate from the Admin API — a member session does not grant any admin access, and an admin session does not work here.

POST https://example.com/outpost/member-api.php?action=login

Session and cookies

Property Value
Cookie nameoutpost_member
Session lifetime30 days
Cookie flagsHttpOnly, SameSite=Lax

CSRF tokens work the same way as the Admin API. After login, call member-api.php?action=me to get a CSRF token, then pass it as X-CSRF-Token on all POST, PUT, and DELETE requests.

Public endpoints (no auth required)

These four endpoints are available without a member session.

Register

POST member-api.php?action=register

Creates a new member account. New members are assigned the free_member role by default.

FieldRequiredDescription
usernameYesUnique username. Letters, numbers, dots, hyphens, underscores.
emailYesValid email address. Must be unique across all users.
passwordYesMinimum 8 characters.

Success response — HTTP 201 (auto-login, verification disabled):

{
  "success": true,
  "member": {
    "id": 50,
    "username": "janesmith",
    "email": "jane@example.com",
    "role": "free_member"
  }
}

Success response — HTTP 200 (verification required):

{
  "success": true,
  "requires_verification": true
}

When requires_verification is true, the member is not logged in. A verification email has been sent. The client should display a "check your email" message instead of redirecting.

Error responses:

{ "error": "Username already exists" }
{ "error": "Email already in use" }
{ "error": "Password must be at least 8 characters" }

curl example:

curl -X POST https://example.com/outpost/member-api.php?action=register \
  -H "Content-Type: application/json" \
  -d '{"username":"janesmith","email":"jane@example.com","password":"mypassword"}'

JavaScript example:

const res = await fetch('/outpost/member-api.php?action=register', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: 'janesmith',
    email: 'jane@example.com',
    password: 'mypassword',
  }),
});

const data = await res.json();
if (data.error) {
  showError(data.error);
} else {
  // Registration successful — member is now logged in
  redirect('/dashboard');
}

Login

POST member-api.php?action=login

Authenticates a member and sets the outpost_member session cookie. You can identify the member by either username or email.

FieldRequiredDescription
usernameYes*Username or email address. Use either username or email — both are accepted.
emailYes*Alternative to username.
passwordYesMember password.

Success response:

{
  "success": true,
  "member": {
    "id": 50,
    "username": "janesmith",
    "role": "free_member"
  },
  "csrf_token": "a1b2c3d4e5f6..."
}

Error response (HTTP 401):

{ "error": "Invalid username or password" }
{ "error": "Account is suspended" }
{ "error": "Please verify your email address before signing in.",
  "needs_verification": true, "email": "j***e@example.com" }

When needs_verification is true, the email address is partially masked. The client should display the error and offer a Resend verification email action via the resend-verify endpoint.

curl example:

curl -c member-cookies.txt -X POST \
  https://example.com/outpost/member-api.php?action=login \
  -H "Content-Type: application/json" \
  -d '{"username":"janesmith","password":"mypassword"}'

JavaScript example:

const res = await fetch('/outpost/member-api.php?action=login', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: 'janesmith',
    password: 'mypassword',
  }),
});

const data = await res.json();
if (data.error) {
  showError(data.error);
} else {
  // Store CSRF token for subsequent mutations
  sessionStorage.setItem('csrf', data.csrf_token);
  redirect('/members/dashboard');
}

Forgot password

POST member-api.php?action=forgot

Initiates a password reset for the given email address. A reset link is emailed to the member. Always returns {"success": true} regardless of whether the address exists, to prevent user enumeration. The reset link expires after 1 hour.

FieldRequiredDescription
emailYesThe member's email address.

Request body:

{ "email": "jane@example.com" }

Response:

{ "success": true }

The reset link sent in the email points to:

/outpost/member-pages/reset-password.php?token=<64-char-hex-token>
Note SMTP must be configured in Settings for password reset emails to be delivered. If SMTP is not configured, the token is still written to the database but no email is sent.

Reset password

POST member-api.php?action=reset

Completes a password reset using the token from the emailed link. Only tokens for free_member and paid_member accounts are accepted here — for admin accounts, use api.php?action=auth/reset.

FieldRequiredDescription
tokenYes64-character hex token from the reset email.
passwordYesNew password, minimum 8 characters.

Request body:

{
  "token": "a1b2c3...64-char-hex...",
  "password": "new-password-min-8"
}

Success response:

{ "success": true }

Error responses:

{ "error": "Reset link is invalid or has expired" }
{ "error": "Password must be at least 8 characters" }

curl example:

curl -X POST https://example.com/outpost/member-api.php?action=reset \
  -H "Content-Type: application/json" \
  -d '{"token":"a1b2c3...","password":"newpassword123"}'

Resend verification email

POST member-api.php?action=resend-verify

Resends the email verification link for an unverified member. Always returns {"success": true} regardless of whether the email exists, to prevent user enumeration. Rate-limited to one request per 60 seconds per session.

FieldRequiredDescription
emailYesThe member's email address.

Request body:

{ "email": "jane@example.com" }

Response:

{ "success": true }

The verification link sent in the email points to:

/outpost/member-pages/verify-email.php?token=<64-char-hex-token>
Note This endpoint only resends for members who have email_verified = NULL and a pending verify_token. Members who are already verified or who were created before email verification was enabled are not affected.

Authenticated endpoints

The following endpoints require an active member session (set by login or register). POST, PUT, and DELETE requests also require an X-CSRF-Token header with the token from me.

Get current member

GET member-api.php?action=me

Returns the currently logged-in member and a fresh CSRF token. Use this to check session state on page load.

Response:

{
  "member": {
    "id": 50,
    "username": "janesmith",
    "email": "jane@example.com",
    "role": "free_member",
    "display_name": "Jane Smith",
    "avatar": "/outpost/uploads/avatar-abc.jpg",
    "member_since": "2024-01-15 09:00:00"
  },
  "csrf_token": "a1b2c3d4e5f6..."
}

If no session is active, returns HTTP 401:

{ "error": "Authentication required" }

JavaScript example — check session on page load:

async function checkSession() {
  const res = await fetch('/outpost/member-api.php?action=me', {
    credentials: 'include',
  });

  if (res.status === 401) {
    // Not logged in — redirect to login page
    location.href = '/members/login';
    return;
  }

  const { member, csrf_token } = await res.json();
  sessionStorage.setItem('csrf', csrf_token);
  displayMemberInfo(member);
}

checkSession();

Logout

POST member-api.php?action=logout

Destroys the member session and clears the cookie. Requires an active session and a valid CSRF token.

Request body: empty or {}

Response:

{ "success": true }

curl example:

curl -b member-cookies.txt -X POST \
  https://example.com/outpost/member-api.php?action=logout \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: a1b2c3d4e5f6..." \
  -d '{}'

JavaScript example:

const csrf = sessionStorage.getItem('csrf');

const res = await fetch('/outpost/member-api.php?action=logout', {
  method: 'POST',
  credentials: 'include',
  headers: {
    'Content-Type': 'application/json',
    'X-CSRF-Token': csrf,
  },
  body: JSON.stringify({}),
});

sessionStorage.removeItem('csrf');
location.href = '/';  // redirect to home

Updates the member's display name or email. Pass only the fields you want to change.

Full login flow example

This example shows a complete member login form implementation with error handling and CSRF token management.

// login.js

let csrfToken = null;

async function loadCsrf() {
  // Only called if already logged in — use for CSRF bootstrapping
  const res = await fetch('/outpost/member-api.php?action=me', {
    credentials: 'include',
  });
  if (res.ok) {
    const data = await res.json();
    csrfToken = data.csrf_token;
    // Already logged in — redirect
    location.href = '/members/dashboard';
  }
}

document.getElementById('login-form').addEventListener('submit', async (e) => {
  e.preventDefault();

  const username = document.getElementById('username').value;
  const password = document.getElementById('password').value;

  const res = await fetch('/outpost/member-api.php?action=login', {
    method: 'POST',
    credentials: 'include',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ username, password }),
  });

  const data = await res.json();

  if (data.error) {
    document.getElementById('error').textContent = data.error;
    return;
  }

  // Store CSRF token from login response
  sessionStorage.setItem('csrf', data.csrf_token);

  // Redirect to dashboard
  location.href = '/members/dashboard';
});

loadCsrf();

JWT Bearer Token Authentication

For mobile apps, native clients, and headless frontends that cannot use cookies, the Member API supports JWT bearer tokens as a stateless alternative to session-based auth.

When to use tokens vs sessions Use session-based auth (login / me / cookies) for server-rendered theme pages and browser-based SPAs. Use JWT tokens (token endpoints) for mobile apps, React Native, Flutter, or any client that cannot maintain PHP session cookies.

Get a token

POST member-api.php?action=token

Exchange email/username and password for a JWT bearer token. The token is valid for 30 days.

FieldRequiredDescription
emailYesEmail address or username.
passwordYesMember password.

Success response:

{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "member": {
    "id": 50,
    "name": "Jane Smith",
    "email": "jane@example.com",
    "tier": "free"
  },
  "expires_at": "2026-04-19T12:00:00+00:00"
}

Error responses:

{ "error": "Invalid credentials" }           // HTTP 401
{ "error": "Your account has been suspended" } // HTTP 403

Rate limited: 10 requests per 60 seconds per IP address.

curl example:

curl -X POST https://example.com/outpost/member-api.php?action=token \
  -H "Content-Type: application/json" \
  -d '{"email":"jane@example.com","password":"mypassword"}'

React Native example:

const res = await fetch('https://example.com/outpost/member-api.php?action=token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email: 'jane@example.com', password: 'mypassword' }),
});

const { token, member, expires_at } = await res.json();
// Store token securely (e.g. SecureStore on iOS/Android)
await SecureStore.setItemAsync('outpost_token', token);

Register with token

POST member-api.php?action=token/register

Create a new member account and receive a JWT token in one step. If email verification is enabled, a token is not returned until the member verifies their email.

FieldRequiredDescription
nameYesUsername (also accepts username).
emailYesValid email address.
passwordYesMinimum 8 characters.

Success response (HTTP 201):

{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "member": {
    "id": 51,
    "name": "janesmith",
    "email": "jane@example.com",
    "tier": "free"
  },
  "expires_at": "2026-04-19T12:00:00+00:00"
}

Verification required response (HTTP 200):

{
  "requires_verification": true,
  "message": "Please check your email to verify your account."
}

Refresh token

POST member-api.php?action=token/refresh

Exchange a valid (non-expired) bearer token for a new token with a fresh 30-day expiry. Use this to extend sessions without requiring the user to re-enter credentials.

Requires Authorization: Bearer <token> header.

Success response:

{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...(new)...",
  "member": { ... },
  "expires_at": "2026-04-19T12:00:00+00:00"
}

Using bearer tokens with existing endpoints

All authenticated member-api endpoints (me, profile, password, logout) also accept bearer tokens as an alternative to session cookies. When using a bearer token, CSRF tokens are not required.

// Update profile via bearer token — no CSRF needed
const res = await fetch('https://example.com/outpost/member-api.php?action=profile', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`,
  },
  body: JSON.stringify({ display_name: 'Jane M. Smith' }),
});

CORS configuration

By default, the Member API allows cross-origin requests only from localhost (for development). To allow requests from mobile apps or external frontends, set the api_cors_origins setting:

ValueBehavior
(not set)Only localhost origins are allowed (development default).
*All origins are allowed. Use this for mobile apps that send requests from native code.
https://app.example.com, https://www.example.comOnly the listed origins are allowed (comma-separated).

Set via the Admin API:

PUT api.php?action=settings
{ "api_cors_origins": "*" }

Or insert directly into the SQLite database:

INSERT INTO settings (key, value) VALUES ('api_cors_origins', '*')
  ON CONFLICT(key) DO UPDATE SET value = excluded.value;

Member roles

Members can have one of two roles:

RoleDescription
free_memberRegistered member with no paid access. Can access pages with visibility = members.
paid_memberMember with paid access. Can access pages with visibility = members and visibility = paid.

Role upgrades are performed by admins via the Admin API members endpoint. Members cannot change their own role.

Lodge endpoints

Lodge endpoints provide member-owned content management. They require an active member session and are only available on collections with Lodge enabled. See Lodge documentation for full configuration details.

Lodge file upload

POST member-api.php?action=lodge/upload&item_id=123

Uploads a file and attaches it to an item owned by the current member. Uses multipart/form-data. MIME type is validated against an allowlist.

FieldRequiredDescription
fileYesThe file (multipart form field).
item_idYesQuery param — the item to attach the file to.

Success response:

{
  "success": true,
  "url": "/outpost/uploads/my-image.jpg",
  "filename": "my-image.jpg"
}

Error responses:

{ "error": "Item not found or not owned by you" }  // 404
{ "error": "File type not allowed" }                // 400
{ "error": "File too large" }                       // 400

JavaScript example — upload with Lodge:

const csrf = sessionStorage.getItem('csrf');
const form = new FormData();
form.append('file', fileInput.files[0]);

const res = await fetch('/outpost/member-api.php?action=lodge/upload&item_id=123', {
  method: 'POST',
  credentials: 'include',
  headers: { 'X-CSRF-Token': csrf },
  body: form,
});

const data = await res.json();
if (data.success) {
  console.log('Uploaded:', data.url);
}
