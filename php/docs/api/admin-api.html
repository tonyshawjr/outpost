Admin API

The Admin API covers all CMS management operations: pages, collections, items, media, users, folders, navigation menus, settings, and more. All endpoints require an active admin session and a CSRF token on mutations.

Auth required All Admin API endpoints require a valid admin session cookie (outpost_session). POST, PUT, and DELETE requests also require an X-CSRF-Token header. See API Overview & Auth for the login flow.

Pages

GET api.php?action=pages

Returns all pages for the active theme. Excludes collection item URL paths and the internal __global__ page. Supports an optional search query parameter to filter by path or title.

Query paramTypeDescription
searchstringOptional. Filter pages by path or title (substring match).

Response:

{
  "pages": [
    {
      "id": 1,
      "path": "/",
      "title": "Home",
      "meta_title": "Welcome",
      "meta_description": "...",
      "status": "published",
      "visibility": "public",
      "published_at": "2024-01-15 09:00:00",
      "updated_at": "2024-03-01 14:22:00"
    }
  ]
}
GET api.php?action=pages&id=1

Returns a single page with all of its fields for the active theme, plus any registry fields that have not yet been populated.

Response:

{
  "page": {
    "id": 1,
    "path": "/",
    "title": "Home",
    "meta_title": "",
    "meta_description": "",
    "status": "published",
    "visibility": "public",
    "active_theme": "personal",
    "fields": [
      {
        "id": 5,
        "field_name": "hero_title",
        "field_type": "text",
        "content": "Welcome",
        "default_value": "Hello World",
        "sort_order": 0
      }
    ]
  }
}
PUT api.php?action=pages&id=1

Updates page metadata. Clears the page HTML cache automatically.

FieldTypeDescription
titlestringPage title shown in admin listings.
meta_titlestringSEO meta title tag.
meta_descriptionstringSEO meta description tag.
statusstringpublished or draft.
visibilitystringpublic, members, or paid.
DELETE api.php?action=pages&id=3

Deletes a page, its template file, all associated fields, and field registry entries. Clears the full template cache.

Query paramTypeDescription
idintegerRequired. The page ID to delete.
Protected pages — The homepage (/) and the internal globals page (__global__) cannot be deleted. The endpoint returns an error for these paths.

What gets deleted:

  • The .html template file from the active theme directory (e.g. themes/personal/about.html)
  • The page row in the pages table
  • All fields for that page (cascade via foreign key)
  • All page_field_registry entries for that path

Response:

{ "success": true }

Fields (bulk update)

PUT api.php?action=fields/bulk

Saves content for one or more fields at once. Rich-text fields are sanitized before storage. Clears page cache for all affected pages; if any global field is updated, all page caches are cleared.

Request body:

{
  "fields": [
    { "id": 5, "content": "Welcome to our site" },
    { "id": 6, "content": "<p>Rich text here</p>" }
  ]
}

Response: {"success": true, "updated": 2}

Global fields

GET api.php?action=globals

Returns all global fields (the __global__ page fields). These are shared across all pages and themes.

Response:

{
  "fields": [
    {
      "id": 12,
      "field_name": "site_name",
      "field_type": "text",
      "content": "My Site",
      "default_value": "",
      "options": ""
    }
  ]
}

Collections

GET api.php?action=collections

Returns all collections with item counts.

POST api.php?action=collections

Creates a new collection. The slug must be lowercase letters, numbers, and hyphens only, and must be unique.

FieldRequiredDescription
slugYesURL-safe identifier, e.g. blog.
nameYesHuman-readable name, e.g. Blog Posts.
singular_nameNoSingular form, e.g. Blog Post. Defaults to name.
schemaNoJSON object defining the field schema.
url_patternNoURL pattern, e.g. /blog/{slug}. Defaults to /{slug}/{slug}.
sort_fieldNoField to sort items by. Defaults to created_at.
sort_directionNoASC or DESC. Defaults to DESC.
items_per_pageNoDefault pagination size. Defaults to 10.

Response: {"success": true, "id": 3} with HTTP 201.

PUT api.php?action=collections&id=3

Updates an existing collection. Same fields as create (except slug which cannot be changed).

DELETE api.php?action=collections&id=3

Deletes a collection and all its items.

Collection items

GET api.php?action=items&collection=blog

Returns all items in a collection, decoded from JSON storage. Supports an optional status filter.

Query paramTypeDescription
collectionstringRequired. Collection slug.
statusstringOptional. Filter by published, draft, or scheduled.

Response:

{
  "items": [
    {
      "id": 10,
      "collection_id": 3,
      "slug": "my-first-post",
      "status": "published",
      "data": { "title": "My First Post", "body": "..." },
      "published_at": "2024-02-01 08:00:00",
      "updated_at": "2024-02-01 08:00:00"
    }
  ],
  "collection": { "id": 3, "slug": "blog", "name": "Blog Posts", ... }
}
POST api.php?action=items

Creates a new collection item.

FieldRequiredDescription
collection_idYesNumeric ID of the collection.
slugYesURL-safe slug, unique within the collection.
dataNoObject containing all field values.
statusNodraft (default), published, or scheduled.
scheduled_atNoISO datetime string, used when status is scheduled.
PUT api.php?action=items&id=10

Updates a collection item. Any fields omitted from the body are left unchanged. Clears the full HTML cache so listing pages reflect the change.

FieldDescription
slugNew slug (must be unique in the collection).
dataObject containing field values to replace.
statusdraft, published, or scheduled.
sort_orderInteger sort position.
scheduled_atISO datetime for scheduled publish.
published_atOverride the publish timestamp.
DELETE api.php?action=items&id=10

Permanently deletes a collection item.

PUT api.php?action=items/inline

Partially updates specific fields in a collection item's data JSON. Designed for on-page editing — merges new field values into the existing data without replacing unmodified fields. Creates a revision snapshot, sanitizes richtext fields, and supports optimistic locking.

FieldRequiredDescription
idYesNumeric ID of the collection item.
fieldsYesAssociative object of field name/value pairs to merge into the item's data.
_versionNoThe updated_at timestamp the client last saw. If provided and the server value differs, returns 409 conflict.

Request body:

{
  "id": 42,
  "fields": { "title": "New Title", "body": "<p>Updated</p>" },
  "_version": "2026-03-06 14:30:00"
}

Response:

{ "success": true, "updated_at": "2026-03-06 14:30:01" }

Conflict response (409):

{
  "error": "This item was modified by another user. Reload to see the latest version.",
  "conflict": true,
  "server_updated_at": "2026-03-06 14:30:05"
}
PUT api.php?action=items/bulk-status

Updates the status of multiple collection items in a single request. Preserves existing published_at timestamps on items that were already published.

FieldRequiredDescription
idsYesArray of integer item IDs.
statusYesdraft or published.

Request body:

{
  "ids": [10, 14, 22],
  "status": "published"
}

Response:

{ "success": true, "count": 3 }
DELETE api.php?action=items/bulk-delete

Permanently deletes multiple collection items in a single request.

FieldRequiredDescription
idsYesArray of integer item IDs.

Request body:

{
  "ids": [10, 14, 22]
}

Response:

{ "success": true, "count": 3 }
POST api.php?action=items/preview-token

Generates a secure preview token for a collection item, allowing draft or scheduled items to be viewed on the live site via ?preview=TOKEN in the URL. The token is regenerated on each call.

FieldRequiredDescription
idYesInteger item ID.

Response:

{
  "success": true,
  "token": "a1b2c3d4e5f6..."
}
Preview URL format Append ?preview=TOKEN to the item's normal URL. The router validates the token against the item's stored preview_token and serves the page even if the item is not published. Preview pages are never cached and display a "Preview Mode" banner at the bottom of the page.

Media

GET api.php?action=media

Returns all media files sorted newest first.

POST api.php?action=media/upload

Uploads a file. Use multipart/form-data. The CSRF token must be sent as a form field named csrf_token (not as a header, because the request is multipart).

Form fields:

FieldDescription
fileThe file to upload.
csrf_tokenCSRF token from auth/me.

Response (HTTP 201):

{
  "success": true,
  "media": {
    "id": 7,
    "filename": "photo-abc123.jpg",
    "original_name": "photo.jpg",
    "path": "/outpost/uploads/photo-abc123.jpg",
    "thumb_path": "/outpost/uploads/thumbs/photo-abc123.jpg",
    "mime_type": "image/jpeg",
    "file_size": 204800,
    "width": 1200,
    "height": 800,
    "alt_text": ""
  }
}

curl example:

curl -b cookies.txt \
  -F "file=@/path/to/photo.jpg" \
  -F "csrf_token=a1b2c3..." \
  "https://example.com/outpost/api.php?action=media/upload"
PUT api.php?action=media&id=7

Updates media metadata. Currently supports alt_text.

Request body:

{ "alt_text": "A sunset over the mountains" }
POST api.php?action=media/transform

Applies a resize or crop transformation to a raster image. SVGs are not supported. Requires the GD extension. The original file is overwritten and the thumbnail is regenerated.

Request body (resize — provide width, height, or both; aspect ratio is maintained if only one is given):

{ "id": 7, "action": "resize", "width": 800 }

Request body (crop — x, y are the top-left origin; width, height define the crop region):

{ "id": 7, "action": "crop", "x": 100, "y": 50, "width": 600, "height": 400 }

Both operations return the updated media object. Maximum dimension is 4800px.

DELETE api.php?action=media&id=7

Deletes a media file and its thumbnail from disk, and removes the database record.

Settings

GET api.php?action=settings

Returns all site settings as a flat key-value object. Requires the settings.* capability.

Response:

{
  "settings": {
    "site_name": "My Site",
    "site_url": "https://example.com",
    "active_theme": "personal",
    "cache_enabled": "1"
  }
}
PUT api.php?action=settings

Updates one or more settings. Send a flat key-value object. Any key present in the body is upserted. Requires the settings.* capability.

Request body:

{
  "site_name": "My Updated Site",
  "cache_enabled": "1"
}

Cache

POST api.php?action=cache/clear

Clears all HTML page caches and compiled template caches. Requires the cache.* capability.

Response: {"success": true, "message": "Cache cleared"}

Users

Note User endpoints require the users.* capability. Creating or modifying users with admin or developer roles requires the super_admin or admin role. Deleting users requires admin role or above.
GET api.php?action=users

Returns all admin users (not members). Password hashes are never returned.

GET api.php?action=users&id=2

Returns a single admin user by ID.

Response:

{
  "user": {
    "id": 2,
    "username": "editor1",
    "email": "editor@example.com",
    "display_name": "Jane Editor",
    "avatar": "",
    "bio": "",
    "role": "editor",
    "created_at": "2024-01-01 10:00:00",
    "last_login": "2024-03-01 09:30:00"
  }
}
POST api.php?action=users

Creates a new admin user. Password must be at least 8 characters.

FieldRequiredDescription
usernameYesUnique login name.
passwordYesMinimum 8 characters.
emailNoEmail address.
display_nameNoDisplay name.
roleNoRole slug. Defaults to admin. Valid values: super_admin, admin, developer, editor.
PUT api.php?action=users&id=2

Updates a user. Pass only the fields you want to change. To change the password, include password in the body. Omitting it leaves the password unchanged.

DELETE api.php?action=users&id=2

Deletes a user. Cannot delete your own account.

Code editor

Code editor endpoints require the code.* capability.

GET api.php?action=code/files

Returns a directory listing of editable theme files.

GET api.php?action=code/read

Returns the contents of a specific file. Pass the relative path as a query parameter.

PUT api.php?action=code/write

Writes content to a file. The file must be within the active theme directory.

POST api.php?action=code/create

Creates a new file or folder. Body: { path, type: "file"|"folder", content? }. File extensions must be in the allowed list.

POST api.php?action=code/rename

Renames or moves a file/folder within the theme. Body: { oldPath, newPath }. Path traversal is blocked.

DELETE api.php?action=code/delete

Deletes a file or folder (recursive for directories). Body: { path }.

GET api.php?action=code/search&q=term

Full-text search across all theme files. Returns up to 200 matching results with line numbers and context.

GET api.php?action=code/context

Returns the Outpost context for editor autocomplete: global field names, collection slugs, and collection schemas.

Channels

Manage external data source channels — connect to REST APIs, RSS feeds, and CSV files, sync data into SQLite cache, and render in templates. Requires the settings.* capability.

GET api.php?action=channels

Returns all channels with item counts and sync status.

Response shape:

{
  "channels": [
    {
      "id": 1,
      "slug": "listings",
      "name": "Property Listings",
      "type": "api",
      "status": "active",
      "cache_ttl": 3600,
      "url_pattern": "/listing/{slug}",
      "last_sync_at": "2026-03-05 10:00:00",
      "item_count": 47,
      "created_at": "2026-03-05 09:00:00"
    }
  ]
}
GET api.php?action=channels&id=1

Returns a single channel with full configuration, field map, and item count.

POST api.php?action=channels

Creates a new channel. The type field must be "api", "rss", or "csv" (default: "api"). Type is immutable after creation.

Request body (REST API channel):

{
  "name": "Property Listings",
  "slug": "listings",
  "type": "api",
  "config": {
    "url": "https://api.example.com/v1/listings",

    "auth_type": "bearer",
    "auth_config": { "bearer_token": "sk_live_..." },

    "params": { "status": "active" },
    "data_path": "data.listings",
    "id_field": "id",
    "slug_field": "slug",
    "pagination": { "type": "offset", "page_param": "page", "limit_param": "limit", "per_page": 50 }
  },
  "field_map": [
    { "source": "title", "type": "string" },
    { "source": "price", "type": "number" },
    { "source": "images.0.url", "type": "string" }
  ],
  "cache_ttl": 3600,
  "url_pattern": "/listing/{slug}",
  "sort_field": "updated_at",
  "sort_direction": "desc",
  "max_items": 100
}

Request body (RSS channel):

{
  "name": "Company Blog",
  "slug": "blog-feed",
  "type": "rss",
  "config": {
    "url": "https://example.com/feed",
    "id_field": "guid",
    "slug_field": "title"
  },
  "field_map": [
    { "source": "title", "type": "string" },
    { "source": "link", "type": "string" },
    { "source": "description", "type": "string" },
    { "source": "pubDate", "type": "string" },
    { "source": "content", "type": "string" }
  ],
  "cache_ttl": 3600
}

Request body (CSV channel):

{
  "name": "Team Directory",
  "slug": "team",
  "type": "csv",
  "config": {
    "url": "https://docs.google.com/spreadsheets/d/.../pub?output=csv",

    "csv_has_headers": true,
    "id_field": "name",
    "slug_field": "name"
  },

    { "source": "name", "type": "string" },
    { "source": "role", "type": "string" },
    { "source": "email", "type": "string" }
  ],

}

Schema discovery

POST api.php?action=channels/discover

Fetches a URL with the provided configuration and returns the discovered schema. Used by the admin channel builder wizard to discover source structure before saving the channel. Dispatches by type: REST API returns JSON schema tree, RSS returns standard feed fields, CSV returns column headers.

Request body (REST API):

{
  "type": "api",
  "url": "https://api.example.com/v1/listings",
  "method": "GET",
  "auth_type": "bearer",
  "auth_config": { "bearer_token": "sk_live_..." },
  "headers": {},
  "params": {},
  "data_path": "data.listings"
}

Request body (RSS):

{
  "type": "rss",
  "url": "https://example.com/feed"
}

Request body (CSV):

{
  "type": "csv",
  "url": "https://docs.google.com/.../pub?output=csv",
  "csv_delimiter": ",",
  "csv_has_headers": true
}

Response:

{
  "success": true,
  "schema": {
    "id": "number",
    "title": "string",
    "price": "number",
    "active": "boolean",
    "tags": "array",
    "address": {
      "street": "string",
      "city": "string"
    }
  },
  "sample": [ { "id": 1, "title": "123 Main St", "price": 350000 } ],
  "total_items": 47
}

Search

GET api.php?action=search/content

Full-text search across pages, collection items, and media. Returns up to 26 results grouped by type. Requires at least 2 characters.

Query paramTypeDescription
qstringSearch query (minimum 2 characters).

Response shape:

{
  "results": [
    { "type": "page",  "id": 1, "title": "About", "subtitle": "/about", "meta": {} },
    { "type": "item",  "id": 5, "title": "My Post", "subtitle": "Posts", "status": "draft", "meta": { "collection_slug": "post" } },
    { "type": "media", "id": 3, "title": "hero.jpg", "subtitle": "image/jpeg", "meta": {} }
  ]
}

Results are ordered by recency within each type: pages (up to 8), collection items (up to 12), media (up to 6). The admin uses this to power the ⌘K command palette.

Sync key management

These endpoints manage the key used by the Outpost Builder sync. Requires the settings.* capability.

GET api.php?action=sync/key

Returns metadata about the current sync API key (masked). Auto-generates a key on first access.

Response:

{
  "key_set": true,
  "key_masked": "a1b2c3d4••••••••••••••••••••••••••••••••••••••••••••••••ef01",
  "last_pull": "2024-03-01 10:00:00",
  "last_push": "2024-03-01 09:55:00"
}
POST api.php?action=sync/key/regenerate

Generates a new sync API key. The old key is invalidated immediately. The new key is returned once — store it in your Builder configuration.

Response:

{
  "success": true,
  "key": "full-64-char-hex-key-shown-once",
  "key_masked": "a1b2c3d4••••••••••••••••••••••••••••••••••••••••••••••••ef01"
}

API keys

Manage Bearer token API keys for headless access. These endpoints require the settings.* capability. Keys allow external tools, CI/CD pipelines, and static site generators to authenticate with the Admin API without browser sessions.

GET api.php?action=apikeys

Lists all API keys. Returns metadata only — the actual key values are never retrievable after creation.

Response:

{
  "keys": [
    {
      "id": 1,
      "name": "CI/CD Pipeline",
      "key_prefix": "op_a1b2c3d4",
      "username": "admin",
      "display_name": "Admin",
      "last_used_at": "2026-03-05 14:30:00",
      "created_at": "2026-02-01 10:00:00"
    }
  ]
}
POST api.php?action=apikeys

Creates a new API key. The full key is returned once in the response — store it securely. It cannot be retrieved later. The key is linked to the currently authenticated user and inherits that user's role and permissions.

Request body:

{
  "name": "CI/CD Pipeline"
}

Response:

{
  "success": true,
  "key": "op_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2",
  "id": 1,
  "prefix": "op_a1b2c3d4"
}
One-time display The key field is the only time the full key is returned. It is stored as a bcrypt hash — there is no way to retrieve it after this response.
DELETE api.php?action=apikeys&id=N

Revokes (deletes) an API key. Any requests using this key will immediately receive 401 Unauthorized.

Response:

{
  "success": true
}

Using an API key

Pass the key as a Bearer token in the Authorization header. No session cookie or CSRF token is required:

curl -H "Authorization: Bearer op_a1b2c3d4e5f6..." \
     "https://example.com/outpost/api.php?action=pages"

API key requests are subject to the same role-based capability checks as session-authenticated requests. Rate limiting uses IP-based tracking instead of session-based.

Revisions

Content revision history — automatic snapshots on every save. Keeps the last 25 revisions per entity.

GET api.php?action=revisions&entity_type=page&entity_id=N

Lists revisions for a page or collection item (newest first, max 25). Pass entity_type (page or item) and entity_id.

Response:

{
  "revisions": [
    {
      "id": 42,
      "entity_type": "page",
      "entity_id": 5,
      "created_at": "2025-03-01 10:00:00",
      "created_by": 1,
      "user": "Admin",
      "field_count": 4
    }
  ]
}
POST api.php?action=revisions/restore

Restores a revision. The current content is automatically snapshotted first so the restore itself is undoable. Body: { entity_type, entity_id, revision_id }.

Response:

{
  "success": true
}

Webhooks

Send HTTP POST notifications to external URLs when content changes. Requires settings.* capability (admin+ only).

GET api.php?action=webhooks

Lists all configured webhooks (id, name, url, events, active status).

GET api.php?action=webhooks&id=N

Returns a single webhook's full details including custom headers (secret is never returned).

POST api.php?action=webhooks

Creates a new webhook. Body: { name, url, events, headers, active }. Returns the auto-generated signing secret (shown only once).

Response:

{
  "success": true,
  "id": 1,
  "secret": "a1b2c3..."
}
PUT api.php?action=webhooks&id=N

Updates a webhook. Body can include: name, url, events, headers, active.

DELETE api.php?action=webhooks&id=N

Deletes a webhook and all its delivery history.

POST api.php?action=webhooks/regenerate-secret&id=N

Regenerates the signing secret for a webhook. Returns the new secret (shown only once).

GET api.php?action=webhooks/deliveries&id=N

Returns the last 50 delivery attempts for a webhook (event, status, HTTP code, attempts, timestamps).

POST api.php?action=webhooks/test&id=N

Sends a test event to the webhook URL and returns the delivery result.

Webhook payload

Each webhook delivery sends a JSON POST with these headers:

  • Content-Type: application/json
  • X-Outpost-Event — the event name (e.g. entry.published)
  • X-Outpost-Signature — sha256=<HMAC-SHA256 of body>
  • X-Outpost-Delivery — unique delivery ID
  • User-Agent: Outpost-CMS/1.0

Body format:

{
  "event": "entry.published",
  "timestamp": "2026-03-05T12:00:00Z",
  "data": { ... }
}

Supported events

  • entry.created, entry.updated, entry.published, entry.unpublished, entry.deleted
  • page.updated, page.published, page.unpublished, page.deleted
  • media.created, media.deleted
  • member.created, member.updated, member.deleted
  • form.submitted
  • cache.cleared
  • * — wildcard, matches all events

Retry schedule

Failed deliveries are retried up to 5 times via the cron endpoint: 1 min, 5 min, 30 min, 2 hours, 12 hours. Successful deliveries are cleaned up after 7 days, exhausted failures after 30 days.

Updates

Built-in auto-updater. Requires super_admin role. See Auto-Updater for the full workflow.

GET api.php?action=updates/check

Checks for a newer version by comparing OUTPOST_VERSION against the latest GitHub Release. Response is cached for 1 hour.

{
  "available": true,
  "current_version": "1.0.0-beta.13",
  "latest_version": "1.0.0-beta.14",
  "release_notes": "...",
  "download_url": "https://github.com/..."
}
POST api.php?action=updates/apply

Downloads a release zip and applies the update. Body: { url }. The URL must point to github.com/tonyshawjr/outpost/releases. All zip paths are validated before extraction. Only core files are replaced — themes, data, and uploads are never touched.

Cron

Cron endpoints handle scheduled tasks (publishing, webhook retries, analytics cleanup). The public cron endpoint requires a secret key.

GET api.php?action=cron&key=SECRET

Runs all scheduled tasks: publishes scheduled items, retries failed webhook deliveries, prunes old analytics data. Call this from an external cron job (e.g. every 5 minutes). The key parameter must match the stored cron secret.

GET api.php?action=cron/key

Returns the current cron secret key. Requires super_admin role.

POST api.php?action=cron/key/regenerate

Regenerates the cron secret key. The previous key is immediately invalidated. Requires super_admin role.

Goals

Analytics goal tracking. Requires analytics.* capability.

GET api.php?action=goals

Lists all configured goals with their current conversion counts.

POST api.php?action=goals

Creates a new goal. Body: { name, type, target_url?, target_event? }.

PUT api.php?action=goals&id=N

Updates an existing goal.

DELETE api.php?action=goals&id=N

Deletes a goal.

Dashboard analytics

GET api.php?action=dashboard/events

Returns custom event analytics (outbound clicks, downloads, form submissions) aggregated by event name.

GET api.php?action=dashboard/events/detail&name=event_name

Returns detailed event data for a specific event name, including per-URL breakdowns.

GET api.php?action=dashboard/goals

Returns goal completion data for the dashboard, including conversion counts per goal over the selected time range.

Revisions diff

GET api.php?action=revisions/diff&id=N

Returns a field-level diff between a revision and the current content. Shows which fields changed, with before/after values.
