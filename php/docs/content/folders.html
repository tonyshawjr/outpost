Folders

Classify and group collection items using flexible category and tag systems. Folders can be flat (like tags) or hierarchical (like categories with parent/child labels), and their labels are filterable from templates.

Renamed in v1.0.0-beta.7 — "Taxonomies" are now called Folders and "Terms" are now called Labels. The old API endpoints (action=taxonomies, action=terms) still work for backward compatibility.

What are folders?

A folder is a labeling system for your content. Where a collection holds the items themselves, a folder provides the classification layer that organizes them. You define a folder (e.g., "Categories"), populate it with labels (e.g., "Technology", "Design", "Business"), and then assign those labels to individual collection items when editing them in the admin.

Once in place, templates can loop over all labels in a folder to build filter navigation, and collection loops can be filtered by a label selected from the URL query string. This is the standard pattern for blog categories, project tags, portfolio types, and any other content classification need.

Folder types

TypeStructureUse cases
Flat A simple list of peer labels — no hierarchy, no parent/child relationship. Tags, Topics, Skills, Services, Industries, Technologies
Hierarchical Labels can have a parent, forming a tree. Each label stores a parent_id pointing to its parent. Categories (Technology → JavaScript), Locations (Europe → France → Paris)

Creating a folder

  1. Go to Admin → Folders and click New Folder.
  2. Enter a Name (e.g., Category), a Slug (e.g., category), and choose Flat or Hierarchical.
  3. Select the Collection this folder applies to.
  4. Save. The folder appears in the sidebar under Folders.
  5. Click the folder name to open its label list. Add labels with New Label.
Note — A folder belongs to one collection. If you need the same labeling on two collections (e.g., both Posts and Tutorials share a Topics folder), create two separate folders with the same labels.

Label fields

Every label has the following built-in fields:

FieldTypeDescription
name string Display name shown to visitors (e.g., "JavaScript")
slug string URL-safe identifier auto-generated from name (e.g., "javascript") — used in filter URLs
description string Optional plain-text description of the label
parent_id integer ID of the parent label. 0 or empty means top-level. Hierarchical folders only.
count integer Number of published collection items currently tagged with this label
url string The archive URL for this label (if a label archive route is configured)
data.* mixed Custom fields added to this folder's label schema (see below)

Assigning labels to collection items

When you open any item in the admin collection editor, the sidebar shows a section for each folder attached to that collection. Click labels to toggle them on or off. Multiple labels can be assigned from the same folder — a blog post can be in both "Technology" and "Design" at once.

Assignments are saved as part of the item when you save. The count on each label is recalculated automatically.

Tip — The count field reflects only published items. Draft items with that label do not increment the count.

Looping over all labels in a folder

Use <outpost-each folder="slug"> to iterate over every label. Replace slug with the folder's slug. This is used to build filter navigation, tag clouds, and category lists.

<!-- Build a category filter bar -->
<nav class="category-filter">
  <a href="/blog">All</a>
  <outpost-each folder="category">
    <a data-outpost="filter_url" data-type="link" href="/blog?category=default">
      <span data-outpost="name">Label Name</span>
      <span class="count" data-outpost="count" data-type="number">0</span>
    </a>
  </outpost-each>
</nav>

All built-in label variables are available in the loop body:

<outpost-each folder="tag">
  <li>
    <a data-outpost="filter_url" data-type="link" href="/blog?tag=default"><span data-outpost="name">Label Name</span></a>
    <outpost-if field="description" exists>
      <p class="label-desc"><span data-outpost="description">Description text</span></p>
    </outpost-if>
    <span><span data-outpost="count" data-type="number">0</span> posts</span>
  </li>
</outpost-each>

Filtering a collection loop by folder label

Add the filteredby attribute to an <outpost-each> collection loop to filter items by a URL query parameter. When a visitor loads /blog?category=design, the loop returns only items tagged with the "design" label. Without the query parameter, all published items are returned.

<!-- URL: /blog?category=design — shows only items tagged "design" -->
<outpost-each collection="post" filteredby="category">
  <article>
    <h2><a data-outpost="url" data-type="link" href="#"><span data-outpost="title">Post Title</span></a></h2>
    <p data-outpost="excerpt">Post excerpt.</p>
  </article>
</outpost-each>
Note — The value of filteredby is the URL query string key, not the folder slug. Use a short, readable key. It will appear in visitor-facing URLs like /blog?category=technology.

You can combine filteredby with other loop attributes like paginate and orderby:

<outpost-each
    collection="post"
    filteredby="category"
    orderby="published_at"
    paginate="10">
  <h2 data-outpost="title">Post Title</h2>
</outpost-each>

Filtering works alongside pagination transparently — each page of results contains only items matching the active label.

Full filter + list pattern

The most common use of folders is a filter bar paired with a filtered item list. Here is a complete blog.html template implementing this pattern:

<!-- blog.html — category filter + paginated post list -->
<nav class="filter-bar">
  <a href="/blog">All Posts</a>
  <outpost-each folder="category">
    <a data-outpost="filter_url" data-type="link" href="/blog?category=default">
      <span data-outpost="name">Label Name</span>
    </a>
  </outpost-each>
</nav>

<div class="post-list">
<outpost-each collection="post" filteredby="category" orderby="published_at" paginate="12">
  <article>
    <outpost-if field="featured_image" exists>
      <img data-outpost="featured_image" data-type="image" src="default.jpg" alt="Post image">
    </outpost-if>
    <h2><a data-outpost="url" data-type="link" href="#"><span data-outpost="title">Post Title</span></a></h2>
    <p data-outpost="excerpt">Post excerpt.</p>
    <div class="post-tags">
      <outpost-each folder="category" scope="item">
        <span class="tag" data-outpost="name">Tag Name</span>
      </outpost-each>
    </div>
  </article>
</outpost-each>
</div>

Displaying an item's own labels

Inside a single-item template or a collection loop, you can access the labels assigned to that specific item using <outpost-each folder="slug" scope="item">:

<!-- post.html (single-item template) -->
<h1 data-outpost="title">Post Title</h1>

<!-- Show this post's categories as links -->
<outpost-if field="labels.category" exists>
  <div class="post-categories">
    <outpost-each folder="category" scope="item">
      <a data-outpost="filter_url" data-type="link" href="/blog?category=default"><span data-outpost="name">Category Name</span></a>
    </outpost-each>
  </div>
</outpost-if>

<div data-outpost="body" data-type="richtext">Post body content.</div>

Hierarchical folders

For hierarchical folders, the loop returns all labels in a single flat list ordered by creation. Each label carries a parent_id — labels with parent_id == "0" are top-level; all others are children. You construct the visual hierarchy in your template:

<ul class="category-tree">
<outpost-each folder="category">
  <outpost-if field="parent_id" is="0">
    <li class="parent">
      <strong data-outpost="name">Parent Name</strong>
    </li>
  </outpost-if>
  <outpost-if field="parent_id" not="0">
    <li class="child">
      <a data-outpost="filter_url" data-type="link" href="/blog?category=default">— <span data-outpost="name">Child Name</span></a>
    </li>
  </outpost-if>
</outpost-each>
</ul>
Tip — When creating hierarchical labels in the admin, create parent labels first. The loop returns labels in creation order, so parents will naturally appear before their children in the flat output list, making visual nesting easier without sorting logic.

Multiple folders on one collection

A collection can have more than one folder attached to it. For example, a blog might use both a hierarchical "Category" folder and a flat "Tag" folder. Each appears as a separate label selector in the item editor, and each can be looped and filtered independently in templates:

<!-- Category nav — hierarchical -->
<div class="category-nav">
  <outpost-each folder="category">
    <a data-outpost="filter_url" data-type="link" href="/blog?category=default"><span data-outpost="name">Category Name</span></a>
  </outpost-each>
</div>

<!-- Tag cloud — flat -->
<div class="tag-cloud">
  <outpost-each folder="tag">
    <a data-outpost="filter_url" data-type="link" href="/blog?tag=default" class="tag"><span data-outpost="name">Tag Name</span></a>
  </outpost-each>
</div>

<!-- Items filtered by category only (tag filter could be added too) -->
<outpost-each collection="post" filteredby="category">
  <h2 data-outpost="title">Post Title</h2>
</outpost-each>

Custom fields on labels

You can add custom fields to a folder's labels. Open the folder in Admin → Folders → [Folder] → Settings and add fields to the label schema. Each label then has those extra fields editable on its detail page.

Custom field values are accessed via data-outpost="data.field_name":

<!-- Category with a brand color and icon custom field -->
<outpost-each folder="category">
  <a data-outpost="filter_url" data-type="link" href="/blog?category=default">
    <outpost-if field="data.icon" exists>
      <img data-outpost="data.icon" data-type="image" src="default.png" alt="" width="20">
    </outpost-if>
    <span data-outpost="name">Label Name</span>
    <span data-outpost="count" data-type="number">0</span>
  </a>
</outpost-each>

Available custom field types for labels:

TypeUse case
textSingle-line string — labels, slugs, short descriptions
textareaMulti-line plain text — extended descriptions
imageMedia picker — icon, thumbnail, cover image per label
colorHex color picker — brand color per category
numberNumeric value — sort order, display weight

Database tables

Folders use three tables in the SQLite database:

TablePurposeKey columns
folders One row per folder id, name, slug, type (flat/hierarchical), collection_id, schema (JSON)
labels One row per label id, folder_id, name, slug, description, parent_id, count, data (JSON)
item_labels Junction table — maps items to labels item_id, label_id

Template tag quick reference

TagWhat it does
<outpost-each folder="slug">...</outpost-each> Loop over all labels in the named folder
<span data-outpost="name">Default</span> Label display name
<span data-outpost="slug">Default</span> Label URL slug
<span data-outpost="count" data-type="number">0</span> Published item count for this label
<span data-outpost="description">Default</span> Optional description set on the label
<span data-outpost="parent_id">0</span> Parent label ID (hierarchical only); 0 = top-level
<span data-outpost="data.field">Default</span> Custom field value on this label
<outpost-each folder="slug" scope="item">...</outpost-each> Loop over labels assigned to a specific collection item
filteredby="key" Collection loop attribute — filter by ?key=label-slug in URL
