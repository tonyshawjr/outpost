Media

Upload and manage images, documents, and video files from the admin. Files are stored on disk, auto-processed for web delivery, and referenced by URL in page fields, collection items, global fields, and templates.

Media library

The media library is at Admin → Media. It shows all uploaded files as a responsive grid of thumbnails. Click any file to open its detail panel, which shows the full path, pixel dimensions (for images), file size, and a Copy Path button you can use to paste the URL into any text or link field.

You can also upload files directly from within the page editor and collection item editor — clicking an image field opens the media library in an inline modal. Newly uploaded files are immediately available for selection without leaving the editor.

Supported file types

CategoryExtensions
Images (raster) jpg, jpeg, png, gif, webp, avif
Images (vector) svg
Documents pdf
Video mp4, webm
Audio mp3, wav

Upload limits and auto-processing

BehaviorDetail
Max file size 10 MB per file by default. Configure via OUTPOST_MAX_UPLOAD_SIZE in config.php.
Image resizing Images wider than 2400px are automatically scaled down to 2400px. Aspect ratio is always preserved. Smaller images are not upscaled.
Thumbnail generation A thumbnail is generated at max 400×300px (aspect ratio preserved) and stored alongside the original with a thumb_ prefix in the filename.
Dimensions recorded Pixel width and height are stored in the database for all uploaded images.
Filename sanitization Filenames are sanitized on upload — special characters are replaced with hyphens, and a Unix timestamp prefix is added to prevent naming collisions.
Note — Processing (resize + thumbnail + WebP conversion) only applies to raster images (jpg, png, gif, webp, avif). SVG, PDF, video, and audio files are stored as-is without processing.

WebP auto-conversion

When enabled, Outpost automatically converts uploaded JPEG and PNG images to WebP format using the GD library. This typically reduces file size by 25–50% with no visible quality loss.

BehaviorDetail
Converted formats JPEG (.jpg, .jpeg) and PNG (.png) only.
Skipped formats GIF (may be animated), SVG (vector), WebP (already optimized), AVIF, PDF, and video files are never converted.
Alpha preservation PNG transparency is preserved in the converted WebP file.
File replacement The original JPEG/PNG is replaced on disk with a .webp file. The database stores the .webp path and image/webp MIME type.
Quality setting Configurable via OUTPOST_WEBP_QUALITY in config.php (default: 85). Range: 0–100.
Existing files Already-uploaded images are not retroactively converted. Only new uploads are affected.

To disable WebP conversion, set the constant in config.php:

define('OUTPOST_WEBP_AUTO_CONVERT', false);
Note — WebP conversion requires the PHP GD extension with WebP support. If GD is not available or does not support WebP, uploads proceed normally without conversion.

File storage paths

All uploaded files are stored under outpost/uploads/. Files are never put in subdirectories — everything is flat in the uploads folder.

FileStored path example
Original upload outpost/uploads/1700000000_hero-photo.jpg
Generated thumbnail outpost/uploads/thumb_1700000000_hero-photo.jpg
PDF document outpost/uploads/1700000001_company-brochure.pdf
SVG icon (sanitized) outpost/uploads/1700000002_logo.svg

The URL stored in the database (and output by the data-type="image" attribute) is the web-root-relative path:

/outpost/uploads/1700000000_hero-photo.jpg
Warning — Never delete the outpost/uploads/ directory. It contains all user media and is not regenerated automatically. When re-deploying, always preserve this directory. See Deploy for the correct deploy sequence.

SVG sanitization

SVG files can contain embedded scripts that execute in a browser if served with image/svg+xml content-type. Outpost automatically sanitizes uploaded SVGs before saving them to disk:

  • <script> tags and their contents are removed entirely.
  • on* event handler attributes (onclick, onload, onerror, etc.) are stripped from all elements.
  • External resource references (xlink:href pointing to external URLs) are removed.

The sanitized SVG is written to disk in place of the raw upload. The result is safe to serve directly and embed inline in HTML. See Sanitization for implementation details.

Using media in templates — the data-type="image" attribute

Image fields in page templates, collection items, and global fields use the data-type="image" attribute. This both outputs the stored URL and registers the field as an image picker widget in the admin editor — so editors see a proper media browser, not a raw text box.

Page and collection image fields

<!-- Page template — hero image field -->
<img
  data-outpost="hero_image"
  data-type="image"
  src="default.jpg"
  alt="Hero image"
  width="1200"
  height="630"
  loading="lazy"
>

<!-- Collection item — featured image (inside outpost-each) -->
<outpost-if field="featured_image" exists>
  <img
    data-outpost="featured_image"
    data-type="image"
    src="default.jpg"
    alt="Featured image"
    loading="lazy"
  >
</outpost-if>

<!-- As a CSS background image -->
<div data-outpost="banner_image" data-type="image" style="background-image: url('default.jpg')">
  ...
</div>

Global image fields

<!-- Site logo in the header partial -->
<img
  data-outpost="logo"
  data-scope="global"
  data-type="image"
  src="default-logo.png"
  alt="Site logo"
  height="32"
>

<!-- Author profile photo — only render if the global is set -->
<outpost-if field="profile_photo" scope="global" exists>
  <img
    data-outpost="profile_photo"
    data-scope="global"
    data-type="image"
    src="default-avatar.jpg"
    alt="Author photo"
    class="avatar"
  >
</outpost-if>

Conditional rendering for optional images

Use <outpost-if field="field_name" exists> to avoid rendering a broken <img> when the image field is empty:

<outpost-if field="featured_image" exists>
  <figure class="post-cover">
    <img data-outpost="featured_image" data-type="image" src="default.jpg" alt="Post cover">
  </figure>
</outpost-if>

Focal point

Every image has a focal point — the spot that should remain visible when the image is cropped by CSS object-fit: cover or background-size: cover. The default is 50% 50% (center). Click anywhere on the image preview in the media detail panel to set a custom focal point.

Setting a focal point

  1. Open the media library and click an image to open its detail panel.
  2. Click anywhere on the image preview — a white dot appears at the click position.
  3. The focal point auto-saves immediately. No button click required.
  4. Click Reset focal point below the preview to return to center (50/50).

Using focal points in templates — the data-focal attribute

The data-focal attribute outputs the focal point as a CSS-ready X% Y% string. Use it with object-position or background-position.

<!-- Image with focal-point-aware cropping -->
<img
  data-outpost="hero_image"
  data-type="image"
  src="default.jpg"
  alt="Hero image"
  style="object-fit: cover;"
  data-focal
>

<!-- Collection item focal point -->
<outpost-each collection="post">
  <img
    data-outpost="featured_image"
    data-type="image"
    src="default.jpg"
    style="object-fit: cover;"
    data-focal
  >
</outpost-each>
Tip — If no custom focal point has been set, data-focal outputs 50% 50% (the CSS default). It is safe to always include data-focal on cover images — there is no performance penalty.

Media folders

Organize media files into folders using the folder sidebar in the media library. Folders support up to 3 levels of nesting (e.g., Photos → Events → 2026). Each folder has an auto-generated slug for use in template tags.

Multi-folder assignment

Files can belong to multiple folders simultaneously. When viewing a file's detail panel, the Folders section shows all assigned folders as chips. You can:

  • Remove from folder: Click the × button on any folder chip.
  • Add to folder: Use the "Add to folder" dropdown to assign the file to additional folders.
  • Drag to add: Drag a file from the grid onto a folder in the sidebar to add it (not move it).

Creating folders

  • Click New Folder at the bottom of the folder sidebar.
  • Type a name and press Enter.
  • Bulk create: Type comma-separated names (e.g., Spring, Summer, Fall) to create multiple folders at once (max 50).
  • Right-click a folder and choose New subfolder to nest folders (max 3 levels).

Filtering by folder

  • All Files — shows every file across all folders.
  • Unfiled — shows files not assigned to any folder.
  • Click any folder name to show only files in that folder.

Right-click context menu

Right-click any file in the media grid to open a context menu with:

  • Folder badges — see which folders the file belongs to at a glance.
  • Add to folder — quickly assign the file to another folder without opening the detail panel.
  • Copy Path — copy the file's URL to the clipboard.
  • Delete — delete the file.

Resizable detail sidebar

The detail sidebar on the right side of the media library can be resized by dragging the handle on its left edge. The width range is 220–500px and persists across sessions via localStorage.

Managing folders

  • Rename: Right-click a folder → Rename, or double-click the folder name.
  • Delete: Right-click → Delete. Files in a deleted folder become unfiled. Subfolders are re-parented to the deleted folder's parent.
  • Copy template tag: Right-click a folder → Copy template tag to copy the <outpost-each folder="slug"> tag for use in templates.

Folder slugs

Each folder has an auto-generated slug derived from its name (lowercase, hyphens, no special characters). Slugs are unique across all folders. The slug is used in the <outpost-each> template tag — see Gallery Loops for details.

Role-based folder restrictions

Admins can restrict editors to specific media folders. When a user with the Editor role has folder grants configured (via Admin → Users → Edit User → Media Folder Access), they can only:

  • See and browse their granted folders
  • Upload files to their granted folders
  • Move/assign files to their granted folders
  • Delete files only from their granted folders

Editors with no folder grants configured have unrestricted access to all folders (same as previous behavior). Only admins and super admins can create, rename, or delete folders.

Note — Folders are a logical organization layer. Files remain in outpost/uploads/ on disk. Assigning a file to a folder does not change its URL.

Bulk upload

Drop multiple files at once onto the media library or use the file picker to select multiple files. Outpost uploads them in a queue with a progress drawer in the bottom-right corner of the screen.

  • Up to 2 files upload concurrently.
  • Each file shows an individual progress bar.
  • For images converted to WebP, the savings are displayed (e.g., "-340 KB").
  • Cancel individual uploads while they are in progress.
  • The queue auto-collapses 3 seconds after all uploads complete.

Documents and non-image files

For PDFs, audio, video, and other non-image files, use the data-type="link" attribute in your template. This outputs the raw URL for use in an anchor tag or media element:

<!-- PDF download link -->
<a data-outpost="brochure_pdf" data-type="link" href="#" download>
  Download Brochure (PDF)
</a>

<!-- Embedded video file -->
<video controls preload="metadata">
  <source data-outpost="demo_video" data-type="link" src="#" type="video/mp4">
</video>

<!-- Audio player -->
<audio controls>
  <source data-outpost="episode_audio" data-type="link" src="#" type="audio/mpeg">
</audio>

Gallery fields

For collection schema fields of type gallery, the field holds an ordered array of image URLs. Loop over them with <outpost-each repeat="gallery_field">:

<div class="gallery">
  <outpost-each repeat="gallery">
    <a data-outpost="url" data-type="link" href="#">
      <img
        data-outpost="url"
        data-type="image"
        src="default.jpg"
        alt=""
        loading="lazy"
      >
    </a>
  </outpost-each>
</div>

Gallery fields are reordered in the admin via drag-and-drop. The order in the template output matches the order set in the editor. See Gallery Loops for the full reference.

Responsive images

Outpost stores the original image at up to 2400px wide and generates a 400px thumbnail. The CMS does not create intermediate sizes (e.g., 640px, 960px). For responsive images, use CSS sizing or the HTML srcset attribute manually:

<img
  data-outpost="hero_image"
  data-type="image"
  src="default.jpg"
  alt="Hero image"
  style="width: 100%; height: auto;"
  loading="lazy"
  decoding="async"
>
Tip — Add loading="lazy" and decoding="async" to all non-hero images. These native HTML attributes defer loading of off-screen images and decode images asynchronously, improving page performance with no JavaScript required.

Copying file paths

In the media library, click any file to open its detail panel. Click Copy Path to copy the web-root-relative path (e.g., /outpost/uploads/1700000000_photo.jpg) to the clipboard. Paste it directly into:

  • A URL field in a page or collection item editor
  • A richtext editor body (for inline images in content)
  • A template file in the code editor
  • CSS as a background-image URL

Editing images

Select an image in the media grid to open its detail panel. The panel includes tools for editing alt text, resizing, and cropping — all without leaving the media library.

Alt text

Every media file has an alt_text field stored in the database. Type a description in the Alt text input and press Enter or click Save. Alt text is returned in both the Admin API and Public Content API responses, so you can use it in templates for accessibility.

Resize

Enter new pixel dimensions in the Width and Height fields. A lock icon between them toggles aspect-ratio locking (on by default) — changing one dimension automatically calculates the other. Click Apply Resize to process. The original file is overwritten and the thumbnail is regenerated.

  • Maximum dimension: 4800px per side
  • You can provide just width or just height — the other is calculated automatically
  • Applies to raster images only (JPEG, PNG, GIF, WebP). SVGs cannot be resized.

Crop

Click Crop Image to reveal the crop controls. Enter the origin (X, Y) and the crop region size (W, H) in pixels. Click Apply Crop to process. Values are clamped to image bounds on the server.

Warning — Both resize and crop operations are destructive. The original file is overwritten on disk. There is no undo. If you need to preserve the original, download it first.
Note — Image editing requires the PHP GD extension. If GD is not installed, the resize and crop controls will still appear in the admin, but the server will return an error when you try to apply a transformation. Most PHP installations include GD by default.

Deleting files

Deleting a file from the media library removes the record from the database and deletes both the original file and its thumbnail from disk. Any templates or content fields that still reference the deleted path will output a broken image or link.

Note — There is no "find references" feature. If you need to replace an image, upload the new file first, update all fields and templates that reference the old path, and then delete the old file.

Public URL access

All files in outpost/uploads/ are publicly accessible at /outpost/uploads/filename. There is no authentication on upload URLs — any uploaded file can be accessed by anyone with the URL. Do not upload sensitive or private files through the media library.

Configuring the upload limit

The default maximum upload size is 10 MB. To change it, define the constant in config.php before Outpost initializes:

// config.php — set upload limit to 25 MB
define('OUTPOST_MAX_UPLOAD_SIZE', 25 * 1024 * 1024);
Note — PHP's own upload limits (upload_max_filesize and post_max_size in php.ini) also apply. OUTPOST_MAX_UPLOAD_SIZE can only set a limit lower than PHP's — it cannot exceed PHP's configured maximum.
