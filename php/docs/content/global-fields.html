Global Fields

Site-wide content available in every template — site name, logo, social links, analytics IDs, contact information, footer text, and any other value shared across the whole site. Global fields auto-scaffold from templates and are edited from one place in the admin.

What are global fields?

Global fields hold content that is shared across the entire site, not scoped to a single page or collection item. Where a page field like {{ hero_title }} only appears on the specific page that defines it, a global field like {{ @site_name }} is available in every template and every partial across every page of the site.

They live in Admin → Globals. Like page fields, global fields are auto-scaffolded — the admin discovers them automatically by scanning the active theme's templates for any tag using the @ prefix. You never register them manually; just use them in your templates and they appear.

Template syntax

Global fields use the @ prefix to distinguish them from page-scoped fields:

{# Plain text — auto-escaped #}
{{ @site_name }}

{# Image field — opens media picker in admin #}
<img src="{{ @logo | image }}" alt="{{ @site_name }}">

{# Richtext — render HTML as-is (no escaping) #}
{{ @bio | raw }}

{# Link field — outputs the raw URL for use in href #}
<a href="{{ @twitter_url | link }}">Twitter</a>

{# Conditional — only render if the global has a value #}
{% if @ga4_id %}
  <script async src="https://www.googletagmanager.com/gtag/js?id={{ @ga4_id }}"></script>
{% endif %}

See Template Syntax → Global Fields for the complete syntax reference.

Auto-scaffolding

You do not register global fields in advance. When you add a new {{ @field_name }} tag anywhere in the active theme — including partials and all page templates — that field appears automatically in Admin → Globals the next time any page is rendered or the admin is opened.

{# Add this to head.html — a "favicon" field appears in Admin → Globals immediately #}
<link rel="icon" href="{{ @favicon | image }}">
Tip — Define all your site identity, social, analytics, and contact fields in your partials (head.html, nav.html, footer.html) during theme development. They all scaffold into Globals automatically at first render.

Field types for globals

The admin input type is inferred from the filter in the template tag. Use the appropriate filter to get the right editor widget:

Template tagAdmin field typeUse for
{{ @field_name }} Text input Site name, tagline, phone number, GA4 ID, email address
{{ @field_name | textarea }} Textarea Multi-line plain text — short bio, address block
{{ @field_name | raw }} Richtext (WYSIWYG) Footer text with links, bio with formatting
{{ @field_name | image }} Image (media picker) Logo, favicon, profile photo, OG image
{{ @field_name | link }} Link / URL input Social profile URLs, CTA links
{{ @field_name | select }} Select dropdown Theme mode toggle, language selection

Managing globals in admin

Navigate to Admin → Globals. All global fields from all templates and partials in the active theme appear on a single form, grouped and ordered by the sequence they appear in the template source. Each field renders the appropriate editor widget based on its filter type.

Press Cmd/Ctrl+S or click Save to save all globals in one request. Saving global fields clears the HTML page cache so visitors immediately see the updated values.

Common use cases with examples

Site identity and branding

{# In partials/head.html #}
<title>{{ meta.title }}{{ @site_name }}{{ /meta.title }} — {{ @site_name }}</title>
<meta name="description" content="{{ meta.description }}{{ @site_tagline }}{{ /meta.description }}">
<link rel="icon" href="{{ @favicon | image }}">

{# In partials/nav.html #}
<a href="/" class="logo">
  {% if @logo %}
    <img src="{{ @logo | image }}" alt="{{ @site_name }}" height="32">
  {% else %}
    {{ @site_name }}
  {% endif %}
</a>

Author / personal blog profile

<aside class="author-bio">
  {% if @profile_photo %}
    <img
      src="{{ @profile_photo | image }}"
      alt="{{ @author_name }}"
      class="avatar"
    >
  {% endif %}
  <h3>{{ @author_name }}</h3>
  <p>{{ @bio_short }}</p>
  <div class="social-links">
    {% if @twitter_url %}
      <a href="{{ @twitter_url | link }}">Twitter</a>
    {% endif %}
    {% if @github_url %}
      <a href="{{ @github_url | link }}">GitHub</a>
    {% endif %}
    {% if @linkedin_url %}
      <a href="{{ @linkedin_url | link }}">LinkedIn</a>
    {% endif %}
  </div>
</aside>

Google Analytics 4

{# In head.html — only outputs GA script if ga4_id has been set in admin #}
{% if @ga4_id %}
<script async src="https://www.googletagmanager.com/gtag/js?id={{ @ga4_id }}"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', '{{ @ga4_id }}');
</script>
{% endif %}

Footer with contact info

<footer>
  {{ @footer_text | raw }}

  <address>
    {% if @contact_email %}
      <a href="mailto:{{ @contact_email }}">{{ @contact_email }}</a>
    {% endif %}
    {% if @phone %}
      <a href="tel:{{ @phone }}">{{ @phone }}</a>
    {% endif %}
    {% if @address %}
      <span>{{ @address | textarea }}</span>
    {% endif %}
  </address>
</footer>

Promotional banner (optional global)

{# Only show if the admin has filled in the promo_banner field #}
{% if @promo_banner %}
  <div class="promo-bar">
    {{ @promo_banner }}
    {% if @promo_url %}
      <a href="{{ @promo_url | link }}">Learn more →</a>
    {% endif %}
  </div>
{% endif %}

Globals in conditionals

Use {% if @field_name %} to render a block only when a global has content. This is the standard pattern for optional fields — analytics IDs, social links, and promotional banners that may not always be active:

{# Optional social icons — only show links that have been filled in #}
<div class="social">
  {% if @instagram_url %}
    <a href="{{ @instagram_url | link }}">Instagram</a>
  {% endif %}
  {% if @twitter_url %}
    <a href="{{ @twitter_url | link }}">Twitter</a>
  {% endif %}
  {% if @youtube_url %}
    <a href="{{ @youtube_url | link }}">YouTube</a>
  {% endif %}
</div>

Storage and caching

Global field values are stored in the settings table in the SQLite database as key-value pairs. The key is the field name (e.g., site_name) and the value is the content string (or a JSON string for richtext and image fields).

When a global field is saved in Admin → Globals, the entire HTML page cache is invalidated so all pages are regenerated on next request with the updated values. This ensures that globals used in partials (like the site name in the page title) are consistent across the site immediately after saving.

Migrating a field from page-scoped to global

If you initially put a field in one page template ({{ author_name }}) and later decide it should be available everywhere, rename it with the @ prefix ({{ @author_name }}). The new global field scaffolds automatically. Content from the old page-scoped field does not carry over — copy the value manually to Admin → Globals after making the change.

Note — Removing a tag from a template does not delete the stored value. The field simply stops appearing in the admin form. If you later re-add the tag, the previously stored value is still there.
