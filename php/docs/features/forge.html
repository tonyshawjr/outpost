Forge

A visual tag builder inside the Code Editor. Select HTML, right-click (or press Cmd+E), and Forge wraps it in the correct Outpost Liquid template tag — with smart detection that suggests the right tag type based on what you selected.

How it works

  1. Open any .html or .htm file in the Code Editor.
  2. Select the HTML you want to tag.
  3. Right-click to open the Forge context menu, or press Cmd+E (Mac) / Ctrl+E (Windows/Linux) to jump straight to Make Editable.
  4. Choose an action and fill in the short form.
  5. Click confirm — the selected text is replaced with the tagged version.

When you have a selection in an HTML file, the status bar shows a Cmd+E to tag hint as a reminder.

Smart detection

Forge analyzes the selected HTML and reorders the context menu so the most likely action appears first (highlighted in green). It also pre-selects the field type in the Make Editable form.

Selected HTML Suggested action Pre-selected type
<img>, <picture>, <figure>, or src="...jpg" Make Editable image
<a href="..."> Make Editable link
<h1>–<h6> Make Editable text
Short plain text (<100 chars) Make Editable text
Long plain text (≥100 chars) Make Editable textarea
HTML with child elements Make Editable richtext
<ul> or <ol> with 2+ links Menu Loop —
<nav> with links Extract Partial (with menu option) —
<header>, <footer> Extract Partial —
<title> or <meta name="description"> Meta Tag title / description
<form> Form —
Multiple repeated sibling elements Collection Loop —
<section>, <article>, large <div> Conditional —

Actions

The Forge menu offers seven actions. Each opens a short form and inserts the correct Liquid syntax.

Make Editable

Wraps the selection in an output tag so the content becomes editable in the admin.

  • Field name — auto-generated from the selected text, in snake_case
  • Type — text, richtext, image, link, textarea, select, color, number, date, toggle
  • Scope — Page field or Global field
  • Use as default — wraps the selected text as a fallback value. ON by default when text is selected. Works for all types including images and links.

Examples of output:

<!-- Simple text field -->
{{ headline }}

<!-- Image with filter -->
{{ hero_image | image }}

<!-- Global field -->
{{ @site_name }}

<!-- Wrapping default (selected text becomes fallback) -->
{{ headline }}Welcome to our site{{ /headline }}

<!-- Image with wrapping default (preserves original URL) -->
{{ hero_image | image }}https://example.com/photo.jpg{{ /hero_image }}

<!-- Link with wrapping default -->
{{ cta_link | link }}https://example.com{{ /cta_link }}

Collection Loop

Wraps the selection in a {% for %} loop that iterates over a collection.

  • Collection — dropdown of existing collections
  • Item variable — defaults to item
  • Limit — optional maximum number of items
  • Order by — optional sort field
{% for item in collection.post limit:3 %}
  <h2>{{ item.title }}</h2>
  {{ item.body | raw }}
{% endfor %}

Field mapper: When you select HTML with images, headings, paragraphs, or links, Forge detects these content elements and shows a field mapper. It auto-guesses which collection fields match each element. You can adjust the mappings before confirming.

Menu Loop

Wraps navigation links in a {% for %} loop that iterates over an admin-managed menu.

  • Menu — dropdown of existing menus (or type a slug)
  • Link variable — defaults to link
  • Auto-replace links — when 2+ links are detected, Forge offers to replace static href and text with {{ link.url }} and {{ link.label }} automatically
{% for link in menu.main %}
  <li><a href="{{ link.url }}">{{ link.label }}</a></li>
{% endfor %}

Conditional

Wraps the selection in an {% if %} block.

  • Expression — a field name, @global, or item.field
  • Operator — truthy (default), ==, or !=
  • Value — comparison value (shown when operator is == or !=)
<!-- Truthy check -->
{% if show_cta %}
  <div class="cta">...</div>
{% endif %}

<!-- String comparison -->
{% if item.status == "featured" %}
  <span class="badge">Featured</span>
{% endif %}

Extract Partial

Moves the selected HTML into a new partial file and replaces the selection with an {% include %} tag.

  • Partial name — the filename (without extension or directory)

What happens when you confirm:

  1. A new file is created at partials/{name}.html containing your selected HTML.
  2. The selection is replaced with {% include '{name}' %}.
  3. The file tree refreshes and the new partial opens in a tab.
<!-- Before: selected HTML -->
<nav class="main-nav">...</nav>

<!-- After: replaced with -->
{% include 'main-nav' %}

Smart nav detection: When you extract a <nav> element that contains 2 or more links, Forge offers a “Connect to admin menu” checkbox. When enabled, it wraps the navigation links in a menu loop before saving the partial, so the nav is immediately dynamic.

Meta Tag

Wraps the selection in a meta tag for SEO.

  • Type — Title or Description
{{ meta.title }}My Page Title{{ /meta.title }}
{{ meta.description }}Page description here{{ /meta.description }}

Form

Inserts a form tag that renders a form created in the admin.

  • Form — dropdown of existing forms, or a slug you type in
{% form 'contact' %}

Keyboard shortcut

Cmd+E (Mac) / Ctrl+E (Windows/Linux) opens the Make Editable popover directly — since it's the most common action. The field type is pre-selected by smart detection, so tagging a field is often just: select text → Cmd+E → Enter.

Requirements

  • Forge only activates on .html and .htm files. It does not appear when editing CSS, JS, JSON, or other file types.
  • You must have a text selection — Forge does not appear on empty right-clicks.
  • The Code Editor must be open with a file loaded.

Forge Playground

Outpost ships with a 6-page demo website in themes/forge-playground/ — a marketing site for Forge itself, built with pure HTML and CSS. It’s designed as a hands-on tutorial:

  • index.html — Hero, feature grid, how-it-works steps, testimonials
  • features.html — Deep dive into each Forge action
  • blog.html — Blog cards for practicing Collection Loop and field mapping
  • gallery.html — Image grid for practicing Make Editable (image type)
  • about.html — Team grid, values section
  • contact.html — Contact form for practicing Form action

The Playground ships without a theme.json — it’s just flat HTML. When you open it in the Code Editor, a banner prompts you to “Forge a Theme” which guides you through creating the theme.json file.

After you’re done experimenting, click Reset to restore all files to their pristine state. This removes theme.json, all template tags, and any partials you created.

Preview tabs

Single-clicking a file in the file tree opens it as a preview tab (shown with italic text). If you click another file without editing, the preview tab is replaced. To keep a tab open permanently:

  • Double-click the file in the tree, or
  • Edit the file contents (any change pins the tab automatically)

Files opened from Find in Files, the command palette, or newly created files are always pinned.

Tip — Forge is designed for quick tagging during theme development. For a full reference of all available template tags and filters, see Template Syntax.
