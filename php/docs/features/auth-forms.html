Auth Forms

Build custom login, register, and forgot-password pages in your theme. Drop a data-outpost-auth attribute on a form and Outpost handles the rest.

Overview

Outpost includes built-in member login pages at /outpost/member-pages/login.php, but most themes need branded auth pages that match the site design. The auth client lets you build your own forms in plain HTML — Outpost auto-discovers them and wires them to the member API.

Note — Auth forms use session-based login (not JWT tokens). This means Lodge, member-only pages, and all session-dependent features work immediately after login. No extra configuration needed.

Login Form

Create a login.html page in your theme with a form that has data-outpost-auth="login":

<form data-outpost-auth="login" data-auth-redirect="/" class="space-y-4">
  <div data-auth-error style="display:none;"></div>

  <div>
    <label for="email">Email</label>
    <input name="email" type="email" required placeholder="you@example.com">
  </div>

  <div>
    <label for="password">Password</label>
    <input name="password" type="password" required placeholder="Your password">
  </div>

  <button type="submit">Sign In</button>

  <p><a href="/forgot-password">Forgot password?</a></p>
  <p>Don't have an account? <a href="/register">Create one</a></p>
</form>

Required elements

ElementAttributeDescription
<form>data-outpost-auth="login"Identifies this as a login form
<input>name="email"Email or username field
<input>name="password"Password field
<button>type="submit"Submit button (text changes to "Please wait..." during submission)

Optional elements

AttributeDescription
data-auth-redirect="/"Where to redirect after successful login. Defaults to /. Also checks for ?return= URL param (set by Lodge redirect).
data-auth-errorElement to display error messages (hidden by default, shown on error)
data-auth-successElement to display success messages

Register Form

<form data-outpost-auth="register" data-auth-redirect="/" class="space-y-4">
  <div data-auth-error style="display:none;"></div>
  <div data-auth-success style="display:none;"></div>

  <div>
    <label>Name</label>
    <input name="name" type="text" required>
  </div>
  <div>
    <label>Email</label>
    <input name="email" type="email" required>
  </div>
  <div>
    <label>Password</label>
    <input name="password" type="password" required>
  </div>

  <button type="submit">Create Account</button>
</form>

If email verification is enabled in settings, the form hides and a success message is shown after registration. Otherwise, the user is logged in immediately and redirected.

Forgot Password Form

<form data-outpost-auth="forgot-password" class="space-y-4">

  <div data-auth-success style="display:none;"></div>

  <div>
    <label>Email</label>
    <input name="email" type="email" required>
  </div>

  <button type="submit">Send Reset Link</button>
</form>

Lodge Integration

When Lodge is enabled and a visitor tries to access a protected route (e.g., /dashboard), they are redirected to the login page with a ?return=/dashboard parameter. After successful login, the auth client reads this parameter and redirects back to the original page.

To use your custom login page instead of the built-in one, go to Members > Lodge > Configuration and select your login page from the dropdown.

Security

  • Rate limiting — Login: 10 attempts per minute per IP. Registration: 5 per 5 minutes. Forgot password: 3 per 5 minutes.
  • Session-based auth — Uses HttpOnly session cookies. No tokens stored in localStorage.
  • CSRF protection — A CSRF token is returned on login and stored as a cookie for subsequent form submissions.
  • Password hashing — bcrypt with cost 10.
  • Account suspension — Suspended accounts cannot log in.
  • Email verification — Optional. When enabled, unverified accounts are blocked from login.

API Endpoints

The auth client calls these endpoints on /outpost/member-api.php:

ActionMethodBodyResponse
loginPOST{ email, password }Session cookie set + { success, member, csrf_token }
registerPOST{ username, email, password }{ success, member } or { requires_verification, message }
forgot-passwordPOST{ email }{ success, message }
Tip — You can style the form however you want. The data-outpost-auth attribute only controls the JS behavior. All CSS classes, layout, and structure are yours to define. The auth client JS is auto-injected when the attribute is detected — no script tags needed.

Social Login (OAuth)

OAuth support for Google, Facebook, and Apple sign-in is planned for a future release. You can add social login buttons to your form now as visual placeholders — they will be enabled when OAuth is available.
