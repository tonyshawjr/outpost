Analytics

Outpost includes built-in, privacy-respecting analytics. No external service, no cookies, no personal data — traffic data stays on your server and is viewable directly in the admin panel.

Built-in tracker

Outpost automatically injects a lightweight tracking snippet before the closing </body> tag on every page rendered by the template engine. No changes to your theme templates are required. The snippet fires a request to /outpost/track.php using fetch() with an Image() pixel as a fallback for older browsers.

Note — The tracker only fires on pages rendered through Outpost's template engine. Static files (images, CSS, JS) are not tracked.

What is tracked

  • Page path — the URL path of the visited page
  • Referrer domain — extracted from the browser's Referer header
  • Device type — desktop, mobile, or tablet (inferred from user agent string)
  • Session ID — a random ID stored in sessionStorage to identify unique sessions; not persisted across browser sessions
  • Bot detection — common bot and crawler user agents are detected and flagged; bot hits are recorded with is_bot=1 and excluded from all reports
  • Timestamp — when the pageview occurred

What is not tracked

  • IP addresses are not stored in the analytics table (they are used briefly for rate limiting, then discarded)
  • No personally identifiable information
  • No cookies — the session ID uses sessionStorage, which does not persist across browser sessions and is not a cookie
  • No cross-site tracking
Tip — Because Outpost analytics uses no cookies and collects no personal data, it generally does not require a cookie consent banner for GDPR compliance. Consult your legal adviser for your specific situation.

Data storage

All analytics data is stored in the analytics_hits table in your SQLite database (outpost/data/cms.db). No data is sent to any external service.

The Analytics dashboard

Navigate to Admin → Analytics (requires developer or admin role). Use the period selector to view data for the last 7 days, 30 days, 90 days, or 12 months.

Traffic section

  • Pageviews & Unique Visitors — a line chart comparing total pageviews against unique visitor sessions over the selected period
  • Top pages — table listing each path with pageviews, unique visitors, and average time on page
  • Referrers — domains that sent traffic, with a traffic share percentage for each
  • Device breakdown — proportional split between desktop, mobile, and tablet visits

SEO health section

Outpost runs automated checks across all your pages and collection items and produces a score from 0 to 100. Checks include:

  • Missing meta descriptions
  • Meta titles that are too long (over 60 characters) or too short (under 10 characters)
  • Missing Open Graph images
  • Pages or items with very short content
  • Duplicate meta descriptions across pages
  • Collection items without excerpts

Results are grouped into Critical issues, Warnings, and Passing checks. Clicking a critical issue or warning shows the specific pages affected.

Content section

  • Publishing cadence — a chart showing how many collection items were published per week over time, useful for understanding content output consistency
  • Top content by pageviews — cross-references analytics hit data with your collection items to show which pieces of content are driving the most traffic

Members section

This section appears when member data is present in the database:

  • Member growth chart — total members, paid members, and free members over time
  • Tier breakdown — proportional split of members across membership tiers

GA4 integration

Outpost's built-in analytics and Google Analytics 4 can run in parallel. To add GA4, store your Measurement ID in a global field and inject the GA4 snippet conditionally in your theme's head.html partial.

Setup

  1. Create a global field named ga4_id in Admin → Globals.
  2. Enter your GA4 Measurement ID (e.g. G-XXXXXXXXXX) as the field value.
  3. Add the following to partials/head.html in your theme:
{% if @ga4_id %}
<script async src="https://www.googletagmanager.com/gtag/js?id={{ @ga4_id }}"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', '{{ @ga4_id }}');
</script>
{% endif %}

When @ga4_id is empty the block is omitted entirely, so you can leave the field blank on development sites without modifying your theme files.

Note — GA4 tracks in addition to Outpost's built-in analytics. Both systems record pageviews independently. Numbers will differ slightly because GA4 uses different bot filtering and session definitions.

Excluding system paths from reports

The analytics reports automatically exclude the following paths:

  • /outpost/ and all admin sub-paths
  • /sync-api (the builder sync endpoint)
  • Bare collection slug paths (e.g. /post) — only individual item URLs are counted

These exclusions are built into the report queries and cannot be configured.
