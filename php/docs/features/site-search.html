Site Search

Simple site-wide search across all pages and collection items. Drop in a template tag, no configuration needed.

Overview

Outpost includes a built-in search system that indexes all published pages and collection items. Add the <outpost-search> tag to any template and visitors get an instant search widget with debounced input, AJAX-powered results, and highlighted excerpts. No external search services, no JavaScript frameworks, no configuration.

Template tag

The simplest way to add search is the <outpost-search> custom element:

<outpost-search></outpost-search>

The tag accepts these optional attributes:

AttributeDefaultDescription
placeholderSearch…Placeholder text for the search input
collection(none)Restrict results to a single collection slug
limit20Maximum results to show (max 50)

Example usage

<!-- Basic search widget -->
<outpost-search placeholder="Search the site..."></outpost-search>

<!-- Restrict to a specific collection -->
<outpost-search collection="post" limit="10"></outpost-search>
Tip — Place the search tag in a partial (e.g. partials/nav.html) to make it available on every page, or add it to a dedicated search page template.

Custom HTML

For full control over the search markup, use data attributes instead of the custom element:

<div data-outpost-search data-limit="20">
  <input type="search" data-outpost-search-input placeholder="Search...">
  <div data-outpost-search-results></div>
</div>

The Outpost search client JS detects [data-outpost-search] containers and wires up the input and results automatically. You can style the container and input however you like.

How it works

Outpost maintains a search index table that stores stripped text from all published pages and collection items. When a visitor types a query:

  1. The client JS debounces input and sends a request to the search API.
  2. The API splits the query into words and uses AND logic — all words must match.
  3. Title matches are ranked higher than body matches.
  4. Results include the page/item URL, a text excerpt with <mark>-highlighted matching terms, and the featured image if available.
  5. Results are rendered into the results container as HTML.
Note — Search is case-insensitive. Partial word matching is supported — searching for “deploy” will match “deployment” and “deploying”.

Search API

The search endpoint is public (no authentication required):

GET /outpost/api.php?action=search/site&q=keyword&limit=20&offset=0
ParameterRequiredDescription
qYesSearch query string
limitNoMax results (default 20, max 50)
offsetNoPagination offset (default 0)

Response format:

{
  "results": [
    {
      "url": "/post/my-article",
      "title": "My Article",
      "excerpt": "...matched text with <mark>keyword</mark>...",
      "image": "/outpost/content/uploads/hero.jpg",
      "type": "collection_item",
      "collection": "post"
    }
  ],
  "total": 42,
  "query": "keyword"
}
Rate limit — The search API is rate limited to 30 requests per minute per IP. Exceeding this returns a 429 Too Many Requests response.

Search index

The search index is built and maintained automatically:

  • Auto-build — if the index is empty on the first search query, it builds automatically from all published content
  • Incremental updates — when a page or collection item is saved/published, its index entry is updated immediately
  • Manual rebuild — POST /outpost/api.php?action=search/reindex (admin authentication required) rebuilds the entire index from scratch

The index stores stripped plain text (no HTML) from titles, body content, and excerpt fields.

Search analytics

Every search query is logged to the analytics_searches table. You can view search analytics in the Analytics dashboard under the Search Analytics section. This shows:

  • Top search queries
  • Queries with zero results (content gaps)
  • Search volume over time
Tip — Use the “zero results” report to identify content your visitors are looking for but cannot find. This is a powerful signal for what content to create next.

URL state

Search queries are reflected in the ?q= URL parameter. This means:

  • Search results pages are shareable — copy the URL and send it to someone
  • On page load, if ?q= is present in the URL, the search runs automatically and results are displayed
  • Browser back/forward navigation works with search history

Styling

The search widget injects minimal CSS automatically. You can override any styles by targeting these classes in your site’s CSS:

SelectorDescription
.outpost-searchThe search widget container
.outpost-search-inputThe search input field
.outpost-search-resultsThe results container
.outpost-search-resultIndividual result item
.outpost-search-result-titleResult title
.outpost-search-result-excerptResult excerpt text
.outpost-search-result-excerpt markHighlighted matching terms within excerpts
.outpost-search-result-imageResult thumbnail image
.outpost-search-empty“No results found” message
Note — The injected CSS uses low-specificity selectors so your site styles take precedence automatically. You do not need !important to override them.
