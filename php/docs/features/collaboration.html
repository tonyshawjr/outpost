Collaboration & Comments

Team discussion on content and client feedback via shareable review links — all without leaving Outpost.

Overview

Outpost 3.3 adds two collaboration features:

  1. Team Comments — Leave threaded comments on collection items directly in the admin panel. @mention teammates, resolve threads when done, and reply to keep discussions organized.
  2. Client Review Links — Generate a shareable URL that injects a feedback overlay on your live site. Clients click elements to leave pinned comments — no admin account needed.

Team Comments

Leaving a comment

  1. Open a collection item. Navigate to any collection and click an item to edit it.
  2. Switch to the Comments tab. In the right sidebar, click the Comments tab (next to Post, SEO, and History).
  3. Type your comment. Use the textarea at the bottom. Type @username to mention a team member — an autocomplete dropdown appears.
  4. Press Cmd+Enter or click Comment. Your comment appears immediately.

Replying

Click Reply on any comment to add a threaded reply. Replies are nested below the parent comment.

Resolving threads

When a discussion is finished, click the Resolve button (green checkmark). Resolved comments fade out but are still visible — click Reopen to bring them back.

Filtering

Use the filter tabs at the top of the Comments panel to show All, Open, or Resolved comments.

Deleting

Comment authors and admins can delete comments. Click Delete on any comment you own (or any comment if you’re an admin).

Client Review Links

What are review links?

A review link is a special URL you share with clients. When they open it, your live site loads normally but with a lightweight feedback overlay. Clients can click any element on the page to leave a comment — all without logging in or creating an account.

This is perfect for:

  • Getting design feedback from clients before launch
  • Collecting content corrections (“change this headline”, “swap this photo”)
  • QA reviews with team members who don’t need admin access

Creating a review link

  1. Open Review Links. Click Review Links in the sidebar (under the admin section, visible to admins only).
  2. Click + New Review Link.
  3. Name it. Give it a descriptive name like “Client Review — Homepage”.
  4. Page scope (optional). Enter a specific page path (e.g., /about) to restrict the review to that page. Leave empty to allow feedback on any page.
  5. Expiry (optional). Set a date/time when the link automatically expires.
  6. Click Create Link. Copy the URL and share it with your client.

How the review overlay works

When someone visits your site with ?review=TOKEN in the URL:

  1. A “Leave Feedback” button appears in the bottom-right corner.
  2. Clicking it enters selection mode — hovering highlights elements with a blue outline.
  3. Clicking an element opens a feedback panel where they enter their name (first time only, saved in their browser) and type their comment.
  4. Submitted comments appear as numbered pins on the page, visible to everyone with the same review link.
  5. Clicking a pin shows the comment thread for that element.

The overlay script is lightweight (~8KB, vanilla JavaScript) and does not interfere with your site’s design or performance.

Managing review links

From the Review Links page you can:

  • Copy link — Copy the full URL to your clipboard.
  • Activate/Deactivate — Temporarily disable a link without deleting it. Deactivated links show “Invalid token” when accessed.
  • Delete — Permanently remove the link. Existing feedback remains in the database.

Viewing review feedback

All review comments appear alongside team comments in the admin panel. External reviewer comments are marked with an “External” badge so you can distinguish them from team discussions.

API Reference

Comments (authenticated)

MethodEndpointDescription
GETcommentsList comments. Params: entity_type, entity_id, page_path, status
POSTcommentsCreate a comment. Body: {entity_type, entity_id, body, parent_id, page_path}
PUTcomments?id=XUpdate comment body or status (open/resolved)
DELETEcomments?id=XDelete a comment (author or admin only)
GETcomments/countOpen comment counts grouped by entity
GETcomments/activityRecent comments across all content

Review Tokens (admin only)

MethodEndpointDescription
GETreview-tokensList all review tokens
POSTreview-tokensCreate a token. Body: {name, page_path, expires_at}
PUTreview-tokens?id=XToggle active/inactive
DELETEreview-tokens?id=XDelete a token

Public Review (token auth, no login required)

MethodEndpointDescription
POSTreview/commentSubmit feedback. Body: {token, author_name, body, page_path, element_selector}
GETreview/commentsList feedback. Params: token, page_path

Database Tables

Three tables are created automatically on first use:

  • comments — Stores all comments (team and external). Supports threading via parent_id, entity targeting (entity_type/entity_id), and element pinning (element_selector).
  • review_tokens — Shareable review links with name, page scope, expiry, and active flag.
  • comment_mentions — Records @mentions for future notification delivery.
