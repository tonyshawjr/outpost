Editorial Workflow

Review and approval gates, an editorial calendar, and bulk scheduling for team content workflows.

Overview

Outpost's editorial workflow adds structure to team content management. It consists of three features that work together:

  • Review & Approval — require admin approval before editors can publish content
  • Editorial Calendar — a month-grid view of scheduled and published items across collections
  • Bulk Scheduling — schedule multiple items to a specific date/time at once

Review & Approval

Enable Require Review on any collection to add an approval gate. When enabled, editors who click "Publish" will have their item set to pending_review instead of published. An admin must then approve the item before it goes live.

Enabling review

  1. Go to Admin → Collections.
  2. Click the edit icon on the collection you want to gate.
  3. Check the Require review before publishing checkbox.
  4. Click Save.
Note — Only admins, super_admins, and developers can toggle the Require Review setting. Editors cannot change it, which prevents them from bypassing their own review gate.

How it works

User roleBehavior on a review-required collection
EditorClicking "Publish" sets status to pending_review. The button label shows "Submit for Review".
Admin / Super AdminCan publish directly (no gate). Can also approve or reject pending items.
DeveloperSame as Admin — can publish directly and approve/reject.

Approving and rejecting items

Admins can approve or reject pending items in two places:

  • Item editor — open a pending item and click Approve or Reject in the top bar.
  • Collection items list — filter by "Pending", select items, and use the Approve bulk action.

Approving sets the item to published with a published_at timestamp. Rejecting returns it to draft.

Pending review status

Items with pending_review status are treated identically to drafts on the front end:

  • Not included in collection loops ({% for %})
  • Not accessible via direct URL
  • Not included in the sitemap

In the admin, pending items show an amber status indicator and appear in the "Pending" filter tab and sidebar count.

Scheduling on review-required collections

Editors cannot schedule items on collections with Require Review enabled. Since scheduled items auto-publish via the lazy check, allowing editors to schedule would bypass the review gate entirely. Only admins can schedule items on review-required collections.

Editorial Calendar

The calendar provides a month-grid view of all scheduled and published items across your collections. Access it from Admin → Calendar in the sidebar.

  • Items are plotted by their published_at or scheduled_at date
  • Color-coded dots indicate status: green (published), blue (scheduled), amber (pending review)
  • Click any item pill to open it in the collection editor
  • Use the collection filter dropdown to focus on a single collection
  • Navigate between months with the prev/next arrows or jump to today

Bulk Scheduling

Schedule multiple items at once from the collection items list:

  1. Go to Admin → Collections → your-collection.
  2. Select the items you want to schedule using the checkboxes.
  3. Click Schedule in the bulk actions bar.
  4. Pick a future date and time in the modal.
  5. Click Schedule to confirm.

All selected items will be set to scheduled with the chosen timestamp. They will auto-publish when the time arrives.

Webhook events

The editorial workflow fires these webhook events:

EventWhen
entry.submitted_for_reviewAn editor submits an item for review (status set to pending_review)
entry.approvedAn admin approves a pending item
entry.rejectedAn admin rejects a pending item
entry.scheduledAn item is scheduled (via individual or bulk scheduling)

API endpoints

EndpointMethodDescription
items/approvePUTApprove pending items. Body: { "ids": [1,2,3] }. Admin+ only.
items/rejectPUTReject pending items back to draft. Body: { "ids": [1,2,3] }. Admin+ only.
items/bulk-schedulePUTSchedule items. Body: { "ids": [1,2,3], "scheduled_at": "2026-04-01T09:00:00" }
calendarGETItems by date range. Params: start, end (YYYY-MM-DD), optional collection (slug filter).
