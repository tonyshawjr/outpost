WordPress Import

Migrate posts from an existing WordPress site into Outpost using WordPress's native XML export format. Posts become collection items, preserving titles, content, excerpts, slugs, and publish dates.

How it works

Outpost reads WordPress eXtended RSS (WXR) — the XML format produced by WordPress's built-in Export tool. Each post in the XML file is mapped to a collection item in a collection of your choice. The import is non-destructive by default: existing items with matching slugs are skipped unless you choose to overwrite them.

Step 1 — Export from WordPress

  1. Log in to your WordPress admin panel.
  2. Go to Tools → Export.
  3. Select Posts (or "All content" — only posts will be imported by Outpost; pages and other content types in the file are ignored).
  4. Click Download Export File.

You will receive a .xml file. This is the file you upload to Outpost.

Tip — For very large WordPress sites (thousands of posts), use a plugin like "Export Media Library" if you also need to migrate images. Outpost's importer handles the post content; images referenced in the HTML body will still point to your old WordPress domain until you update them manually.

Step 2 — Import into Outpost

  1. Navigate to Admin → Import (gear icon in the sidebar, or go to /outpost/#import).
  2. Upload the .xml WXR file.
  3. Select the target collection — the collection where imported posts will be created as items.
  4. Choose your import options:
    • Posts to import — "Published only" imports only posts with publish status; "All" also imports drafts.
    • On duplicate — "Skip" keeps existing items untouched; "Overwrite" replaces existing items that match by slug.
  5. Click Import.

When the import completes, a results summary shows how many items were imported and how many were skipped.

Warning — The target collection must already exist before importing. Create your collection in Admin → Collections first, then run the import.

What gets imported

WordPress field Outpost field Notes
Post title title Plain text
Post content body Stored as HTML (richtext field)
Post excerpt excerpt Plain text
Post slug Item slug Auto-sanitized to lowercase alphanumeric and hyphens
Published date published_at ISO 8601 timestamp
Post status (publish) published Other statuses map to draft

What does not import

  • Categories and tags — folder data is not mapped. You will need to assign folders to imported items manually in the Outpost admin.
  • Featured images — image URLs referenced in the post content body are preserved as-is, but image files are not downloaded from the old WordPress server. You will need to download and re-upload images to Outpost's Media library, then update the references in your content.
  • Authors — post author data is not imported.
  • Comments — not imported.
  • WordPress metadata and custom fields — ACF fields, post meta, and plugin-specific data are not imported.
  • WordPress Pages — only posts are imported, not WordPress Pages, custom post types, or media attachments.
Note — WordPress's block editor (Gutenberg) stores post content as HTML with embedded block comment markers (<!-- wp:paragraph --> etc.). These comments are harmless but will be present in the imported body field. They do not affect rendering in Outpost's richtext output.

Duplicate handling

Items are matched by slug. If a collection item with the same slug already exists in the target collection:

  • Skip — the existing Outpost item is left exactly as-is. The import count shows it as "skipped".
  • Overwrite — the existing item's title, body, excerpt, and published_at are replaced with the WordPress version. Status, folders, and any Outpost-specific fields not present in the WordPress export are preserved.
Warning — The "Overwrite" option replaces content without a confirmation prompt. Run with "Skip" first to see what would be affected, then run again with "Overwrite" if needed.

After importing

Review imported items in Admin → Collections → select your collection. Things to check after a large import:

  • Scan for formatting issues in richtext content — WordPress HTML may include plugin-specific shortcodes or non-standard markup that won't render correctly.
  • Update image URLs in the body content to point to images you've re-uploaded to Outpost's Media library.
  • Assign categories, tags, or other folders to imported items.
  • Review items imported from draft status and decide whether to publish them.
