Grammar & Spelling

Inline grammar and spelling suggestions in the rich-text editor, powered by LanguageTool. Toggle the check on, and mistakes are underlined where you write — click one to see the fix and apply it in a single tap. Your text is proxied through Outpost so any API key stays server-side.

Overview

Grammar & Spelling adds a proofreading pass to the post/rich-text editor. When you switch it on, Outpost sends the editor’s plain text to a LanguageTool server, gets back a list of issues, and draws a wavy underline under each one. Clicking an underline opens a small popover with the explanation and one-click replacement buttons.

Everything runs through a small PHP proxy (grammar.php). The browser never talks to LanguageTool directly — the server URL and any credentials are configured by an admin and, in the case of a Premium key, encrypted at rest. By default Outpost uses LanguageTool’s free public API, so there is nothing to install to get started.

Off by default. Grammar checking is not running until you toggle it on in the editor toolbar. Outpost never sends your draft anywhere unless you turn the check on for that editor.

Where to find it

  1. Open any content with a rich-text body — a page or a collection item’s rich-text field.
  2. In the editor toolbar, find the Check grammar & spelling button (the spell-check icon at the far right of the toolbar).
  3. Click it to toggle checking on. The button highlights while it is active.
  4. Mistakes are underlined as you write. Outpost re-checks automatically a short moment after you stop typing (debounced), so you don’t get flicker on every keystroke.
  5. Click the button again to turn checking off — all underlines clear immediately.

The toggle is per-editor and starts in the off position every time the editor loads.

How it works

Underlines by issue type

Each suggestion is drawn as a colored wavy underline, classed by the kind of issue LanguageTool reported:

UnderlineIssue typeWhat it means
lt-spellMisspellingA word LanguageTool doesn’t recognize.
lt-grammarGrammarAgreement, tense, punctuation, and similar rule violations.
lt-styleStyleWordiness, redundancy, or phrasing the style rules flag.

The underlines are drawn as editor decorations, so they never become part of your saved content — they are purely visual hints layered over the text.

Click-to-fix popover

Click any underlined span while checking is on and a popover appears anchored to the mistake. It shows:

  • The message — LanguageTool’s explanation of the issue.
  • One button per suggested replacement (up to eight). Click one and Outpost replaces exactly the flagged span with your choice — the surrounding text is untouched.
  • A Dismiss button to close the popover without changing anything. If LanguageTool offered no replacement, the popover says so.

After you apply a fix, that underline disappears and Outpost re-checks shortly after, so any downstream issues stay current.

Tip — Language is auto-detected. Outpost sends auto as the language, so LanguageTool figures out what you’re writing in without you picking a locale.

The free public API vs. self-hosting

Grammar checking needs a LanguageTool server. You have two choices, and switching between them is a single field.

Free public API (default)

Leave the Server URL blank and Outpost uses LanguageTool’s public endpoint at https://api.languagetool.org/v2/check. Nothing to install — it just works. This is the right choice for occasional proofreading on a single site.

Self-hosted (for heavy or private use)

Run your own LanguageTool server and point Outpost at it. The API is identical, so nothing else changes — you only swap the URL. The simplest way is the official Docker image:

docker run -d -p 8010:8010 erikvl87/languagetool

Then set the Server URL to your instance’s check endpoint, for example:

http://localhost:8010/v2/check

Self-hosting keeps your text on your own infrastructure and removes the public rate limit — the right choice for a busy editorial team or any content you’d rather not send to a third party.

Same API, one field. The public API and a self-hosted server speak the same protocol. Moving from one to the other is just editing the Server URL in Settings — no code changes, no re-configuration.

Optional LanguageTool Premium

If you have a paid LanguageTool account, you can supply Premium credentials for higher limits and extra rules. Under Premium (optional) in the settings block, add:

  • Username — your LanguageTool account email.
  • API Key — your Premium API key.

When both are present, Outpost includes them on the request to the configured host. The API key is encrypted at rest (Ranger crypto) and is only ever sent server-side — it never reaches the browser. Leave these blank if you’re not a Premium subscriber; they aren’t needed for the free public API or a self-hosted server.

Rate limits

The LanguageTool public API is rate-limited — roughly 20 requests per minute per IP. Because Outpost debounces checks (it waits until you pause typing), normal editing rarely hits this. If you do exceed it, the check surfaces a clear message:

Grammar check rate limit reached — try again shortly or connect a self-hosted server. If you proofread constantly or have multiple editors working at once, self-host LanguageTool (see above) to remove the limit entirely.

Text is capped at 20 KB per check; anything longer is truncated before it’s sent. Very long documents are checked up to that limit.

Configuring it

Go to Settings → Integrations → Grammar & Spelling:

  • Server URL — blank for the free public API, or your self-hosted check endpoint. It must start with http:// or https://.
  • Premium Username and API Key — only if you have a LanguageTool Premium account.

Reading and writing these settings requires the settings.* capability. The stored server URL and username are shown back to you so you can confirm or update them; the Premium key is decrypted on demand server-side.

Endpoints

Both routes are on the admin API (api.php?action=…). Checking requires the content.* capability; reading and writing settings require settings.*.

RouteMethodDescription
grammar/checkPOSTCheck text. Body: { text, language } (language defaults to auto; text capped at 20 KB). Returns { matches: […] }.
grammar/settingsGET / PUTRead or write the LanguageTool configuration. GET returns the server URL, username, and (decrypted) key. PUT accepts languagetool_server_url, languagetool_username, and languagetool_apikey (stored encrypted).

Match shape

Each item in matches is normalized to the same shape, regardless of which LanguageTool server produced it:

{
  "offset": 12,              // start position in the plain text
  "length": 7,               // length of the flagged span
  "message": "Possible spelling mistake found.",
  "shortMessage": "Spelling",
  "replacements": ["writing", "waiting"],   // up to 8
  "ruleId": "MORFOLOGIK_RULE_EN_US",
  "category": "Possible Typo",
  "issueType": "misspelling"  // drives the underline color
}

The editor maps each match’s offset/length from plain-text coordinates back to positions in the document to draw the underline in the right place, and issueType selects the lt-spell / lt-grammar / lt-style class.

Privacy & security

  • Server-side proxy — text and credentials are sent from Outpost’s server, never from the browser. The LanguageTool URL and key are not exposed to page source.
  • Encrypted Premium key — the API key is stored encrypted (Ranger crypto) in the settings table and decrypted only when a request is made.
  • Opt-in — checking is off until you toggle it, so no draft leaves your server unless you ask for a check.
  • Self-host for full privacy — point the Server URL at your own instance to keep every check on your own infrastructure.
