Compass

Smart filtering, search, and faceted navigation for any collection. You write your own HTML with your own classes and layout. Add data-compass attributes to make elements smart. Compass handles the filtering behavior — you control everything visual.

What is Compass?

Compass is Outpost’s built-in filtering and search system. It turns any collection into a searchable, filterable directory — business listings, event calendars, product catalogs, job boards, member directories — without writing PHP or JavaScript.

Compass is invisible. There are no forced layouts, no generated wrappers, no opinionated markup. You write standard HTML with your own classes and structure. Then you add data-compass attributes to tell Compass what each element does. Compass wires up the behavior — event listeners, API calls, result rendering, URL sync — automatically.

What Compass handles:

  • Text search across multiple fields
  • Dropdown, checkbox, radio, and toggle filters from folder labels or field values
  • Range sliders for numeric fields (price, rating, distance)
  • A–Z alphabetical filtering
  • Proximity / geolocation filtering (find businesses near me)
  • Pagination and sorting
  • URL state sync — every filter selection is reflected in the URL for sharing and bookmarking
  • Result count, active filter pills, and reset

Use cases:

  • Business directories with category, city, and keyword search
  • Event calendars filtered by date, type, and location
  • Job boards with role type, salary range, and location filters
  • Product catalogs with price range, brand, and attribute filters
  • Real estate listings with price, bedrooms, and proximity search
  • Member directories with alphabetical browsing
  • Resource libraries with tag and date filtering

Quick Start

The simplest Compass setup is a search input, a dropdown, and a results container. Write your own HTML, add data-compass attributes, and Compass does the rest:

<input type="text" data-compass="search" data-collection="posts" placeholder="Search...">

<select data-compass="dropdown" data-collection="posts" data-source="folder:categories">
  <option value="">All Categories</option>
</select>

<div data-compass="results" data-collection="posts">
  <outpost-each collection="posts" limit="12">
    <outpost-include partial="post-card"></outpost-include>
  </outpost-each>
</div>

That’s it. The search input and dropdown are your elements — your classes, your layout, your styles. Compass discovers them via data-compass, groups them by data-collection, and wires everything together.

All Data Attributes

Every Compass element is a standard HTML element with a data-compass attribute that declares its role.

AttributeElementDescription
data-compass="search"<input>Text search, debounced 300ms.
data-compass="dropdown"<select>Single-select filter. Auto-populates options if the <select> is empty (aside from a placeholder <option>). Automatically upgrades to a searchable dropdown when 15+ options are present (configurable).
data-compass="checkbox"<div>Multi-select checkboxes. Auto-fills checkbox inputs from the data source.
data-compass="radio"<div>Single-select radio buttons. Auto-fills radio inputs from the data source.
data-compass="range-min"<input type="range">Low end of a numeric range filter.
data-compass="range-max"<input type="range">High end of a numeric range filter.
data-compass="az"<div>A–Z letter buttons. Auto-generated inside the container.
data-compass="toggle"<input type="checkbox">Boolean label/field filter (e.g., “Featured only”).
data-compass="proximity"<button>Geolocation “near me” trigger.
data-compass="results"<div>Results container. Content is replaced when filters change.
data-compass="count"anyText content replaced with the current result count.
data-compass="pager"<div>Pagination buttons, auto-generated inside the container.
data-compass="reset"<button>Clears all active filters.
data-compass="selections"<div>Active filter pills with remove buttons.
data-compass="sort"<select>Sort order selector.
data-compass="submit"<button>Triggers the filter on click (instead of live filtering).

These attributes configure how each Compass element behaves.

Source Types

The data-source attribute determines where filter options come from. There are three source types:

SourceSyntaxDescriptionExample
Folder labels folder:slug Options come from the labels in a collection folder. Labels are managed in the admin under the collection’s folder settings. data-source="folder:categories"
Field values field:fieldname Options are auto-populated from distinct values of a field across all items. Ideal for fields like “city” or “type” where the values aren’t predefined. data-source="field:city"
Label group label:slug A single label used as a toggle filter. Useful for flags like “open-late” or “featured”. data-source="label:open-late"

How It Works

  1. Discovery — Compass JS finds all [data-compass] elements on the page.
  2. Grouping — Elements are grouped by their data-collection value into filter sets.
  3. Binding — Appropriate event listeners are attached (change, input, click) based on the element type.
  4. Filtering — When any filter changes, Compass fetches results from the compass/filter API endpoint.
  5. Rendering — The results container, count, pager, and selection pills are updated.
  6. URL sync — Filter state is pushed to the URL query string for sharing and bookmarking.

Examples by Filter Type

Search

A plain text input. Compass debounces it at 300ms and searches across the fields you specify.

<input
  type="text"
  data-compass="search"
  data-collection="business"
  data-fields="title,description,address"
  placeholder="Search businesses..."
>

Dropdown

A standard <select>. If you include only a placeholder <option>, Compass auto-populates the rest from the data source. Or write your own options for full control.

<!-- Auto-populated from folder labels -->
<select data-compass="dropdown" data-collection="business" data-source="folder:categories">
  <option value="">All Categories</option>
</select>

<!-- Auto-populated from field values -->
<select data-compass="dropdown" data-collection="business" data-source="field:city">
  <option value="">All Cities</option>
</select>

Searchable Dropdowns

When a dropdown has 15 or more options, Compass automatically upgrades it to a searchable dropdown — a custom widget with a type-to-filter input, scrollable option list, and keyboard support. The original <select> is hidden and kept in sync.

You can control this behavior with the data-searchable attribute:

AttributeBehavior
(no attribute)Auto-upgrade when 15+ options (default)
data-searchable="true"Always upgrade, even with few options
data-searchable="false"Never upgrade, always plain <select>
data-searchable-threshold="20"Custom threshold (upgrade at 20+ instead of 15)
 
<!-- Force searchable on a small list -->
<select data-compass="dropdown" data-collection="business"
        data-source="folder:categories" data-searchable="true">
  <option value="">All Categories</option>
</select>

<!-- Keep as plain select even with many options -->
<select data-compass="dropdown" data-collection="business"
        data-source="field:city" data-searchable="false">
  <option value="">All Cities</option>
</select>
Note — The searchable dropdown inherits the width and positioning of the original <select>. All CSS classes on the <select> are copied to the wrapper. Override styles by targeting .compass-searchable-dropdown, .compass-sd-trigger, .compass-sd-panel, and .compass-sd-item.

Checkbox

A container <div>. Compass fills it with labeled checkbox inputs from the data source.

<div data-compass="checkbox" data-collection="business" data-source="folder:amenities"></div>

Radio

A container <div>. Compass fills it with labeled radio inputs from the data source.

<div data-compass="radio" data-collection="event" data-source="field:type"></div>

Range

Two separate range inputs — one for the minimum, one for the maximum. Style and position them however you like.

<label>Min Price</label>
<input type="range" data-compass="range-min" data-collection="product" data-source="field:price" min="0" max="500" step="10">

<label>Max Price</label>
<input type="range" data-compass="range-max" data-collection="product" data-source="field:price" min="0" max="500" step="10">

A–Z

A container <div>. Compass auto-generates letter buttons inside it.

<div data-compass="az" data-collection="business"></div>

Toggle

A plain checkbox input. Compass uses it as a boolean filter tied to a field or label.

<label>
  <input type="checkbox" data-compass="toggle" data-collection="business" data-source="label:open-late">
  Open Late
</label>

<label>
  <input type="checkbox" data-compass="toggle" data-collection="business" data-source="field:featured">
  Featured Only
</label>

Proximity

A button that triggers geolocation. When clicked, Compass uses the browser’s Geolocation API to find nearby items.

<button data-compass="proximity" data-collection="business" data-radius="25" data-unit="miles">
  Near Me
</button>

Sort

A standard <select> with your own sort options.

<select data-compass="sort" data-collection="business">
  <option value="title:asc">A–Z</option>
  <option value="title:desc">Z–A</option>
  <option value="created_at:desc">Newest</option>
  <option value="rating:desc">Top Rated</option>
</select>

Results

A container whose content is replaced with filtered results. The inner HTML serves as the initial content (rendered server-side by Outpost’s template engine) and is swapped when filters change.

<div data-compass="results" data-collection="business">
  <outpost-each collection="business" limit="12">
    <article class="card">
      <img data-outpost="featured_image" data-type="image" src="placeholder.jpg">
      <h3 data-outpost="title">Business Name</h3>
      <p data-outpost="description">Description</p>
      <a data-bind="href:url">View Details</a>
    </article>
  </outpost-each>
</div>

Count, Pager, Selections, Reset

<!-- Result count -- text content is replaced with the number -->
<span data-compass="count" data-collection="business" data-compass-template="{n} businesses found">0 results</span>

<!-- Pagination -- buttons auto-generated inside -->
<div data-compass="pager" data-collection="business" data-per-page="12"></div>

<!-- Active filter pills with remove buttons -->
<div data-compass="selections" data-collection="business"></div>

<!-- Reset all filters -->
<button data-compass="reset" data-collection="business">Clear all filters</button>

<!-- Manual submit trigger (disables live filtering) -->
<button data-compass="submit" data-collection="business">Apply Filters</button>

Legacy Custom Element Syntax

The <outpost-compass> custom element tags from earlier versions still work. They compile to the same data-compass HTML at render time. Use them for quick prototyping; use raw data attributes for full control over your markup.

<!-- These two are equivalent -->
<outpost-compass type="search" collection="business" fields="title,description" placeholder="Search..."></outpost-compass>

Styling

Compass does not generate wrapper elements around your HTML. Your inputs, selects, buttons, and layout divs keep their original markup and classes. Compass only injects minimal CSS for auto-generated content inside containers:

  • Checkbox and radio labels inside data-compass="checkbox" / data-compass="radio" containers
  • Letter buttons inside data-compass="az" containers
  • Page buttons inside data-compass="pager" containers
  • Filter pills inside data-compass="selections" containers

Override auto-generated styles by targeting the Compass CSS classes in your stylesheet:

/* Override Compass accent color */
.compass-az-btn.compass-az-active,
.compass-pager-btn.compass-pager-active { background: #3b82f6; border-color: #3b82f6; }
.compass-pill { background: #eff6ff; color: #1d4ed8; }
.compass-az-btn:hover,
.compass-pager-btn:hover { border-color: #3b82f6; }

/* Override spacing */
.compass-checkbox, .compass-radio { gap: 12px; }

/* Override empty message style */
.compass-empty { font-size: 16px; color: #666; }

Everything else — your search input, your dropdowns, your grid layout, your card styles — is 100% yours.

API Endpoints

Compass has five API endpoints. You typically don’t call these directly — the JS handles them — but they are available for headless usage or custom JavaScript integrations.

GET compass/values

GET api.php?action=compass/values&collection=slug&source=folder:categories

Returns the available filter options for a given source. Used to populate dropdown, checkbox, and radio filters.

Response:

{
  "values": [
    { "slug": "restaurants", "name": "Restaurants", "count": 24 },
    { "slug": "retail", "name": "Retail", "count": 18 },
    { "slug": "services", "name": "Services", "count": 31 }
  ]
}

GET compass/filter

GET api.php?action=compass/filter&collection=business&q=pizza&fields=title,description&categories=restaurants&city=jacksonville&sort=title:asc&page=1&per_page=12

Executes a filter query and returns matching items. All filter parameters are passed as query string parameters. The Compass JS client builds this URL automatically from the current filter state.

Query parameters:

ParameterDescription
collectionCollection slug (required)
per_pageResults per page (default: 12)
pagePage number
qSearch query text
fieldsComma-separated fields to search across
sortSort order, e.g. title:asc
partialPartial template for server-side rendering
facet namesFilter values keyed by facet name (e.g. categories=restaurants, city=jacksonville). Multi-values are comma-separated.

Response:

{
  "html": "<article>...</article>",
  "total": 48,
  "facets": {
    "categories": [
      { "value": "restaurants", "count": 8 }
    ]
  }
}

GET compass/proximity

GET api.php?action=compass/proximity&collection=slug&lat=34.75&lng=-77.43&radius=25&unit=mi

Returns items within a geographic radius, sorted by distance.

Response:

{
  "items": [
    { "id": 1, "title": "Joe's Pizza", "distance": 1.2, "unit": "mi", ... },
    { "id": 2, "title": "Main Street Deli", "distance": 3.8, "unit": "mi", ... }
  ],
  "total": 15
}

GET compass/hierarchy

GET api.php?action=compass/hierarchy&collection=slug&source=folder:categories

Returns nested category tree structure with counts.

POST compass/reindex

POST api.php?action=compass/reindex&collection=slug

Rebuilds the Compass search index for a collection. Requires admin authentication. The index rebuilds automatically on item create/update/delete, but this endpoint forces a full rebuild.

Response:

{
  "success": true,
  "indexed": 156,
  "collection": "business"
}

URL State

Every active filter is reflected in the URL query string. Filtered views are shareable and bookmarkable.

/explore?q=pizza&category=restaurants&city=jacksonville&sort=rating:desc
ParameterFilter TypeExample

Compass reads URL parameters on page load and pre-populates filters accordingly. When a visitor changes a filter, the URL updates without a full page reload (using history.pushState).

Indexing

Compass maintains a lightweight search index for each collection that has Compass filters. The index is stored in the SQLite database (compass_index table) and rebuilds automatically when:

  • A collection item is created, updated, or deleted
  • A folder label is added, renamed, or removed
  • The admin triggers a manual reindex via compass/reindex

The first page load with Compass elements triggers an initial index build if one doesn’t exist. Subsequent updates are incremental (only re-indexing the changed item). Sub-50ms queries on 10k+ items.

Browse vs. Search Mode

Many explore/directory pages have two modes: a browse mode (featured content, carousels, category grid) and a search mode (filtered results). Compass handles this with a simple pattern:

  1. Mark browse sections with data-default-section and the results section with data-compass-section
  2. Hide the results section by default: style="display:none;"
  3. Toggle between them using the compass:stateChange event
  4. Check URL params on page load so direct links like /explore?q=pizza start in search mode immediately — no flash of browse content
<!-- Browse mode (visible by default) -->
<section data-default-section>
  <h2>Featured Businesses</h2>
  <!-- carousels, cards, etc. -->
</section>

<!-- Search mode (hidden until filters are active) -->
<section data-compass-section style="display:none;">
  <span data-compass="count" data-collection="business">0 results</span>
  <div data-compass="results" data-collection="business">
    <template data-compass-template>
      <a href="/business/{{slug}}">{{title}}</a>
    </template>
  </div>
</section>

<!-- Toggle script -->
<script>
(function() {
  function toggleMode(hasFilters) {
    document.querySelectorAll('[data-default-section]').forEach(function(el) {
      el.style.display = hasFilters ? 'none' : '';
    });
    var results = document.querySelector('[data-compass-section]');
    if (results) results.style.display = hasFilters ? '' : 'none';
  }

  // Check URL on load for direct search links
  var params = new URLSearchParams(window.location.search);
  var hasUrlFilters = false;
  params.forEach(function(val) { if (val) hasUrlFilters = true; });
  if (hasUrlFilters) toggleMode(true);

  // React to Compass filter changes
  document.addEventListener('compass:stateChange', function(e) {
    var state = e.detail && e.detail.state ? e.detail.state : {};
    var hasFilters = Object.keys(state).some(function(k) {
      var v = state[k];
      return v && v !== '' && (!Array.isArray(v) || v.length > 0);
    });
    toggleMode(hasFilters);
  });
})();
</script>
Tip — The data-default-section and data-compass-section attributes are not Compass features — they are conventions you define in your site. The toggle script is yours to write however you want. Compass provides the compass:stateChange event; you decide what to do with it.

Client-Side Result Templates

When Compass filters via AJAX, it replaces the results container with new content. If your server doesn’t return pre-rendered HTML (most setups), add a <template data-compass-template> inside your results container. Compass uses it to render each item using {{field}} placeholders:

<div data-compass="results" data-collection="business" class="grid grid-cols-3 gap-6">
  <template data-compass-template>
    <a href="/business/{{slug}}" class="card">
      <h3>{{title}}</h3>
      <p>{{city}}, {{state}}</p>
      <p>{{description}}</p>
    </a>
  </template>

  <!-- Server-rendered initial content (replaced when filtering) -->
  <outpost-each collection="business" limit="12">
    <outpost-include partial="business-card"></outpost-include>
  </outpost-each>
</div>

Available placeholders: any field from the item’s data JSON ({{title}}, {{city}}, {{description}}, etc.), plus {{slug}}, {{id}}, and {{url}} (auto-generated from the collection URL pattern). All values are HTML-escaped.

Note — The <template> tag is cached by Compass on initialization, so it survives when innerHTML is replaced during filtering. You write the card markup once in the template and Compass stamps it out for each result.
