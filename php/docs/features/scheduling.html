Scheduling

Set a future publish date on any collection item and Outpost will automatically make it public at the right time — no cron job or server configuration required.

How scheduling works

Outpost uses an automatic publishing model. When you schedule an item, it is stored in the database with a status of scheduled and a scheduled_at timestamp. No background process or server cron job is needed. On every incoming page request, Outpost automatically checks whether any scheduled items have a scheduled_at time in the past — and if so, promotes them to published on the spot.

The check is throttled to run at most once per 60 seconds (using a file-based lock), so it adds no measurable overhead to page loads. On a site with regular traffic, items typically publish within a minute of the scheduled time.

Note — Scheduling is available for collection items only. Pages use published/draft status without a scheduled option.

Scheduling an item

  1. Open a collection item in Admin → Collections → your-collection → item.
  2. In the Status field (in the right sidebar), select Scheduled.
  3. A date and time picker appears below the status selector. Set your desired publish date and time.
  4. Click Save.

The item is now saved with status = 'scheduled' and the scheduled_at timestamp you chose. It is immediately invisible to public visitors. You can still preview it on the live site by clicking Preview in the item editor — this generates a secure token URL that bypasses draft blocking.

Tip — The date/time picker uses the server's configured time zone. Check your server's time zone setting (see Time zones below) before scheduling time-sensitive content.

How scheduled items behave

While an item has status = 'scheduled', Outpost treats it as unpublished everywhere:

Context Scheduled item behaviour
Collection loops ({% for %}) Not included — excluded by status = 'published' filter
Direct URL access (e.g. /post/my-slug) 404 — the router blocks items that are not published
Sitemap (/sitemap.xml) Not included
Admin collection list Shown with a clock icon and the scheduled date
Admin item editor Fully editable — you can change the date or unpublish

The auto-publish check

On every front-end page request, outpost_maybe_auto_publish() runs after outpost_init(). It uses a file-based throttle (cache/cron_last_run.txt) to ensure the actual database check runs at most once per 60 seconds:

-- SQL run at most once per 60 seconds (simplified)
SELECT id FROM collection_items
WHERE status = 'scheduled'
  AND scheduled_at <= NOW();

-- If any rows found, update them:
UPDATE collection_items
SET status = 'published', published_at = NOW()
WHERE status = 'scheduled'
  AND scheduled_at <= NOW();

This query is lightweight — it runs only when at least one scheduled item exists and exits immediately when there is nothing to promote. The template cache is automatically cleared when items are published.

Note — The auto-publish check fires on front-end page requests, not on admin-only requests. If your site has zero public traffic for an extended period, a scheduled item will publish on the first public page load after its scheduled time.

Unscheduling an item

Open the item in the admin and change the status in the sidebar:

  • Change to Published to publish the item immediately, regardless of the scheduled date.
  • Change to Draft to unpublish the item and remove it from the schedule entirely.

Saving with either status clears the scheduled_at value.

Time zones

Scheduled times are stored as UTC in the database and compared against the server's current time. The date/time picker in the admin displays and accepts times in your server's configured time zone.

To set the server time zone explicitly, add the following to your outpost/config.php:

<?php
// config.php
date_default_timezone_set('America/New_York');
// or
date_default_timezone_set('Europe/London');
// or
date_default_timezone_set('UTC');

You can also set the time zone globally via php.ini:

date.timezone = "America/Chicago"
Warning — If no time zone is configured, PHP defaults to UTC. Confirm your server's effective time zone before scheduling important content. The admin displays times as the server understands them — if the server is set to UTC and you are in a different time zone, the picker will show UTC times.

Database column

The scheduled_at field lives on the collection_items table alongside status:

Column Type Description
status TEXT published, draft, scheduled, or pending_review
scheduled_at TEXT (ISO 8601) The scheduled publish time; NULL when not scheduled
published_at TEXT (ISO 8601) Set to the current time when status changes to published

Status transition diagram

An item's status follows these allowed transitions:

From status To status How
draft published Manual publish in admin
draft pending_review Editor publishes on a collection with Require Review enabled
draft scheduled Set status to Scheduled + pick a date
pending_review published Admin approves the item
pending_review draft Admin rejects the item
scheduled published Automatic (lazy check) or manual override in admin
scheduled draft Manual change in admin (cancels the schedule)
published draft Manual unpublish in admin

Scheduling and collection loops

Collection loops always filter by status = 'published'. This means scheduled items never appear in your theme's {% for %} loops until after the lazy publish check promotes them. There is nothing special to add to your templates — the filtering is built into the loop handler.

{# This loop only shows published items. Scheduled items are excluded automatically. #}
{% for post in collection.post %}
  <article>
    <h2>{{ post.title }}</h2>
    <p>{{ post.excerpt }}</p>
  </article>
{% endfor %}

Advanced: external cron endpoint

For most sites, the built-in auto-publish is all you need. If your site has extremely low traffic and you need guaranteed timing, Outpost also provides a key-authenticated cron endpoint that can be called from an external cron job or monitoring service:

# Optional: external cron endpoint (key from Settings → Advanced)
*/5 * * * * curl -s "https://yourdomain.com/outpost/api.php?action=cron&key=YOUR_KEY" > /dev/null 2>&1

This endpoint also processes webhook retries and delivery cleanup in addition to scheduled publishing.

Tip — You do not need this for normal use. The auto-publish runs on every page load (throttled to once per minute) and handles scheduling automatically.

Bulk scheduling

Select multiple items in Admin → Collections → your-collection, then click Schedule in the bulk actions bar. A date and time picker appears — pick a future date and time, then confirm. All selected items are set to scheduled with the chosen timestamp.

Alternatively, you can use the API endpoint PUT api.php?action=items/bulk-schedule with a JSON body containing ids (array of item IDs) and scheduled_at (ISO 8601 datetime). The scheduled time must be in the future.

Note — On collections with Require Review enabled, editors cannot schedule items (since scheduled items auto-publish, bypassing the review gate). Only admins can schedule items on review-required collections.

Limitations

  • Scheduling is only available for collection items. Pages (Admin → Pages) use published/draft only.
  • Editors on review-required collections cannot schedule items — only admins can.
  • Republishing is not supported — you cannot schedule an already-published item to re-publish at a future time. Scheduling only transitions items from scheduled to published.
  • Unpublishing on a date is not supported — there is no equivalent "schedule to draft" feature.
  • The lazy check fires on front-end requests only. Idle sites with no visitors will not auto-publish until the next request arrives.
