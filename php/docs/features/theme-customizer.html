Theme Customizer

This feature has been removed in Outpost v5.0.

Removed in v5.0 — The Theme Customizer was removed in v5.0. CSS variables and visual customization are managed directly by developers in their CSS files.

What changed

In previous versions, themes could declare customizable properties (colors, fonts, logo, favicon) via a customizer key in theme.json. The admin panel provided a visual editor with live preview for these properties.

In v5.0, the theme concept has been removed entirely. Visual customization is now handled directly by developers editing their CSS files. Site-wide branding can still be configured through Brand Settings in the admin panel.

How to customize your site in v5.0

  • Colors and fonts — define CSS custom properties in your stylesheet at the site root (e.g. css/style.css) and change values there.
  • Logo and favicon — use global fields (data-outpost="site_logo" data-type="image" data-scope="global") in your partials and manage them from Admin → Globals.
  • Brand identity — use Brand Settings for site-wide brand configuration.
