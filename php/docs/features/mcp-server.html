MCP Server

Connect AI tools like Claude Desktop and ChatGPT to manage your Outpost site content directly via the Model Context Protocol.

Overview

The MCP (Model Context Protocol) server is a single PHP file (mcp.php) that lets AI assistants read and write your site content through a standardized JSON-RPC 2.0 protocol. It uses your existing API keys for authentication — no new setup required.

Once connected, an AI assistant can list your collections, create and edit posts, update page fields, manage globals, search content, and browse your media library — all through natural language.

Zero dependencies. The MCP server is pure PHP. No Composer packages, no npm, no build step. It reuses the same database queries and auth system as the REST API.

Setup

1. Create an API Key

Go to Settings → Integrations and create an API key. Copy the key — you’ll only see it once.

2. Configure Claude Desktop

Open your Claude Desktop config file and add the MCP server:

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%\Claude\claude_desktop_config.json
{
  "mcpServers": {
    "outpost-cms": {
      "url": "https://yoursite.com/outpost/mcp.php",
      "headers": {
        "Authorization": "Bearer op_your_api_key_here"
      }
    }
  }
}

Replace yoursite.com with your actual domain and op_your_api_key_here with your API key.

3. Restart Claude Desktop

After saving the config, restart Claude Desktop. The Outpost tools will appear in the tools list.

Tip: The connection info panel in Settings → Integrations shows a pre-filled config snippet you can copy directly.

Connect Claude Code (terminal)

Claude Code can connect to the same endpoint over HTTP and build pages from the terminal using the visual-builder tools. Add the server to your project’s .mcp.json:

{
  "mcpServers": {
    "outpost-builder": {
      "type": "http",
      "url": "https://yoursite.com/outpost/mcp.php",
      "headers": {
        "Authorization": "Bearer op_your_api_key_here"
      }
    }
  }
}

Or register it in one command:

claude mcp add --transport http outpost-builder https://yoursite.com/outpost/mcp.php \
  --header "Authorization: Bearer op_your_api_key_here"
To build pages: read the outpost://builder/guide resource first (the node model and operation vocabulary), then get_page_tree to read a page and apply_page_ops to change it — the same engine the in-app AI builder uses, so edits save and bake to static HTML.

Available Tools

The MCP server exposes tools covering content operations and the visual page builder.

Visual Builder (page building)

ToolDescription
get_page_treeRead a page’s visual-builder node tree plus its CSS class registry — the same tree the in-app builder edits. Returns a version for optimistic locking.
apply_page_opsApply an ordered list of build operations (insert_tree, update, set_classes, move, duplicate, remove, define_class, bind_field) to a page. Validates, saves, and bakes to static HTML. Pass expect_version to avoid clobbering concurrent edits.
get_stylesList the site’s CSS class registry with declarations and usage counts.
get_design_tokensRead the site’s design tokens (color scale, fluid type and spacing scales) so builds stay on-brand.

add_block_to_page / set_block_fieldAdd a block to a page and set its field values.

Authentication

The MCP server uses the same API keys as the REST API. Create keys in Settings → Integrations.

  • Keys are sent via Authorization: Bearer op_xxx... header
  • The key must belong to a user with an admin role
  • The initialize method is the only unauthenticated call (required by MCP protocol)
  • All tool and resource calls require a valid key

Security

  • Payload limit — 1 MB maximum request body. Larger payloads are rejected before parsing.
  • Rate limiting — Mutation tools (create, update, delete) are rate-limited using the same IP-based limiter as the REST API.
  • Input sanitization — All inputs validated. Slugs must match [a-z0-9-]. Richtext fields run through the HTML sanitizer. Search queries escape LIKE wildcards.
  • Error redaction — Error messages never expose file paths or stack traces. Full errors logged server-side only.
  • CORS — Same origin policy as the REST API. Configurable via api_cors_origins setting.
  • Revisions — All write operations create revision snapshots for rollback.

Example Prompts

Once connected, try prompts like:

  • “What collections does my site have?”
  • “Create a new blog post about our spring sale”
  • “Update the site name to Acme Corp”
  • “Show me all draft posts”
  • “Search for posts mentioning SEO”
  • “What fields does the homepage have?”

With Claude Code connected over HTTP, you can also build page layouts:

  • “Read the builder guide, then add a three-column feature grid to the /about page”
  • “Build a hero section on the homepage with a headline, subhead, and a call-to-action button”
  • “Add a pricing section to /pricing using the site’s design tokens”

Technical Details

  • Protocol: MCP 2025-03-26, JSON-RPC 2.0
  • Transport: Streamable HTTP (POST for messages, DELETE for session termination)
  • File: outpost/mcp.php — single file, ~850 lines, zero dependencies
  • Session: Mcp-Session-Id header generated on initialize. Auth is stateless via Bearer token.
