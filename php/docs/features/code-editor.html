Code Editor

A built-in file editor scoped to the site root that lets you edit templates, CSS, and JavaScript directly in the admin panel — no FTP, SSH, or local development setup required for quick changes.

Access

Navigate to Admin → Code Editor. The Code Editor is only available to users with the developer or admin role. Users with the editor role do not see this section in the admin.

Features

  • File tree — a sidebar tree listing all files in the site root (excluding the outpost/ directory)
  • Edit any site file — .html, .css, .js, .json, and other text-based formats
  • Save shortcut — Cmd+S (Mac) or Ctrl+S (Windows/Linux) saves the file without reaching for the button
  • Syntax highlighting — colour-coded highlighting for HTML, CSS, and JavaScript
  • Instant effect — saving a template file automatically invalidates the template cache, so the next page load reflects your changes immediately

Supported file extensions

The editor enforces a whitelist of editable file types:

php  html  css  js  json  xml  svg  txt  md  yml  yaml

Files with extensions not in this list cannot be opened or saved through the code editor.

Note — Writing or overwriting .php files requires the admin or developer role. The whitelist is enforced server-side in the API — it cannot be bypassed from the browser.

What you can edit

The code editor operates on the site root directory, excluding the outpost/ system directory. You can edit:

  • Top-level template files (index.html, blog.html, post.html, etc.)
  • Partial files (partials/head.html, partials/nav.html, partials/footer.html, etc.)
  • Stylesheets and scripts (assets/style.css, assets/main.js)
  • Site configuration (theme.json)

Forge button

When editing an .html file, a Forge button appears in the toolbar. Select any HTML and use Forge to wrap it in Outpost template tags — make fields editable, extract partials, create collection loops, and more. See the Forge documentation for details.

Analyze Site

The Analyze Site button in the toolbar runs Forge's site analysis across all .html files in the site root. It discovers pages, editable fields, navigation menus, and section blocks — then registers them in the CMS database. This is the same analysis that runs during the setup wizard, but can be re-run at any time to pick up new HTML files or fields.

Workflow

  1. Select a file from the file picker at the top of the editor.
  2. Make your changes in the code area.
  3. Press Cmd+S / Ctrl+S or click Save.
  4. Visit your site to see the changes — template cache is cleared automatically on save.
Warning — The code editor saves directly to the live site files on the server. There is no undo after saving. For significant changes, copy the current file contents to a local file before editing, so you can restore them if needed.

Security

Several safeguards restrict what the code editor can access:

  • Files outside the site root or inside the outpost/ directory cannot be opened or written — the API validates all paths to prevent directory traversal.
  • Only whitelisted file extensions are editable.
  • All save operations require an authenticated session with a valid CSRF token.
  • Writing .php files additionally requires the admin or developer role.

Template Reference

The Template Reference panel is available in the admin alongside the Code Editor. It provides a quick-reference card for all template syntax — output tags, filters, conditionals, loops, includes, and more — without leaving the admin. This is particularly useful when writing or modifying templates directly in the browser.

Code editor vs. local editing

Both approaches are valid. Use whichever fits the task:

Situation Recommended approach
Quick CSS tweak, fixing a typo in a template, minor content adjustment Code Editor
Building a new site or major feature Local editing with version control
Changes that need to be reviewed before going live Local editing, deploy via file transfer
Emergency fix on a live site with no local environment available Code Editor
Tip — If you use the Outpost Builder (Electron app), the Builder's sync feature provides a better workflow for local development: edit locally, sync to the server, and see changes reflected without manual file transfers.
