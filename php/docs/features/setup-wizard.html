Setup Wizard

The setup wizard runs automatically on fresh installs, guiding new users through site name confirmation and Forge site analysis in under two minutes.

How It Works

When a user logs in for the first time after installation, Outpost checks the setup_completed setting. If it's 0 (or missing), the admin redirects to the setup wizard instead of the dashboard.

The wizard is a full-screen, 3-step flow:

  1. Site Name — confirm your site's name (auto-detected from the <title> tag in your index.html, or enter manually)
  2. Forge Analysis — Forge scans all .html files in the site root. It discovers pages, <!-- outpost:section --> blocks, data-outpost fields, and <nav> elements for menu creation. Results are displayed for review before applying.
  3. Done — summary of discovered pages, fields, and menus, with a link to the dashboard

Existing Site Detection

When upgrading from a previous version, Outpost automatically detects existing content. If any pages or collections exist in the database, the setup_completed setting is set to 1 during the migration, and the wizard never appears. Forge can always be re-run manually from the admin to scan for new HTML files or fields.

Note — The setup wizard only appears once. After completing or skipping it, the setup_completed setting is permanently set and the wizard will not appear again.

Skipping the Wizard

A "Skip" link is visible on every step. Clicking it sets setup_completed = 1 and redirects to the dashboard with no content seeded.

Getting Started Checklist

After completing the wizard, the dashboard shows a "Getting Started" checklist card with 5 items:

Item Completed When Links To
Upload your logo @site_logo global is non-empty Globals
Edit your homepage Any fields for path / have been updated Page Editor (homepage)
Create your first post Any published collection items exist Collections
Set up your navigation Any menus have items Navigation

The checklist can be dismissed. Once dismissed or all items are completed, the normal dashboard appears.

Contextual Tips

Seven admin sections show a one-line contextual tip below the page header on first visit:

  • Pages — "Pages are auto-discovered from your HTML files by Forge."
  • Collections — "Collections are reusable content types like blog posts, projects, or team members."
  • Media — "Drag and drop files anywhere on this page to upload."
  • Globals — "Globals are site-wide values like your site name, logo, and social links."
  • Navigation — "Create menus and add links to pages, collections, or custom URLs."
  • Forms — "Build forms visually and embed them with {% form 'slug' %} in any template."

Tips are dismissed per-browser via localStorage and never reappear once closed.

API Endpoints

POST setup/apply

Applies wizard choices in a single transaction: updates site name, runs Forge site analysis, registers discovered pages and fields, sets setup_completed.

// Request body
{
  "site_name": "My Awesome Site"
}

// Response
{
  "success": true,
  "summary": { "pages": 5, "fields": 12, "menus": 1 }
}

GET setup/checklist

Returns dynamic checklist completion status.

POST setup/checklist/dismiss

Dismisses the Getting Started checklist permanently.
