Brand Settings

Set your site-wide colors, fonts, logo, and favicon in one place. Brand settings serve as the baseline identity for your entire site — the CSS Framework reads from them and injects CSS custom properties into every page.

How it works

Brand is available at Admin → Brand (in the Design & Build section). It requires the settings.* capability.

Brand settings define your site's core visual identity:

  • Palette — six color roles: primary, secondary, accent, neutral, background, and surface
  • Typography — heading font, body font, and a type scale ratio (Minor Second through Golden Ratio)
  • Identity — logo and favicon images

These values are stored in content/data/brand.json and are included in backups.

Where brand values go

Brand settings flow into your site as CSS custom properties. When the CSS Framework is enabled (via "framework": true in theme.json), brand values are injected directly as --brand-* CSS tokens before </head>. Your site CSS uses these tokens directly:

/* Brand tokens injected automatically */
:root {
  --brand-primary: #2D5A47;
  --brand-accent: #14B8A6;
  --brand-neutral: #27272A;
  --brand-bg: #FFFFFF;
  --brand-surface: #FAFAFA;
  --brand-font-heading: 'Inter', -apple-system, sans-serif;
  --brand-font-body: 'Inter', -apple-system, sans-serif;
  --text-xs: 0.64rem;
  --text-sm: 0.8rem;
  --text-base: 1rem;
  --text-lg: 1.25rem;
  /* ... through text-5xl */
}

The framework reads directly from brand.json every time a page renders.

The framework flag

Add "framework": true to your theme.json (in the site root) to opt into the Outpost CSS Framework:

{
  "name": "My Site",
  "version": "1.0.0",
  "framework": true
}

When enabled, Outpost injects three things before </head> on every page:

  1. Google Fonts links for the brand's heading and body fonts
  2. Framework CSS (/outpost/framework/outpost-framework.css) — lightweight reset, grid system, utility classes, button styles
  3. Brand tokens — :root CSS custom properties for all brand values + computed type scale

Your site CSS then uses var(--brand-primary), var(--brand-font-heading), var(--text-lg), etc.

Note — The framework flag is optional. Sites that do not use the framework can reference brand values directly from the Brand API or use their own CSS custom properties.

Storage

Brand settings are stored in content/data/brand.json:

{
  "colors": {
    "primary": "#2D5A47",
    "secondary": "#C4785C",
    "accent": "#14B8A6",
    "neutral": "#27272A",
    "background": "#FFFFFF",
    "surface": "#FAFAFA"
  },
  "typography": {
    "heading_font": "Inter",
    "body_font": "Inter",
    "type_scale": "1.25"
  },
  "identity": {
    "logo": "",
    "favicon": ""
  }
}

This file lives inside content/, so it is included in backups and safe from the auto-updater.

Logo and favicon sync

When you set a logo or favicon in Brand settings, the values are automatically synced to the @site_logo and @site_favicon global fields. This means your existing template tags — {{ @site_logo | image }} and {{ @site_favicon | image }} — work without any changes.
