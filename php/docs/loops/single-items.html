Single Items

Fetch and render one specific collection item by its URL slug. This is the template tag that powers every individual post, project, product, or detail page.

Basic Syntax

How It Works

When a visitor hits a collection URL like /post/hello-world, the router extracts the slug (hello-world) and sets it as an internal parameter. The <outpost-single> tag reads that slug, queries the collection for a published item matching it, and renders the main block. If no published item with that slug exists, the <outpost-single ... else> block renders instead.

Basic Single Item

<outpost-single collection="post">
 <article>
 <h1 data-outpost="title">Post Title</h1>
 <time data-outpost="published_at"></time>
 <div data-outpost="body" data-type="richtext"></div>
 </article>
</outpost-single>

<outpost-single collection="post" else>
 <p>Post not found.</p>
</outpost-single>

Available Fields

Inside a <outpost-single> block, you have access to all the same fields as in a collection loop: title, slug, published_at, url, status, plus all custom fields defined in the collection schema.
