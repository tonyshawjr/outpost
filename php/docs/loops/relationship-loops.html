Relationship Loops

Link collection items together with relationship fields. Editors hand-pick related items from any collection — the template receives full item data in the order they were selected, making it easy to build curated content connections.

What Is a Relationship Field?

A relationship field is a schema field type that stores references to items from another collection (or the same collection). Instead of the editor typing an ID or URL, they select items from a searchable list in the admin. The template receives the full item data for each selected item — the same shape as what you get inside a regular collection loop.

Common uses include related posts, featured products, curated reading lists, cross-linked team members, and any pattern where an editor needs to hand-pick specific items from a collection.

Note — Relationship fields are defined per collection schema. Go to Admin → Collections → select a collection → Edit Schema → Add Field → choose type Relationship, then configure the target collection and selection options.

Basic Syntax

Loop over a relationship field using the relationship. prefix. Each iteration yields a full item object with the same fields as a collection loop item.

{% for item in relationship.field_name %}
  {{ item.title }}
  {{ item.url }}
{% endfor %}

Where:

  • relationship.field_name identifies the relationship field by its machine name in the schema
  • item is the loop variable representing one related item
  • Each item has the same fields as a regular collection loop item: title, url, slug, published_at, featured_image, excerpt, body, plus all custom fields from the target collection's schema

Available Fields

Related items expose the same fields as items in a {% for x in collection.slug %} loop. All standard and custom fields from the target collection are accessible.

Field Type Description
title string Item title
slug string URL slug
url string Full URL built from the target collection's url_pattern
featured_image string Image URL (empty string if not set)
excerpt string Short description
body html Rich text body — use | raw to render
published_at string Formatted publish date
{custom_field} varies Any field defined in the target collection's schema

Item Order

Items are returned in the order the editor selected them in the admin — not by publish date or alphabetical order. This makes relationship fields ideal for curated lists where the order matters.

Tip — If you want an editorially curated "Featured Posts" section, a relationship field gives the editor full control over both which posts appear and in what order — unlike limit:3 which always shows the most recent items.

Related Posts Example

A blog post detail page with a "Related Posts" section. The post collection has a related_posts relationship field targeting the same post collection.

{% single post from collection.post %}
  <article>
    <h1>{{ post.title }}</h1>
    <time>{{ post.published_at }}</time>
    <div class="prose">{{ post.body | raw }}</div>
  </article>

  {% if post.related_posts %}
    <section class="related-posts">
      <h2>Related Posts</h2>
      <div class="related-grid">
        {% for related in relationship.related_posts %}
          <a href="{{ related.url }}" class="related-card">
            {% if related.featured_image %}
              <img src="{{ related.featured_image }}" alt="{{ related.title }}">
            {% endif %}
            <h3>{{ related.title }}</h3>
            {% if related.excerpt %}
              <p>{{ related.excerpt }}</p>
            {% endif %}
          </a>
        {% endfor %}
      </div>
    </section>
  {% endif %}
{% else %}
  <p>Post not found.</p>
{% endsingle %}

Cross-Collection Relationships

Relationship fields can target a different collection than the one they belong to. This enables cross-linking between content types.

Product with related case studies

{% single product from collection.product %}
  <article>
    <h1>{{ product.title }}</h1>
    {{ product.body | raw }}

    {% if product.case_studies %}
      <section class="case-studies">
        <h2>Case Studies</h2>
        {% for study in relationship.case_studies %}
          <div class="case-study-card">
            {% if study.featured_image %}
              <img src="{{ study.featured_image }}" alt="{{ study.title }}">
            {% endif %}
            <h3><a href="{{ study.url }}">{{ study.title }}</a></h3>
            <p>{{ study.excerpt }}</p>
            {% if study.client %}
              <span class="client">{{ study.client }}</span>
            {% endif %}
          </div>
        {% endfor %}
      </section>
    {% endif %}
  </article>
{% endsingle %}

Event with featured speakers

{% single event from collection.event %}
  <article>
    <h1>{{ event.title }}</h1>
    <p class="event-date">{{ event.date }}</p>
    {{ event.body | raw }}

    {% if event.speakers %}
      <section class="speakers">
        <h2>Speakers</h2>
        <div class="speaker-grid">
          {% for speaker in relationship.speakers %}
            <div class="speaker-card">
              {% if speaker.photo %}
                <img src="{{ speaker.photo | image }}" alt="{{ speaker.title }}"
                     class="speaker-photo">
              {% endif %}
              <h3>{{ speaker.title }}</h3>
              {% if speaker.role %}
                <p class="speaker-role">{{ speaker.role }}</p>
              {% endif %}
            </div>
          {% endfor %}
        </div>
      </section>
    {% endif %}
  </article>
{% endsingle %}

Schema Configuration

When adding a relationship field to a collection schema, you configure the following options:

Option Description
Target collection The collection to select items from (e.g., post, product, member). Can be the same collection or a different one.
Multiple / Single Whether the editor can select multiple items or just one. Multiple returns a list (use a {% for %} loop); single returns one item.
Max items Optional cap on the number of items the editor can select. Useful for constrained layouts like a "Top 3 Picks" section.

Empty Relationship

If the editor has not selected any items, the loop body is silently skipped. Guard the surrounding markup with an {% if %} check to avoid rendering empty containers.

{% if post.related_posts %}
  <section class="related">
    <h2>You might also like</h2>
    {% for related in relationship.related_posts %}
      <a href="{{ related.url }}">{{ related.title }}</a>
    {% endfor %}
  </section>
{% endif %}

Common Use Cases

Use case Field name Source collection Target collection
Related posts related_posts post post (same)
Featured products featured_products page (homepage) product
Author profile author post member
Event speakers speakers event member
Product case studies case_studies product case-study
Curated reading list reading_list page post

Relationship vs. related: Option

The related: loop option and relationship fields both show related content, but they work differently.

Relationship field related: loop option
Editor hand-picks specific items Automatically finds items sharing folder labels
Order is controlled by the editor Order is determined by relevance (shared labels)
Can link across different collections Only works within the same collection
Always shows exactly what was selected Results vary as folder labels change
Tip — Use a relationship field when the editor needs precise control over which items appear. Use related: when you want an automatic, hands-off "similar items" section based on folder label overlap.

Complete Example: Blog Post with Author and Related

A post.html template where the post collection has both an author relationship field (single, targeting member) and a related_posts relationship field (multiple, targeting post).

{% include 'head' %}
{% include 'nav' %}

{% single post from collection.post %}
<main>
  <article class="post">
    {% if post.featured_image %}
      <img src="{{ post.featured_image }}" alt="{{ post.title }}" class="post-hero">
    {% endif %}

    <header class="post-header">
      <h1>{{ post.title }}</h1>
      <div class="post-meta">
        <time>{{ post.published_at }}</time>

        {% for author in relationship.author %}
          <div class="author-byline">
            {% if author.photo %}
              <img src="{{ author.photo | image }}" alt="{{ author.title }}" class="author-avatar">
            {% endif %}
            <a href="{{ author.url }}">{{ author.title }}</a>
          </div>
        {% endfor %}
      </div>
    </header>

    <div class="prose">{{ post.body | raw }}</div>
  </article>

  {% if post.related_posts %}
    <section class="related-posts">
      <h2>You might also like</h2>
      <div class="related-grid">
        {% for related in relationship.related_posts %}
          <article class="related-card">
            {% if related.featured_image %}
              <img src="{{ related.featured_image }}" alt="{{ related.title }}">
            {% endif %}
            <h3><a href="{{ related.url }}">{{ related.title }}</a></h3>
            <p>{{ related.excerpt | or_body }}</p>
            <time>{{ related.published_at }}</time>
          </article>
        {% endfor %}
      </div>
    </section>
  {% endif %}
</main>
{% else %}
<main>
  <p>Post not found.</p>
</main>
{% endsingle %}
