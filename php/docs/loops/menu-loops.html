Menu Loops

Render navigation menus defined in Admin → Navigation. Menu loops give editors full control over link labels, URLs, order, and nesting — without touching any template code.

Basic Syntax

Reference a menu by its slug — the identifier set when the menu was created in the admin. The menu slug main is the conventional name for the primary site navigation.

{% for item in menu.main %}
  {{ item.label }}
  {{ item.url }}
  {{ item.target }}
{% endfor %}

Available Item Fields

Field Description
item.label Display text for the link — set by the editor
item.url Link destination — can be a relative path, absolute URL, or anchor
item.target _self (default, same tab) or _blank (new tab)
item.children Array of child menu items — same structure as top-level items. Empty array if no children.

Basic Horizontal Nav

The simplest possible navigation — a flat list of links with no dropdown support.

<nav class="main-nav">
  {% for item in menu.main %}
    <a href="{{ item.url }}" target="{{ item.target }}">{{ item.label }}</a>
  {% endfor %}
</nav>

Nav with Dropdown Menus

Check item.children to conditionally render a dropdown panel. Nest a second {% for child in item.children %} loop inside.

<nav class="main-nav">
  {% for item in menu.main %}
    <div class="nav-item {% if item.children %}has-dropdown{% endif %}">
      <a href="{{ item.url }}" target="{{ item.target }}" class="nav-link">
        {{ item.label }}
        {% if item.children %}
          <span class="arrow">&#8964;</span>
        {% endif %}
      </a>
      {% if item.children %}
        <ul class="dropdown">
          {% for child in item.children %}
            <li>
              <a href="{{ child.url }}" target="{{ child.target }}">
                {{ child.label }}
              </a>
            </li>
          {% endfor %}
        </ul>
      {% endif %}
    </div>
  {% endfor %}
</nav>

CSS for Hover Dropdowns

.nav-item { position: relative; }
.dropdown {
  display: none;
  position: absolute;
  top: 100%;
  left: 0;
  background: #fff;
  border: 1px solid #e4e4e7;
  border-radius: 8px;
  padding: 8px 0;
  min-width: 180px;
  list-style: none;
  box-shadow: 0 4px 16px rgba(0,0,0,.08);
}
.nav-item.has-dropdown:hover .dropdown,
.nav-item.has-dropdown:focus-within .dropdown {
  display: block;
}
.dropdown a {
  display: block;
  padding: 8px 16px;
  text-decoration: none;
  color: #374151;
}
.dropdown a:hover { background: #f4f4f5; }
Tip — The :focus-within selector ensures dropdowns remain open when a user tabs into the child links, which is important for keyboard accessibility. See Dropdown Navigation for a more complete accessible implementation.

Header and Footer Menus

Create separate menus in the admin — one with slug main for the header, one with slug footer for the footer — then reference each by slug in your partials.

{# In partials/nav.html #}
<header class="site-header">
  <a href="/" class="logo">{{ @site_name }}</a>
  <nav>
    {% for item in menu.main %}
      <a href="{{ item.url }}" target="{{ item.target }}">{{ item.label }}</a>
    {% endfor %}
  </nav>
</header>

{# In partials/footer.html #}
<footer>
  <nav class="footer-nav">
    {% for item in menu.footer %}
      <a href="{{ item.url }}" target="{{ item.target }}">{{ item.label }}</a>
    {% endfor %}
  </nav>
  <p>{{ @footer_text }}</p>
</footer>

Mobile Hamburger Navigation

A toggle-based mobile menu using a minimal JavaScript class swap. The same template works for both mobile and desktop — CSS handles the display differences.

<header class="site-header">
  <a href="/" class="logo">{{ @site_name }}</a>

  <button id="nav-toggle" aria-expanded="false" aria-controls="main-nav">
    <span class="hamburger-icon">&#9776;</span>
  </button>

  <nav id="main-nav" class="main-nav nav-closed">
    {% for item in menu.main %}
      <a href="{{ item.url }}" target="{{ item.target }}" class="nav-link">
        {{ item.label }}
      </a>
      {% if item.children %}
        <ul class="sub-menu">
          {% for child in item.children %}
            <li><a href="{{ child.url }}" target="{{ child.target }}">{{ child.label }}</a></li>
          {% endfor %}
        </ul>
      {% endif %}
    {% endfor %}
  </nav>
</header>

<script>
var toggle = document.getElementById('nav-toggle');
var nav = document.getElementById('main-nav');
toggle.addEventListener('click', function () {
  var isOpen = !nav.classList.contains('nav-closed');
  nav.classList.toggle('nav-closed');
  toggle.setAttribute('aria-expanded', isOpen ? 'false' : 'true');
});
</script>
/* Mobile-first CSS */
.main-nav {
  display: flex;
  flex-direction: column;
  gap: 4px;
}
.main-nav.nav-closed {
  display: none;
}
#nav-toggle {
  display: block; /* show on mobile */
}

@media (min-width: 768px) {
  #nav-toggle { display: none; }
  .main-nav {
    display: flex !important; /* always visible on desktop */
    flex-direction: row;
    align-items: center;
    gap: 8px;
  }
}

Social Links Menu

Create a dedicated menu in the admin with slug social and add each social network as a link item with target: _blank. Render it anywhere in your theme.

<div class="social-links">
  {% for item in menu.social %}
    <a href="{{ item.url }}" target="{{ item.target }}" aria-label="{{ item.label }}">
      {{ item.label }}
    </a>
  {% endfor %}
</div>

Empty Menu

If no menu with the given slug exists, the loop simply renders nothing — no error is thrown. You can add an {% else %} branch for a fallback, though in practice menus are always created alongside the theme.

{% for item in menu.main %}
  <a href="{{ item.url }}">{{ item.label }}</a>
{% else %}
  <a href="/">Home</a>
{% endfor %}

Using menu.target Safely

item.target is always either _self or _blank — it is safe to output directly in a target attribute. The admin enforces these two values; editors cannot set arbitrary attribute values.

<a href="{{ item.url }}" target="{{ item.target }}">{{ item.label }}</a>
Tip — For external links, consider adding rel="noopener noreferrer" when target is _blank. Because Outpost does not expose a conditional for this in template syntax, it is simplest to add rel="noopener noreferrer" to every link unconditionally — it has no negative effect on same-tab links.

Complete nav.html Partial

A fully featured header partial suitable for most sites: logo, main menu with dropdown support, and an optional call-to-action button from a global field.

<header class="site-header">
  <a href="/" class="site-logo">{{ @site_name }}</a>

  <button id="nav-toggle" aria-expanded="false" aria-controls="primary-nav">Menu</button>

  <nav id="primary-nav" class="primary-nav nav-closed">
    {% for item in menu.main %}
      <div class="nav-item {% if item.children %}has-dropdown{% endif %}">
        <a href="{{ item.url }}" target="{{ item.target }}" class="nav-link">
          {{ item.label }}
        </a>
        {% if item.children %}
          <ul class="dropdown">
            {% for child in item.children %}
              <li>
                <a href="{{ child.url }}" target="{{ child.target }}">
                  {{ child.label }}
                </a>
              </li>
            {% endfor %}
          </ul>
        {% endif %}
      </div>
    {% endfor %}

    {% if @cta_label %}
      <a href="{{ @cta_url }}" class="btn btn-primary">{{ @cta_label }}</a>
    {% endif %}
  </nav>
</header>

<script>
var toggle = document.getElementById('nav-toggle');
var nav = document.getElementById('primary-nav');
toggle.addEventListener('click', function () {
  var open = nav.classList.toggle('nav-open');
  nav.classList.toggle('nav-closed', !open);
  toggle.setAttribute('aria-expanded', open ? 'true' : 'false');
});
</script>
