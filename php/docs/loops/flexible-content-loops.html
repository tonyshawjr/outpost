Flexible Content Loops

Build dynamic page layouts from modular content blocks. Flexible content fields let editors compose pages from named layouts — each with its own set of sub-fields — enabling freeform page building within a structured schema.

What Is a Flexible Content Field?

A flexible content field is a schema field type that stores an ordered list of blocks, where each block belongs to a named layout. Unlike a repeater (where every row has the same fields), each layout in a flexible content field defines its own set of typed sub-fields. The editor chooses which layouts to add and in what order, creating a unique page structure per item.

Think of it as a page builder inside your collection schema: you define the available building blocks (layouts) and the editor assembles them into a page. Each layout can have completely different sub-fields — a "hero" layout might have a title, image, and CTA, while a "testimonial" layout has a quote, author, and photo.

Note — Flexible content fields are defined per collection schema. Go to Admin → Collections → select a collection → Edit Schema → Add Field → choose type Flexible Content, then define your layouts and their sub-fields.

Basic Syntax

Loop over a flexible content field using the flexible. prefix. Each iteration yields a block object with a layout property and the sub-fields defined for that layout.

{% for block in flexible.field_name %}
  {{ block.layout }}
  {{ block.sub_field }}
{% endfor %}

Where:

  • flexible.field_name identifies the flexible content field by its machine name in the schema
  • block is the loop variable representing one block in the list
  • block.layout returns the layout type name as a string (e.g., "hero", "testimonial", "text_block")
  • block.sub_field accesses sub-fields defined for that particular layout

Rendering by Layout Type

Since each layout has different sub-fields, use {% if block.layout == "name" %} conditionals to render the correct HTML for each layout type.

{% for block in flexible.page_sections %}
  {% if block.layout == "hero" %}
    <section class="hero">
      <h1>{{ block.title }}</h1>
      <p>{{ block.subtitle }}</p>
      {% if block.image %}
        <img src="{{ block.image | image }}" alt="{{ block.title }}">
      {% endif %}
    </section>
  {% endif %}

  {% if block.layout == "testimonial" %}
    <blockquote class="testimonial">
      {{ block.quote | raw }}
      <cite>{{ block.author }}</cite>
    </blockquote>
  {% endif %}

  {% if block.layout == "text_block" %}
    <section class="prose">
      {{ block.body | raw }}
    </section>
  {% endif %}
{% endfor %}

Sub-Field Filters

Sub-fields use the same output filters as any other field. Apply the appropriate filter based on the sub-field type defined in the layout.

Sub-field type Template syntax Notes
Text {{ block.heading }} Auto-escaped. No filter needed.
Textarea {{ block.description | textarea }} Preserves line breaks as <br> tags.
Richtext {{ block.body | raw }} Outputs HTML as-is. Only use | raw for trusted editor content.
Image {{ block.photo | image }} Resolves to the full public URL of the uploaded image.
Link {{ block.cta_url | link }} Ensures the URL is safe to output in an href attribute.
Select {{ block.style | select }} Outputs the selected option value.

Defining Layouts in the Schema Builder

When you add a flexible content field to a collection schema, you define one or more layouts. Each layout has:

  • Layout name — the identifier used in block.layout comparisons (e.g., hero, cta, gallery)
  • Layout label — the display name shown to editors in the admin (e.g., "Hero Section", "Call to Action")
  • Sub-fields — the fields available when the editor adds a block of this layout type. Each sub-field has a type (text, image, richtext, etc.), machine name, label, and optional configuration.

Example schema definition for a page_sections flexible content field:

Field: page_sections
Type: Flexible Content
Layouts:
  hero        → title (text), subtitle (text), image (image), cta_label (text), cta_url (link)
  text_block  → body (richtext)
  testimonial → quote (richtext), author (text), company (text), avatar (image)
  cta         → heading (text), description (text), button_label (text), button_url (link), style (select)
  gallery     → heading (text), images (gallery)

Inside a Single Item

Use a flexible content loop on a detail page to render the composed page structure for a single item.

{% single page from collection.landing %}
  <article>
    <h1>{{ page.title }}</h1>

    {% for block in flexible.page_sections %}
      {% if block.layout == "hero" %}
        <section class="hero-section">
          <div class="hero-content">
            <h2>{{ block.title }}</h2>
            <p>{{ block.subtitle }}</p>
            {% if block.cta_label %}
              <a href="{{ block.cta_url | link }}" class="btn btn-primary">
                {{ block.cta_label }}
              </a>
            {% endif %}
          </div>
          {% if block.image %}
            <img src="{{ block.image | image }}" alt="{{ block.title }}" class="hero-image">
          {% endif %}
        </section>
      {% endif %}

      {% if block.layout == "text_block" %}
        <section class="content-section">
          <div class="prose">{{ block.body | raw }}</div>
        </section>
      {% endif %}

      {% if block.layout == "testimonial" %}
        <section class="testimonial-section">
          <blockquote>
            {{ block.quote | raw }}
            <footer>
              {% if block.avatar %}
                <img src="{{ block.avatar | image }}" alt="{{ block.author }}" class="avatar">
              {% endif %}
              <div>
                <strong>{{ block.author }}</strong>
                {% if block.company %}
                  <span>{{ block.company }}</span>
                {% endif %}
              </div>
            </footer>
          </blockquote>
        </section>
      {% endif %}

      {% if block.layout == "cta" %}
        <section class="cta-section {% if block.style == "dark" %}cta-dark{% endif %}">
          <h2>{{ block.heading }}</h2>
          {% if block.description %}
            <p>{{ block.description }}</p>
          {% endif %}
          {% if block.button_label %}
            <a href="{{ block.button_url | link }}" class="btn">{{ block.button_label }}</a>
          {% endif %}
        </section>
      {% endif %}
    {% endfor %}
  </article>
{% else %}
  <p>Page not found.</p>
{% endsingle %}

Empty Flexible Content

If an editor has not added any blocks to a flexible content field, the loop body is silently skipped. No error is thrown. Use an {% if %} check if you need to conditionally show a section wrapper alongside the loop.

{% if page.page_sections %}
  <div class="page-builder">
    {% for block in flexible.page_sections %}
      {# ... layout rendering ... #}
    {% endfor %}
  </div>
{% else %}
  <div class="prose">{{ page.body | raw }}</div>
{% endif %}

Block Order

Blocks are returned in the order the editor arranged them in the admin. The flexible content field supports drag-and-drop reordering — the template always reflects the saved order with no additional configuration.

Common Use Cases

The following table shows typical flexible content configurations for common page-building patterns.

Use case Flexible content field Layouts
Landing page page_sections hero, features, testimonial, cta, text_block
Case study content_blocks intro, challenge, solution, results, quote
Product page sections hero, specs, gallery, reviews, faq, cta
About page about_sections mission, team, timeline, values, contact

Flexible Content vs. Repeaters

Both field types store ordered lists of data, but they serve different purposes.

Use flexible content when... Use a repeater when...
Each block can have a different structure (different layouts) Every row has the same fields (uniform structure)
Editors need to compose freeform page layouts Editors are filling in a list of identical items
The template needs block.layout to decide what to render The template renders every row the same way
Landing pages, case studies, product pages FAQ lists, feature bullets, team members, pricing tiers

Complete Example: Landing Page Builder

A full landing.html template for a landing collection with a sections flexible content field containing hero, features, testimonials, and cta layouts.

{% include 'head' %}
{% include 'nav' %}

{% single page from collection.landing %}
<main>
  {% for block in flexible.sections %}

    {% if block.layout == "hero" %}
      <section class="hero">
        <div class="hero-inner">
          <h1>{{ block.heading }}</h1>
          <p class="hero-sub">{{ block.subheading }}</p>
          {% if block.cta_label %}
            <a href="{{ block.cta_url | link }}" class="btn btn-primary">
              {{ block.cta_label }}
            </a>
          {% endif %}
        </div>
        {% if block.background_image %}
          <img src="{{ block.background_image | image }}" alt="" class="hero-bg">
        {% endif %}
      </section>
    {% endif %}

    {% if block.layout == "features" %}
      <section class="features">
        <h2>{{ block.heading }}</h2>
        {% if block.description %}
          <p class="features-desc">{{ block.description }}</p>
        {% endif %}
        {{ block.features_content | raw }}
      </section>
    {% endif %}

    {% if block.layout == "testimonials" %}
      <section class="testimonials">
        <blockquote>
          <p>"{{ block.quote }}"</p>
          <footer>
            {% if block.avatar %}
              <img src="{{ block.avatar | image }}" alt="{{ block.author }}" class="avatar">
            {% endif %}
            <cite>{{ block.author }}{% if block.role %}, {{ block.role }}{% endif %}</cite>
          </footer>
        </blockquote>
      </section>
    {% endif %}

    {% if block.layout == "cta" %}
      <section class="cta-banner">
        <h2>{{ block.heading }}</h2>
        <p>{{ block.body_text }}</p>
        {% if block.button_text %}
          <a href="{{ block.button_url | link }}" class="btn btn-primary">
            {{ block.button_text }}
          </a>
        {% endif %}
      </section>
    {% endif %}

  {% endfor %}
</main>
{% else %}
<main>
  <p>Page not found.</p>
</main>
{% endsingle %}

{% include 'footer' %}
Tip — Break large flexible content templates into partials. Create a partial for each layout type (e.g., partials/block-hero.html, partials/block-testimonial.html) and include them with {% include 'block-hero' %} to keep your main template readable.
