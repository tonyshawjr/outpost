Roadmap

Where Outpost is headed — what’s shipped, what’s next, and why.

Outpost is a powerful, zero-config CMS for everyone — from freelancers building client sites to developers who want full control. No Composer, no build step, no managed infrastructure. It runs wherever PHP 8.x runs.

What’s Shipped

v4.0 — Smart Forge + Frontend Drawer

The product reinvention. Drop in HTML, everything is automatically editable, all editing moves to the frontend.

  • Smart Forge — one button scans an entire HTML file and auto-converts ALL content elements to editable fields. What takes 20 right-clicks today takes 1 click.
  • Frontend editing drawer — when logged in, the “Edit in Outpost” button opens a Storyblok-style drawer. Every editable field on the page is listed in the drawer. Edit text, swap images, change buttons — all from the frontend.
  • Pages section removed — no more backend page editing. All content editing moves to the frontend drawer.
  • Drawer design — section grouping, repeater fields, rich text editor, image picker, save & instant preview.

Core Constraints

These define what Outpost is. Breaking them makes it something else.

  1. Zero server-side dependencies — No Composer, no npm on the server, no build step to deploy
  2. SQLite only — single-file database enables trivial backups and shared hosting compatibility
  3. Liquid templates — no PHP in theme files; content/presentation separation is enforced (Smart Forge in v4.0 makes Liquid optional for basic content)
  4. Drop-in install — copy files, visit /outpost/, done
