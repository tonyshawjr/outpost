Roadmap

Where Outpost is headed — what’s shipped, what’s next, and why.

Outpost is a powerful, zero-config CMS for everyone — from freelancers building client sites to developers who want full control. No Composer, no build step, no managed infrastructure. It runs wherever PHP 8.x runs.

What’s Shipped

v1.0 — Stable Release

Feature-complete across all core subsystems. Everything below was built and shipped during beta:

  • Content management (pages, collections, globals, folders, menus, navigation)
  • Flexible Content, Relationship, and Conditional Logic fields (ACF Pro parity)
  • Block editor (TipTap richtext, image, markdown, HTML, divider)
  • Media library with search, filter, crop, resize, alt text
  • Liquid template engine with partials, conditionals, loops, globals, meta tags
  • Built-in analytics (pageviews, referrers, top pages — zero external dependencies)
  • Member system (registration, login, email verification, paid tiers, visibility gates)
  • Visual form builder with 14 field types and enhanced submissions inbox
  • Channels — connect to REST APIs, RSS feeds, and CSV files
  • On-page editing for text, richtext, textarea, and image fields
  • Code editor (VS Code-style IDE with tabs, file tree, autocomplete, find in files)
  • Revision history with field-level diffs and one-click rollback
  • TOTP two-factor authentication with backup codes
  • API key authentication for headless use
  • Webhooks, scheduled publishing, bulk operations
  • WordPress import
  • 6 roles with capability-based permissions
  • Dark mode, mobile-responsive admin
  • Security hardening (37 findings addressed)
  • 53 pages of developer documentation

What’s Next

v2.0 — Onboarding & Setup Wizard

First impressions matter. A fresh install currently drops you into a blank admin. v2.0 fixes that.

  • Setup wizard — runs on first visit: site name, choose a theme, set admin credentials, pick starter content
  • Starter content packs — pre-built content for common use cases (blog, portfolio, business, documentation)
  • Getting started dashboard — replaces the default dashboard with a checklist for new sites
  • Contextual tips — subtle inline hints on first use of each admin section
  • Empty states — every list page has a helpful empty state with a clear call-to-action

v2.1 — Collection Folders

Bring the media folder system to collections. Organize posts, products, and case studies the same way you organize media.

  • Folder sidebar for collections (same as media)
  • Drag-to-folder from the item list
  • Multi-folder assignment
  • {% for item in collection_folder.slug %} template loops

v2.2 — Theme Gallery

More starting points, less blank-page anxiety. Ship 4–5 polished themes with full customizer support.

  • Business — homepage with services, team, testimonials, contact form
  • Portfolio — grid gallery, project case studies, about page
  • Blog — clean reading experience, categories, author pages
  • Documentation — sidebar navigation, search, code blocks
  • Landing page — single-page marketing site with CTA blocks

v2.3 — Visual Tag Builder

Turn any static HTML into an Outpost theme without writing a single template tag. Import HTML, select content, click “Make Editable” — Outpost wraps it in the correct tags for you.

  • Interactive “Make Editable” workflow in the code editor
  • Field type and scope selector popover
  • Collection loop, conditional, and partial extraction
  • Autocomplete with inline descriptions

v2.4 — Headless-First

Position Outpost as the zero-config headless CMS alongside the traditional themed approach.

  • GraphQL API alongside REST
  • Content webhooks v2 with full content diffs
  • Static site preview via webhook triggers
  • Real-time content events (SSE or WebSocket)

v2.5 — Deeper Analytics

  • Content performance cohorts
  • Goal funnels (visitor → member → paid)
  • Search analytics
  • Optional geo enrichment via MaxMind GeoLite2

v2.6 — Commerce

Lightweight digital product sales via Stripe.

  • Sell downloads, courses, and one-time purchases

v2.7 — Collaborative Editing

  • Real-time multi-user editing

Long-Term (v3.x+)

  • Internationalization — multi-language content, locale routing, one-click duplicate-as-locale
  • Theme Marketplace — browse, preview, and install themes from within the admin
  • Channels Phase 3 & 4 — inbound webhooks, outbound channels, filtering, channel-to-channel piping

Core Constraints

These define what Outpost is. Breaking them makes it something else.

  1. Zero server-side dependencies — No Composer, no npm on the server, no build step to deploy
  2. SQLite only — single-file database enables trivial backups and shared hosting compatibility
  3. Liquid templates only — no PHP in theme files; content/presentation separation is enforced
  4. Drop-in install — copy files, visit /outpost/, done

Not Planned

These are explicitly out of scope. Acknowledging them keeps the codebase focused.
