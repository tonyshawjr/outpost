Roadmap

Where Outpost is headed — what’s shipped, what’s next, and why.

Outpost is a powerful, zero-config CMS for everyone — from freelancers building client sites to developers who want full control. No Composer, no build step, no managed infrastructure. It runs wherever PHP 8.x runs.

v4.0 — Smart Forge + Frontend Editor

The product reinvention. Drop in HTML, everything is automatically editable, all editing moves to the frontend.

  • Smart Forge — one button scans an entire HTML file and auto-converts ALL content elements to editable data-outpost fields.
  • Frontend editing drawer — Storyblok-style drawer with block-grouped field panels.
  • Theme manifest system — scans templates to build a manifest of fields, globals, and collections per page.
  • Editor overlay — page scaling with live preview in iframe.

v4.1 — Template Engine v2: Data Attribute Architecture

Complete replacement of template syntax with HTML data attributes and custom elements.

  • data-outpost fields — editable fields defined by attributes on HTML elements, not custom tags.
  • Block grouping — <!-- outpost:hero --> comments group fields into editor sections.
  • Block settings — CSS custom properties injected from structured comment definitions.
  • Custom elements — <outpost-each>, <outpost-single>, <outpost-if>, <outpost-menu>, <outpost-include> for structural logic.
  • Clean output — zero CMS fingerprints on the live site. All Outpost syntax stripped from public HTML.
  • Auto-hide — elements with empty fields automatically removed from output.

v4.1.1 — Developer Tooling

Code editor and Smart Forge updates for v2 template syntax.

  • v2 syntax highlighting — high-contrast, colorblind-accessible colors for data-outpost, data-type, data-scope, <outpost-*> elements, and block comments in the code editor.
  • v2 autocomplete — context-aware completions for all v2 attributes and custom elements.
  • Smart Forge v2 — outputs data-outpost attributes and <!-- outpost:section --> block comments instead of Liquid syntax.
  • Editor field display fix — frontend editor now correctly shows page fields and global fields from v2 templates.

v4.2 — Click-to-Edit Bridge, EditDrawer & Skeleton Theme

Click any element in the live preview to edit it, plus a Storyblok-style editing drawer and a full v2 reference theme.

  • Block-level targeting — click a block background to see all its fields; click an element to jump to that specific field
  • Editor mode rendering — v2 engine keeps data-outpost attributes in output when loaded with ?_outpost_editor=1
  • Block attribute injection — engine adds data-outpost-block="name" to block wrappers in editor mode
  • Reverse highlighting — sidebar field hover highlights the corresponding element in the preview
  • Storyblok-style EditDrawer — three-level drill-down sidebar (section list → section detail with General/Style tabs → field detail), breadcrumb navigation, bridge click integration jumps to matching section/field
  • Skeleton theme v2 showcase — complete theme rewrite with 7 homepage blocks, outpost-settings (colors, layout variants, ranges, toggles), repeaters, collection loops with pagination, folder taxonomy loops, single items with related posts, global blocks, conditionals, and responsive CSS using var() and [data-layout] selectors

v4.4 — JWT Auth & Mobile API

Bearer token authentication and CORS support for mobile apps and headless usage.

  • CORS support — configurable allowed origins for cross-domain API access from mobile apps and SPAs
  • Dual auth — all endpoints accept both session cookies (web) and Bearer tokens (apps). CSRF skipped for token auth.

  • 12 filter types — search, dropdown, checkbox, radio, range, A–Z, toggle, proximity, pager, sort
  • Results container — <div data-compass="results"> with any layout
  • Helper tags — result count, reset, active filter pills
  • URL state sync — all filters reflected in shareable URLs
  • Indexed search — SQLite-based index with incremental updates, sub-50ms queries
  • Proximity search — geolocation-based distance filtering
  • Minimal CSS — ~95 lines, only auto-generated content styled (v4.8.4)

Core Constraints

These define what Outpost is. Breaking them makes it something else.

  1. Zero server-side dependencies — No Composer, no npm on the server, no build step to deploy
  2. SQLite only — single-file database enables trivial backups and shared hosting compatibility
  3. HTML data-attribute templates — no PHP in theme files; content/presentation separation enforced via data-outpost attributes and <outpost-*> elements
  4. Drop-in install — copy files, visit /outpost/, done

Not Planned

These are explicitly out of scope.

Plugin ecosystemRequires a package manager — violates the zero-config promise
