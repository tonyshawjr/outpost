Global Fields

Global fields hold site-wide values — the same content available on every page and every template. Managed in Admin → Globals, referenced in templates with the @ prefix.

What Are Global Fields?

Most CMS fields are tied to a specific page or collection item. Global fields are different — they exist once and are available everywhere. A global field is the right choice for content that appears in multiple places:

  • The site name shown in the browser tab and nav bar
  • The logo image used in the header and footer
  • Social media URLs linked from multiple pages
  • The author bio shown on every blog post
  • A Google Analytics ID included on every page

If you find yourself copying the same value into multiple pages, that value belongs in a global field.

Auto-scaffolding

The first time a template references a global field that does not yet exist, Outpost creates it automatically in Admin → Globals. There is no schema file to update and no migration to run. Visit any page that uses the new global and the field will appear in the admin ready to edit.

Tip — This means you can design your template first and fill in content later. Add data-outpost="site_name" data-scope="global" to your nav partial, deploy, and the field will appear in Globals automatically.

Auto-migration

If you move content from a page field to a global, rename it with the @ prefix in your template. Outpost detects the change and automatically migrates the existing value from the page field to the global field on the next page load. No data is lost.

Checking for a Value

Use <outpost-if field="name" scope="global" exists> to conditionally render elements when a global field has been filled in. This is important for optional fields like social media links — you do not want to render an empty anchor tag if the URL has not been set.

In the templates engine, global fields use data-scope="global" alongside data-outpost. The @ prefix is not needed — the scope attribute tells the engine this is a global field.

Basic Global Output

<span data-outpost="site_name" data-scope="global">My Site</span>
<p data-outpost="bio" data-scope="global" data-type="textarea"></p>
<div data-outpost="about_body" data-scope="global" data-type="richtext"></div>
<img data-outpost="logo" data-scope="global" data-type="image" src="logo.png">
<a data-outpost="website_url" data-scope="global" data-type="link" href="#">Website</a>
<span data-outpost="brand_color" data-scope="global" data-type="color">#000000</span>

Checking for a Value

Use <outpost-if> with scope="global" to conditionally render global content.

<outpost-if field="profile_photo" scope="global" exists>
 <img data-outpost="profile_photo" data-scope="global" data-type="image" src="" alt="">
</outpost-if>

Best Practices

  • Use global fields for anything that appears on more than one page — site name, logo, social links, footer copy, GA4 ID.
  • Use page fields for content specific to one page — page title, hero image, body text.
  • Always wrap optional globals like social links in <outpost-if field="field" scope="global" exists> so empty fields do not render broken HTML.
  • Define all your global fields in your partials/head.html or partials/nav.html so they are scaffolded on the first page load — even if those partials are included on every page.
