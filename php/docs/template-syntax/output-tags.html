Output Tags & Filters

Output tags render field values into your HTML. Every field you reference in a template is automatically registered in the admin editor — no configuration required.

Basic Text Output

The simplest output tag renders a field's value, HTML-escaped, directly into your markup.

{{ field_name }}

HTML-escaping means characters like <, >, and & are converted to safe entities automatically, protecting against XSS. The field is registered in the admin page editor the first time the template is rendered.

<header>
  <h1>{{ page_title }}</h1>
  <p class="lead">{{ page_subtitle }}</p>
</header>

With a Default Value

Wrap a default value between the opening and closing tag. The default is shown as a placeholder in the admin editor, and output on the front end when the field is empty. The closing tag uses the field name prefixed with /.

{{ hero_heading }}Welcome to my site{{ /hero_heading }}
{{ cta_label }}Get in touch{{ /cta_label }}
<section class="hero">
  <h1>{{ hero_heading }}Build something great.{{ /hero_heading }}</h1>
  <p>{{ hero_subheading }}We design and develop digital products.{{ /hero_subheading }}</p>
  <a href="{{ cta_url | link }}" class="btn">{{ cta_label }}Learn more{{ /cta_label }}</a>
</section>
Note — The inline syntax {{ field "Default" }} is also supported for backwards compatibility, but wrapping tags are recommended because they support multiline content and HTML in defaults.

Filters

Append a filter with the pipe character | to change how a field is rendered and what input type appears in the admin editor.

| raw — Rich Text / HTML

Outputs the field value without HTML escaping. Use this for fields that contain richtext editor content (TipTap). Content is sanitized before it is stored, so the output is safe.

{{ field_name | raw }}
<article>
  <h1>{{ post_title }}</h1>
  <div class="prose">
    {{ post_body | raw }}
  </div>
</article>
Warning — Only use | raw for content managed through the CMS richtext editor. Never use it for fields where an editor could enter arbitrary HTML from an untrusted source.

| image — Image URL

Opens the media picker in the admin editor. Outputs the file path or URL of the selected image. Use it in src attributes and CSS background properties.

{{ field_name | image }}
<!-- Standard image tag -->
<img src="{{ hero_image | image }}" alt="{{ hero_image_alt }}">

<!-- CSS background — note single quotes inside the attribute value -->
<div class="banner" style="background-image: url('{{ banner_photo | image }}')">
  <h2>{{ banner_heading }}</h2>
</div>

<!-- With a conditional to avoid an empty src attribute -->
{% if featured_image %}
  <img src="{{ featured_image | image }}" alt="{{ post_title }}" class="post-thumb">
{% endif %}

| link — URL / Link Input

Renders a URL input field in the admin editor. Outputs the URL as a plain string — use it in href attributes.

{{ field_name | link }}
<nav>
  <a href="{{ primary_cta_url | link }}" class="btn btn-primary">
    {{ primary_cta_label }}Get started{{ /primary_cta_label }}
  </a>
  <a href="{{ secondary_cta_url | link }}">
    {{ secondary_cta_label }}Learn more{{ /secondary_cta_label }}
  </a>
</nav>

| textarea — Multi-line Text

Provides a multi-line textarea in the admin editor. Newlines in the stored value are converted to <br> tags on output. Ideal for bios, addresses, and short descriptions where paragraph breaks matter but full richtext is unnecessary.

{{ field_name | textarea }}
<!-- Author bio -->
<aside class="author-card">
  <img src="{{ author_photo | image }}" alt="{{ author_name }}">
  <h3>{{ author_name }}</h3>
  <p>{{ author_bio | textarea }}</p>
</aside>

<!-- Contact address -->
<address>{{ office_address | textarea }}</address>

| select — Dropdown Selector

The admin editor shows a dropdown with options defined in the collection schema. Outputs the selected option's string value.

{{ field_name | select }}
<!-- Display a category badge -->
<article class="post-card">
  <span class="category">{{ post_category | select }}</span>
  <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
</article>

<!-- Use the value to add a CSS class -->
<div class="alert alert--{{ alert_type | select }}">
  {{ alert_message }}
</div>

| toggle — Boolean / Checkbox

Renders a toggle switch in the admin editor. Returns true or false — do not use as direct output. Use it inside a {% if %} tag instead.

{% if show_sidebar %}
  <aside class="sidebar">...</aside>
{% endif %}

{% if is_featured %}
  <span class="badge">Featured</span>
{% endif %}
Note — The filter declaration {{ field | toggle }} only needs to appear once in the template to register the field as a toggle type. In practice you will always reference the field inside {% if %} — the declaration is implicit.

| color — Color Picker

Opens a color picker in the admin editor. Outputs a hex color string like #14b8a6. Use it in inline CSS or pass it to CSS custom properties.

{{ field_name | color }}
<section style="background-color: {{ section_bg | color }}; color: {{ section_text | color }};">
  <h2>{{ section_heading }}</h2>
</section>

<!-- Set a CSS custom property for use across child elements -->
<div style="--brand: {{ brand_color | color }};">
  <button style="background: var(--brand);">{{ cta_label }}</button>
</div>

| number — Numeric Input

Renders a number input in the admin editor. Outputs the value as a string.

{{ field_name | number }}
<!-- Limit displayed in UI -->
<p>Showing up to {{ max_results | number }} results.</p>

<!-- Pass to JavaScript -->
<div id="slider" data-min="{{ slider_min | number }}" data-max="{{ slider_max | number }}"></div>

| date — Date Input

Renders a date picker in the admin editor. Outputs the stored date in ISO 8601 format: 2025-03-04. Format or localize it with JavaScript on the front end.

{{ field_name | date }}
<!-- Store as ISO and format client-side -->
<time datetime="{{ event_date | date }}">{{ event_date | date }}</time>

<script>
  document.querySelectorAll('time[datetime]').forEach(el => {
    const d = new Date(el.getAttribute('datetime'));
    el.textContent = d.toLocaleDateString('en-US', { year:'numeric', month:'long', day:'numeric' });
  });
</script>

| focal — Focal Point Position

Outputs the image's focal point as a CSS-ready X% Y% string. Use with object-position or background-position to ensure the important part of the image stays visible when cropped. Returns 50% 50% (center) if no custom focal point has been set.

{{ field_name | focal }}
<!-- Image with focal-point-aware cropping -->
<img
  src="{{ hero_image | image }}"
  alt="{{ hero_title }}"
  style="object-fit: cover; object-position: {{ hero_image | focal }};"
>

<!-- CSS background image -->
<div style="background-image: url('{{ banner | image }}'); background-size: cover; background-position: {{ banner | focal }};">
  ...
</div>

<!-- Collection item focal point -->
{% for post in collection.post %}
  <img
    src="{{ post.featured_image | image }}"
    style="object-fit: cover; object-position: {{ post.featured_image | focal }};"
  >
{% endfor %}
Note — Set the focal point by clicking on an image preview in the media library detail panel. The | focal filter looks up the stored coordinates for the media file and outputs them as percentages.

| or_body — Excerpt with Body Fallback

If the field (typically excerpt) is empty, this filter automatically generates a 40-word excerpt from the body field of the same item. This is the cleanest way to handle list pages where some items have manual excerpts and some do not.

{{ field_name | or_body }}
<!-- Blog post cards -- works whether or not an excerpt exists -->
{% for post in collection.post limit:10 %}
<article class="post-card">
  {% if post.featured_image %}
    <a href="{{ post.url }}">
      <img src="{{ post.featured_image }}" alt="{{ post.title }}">
    </a>
  {% endif %}
  <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
  <p>{{ post.excerpt | or_body }}</p>
  <a href="{{ post.url }}">Read more &rarr;</a>
</article>
{% endfor %}

| edit — On-Page Editing (Opt-In)

Append | edit to any field tag to enable on-page editing for that field. When a logged-in admin views the front-end site, fields marked with | edit become click-to-edit. Fields without | edit are only editable in the admin panel.

<!-- Editable on the front-end -->
{{ headline | edit }}
{{ body | raw | edit }}
{{ hero_image | image | edit }}
{{ @site_name | edit }}

<!-- NOT editable on front-end (admin panel only) -->
{{ headline }}
{{ body | raw }}
{{ hero_image | image }}

The | edit modifier works with all field types that support on-page editing: text, textarea, richtext (| raw), and image. It also works with global fields (@ prefix) and collection item fields inside {% single %} blocks:

<!-- Global field editable on front-end -->
{{ @company_name | edit }}

<!-- Collection item field editable in single view -->
{% single post from collection.post %}
  <h1>{{ post.title | edit }}</h1>
  {{ post.body | raw | edit }}
{% endsingle %}
Note — The | edit modifier only controls front-end editing. All fields remain fully editable in the admin panel regardless of this modifier. The field scanner strips | edit before detecting types, so {{ hero | image | edit }} correctly registers as an image field.

Loop Variable Fields

Inside a {% for %} or {% single %} block, access item fields using dot notation: {{ item.field }}. The same filters apply.

Tag Description
{{ item.field }} Escaped text output for any custom field.
{{ item.field | raw }} Raw HTML output — for richtext fields.
{{ item.url }} Auto-built URL from the collection's url_pattern (e.g. /post/my-slug).
{{ item.slug }} The item's URL slug.
{{ item.id }} Database integer ID.
{{ item.published_at }} Formatted publish date (e.g. "March 4, 2026").
{{ item.status }} Item status: published, draft, or scheduled.
{{ item.created_at }} Creation timestamp.
{{ item.updated_at }} Last-modified timestamp.
{{ item.featured_image }} Direct access to the featured image field (escaped URL).

Real-world Examples

Blog Post Card

{% for post in collection.post limit:6 %}
<article class="card">
  {% if post.featured_image %}
    <a href="{{ post.url }}" class="card-thumb">
      <img src="{{ post.featured_image }}" alt="{{ post.title }}">
    </a>
  {% endif %}
  <div class="card-body">
    <span class="label">{{ post.category | select }}</span>
    <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
    <p>{{ post.excerpt | or_body }}</p>
    <time datetime="{{ post.published_at }}">{{ post.published_at }}</time>
  </div>
</article>
{% else %}
<p>No posts published yet.</p>
{% endfor %}

Hero Section

<section class="hero" style="background-image: url('{{ hero_bg | image }}');">
  <div class="hero-inner">
    <h1>{{ hero_heading }}We build things people love.{{ /hero_heading }}</h1>
    <div class="hero-body">{{ hero_body | raw }}</div>
    {% if cta_url %}
      <a href="{{ cta_url | link }}" class="btn">{{ cta_label }}Get started{{ /cta_label }}</a>
    {% endif %}
  </div>
</section>

Portfolio Item (Single Page)

{% single project from collection.portfolio %}
<article class="project">
  <header>
    <h1>{{ project.title }}</h1>
    <span class="tag">{{ project.type | select }}</span>
    {% if project.client_url %}
      <a href="{{ project.client_url | link }}" target="_blank">View live site &rarr;</a>
    {% endif %}
  </header>
  {% if project.hero_image %}
    <img src="{{ project.hero_image }}" alt="{{ project.title }}" class="project-hero">
  {% endif %}
  <div class="prose">{{ project.body | raw }}</div>
  <div class="project-meta">
    <span>Completed: {{ project.completion_date | date }}</span>
    <span style="color: {{ project.brand_color | color }};">Brand color</span>
  </div>
</article>
{% else %}
<p>Project not found.</p>
{% endsingle %}
