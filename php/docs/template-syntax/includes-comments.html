Includes & Comments

Split templates into reusable partials with <outpost-include />, and use standard HTML comments for developer notes.

Includes

The <outpost-include /> element inserts another template file inline. It is the primary way to keep themes DRY — header, navigation, and footer markup is written once and shared across every page template.

This loads partials/partial-name.html from the site root directory. Do not include the file extension in the tag name.

Standard Partial Names

By convention, themes use these partial names. Sticking to these names makes themes easier to understand and share.

Tag File Contains
<outpost-include partial="head" /> partials/head.html DOCTYPE through opening <body> — charset, viewport, title, CSS, GA4
<outpost-include partial="nav" /> partials/nav.html Site header and navigation
<outpost-include partial="footer" /> partials/footer.html Footer content, closing </body></html>
<outpost-include partial="sidebar" /> partials/sidebar.html Sidebar content (optional)

Partial Name Restrictions

Partial names must be simple file names without path separators. Forward slashes, backslashes, and .. are rejected for security. All partials must live directly in the partials/ directory.

Context Inheritance

Partials have full access to all CMS functions and global variables. Loop variables from the parent template are also accessible inside an included partial. This enables component-style composition — write a card partial once and include it inside any loop.
