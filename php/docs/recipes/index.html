Each recipe is a complete walkthrough: collection setup, theme file structure, and copy-paste template code. Pick the one that matches your project and adapt it to fit.

What Outpost is great for

Outpost is intentionally focused. It handles the things that most content-driven sites need — and stays out of your way for everything else.

Use case How Outpost handles it
Blog or publication Collections with richtext, folders, pagination, SEO meta, and scheduled publishing
Marketing / brochure site Pages with richtext blocks, global fields for repeated content (phone, address, tagline)
Portfolio Collections with gallery fields, folders for project categories, filterable loop
Membership / newsletter Built-in member registration, login, free/paid tiers, per-page visibility
Headless backend Public content API — no auth required, CORS open, returns published content as JSON
Multi-author site User roles (admin, editor, author), author field on collection items
Docs / knowledge base Hierarchical folders, navigation menus with nesting, full-text search via API
Agency client sites Theme system with theme.json, Sync API for builder tools, no Composer dependencies

How these recipes are structured

Each recipe follows the same format:

  1. Admin setup — what to create in the Outpost dashboard (collections, fields, folders, globals, menus)
  2. Theme file structure — which template files you need and what each one does
  3. Complete template code — copy-paste examples for every file, using real Outpost template syntax
  4. Tips and variations — callouts for common gotchas, performance notes, and extensions
Tip The recipes build on each other. If you are new to Outpost, start with the Personal Blog recipe — it covers all the core concepts at length. The other recipes assume you have already worked through the blog example or are comfortable with collections, loops, and partials.

Primitives at a glance

All of Outpost's functionality is built from a small set of primitives. Understanding these makes any recipe straightforward to adapt:

Primitive What it is Template access
Collection A set of structured items with a schema you define (fields) {% for item in collection.slug %}
Page A standalone page with richtext blocks and a route Rendered directly by the router to a template
Folder A set of classification labels attached to a collection {% for label in folder.slug %}
Global field Site-wide key/value data (phone, logo, tagline) {{ @field_name }}
Menu An ordered list of links with optional nesting {% for item in menu.slug %}
Member A registered user with free or paid status {% if member %}, {% if paid_member %}
