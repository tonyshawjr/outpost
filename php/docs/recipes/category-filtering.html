Category Filtering

Let visitors filter your blog, portfolio, or resource library by folder label. Outpost handles the filtering server-side — you just add a filter nav and the filteredby modifier to your loop.

Folder Loops Pagination filteredby

How it works

Category filtering in Outpost is URL-driven. When a visitor clicks a category link, the page reloads with a query parameter such as ?category=design. The filteredby:category modifier on the collection loop reads that parameter and returns only items tagged with the matching label. No JavaScript is required for the filtering itself.

  1. Create a folder (e.g. category) and attach it to your collection
  2. Tag your collection items with one or more labels
  3. Add a filter nav using a folder loop
  4. Add filteredby:category to the collection loop
  5. Optionally add paginate:N — pagination preserves the filter param automatically

1. Folder and collection setup

In this recipe the collection is post and the folder is category. The same pattern applies to any collection and folder combination.

What to createSettings
Collection Name: Posts, Slug: post, URL pattern: /post/{slug}
Folder Name: Category, Slug: category, Attached to: post

Add some labels to the folder — for example: Design, Engineering, Business, Tutorial. Then tag several posts with each label.

2. Complete filterable archive template

This is the full blog.html template. It combines a folder loop (filter links), a filterable collection loop, and pagination.

{% include 'head' %}
{% include 'nav' %}

<main>
  <header class="archive-header">
    <h1>Blog</h1>
  </header>

  {#
    Filter navigation — folder loop renders all labels.
    Each link appends ?category=slug to the current page URL.
    The "All" link clears the filter by linking to the base URL.
  #}
  <nav class="filter-nav" aria-label="Filter by category">
    <a href="/blog"
       class="filter-link"
       data-label="">All</a>
    {% for label in folder.category %}
      <a href="/blog?category={{ label.slug }}"
         class="filter-link"
         data-label="{{ label.slug }}">
        {{ label.name }}
        <span class="filter-count">({{ label.count }})</span>
      </a>
    {% endfor %}
  </nav>

  {#
    Collection loop with both filteredby and paginate.
    - filteredby:category reads ?category=slug from the URL
    - paginate:12 adds server-side pagination
    - When no ?category= param is present, all posts are shown
  #}
  <div class="posts-grid">
    {% for post in collection.post filteredby:category paginate:12 %}
      <article class="post-card">
        {% if post.featured_image %}
          <a href="{{ post.url }}" class="post-card-thumb">
            <img src="{{ post.featured_image | image }}"
                 alt="{{ post.title }}"
                 loading="lazy">
          </a>
        {% endif %}
        <div class="post-card-body">
          <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
          {% if post.excerpt %}
            <p>{{ post.excerpt }}</p>
          {% endif %}
          <span class="post-date">{{ post.published_at }}</span>
        </div>
      </article>
    {% endfor %}
  </div>

  {# Pagination — automatically includes ?category= in page links #}
  {% pagination %}
</main>

{% include 'footer' %}

3. Pagination preserves the filter

When you combine filteredby:category and paginate:12, Outpost's pagination tag automatically carries the ?category= parameter into all page links. A visitor filtering by "Design" who clicks page 2 will see ?category=design&page=2 — not lose their filter.

Note This automatic parameter preservation only works with {% pagination %}. If you build your own pagination links manually, you will need to include the ?category= param yourself.

4. Active state with JavaScript

The Liquid template engine does not have access to the current URL query string inside a loop, so active-state highlighting must be done client-side. This small script reads the ?category= parameter and adds an active class to the matching filter link.

<script>
  const params = new URLSearchParams(location.search);
  const active = params.get('category') || '';

  document.querySelectorAll('.filter-link').forEach(link => {
    if (link.dataset.label === active) {
      link.classList.add('active');
      link.setAttribute('aria-current', 'true');
    }
  });
</script>

Add the corresponding CSS:

.filter-link {
  color: inherit;
  text-decoration: none;
  padding: 4px 10px;
  border-radius: 20px;
  font-size: 0.875rem;
  transition: background 0.15s;
}

.filter-link:hover {
  background: #f0f0f0;
}

.filter-link.active {
  background: #111;
  color: #fff;
}

.filter-count {
  opacity: 0.5;
  font-size: 0.8em;
}

5. Filtering a portfolio grid

The same pattern works for any collection. Here is the equivalent for a project collection with a category folder:

<nav class="filter-nav">
  <a href="/work" class="filter-link" data-label="">All</a>
  {% for label in folder.category %}
    <a href="/work?category={{ label.slug }}"
       class="filter-link"
       data-label="{{ label.slug }}">
      {{ label.name }}
    </a>
  {% endfor %}
</nav>

<div class="project-grid">
  {% for project in collection.project filteredby:category %}
    <a href="{{ project.url }}" class="project-card">
      {% if project.cover_image %}
        <img src="{{ project.cover_image | image }}" alt="{{ project.title }}">
      {% endif %}
      <span>{{ project.title }}</span>
    </a>
  {% endfor %}
</div>

6. Multiple folder filters

The filteredby modifier can only be used once per loop — it reads a single query parameter. If you need to filter by two folders at once (e.g. category + tag), you have two options:

  • Server-side: Use the Content API with filteredby and label params, then apply a second filter client-side in JavaScript
  • Client-side only: Load all items from the API and filter entirely in JavaScript using Array.filter()
// Client-side multi-filter example
const res = await fetch('/outpost/content/items?collection=post&limit=100');
const { data } = await res.json();

const category = new URLSearchParams(location.search).get('category');
const tag = new URLSearchParams(location.search).get('tag');

const filtered = data.items.filter(post => {
  const matchCategory = !category ||
    post.folder.category?.some(t => t.slug === category);
  const matchTag = !tag ||
    post.folder.tag?.some(t => t.slug === tag);
  return matchCategory && matchTag;
});

renderPosts(filtered);

7. Empty results fallback

When a category has no published posts, the {% for %} block produces no output. Add a fallback message after the loop using a flag variable set inside the loop.

<div class="posts-grid" id="posts-grid">
  {% for post in collection.post filteredby:category paginate:12 %}
    <article class="post-card">
      <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
    </article>
  {% endfor %}
</div>

{# Show a message if the grid has no children — JS approach #}
<p class="no-results" style="display:none">
  No posts found in this category.
</p>

<script>
  const grid = document.querySelector('#posts-grid');
  if (!grid.children.length) {
    document.querySelector('.no-results').style.display = 'block';
  }
</script>
Tip An alternative is to use the Content API to fetch the count for a given label before rendering, and show a message conditionally. But the JavaScript approach above is simpler and avoids an extra network request.

8. Using tag filtering instead of category

The same technique works with any folder slug. To filter by tag instead of category, replace filteredby:category with filteredby:tag and change the query parameter from ?category= to ?tag=:

<nav class="tag-nav">
  {% for label in folder.tag %}
    <a href="/blog?tag={{ label.slug }}"
       class="filter-link"
       data-label="{{ label.slug }}">
      {{ label.name }}
    </a>
  {% endfor %}
</nav>

{% for post in collection.post filteredby:tag paginate:10 %}
  ...
{% endfor %}
{% pagination %}

9. Displaying the current filter label

You may want to show the user a heading like "Showing: Design" above the results. Since the Liquid template does not have access to the query parameter, use JavaScript to inject the label from the active filter link's text.

<h2 class="results-label" id="results-label">All Posts</h2>

<script>
  const params = new URLSearchParams(location.search);
  const activeSlug = params.get('category') || '';
  const label = document.getElementById('results-label');

  if (activeSlug) {
    const activeLink = document.querySelector(
      `.filter-link[data-label="${activeSlug}"]`
    );
    if (activeLink) {
      const name = activeLink.textContent.trim().replace(/\s*\(\d+\)$/, '');
      label.textContent = name;
    }
  }
</script>

10. Search + filter combination

Outpost does not have a built-in template-level search. However you can combine the Content API with a client-side search for a filtered + searched experience:

<input type="text" id="search" placeholder="Search posts..." />
<div id="results"></div>

<script>
const BASE = '/outpost/content';
let allPosts = [];

async function loadAll() {
  const res = await fetch(`${BASE}/items?collection=post&limit=100`);
  const { data } = await res.json();
  allPosts = data.items;
  render(allPosts);
}

function render(posts) {
  const container = document.getElementById('results');
  container.innerHTML = posts.map(p => `
    <article>
      <h2><a href="${p.url}">${p.title}</a></h2>
      <p>${p.data.excerpt || ''}</p>
    </article>
  `).join('');
}

document.getElementById('search').addEventListener('input', e => {
  const q = e.target.value.toLowerCase();
  const filtered = allPosts.filter(p =>
    p.title.toLowerCase().includes(q) ||
    (p.data.excerpt || '').toLowerCase().includes(q)
  );
  render(filtered);
});

loadAll();
</script>
Tip For a Liquid-only site, the best approach to search is to redirect to the archive page with a ?search=query parameter, then use the JavaScript content API approach above to filter results client-side after loading.

11. Per-page counts in filter links

The label.count property in the folder loop returns the total number of published items tagged with that label in the attached collection — regardless of any current filter. This is useful for giving users a preview of how many results they will see.

{% for label in folder.category %}
  {# label.count = total published items with this label #}
  {% if label.count > 0 %}
    <a href="/blog?category={{ label.slug }}"
       class="filter-link"
       data-label="{{ label.slug }}">
      {{ label.name }}
      <span class="filter-count">({{ label.count }})</span>
    </a>
  {% endif %}
{% endfor %}

The {% if label.count > 0 %} condition hides folder labels that have no published items. This keeps the filter nav clean and avoids linking to a filter that would return zero results.

12. Summary of all filter modifiers

A reference table of all the modifiers you can combine on a collection loop for filtering and pagination:

ModifierExampleEffect
limit:N limit:6 Return at most N items
paginate:N paginate:10 Show N items per page; read ?page= from URL
filteredby:folder filteredby:category Filter by the given folder; reads ?folder=slug
orderby:field orderby:title Sort by a field name
order:asc order:asc Sort ascending (default is descending by published_at)
related:var related:post Items sharing folder labels with the given single item variable

Modifiers can be combined in a single loop tag:

{# Filter by category, sort by title ascending, paginate 8 per page #}
{% for post in collection.post filteredby:category orderby:title order:asc paginate:8 %}
  <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
{% endfor %}
{% pagination %}
