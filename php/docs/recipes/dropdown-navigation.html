Dropdown Navigation

Build a multi-level navigation menu with parent items and nested dropdowns. This recipe covers the Liquid template, CSS hover and focus-within approach, JavaScript click-toggle for mobile, and accessibility attributes.

Navigation Menus Accessibility Mobile

1. Create the menu in Admin

Go to Admin → Navigation → New Menu. Give it the slug main. Add your top-level items — pages, custom URLs, or collection links. To create a dropdown, drag a child item under a parent item to indent it.

Note Outpost supports a maximum of two nesting levels: top-level items and their direct children. Grandchildren (three levels deep) are not supported. Plan your nav structure accordingly.

Available properties on each menu item:

PropertyDescription
item.labelThe link text
item.urlThe href value
item.target_self or _blank
item.childrenArray of child items (truthy when children exist)
child.labelChild link text
child.urlChild link href
child.targetChild link target

2. Complete dropdown template

This template handles both flat items (no children) and dropdown items (with children). It goes in partials/nav.html.

<header class="site-header">
  <div class="header-inner">

    {# Logo #}
    <a href="/" class="site-logo">{{ @site_name }}</a>

    {# Desktop nav #}
    <nav class="site-nav" id="site-nav" aria-label="Main navigation">
      {% for item in menu.main %}

        {% if item.children %}

          {# Parent item — has a dropdown #}
          <div class="nav-item has-dropdown">
            <button class="nav-link nav-parent"
                    aria-expanded="false"
                    aria-haspopup="true">
              {{ item.label }}
              <svg class="chevron"
                   width="10" height="6"
                   viewBox="0 0 10 6"
                   fill="none"
                   aria-hidden="true">
                <path d="M1 1L5 5L9 1"
                      stroke="currentColor"
                      stroke-width="1.5"
                      stroke-linecap="round"/>
              </svg>
            </button>

            <ul class="dropdown" role="menu">
              {% for child in item.children %}
                <li role="none">
                  <a href="{{ child.url }}"
                     class="dropdown-link"
                     role="menuitem"
                     target="{{ child.target }}">
                    {{ child.label }}
                  </a>
                </li>
              {% endfor %}
            </ul>
          </div>

        {% else %}

          {# Flat item — no dropdown #}
          <a href="{{ item.url }}"
             class="nav-link"
             target="{{ item.target }}">
            {{ item.label }}
          </a>

        {% endif %}
      {% endfor %}
    </nav>

    {# Mobile hamburger button #}
    <button class="nav-hamburger"
            id="nav-hamburger"
            aria-label="Toggle navigation"
            aria-controls="site-nav"
            aria-expanded="false">
      <span></span>
      <span></span>
      <span></span>
    </button>

  </div>
</header>

4. JavaScript — click toggle (mobile-friendly)

On mobile the hover CSS approach does not work. This script adds click-toggle behaviour using aria-expanded on the parent button. It also handles the hamburger menu toggle.

// Dropdown toggle for parent nav items
document.querySelectorAll('.has-dropdown .nav-parent').forEach(btn => {
  btn.addEventListener('click', () => {
    const expanded = btn.getAttribute('aria-expanded') === 'true';
    const newState = !expanded;

    // Close all other open dropdowns first
    document.querySelectorAll('.has-dropdown .nav-parent').forEach(other => {
      if (other !== btn) {
        other.setAttribute('aria-expanded', 'false');
      }
    });

    btn.setAttribute('aria-expanded', String(newState));
  });
});

// Close dropdowns when clicking outside
document.addEventListener('click', e => {
  if (!e.target.closest('.has-dropdown')) {
    document.querySelectorAll('.has-dropdown .nav-parent').forEach(btn => {
      btn.setAttribute('aria-expanded', 'false');
    });
  }
});

// Hamburger menu toggle
const hamburger = document.getElementById('nav-hamburger');
const nav = document.getElementById('site-nav');

if (hamburger && nav) {
  hamburger.addEventListener('click', () => {
    const open = nav.classList.toggle('is-open');
    hamburger.setAttribute('aria-expanded', String(open));
  });
}

// Keyboard: close dropdown on Escape
document.addEventListener('keydown', e => {
  if (e.key === 'Escape') {
    document.querySelectorAll('.has-dropdown .nav-parent').forEach(btn => {
      btn.setAttribute('aria-expanded', 'false');
      btn.focus();
    });
  }
});
Tip On desktop you get hover behaviour from CSS (fast, no JS needed). On mobile — where hover does not exist — the JavaScript click toggle takes over. Both use aria-expanded as the source of truth, which keeps behaviour consistent and accessible.

5. Animated hamburger icon

Add a cross animation to the hamburger when the nav is open:

/* Animate hamburger to X when open */
.nav-hamburger[aria-expanded="true"] span:nth-child(1) {
  transform: translateY(7px) rotate(45deg);
}

.nav-hamburger[aria-expanded="true"] span:nth-child(2) {
  opacity: 0;
  transform: scaleX(0);
}

.nav-hamburger[aria-expanded="true"] span:nth-child(3) {
  transform: translateY(-7px) rotate(-45deg);
}

6. Accessibility notes

The template and JavaScript in this recipe follow these accessibility practices:

PatternWhy
aria-expanded on the parent button Tells screen readers whether the dropdown is open or closed
aria-haspopup="true" Announces that the button controls a popup menu
role="menu" on <ul> Identifies the list as a navigation menu to assistive technology
role="none" on <li> Removes the default list-item role (it is already wrapped by role="menu")
role="menuitem" on child links Identifies each link as a menu item within the menu role
aria-hidden="true" on the SVG chevron Hides the decorative icon from screen readers
focus-within CSS selector Keeps the dropdown open when keyboard focus is inside it
Escape key closes dropdowns Standard keyboard pattern for menus (ARIA APG)
Note The ARIA Menu Button pattern technically requires arrow key navigation within the dropdown. For a simple site nav this is usually unnecessary overhead. If you need full ARIA compliance (e.g. for a web app), implement ArrowDown, ArrowUp, Home, and End key handlers within the dropdown list.

7. Nesting limit

Outpost's navigation system supports one level of children — parent items and their direct children. Grandchildren (three levels deep) are not stored or exposed in the menu data. If you need three levels, the recommended approach is to use two separate menus: a primary nav and a secondary contextual nav on relevant pages.

{# Supported structure: #}
{% for item in menu.main %}
  {# item = top level #}
  {% if item.children %}
    {% for child in item.children %}
      {# child = one level deep — this is the maximum #}
    {% endfor %}
  {% endif %}
{% endfor %}

{# Not supported — child.children does not exist #}
