Database Schema

Complete reference for every SQLite table in Outpost CMS — columns, types, constraints, and relationships.

Single-file SQLite. Outpost stores everything in one .db file, by default at outpost/data/cms.db. The path is set by OUTPOST_DB_PATH. Back it up by copying that one file.

The schema is created and evolved via idempotent migrations that run automatically on every boot. You never need to run SQL manually to set up or upgrade a site. See Migrations for the full pattern.

Table Index

Outpost uses 18 tables organised into four functional groups.

GroupTables
Contentpages, fields, collections, collection_items
Folder & Navigationfolders, labels, item_labels, menus
Users & Membersusers, members, member_events, settings
Media, Forms & Logsmedia, form_configs, form_submissions, analytics_hits, analytics_searches, activity_log, revisions

pages

One row per page on the site. Pages are auto-discovered on first visit — Outpost creates a stub record the first time a URL is requested, so editors can immediately add content without manual setup.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
titleTEXTHuman-readable page title used in admin lists.
slugTEXT UNIQUEURL-safe identifier derived from the path (e.g. about, blog).
bodyTEXTMain richtext content stored as sanitized HTML.
meta_titleTEXTSEO <title> override. Falls back to title if empty.
meta_descriptionTEXTSEO meta description. Used in <meta name="description">.
og_imageTEXTOpen Graph image path for social sharing previews.
visibilityTEXT'public' (default), 'members', or 'paid'. Controls front-end access gating.
statusTEXT'published' or 'draft'. Draft pages are not served to visitors.
sort_orderINTEGERPosition in the admin pages list. Drag-reorderable.
created_atTEXTISO 8601 UTC timestamp set on insert.
updated_atTEXTISO 8601 UTC timestamp updated on every save.
lockedINTEGER0 (default) or 1. When set to 1, non-admin roles cannot update or delete this page. Only admin and super_admin can toggle this column.

fields

Stores custom field definitions and values for pages. Fields are auto-registered the first time a template tag is encountered. One row per unique (page_id, theme, field_name) combination. In v5 (no theme layer), the theme column is always an empty string ''.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
page_idINTEGER FK → pagesThe page this field belongs to. Cascades on page delete.
themeTEXTLegacy column from the theme system. In v5, always an empty string ''. Retained for schema compatibility.
field_nameTEXTIdentifier matching the template tag (e.g. hero_title).
field_typeTEXTField kind: text, textarea, richtext, image, link, select, toggle, color, number, date, gallery, repeater.
contentTEXTThe stored value. JSON for gallery and repeater types.
default_valueTEXTFallback value shown before an editor fills the field.
optionsTEXTJSON — extra metadata (e.g. select option list, repeater schema).
sort_orderINTEGEROrder in which fields appear in the editor panel.
updated_atTEXTISO 8601 timestamp of last edit.
UNIQUE constraint: (page_id, theme, field_name). The theme column is a legacy artifact from the multi-theme era. In v5, it is always '', making the effective unique key (page_id, field_name).

collections

Defines a content type — a named set of structured items like blog posts, products, or team members. The schema is a JSON object describing the fields each item has.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
nameTEXTDisplay name shown in admin (e.g. "Blog Posts").
slugTEXT UNIQUEURL and template identifier (e.g. post). Used in <outpost-each collection="post">.
schemaTEXTJSON array of field definitions. Each entry: {"name":"title","type":"text"}.
url_patternTEXTURL template for single items, e.g. /post/{slug}. The {slug} token is replaced at render time.
sort_fieldTEXTDefault sort column (e.g. created_at, title).
sort_directionTEXTASC or DESC.
items_per_pageINTEGERDefault items shown per page when no limit: is specified.
template_pathTEXTCustom template file override. If empty, routing uses the default {slug}.html pattern.
statusTEXT'active' or 'archived'.
created_atTEXTISO 8601 timestamp.

collection_items

One row per item in any collection. Field values are stored in the JSON data column, keyed by field name. This schema-less approach allows each collection to have its own field set without requiring a table per collection type.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
collection_idINTEGER FK → collectionsWhich collection this item belongs to.
titleTEXTPrimary title — also stored in data for template access.
slugTEXTURL-safe identifier. Used in url_pattern substitution.
dataTEXTJSON object containing all custom field values keyed by field name.
statusTEXT'published', 'draft', or 'scheduled'.
scheduled_atTEXTISO 8601 — when the item auto-publishes. Only relevant when status = 'scheduled'.
meta_titleTEXTSEO title override for the single item page.
meta_descriptionTEXTSEO description override.
og_imageTEXTOpen Graph image for social sharing.
sort_orderINTEGERManual drag-sort position in the admin item list.
created_atTEXTISO 8601 creation timestamp.
updated_atTEXTISO 8601 last-modified timestamp.
published_atTEXTISO 8601 timestamp of when the item was first published. Used for display dates in templates.
Template access. In collection loops, <span data-outpost="published_at">Date</span> outputs a human-readable date like "March 4, 2026". The raw ISO value is not exposed to templates.

media

Tracks every file uploaded through the Media manager. The actual files are stored in uploads/ on disk; this table holds metadata.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
filenameTEXTThe name of the file on disk (may be renamed from the original to avoid conflicts).
original_nameTEXTThe original filename as uploaded by the user.
mime_typeTEXTMIME type, e.g. image/jpeg, image/svg+xml, application/pdf.
file_sizeINTEGERFile size in bytes.
widthINTEGERImage width in pixels. NULL for non-image files.
heightINTEGERImage height in pixels. NULL for non-image files.
created_atTEXTISO 8601 upload timestamp.

users

Admin user accounts. Passwords are hashed with PHP's password_hash() using bcrypt. Never store or log plain-text passwords.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
nameTEXTDisplay name shown in the admin UI and activity log.
emailTEXT UNIQUELogin email address.
passwordTEXTbcrypt password hash.
roleTEXTAdmin role: super_admin, admin, developer, or editor. See Admin Roles for permissions.
created_atTEXTISO 8601 timestamp.
updated_atTEXTISO 8601 timestamp.

user_collection_grants

Many-to-many relationship restricting which collections an editor-role user can access. If an editor has no rows in this table, they have unrestricted access to all collections (backward compatible).

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
user_idINTEGER FK → usersThe editor user. Cascades on user delete.
collection_idINTEGER FK → collectionsThe granted collection. Cascades on collection delete.
UNIQUE constraint: (user_id, collection_id). Prevents duplicate grants.

members

Front-end member accounts for gated content. Separate from admin users — members cannot log into the Outpost admin panel.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
nameTEXTDisplay name.
emailTEXT UNIQUELogin email address.
passwordTEXTbcrypt password hash.
roleTEXT'free_member' or 'paid_member'. Used to gate visibility = 'paid' pages.
statusTEXT'active' or 'suspended'. Suspended members cannot log in.
email_verifiedTEXTISO 8601 timestamp of when the email was verified. NULL if unverified. Auto-set on registration when verification is disabled.
verify_tokenTEXT64-character hex token for email verification. Cleared after verification.
verify_token_expiresTEXTISO 8601 timestamp when the verification token expires (24 hours).
reset_tokenTEXT64-character hex token for password reset. Cleared after use.
reset_token_expiresTEXTISO 8601 timestamp when the reset token expires (1 hour).
created_atTEXTISO 8601 registration timestamp.
updated_atTEXTISO 8601 last-updated timestamp.

settings

A flat key-value store for site configuration, global field values, and migration tracking flags. Keys are arbitrary strings; values are always TEXT.

ColumnTypeDescription
keyTEXT PKSetting identifier (e.g. site_title, migration_v2_done).
valueTEXTSetting value — always stored as a string, cast as needed in code.

Well-known keys used by Outpost internally:

KeyDescription
active_themeLegacy key from the multi-theme era. Not used in v5 (no theme layer). May still exist in databases that were migrated from earlier versions.
site_titleSite display name.
installedSet to 1 after initial setup wizard completes.
require_email_verification'0' (default) or '1'. When enabled, new members must verify their email before signing in.
migration_*Flags set by one-time data migrations to prevent re-running them.

folders

Defines a classification system (e.g. Categories, Tags, Topics). A folder can be flat or hierarchical. Custom field schemas can be attached to labels.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
nameTEXTDisplay name (e.g. "Categories").
slugTEXT UNIQUEIdentifier used in templates: <outpost-each folder="category">.
typeTEXT'flat' (tags) or 'hierarchical' (categories with parent/child nesting).
schemaTEXTJSON array of custom field definitions attached to each label (e.g. a label image or description).
created_atTEXTISO 8601 timestamp.

labels

Individual folder labels — a category, tag, or any classified label. Custom field values for each label are stored in the JSON data column.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
folder_idINTEGER FK → foldersWhich folder this label belongs to.
nameTEXTDisplay name (e.g. "JavaScript").
slugTEXTURL-safe identifier (e.g. javascript).
descriptionTEXTOptional plain-text description shown in folder listings.
parent_idINTEGER0 for top-level labels; otherwise the id of the parent label (hierarchical only).
dataTEXTJSON object of custom field values defined by the folder's schema.
sort_orderINTEGERManual sort position within the folder.

item_labels

Junction table connecting collection items to folder labels. No primary key — the composite (item_id, label_id) is the natural key.

ColumnTypeDescription
item_idINTEGER FK → collection_itemsThe collection item.
label_idINTEGER FK → labelsThe folder label.
This table is what powers folder filtering in collection loops. When you use filteredby:category in a loop, Outpost joins through item_labels to find items tagged with the requested label slug.

menus

Navigation menu definitions. Items and their nested children are stored as a single JSON array. Outpost supports unlimited nesting depth, though templates typically only render one level of children.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
nameTEXTDisplay name in the admin (e.g. "Main Navigation").
slugTEXT UNIQUEIdentifier used in templates: <outpost-menu name="main">.
itemsTEXTJSON array of menu item objects. Each item: {"label":"Home","url":"/","target":"_self","children":[]}.
created_atTEXTISO 8601 timestamp.

form_configs

Defines a contact form — its fields, notifications, and spam protection settings. The slug matches the name attribute on the HTML <form> element.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
nameTEXTDisplay name in admin (e.g. "Contact Form").
slugTEXT UNIQUEMachine identifier. Must match the HTML form's name attribute for submissions to be recorded.
fieldsTEXTJSON array describing form fields: label, type, required flag.
notify_emailTEXTEmail address that receives a notification on new submission.
redirect_urlTEXTURL to redirect the visitor to after a successful submission.
recaptcha_enabledINTEGER0 (disabled) or 1 (enabled). When enabled, requires valid reCAPTCHA v3 token.
created_atTEXTISO 8601 timestamp.

form_submissions

Every submitted form response. IP addresses are stored for spam review — ensure your privacy policy discloses this if operating in GDPR jurisdictions.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
form_nameTEXTMatches form_configs.slug. Links submission to its form definition.
dataTEXTJSON object of submitted field values, keyed by field name.
ip_addressTEXTSubmitter's IP address, stored for spam review purposes.
created_atTEXTISO 8601 submission timestamp.

analytics_hits

Records a row for every page view tracked by Outpost's built-in analytics. The tracker fires a lightweight JS beacon; no cookies are used and IPs are hashed for privacy.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
urlTEXTThe page path that was viewed (e.g. /blog/my-post).
referrerTEXTThe document.referrer value — where the visitor came from. Empty for direct visits.
ip_hashTEXTSHA-256 hash of the visitor's IP + a daily salt. Never stored as plain text.
user_agentTEXTBrowser user agent string for device/browser breakdown reporting.
country_codeTEXT2-letter ISO country code from MaxMind GeoLite2 lookup. NULL when geo enrichment is not enabled. Added in v2.3.
created_atTEXTISO 8601 timestamp of the page view.

analytics_searches

Records every search query made on the front-end site search. Tracks what visitors searched for, how many results were returned, and which result (if any) they clicked. Added in v2.3.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
queryTEXT NOT NULLThe search query string entered by the visitor.
results_countINTEGERNumber of results returned for the query. Defaults to 0.
clicked_pathTEXTThe URL path of the result the visitor clicked. NULL if they didn't click any result.
session_idTEXTAnonymised session identifier linking the search to a visitor session.
created_atDATETIMETimestamp of the search, defaults to CURRENT_TIMESTAMP.

member_events

Audit log of member lifecycle events — signups, logins, role changes, and email verifications. Used for member analytics and debugging authentication flows. Added in v2.3.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
user_idINTEGER FK → usersThe member who triggered the event. Set to NULL if the member is later deleted (ON DELETE SET NULL).
event_typeTEXT NOT NULLMachine-readable event code: signup, login, role_change, or email_verified.
detailsTEXTOptional JSON or plain-text context (e.g. previous role for role_change events). NULL when not applicable.
session_idTEXTSession identifier at the time of the event. NULL for server-initiated events.
created_atDATETIMETimestamp of the event, defaults to CURRENT_TIMESTAMP.

activity_log

Audit trail of admin actions. Every significant create, update, or delete action is recorded here with the acting user and a description of what changed.

ColumnTypeDescription
idINTEGER PKAuto-increment primary key.
user_idINTEGER FK → usersWhich admin user performed the action.
actionTEXTMachine-readable action code, e.g. created_item, deleted_page, published_item.
targetTEXTHuman-readable description of the affected resource, e.g. Post: "Hello World".
created_atTEXTISO 8601 timestamp of the action.

revisions

Automatic content snapshots. Every save of a page or collection item records the previous state here so it can be browsed and restored. A maximum of 25 revisions are kept per entity; older ones are pruned on each save. See Revision History.

ColumnTypeNotes
idINTEGER PKAuto-increment
entity_typeTEXT NOT NULL'page' or 'item'
entity_idINTEGER NOT NULLFK to pages.id or collection_items.id
dataTEXT NOT NULLJSON snapshot of all field content
metaTEXT NOT NULLJSON snapshot of title/slug/status metadata
created_byINTEGERFK to users.id
created_atTEXTDefault datetime('now')

Inspecting the Schema

You can inspect any table's schema at any time using SQLite's PRAGMA command:

-- List all tables
SELECT name FROM sqlite_master WHERE type = 'table' ORDER BY name;

-- Show columns for a table
PRAGMA table_info(collection_items);

-- Show all indexes
SELECT name, tbl_name FROM sqlite_master WHERE type = 'index';
Never modify the schema manually in a way that breaks Outpost's expectations. Adding new tables or columns is safe. Dropping columns or renaming existing ones will break the CMS. If you need schema changes, write a migration — see Migrations.
