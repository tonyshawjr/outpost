Security

Outpost is hardened against common web vulnerabilities. This page documents the security measures built into the CMS.

Authentication & Sessions

  • Session-based auth with CSRF token validation on all mutations.
  • Bcrypt password hashing (cost 12).
  • Session cookies use the secure flag in production (HTTPS-only).
  • Session lifetime: 24 hours (configurable in config.php).
  • TOTP two-factor authentication with backup codes.
  • TOTP codes are single-use (replay prevention).
  • TOTP verify endpoint rate limited: 5 attempts per 5 minutes per IP.

Rate Limiting

Outpost enforces per-endpoint rate limits to prevent brute-force attacks and abuse. Limits are tracked in the SQLite database and keyed by IP address, session, or API key depending on the endpoint.

EndpointLimitWindowKey
Admin login 5 attempts 60 seconds IP + session
Member login 5 attempts 60 seconds IP + session
TOTP verify 5 attempts 5 minutes IP
Forgot password 5 requests 5 minutes IP
Reset password 10 requests 5 minutes IP
Form submissions 5 per form 60 seconds IP
API mutations 60 requests 60 seconds API key

SSRF Prevention

All outbound HTTP requests (channels, webhooks) are validated before execution to prevent Server-Side Request Forgery attacks.

  • URL scheme must be http or https — file://, ftp://, gopher:// etc. are blocked.
  • Hostname is resolved to IP and checked against blocked ranges:
Blocked RangeCIDR / HostReason
Private (Class A) 10.0.0.0/8 Internal network access
Private (Class B) 172.16.0.0/12 Internal network access
Private (Class C) 192.168.0.0/16 Internal network access
Loopback 127.0.0.0/8 Localhost access
Link-local 169.254.0.0/16 Link-local network
Cloud metadata 169.254.169.254 AWS/GCP instance metadata endpoint
Cloud metadata metadata.google.internal GCP metadata hostname

Additionally, curl handles are restricted via CURLOPT_PROTOCOLS and CURLOPT_REDIR_PROTOCOLS to prevent protocol downgrade after redirects.

File Upload Security

  • Extension allowlist: jpg, jpeg, png, gif, webp, avif, svg, pdf, mp4, webm.
  • MIME type validation — the server checks the file's MIME type matches its extension.
  • SVG sanitization — SVG files are parsed with a DOM parser. The following are removed: <script>, <foreignObject>, all on* event handler attributes, and dangerous URI schemes (javascript:, data:, vbscript:).
  • Path traversal prevention — all file paths are validated with realpath() scoping to ensure uploads cannot escape the uploads/ directory.

HTML Sanitization

All richtext/HTML content is sanitized before storage. See the Sanitization reference for the full allowlist of tags and attributes.

  • Only allowlisted elements and attributes are preserved — everything else is stripped.
  • URI schemes are restricted to http, https, mailto, and relative paths.
  • Dangerous schemes (javascript:, data:, vbscript:) are blocked.
  • Whitespace and control characters are stripped before scheme matching to prevent bypass via java\nscript: or similar tricks.

API Key Security

  • API keys are hashed with bcrypt before storage. The plaintext key is shown exactly once at creation time and cannot be retrieved again.
  • Lookup uses a prefix-based approach for O(1) database lookups — the system does not need to scan and compare against all stored keys.
  • Per-key usage tracking and rate limiting (60 requests per 60 seconds).
// API key format: outpost_XXXX_YYYYYYYYYYYY
// XXXX = prefix (stored plaintext for fast lookup)
// YYYYYYYYYYYY = secret (stored as bcrypt hash)

CORS

Cross-Origin Resource Sharing is handled with strict origin matching to prevent unauthorized cross-origin access.

  • Only exact localhost or 127.0.0.1 origins are accepted during development.
  • No substring matching — this prevents bypass via domains like evil-localhost.com.

Auto-Updater Security

The built-in auto-updater at Settings → Updates includes multiple safeguards.

  • Download URL validation: the URL must point to github.com with the correct repository path.
  • HTTPS enforced for all update downloads.
  • Zip slip prevention: all zip entry paths are validated before extraction to ensure no file can be written outside the target directory.
  • Role gated: only super_admin users can trigger updates.

Path Traversal Prevention

All file operations are scoped to their intended directories using realpath() containment checks.

Template cache cache/templates/ .htaccess deny from all

Access Control

Outpost uses a 6-role capability system. Each role inherits the permissions of roles below it.

RoleScope
super_admin Full access including settings, updates, user management, code editor, and API keys.
admin Full content management, member management, site configuration.
developer Code editor access, template editing, file management.
editor Create and edit pages, collections, media. No settings or code access.
paid_member Access to public, members-only, and paid-only pages on the front end.
free_member Access to public and members-only pages on the front end.

Page Visibility Gates

Page visibility is enforced at the router level in front-router.php. Each page has a visibility setting:

  • public — visible to all visitors (default).
  • members — requires an authenticated member session (free_member or above).
  • paid — requires an authenticated paid_member session.
Suspended member detection. Member sessions are periodically revalidated against the database on a 5-minute interval. If a member is suspended or deleted between page loads, their session is invalidated within 5 minutes.

SQL Injection Prevention

All database queries use PDO prepared statements with parameterized placeholders. No user input is ever interpolated directly into SQL strings.
