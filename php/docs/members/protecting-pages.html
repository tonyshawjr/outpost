Protecting Pages

Gate any CMS page behind a member login or paid membership using a simple visibility setting — no code required. For fine-grained in-page control, use PHP partials.

Page-level visibility

Every page managed through Admin → Pages has a visibility setting. Find it in the right sidebar of the page editor under the SEO / Settings panel. Three options are available:

Setting Who can access
Public Anyone — logged-in or not. This is the default.
Members Only Any logged-in member (both free_member and paid_member).
Paid Members Members with the paid_member role only.

The setting is saved to the pages.visibility column ('public', 'members', or 'paid') and enforced by both front-router.php and index.php before the template is rendered.

What happens when access is denied

The router checks the visitor's member session before rendering any protected page:

  • Not logged in → Members Only page: redirected to the member login page with the current URL passed as a ?redirect= parameter. After login, the member is returned to the page they tried to visit.
  • Free member → Paid Members page: redirected to the upgrade page so they can convert to a paid account.
  • Correct access: the template renders normally.
Note — Suspended members are blocked at login. A suspended member who somehow holds a valid session cookie will be rejected when the session is validated on each request.

Scope of page visibility

Visibility settings apply only to pages stored in the pages table — that is, CMS pages you create in Admin → Pages (e.g. /about, /resources, /community). Collection item pages (individual blog posts, portfolio items, etc.) are always public unless you implement custom PHP logic.

Tip — To gate collection items, use a PHP partial inside your template. See the section below on checking member status in PHP.

Protecting content within a template

For in-page control — showing different content to logged-in vs. anonymous visitors within the same Liquid template — use a PHP partial. Create a file in your theme's partials/ directory and call PHP functions there.

Blocking access from within a partial

<?php
// partials/member-gate.php
// Include this at the top of any template that requires a member login.
require_once $_SERVER['DOCUMENT_ROOT'] . '/outpost/members.php';

if (!OutpostMember::check()) {
    header('Location: /outpost/member-pages/login.php?redirect=' . urlencode($_SERVER['REQUEST_URI']));
    exit;
}
?>

Then include this partial at the top of the Liquid template:

{% include 'member-gate' %}

<!-- rest of template renders only for logged-in members -->
{{ title }}
{{ body | raw }}

Blocking paid-only content

<?php
// partials/paid-gate.php
require_once $_SERVER['DOCUMENT_ROOT'] . '/outpost/members.php';

$member = OutpostMember::current();

if (!$member) {
    header('Location: /outpost/member-pages/login.php?redirect=' . urlencode($_SERVER['REQUEST_URI']));
    exit;
}

if ($member['role'] !== 'paid_member') {
    header('Location: /upgrade');
    exit;
}
?>

Checking member status in PHP

The OutpostMember class in members.php provides two key methods:

OutpostMember::check()

Returns true if a member is currently logged in and active. Returns false otherwise (not logged in, or session invalid).

require_once '/path/to/outpost/members.php';

if (!OutpostMember::check()) {
    // visitor is not logged in — redirect or show teaser
}

OutpostMember::current()

Returns the full member record as an associative array, or null if no member is logged in.

$member = OutpostMember::current();

if ($member) {
    echo 'Welcome, ' . htmlspecialchars($member['display_name']);
}

if ($member && $member['role'] === 'paid_member') {
    // show paid content
}

if ($member && $member['member_expires'] && strtotime($member['member_expires']) < time()) {
    // membership has expired — treat as free or prompt to renew
}

Showing teaser content to non-members

A common pattern is to show a preview of content to anonymous visitors and reveal the full version only to members. Use a PHP partial to branch the output:

<?php
// partials/member-reveal.php
require_once $_SERVER['DOCUMENT_ROOT'] . '/outpost/members.php';
$GLOBALS['_outpost_member'] = OutpostMember::current();
?>

Then in your Liquid template, use a conditional include or render the teaser inline. Because Liquid does not have direct PHP access, the simplest approach is to split the protected section into its own partial and gate it there.

Login redirect

When a protected page redirects to login, it appends the original URL as a query parameter:

/outpost/member-pages/login.php?redirect=/resources/advanced-guide

The login page reads this parameter on successful authentication and sends the member directly to the page they wanted. You do not need to implement this yourself — it is built into the login page.

Warning — Never rely solely on Liquid template conditionals to hide sensitive content. A conditional in a template only controls what is rendered in the HTML output; it does not prevent the server from processing the request. Always use PHP-level checks (either via page visibility settings or PHP partials) to truly block access.
