Releases

Bundle multiple content changes together and publish them all at once — like a staging area for your site updates.

What are Releases?

A release is a way to group several changes and push them live all at the same time. Think of it like preparing a batch of updates behind the scenes, then flipping one switch to make everything go live.

Without releases, every edit you save goes live immediately. That works fine for small tweaks, but sometimes you need to coordinate bigger updates — a homepage redesign plus new blog posts plus updated navigation, all launching together on Monday morning.

Releases solve that problem. Create a release, make your changes, review everything as a bundle, then publish when you're ready.

Why Use Releases?

Here are some real situations where releases make your life easier:

  • Campaign launches. You're redesigning your homepage, updating three blog posts, and adding a new navigation link — all for a Monday campaign launch. With a release, you prepare everything over the week and publish it all at 9 AM Monday.
  • Avoiding a broken-looking site. If you need to change your homepage headline, update the about page, and swap out a hero image, your site might look inconsistent while you're making those changes one by one. A release lets you make all the edits privately and go live in one step.
  • Easy rollback. Something went wrong after publishing? Click "Rollback" and every change in the release is undone instantly. No need to remember what you changed or fix things one by one.
  • Team coordination. Multiple people can add changes to the same release. When the release is published, everyone's work goes live together.

How to Use Releases

Creating a release

  1. Open Releases. Click Releases in the sidebar (under the Content section).
  2. Create a new release. Click the + New Release button.
  3. Name it. Give your release a descriptive name, like "Spring Campaign" or "Homepage Redesign." You can also add an optional description to note what this release is about.
  4. Click Create. Your release is now in Draft status and ready for changes.

Adding changes to a release

Once you have a draft release, changes get added to it as you edit content. Each change captures a "before" and "after" snapshot, so the system knows exactly what changed and can undo it if needed.

Changes can include:

  • Page edits (updating text, images, or settings on any page)
  • Collection item changes (creating, editing, or deleting blog posts, products, etc.)
  • Field updates (changing global fields like site name or tagline)
  • Navigation changes (adding or removing menu items)
  • Collection configuration changes

Reviewing a release

Click on any release to see all the changes it contains. Each change shows:

  • What was changed (page title, collection item, field name, etc.)
  • What type of change it was (create, update, or delete)
  • When the change was added

You can remove individual changes from a draft release if you decide they shouldn't be part of this batch.

Publishing a release

  1. Open the release. Click on the release you want to publish.
  2. Review the changes. Make sure everything looks right.
  3. Click Publish. All changes are applied at once. Your site is updated instantly.

Publishing is atomic — that means either all changes go live, or none of them do. You won't end up with a half-updated site.

Note — Once a release is published, you can no longer add or remove changes from it. If you need to make more changes, create a new release.

Rolling back a release

Made a mistake? Changed your mind? Rollback undoes every change in the release, putting your site back to exactly how it was before.

  1. Find the published release. Go to Releases and click on the one you want to undo.
  2. Click Rollback. All changes are reverted in reverse order.

After rolling back, the release is marked as "Rolled Back." This is a final state — you can't re-publish or modify it. If you want to try again, create a new release.

Warning — Rollback reverts content to its state before the release was published. If someone made additional edits to the same content after the release was published, the rollback may overwrite those changes.

Release Statuses

Every release has one of three statuses:

Status What it means What you can do
Draft In progress. Changes are being collected but nothing is live yet. Add changes, remove changes, edit the name, publish, or delete the release entirely.
Published All changes have been applied and are live on your site. View the changes that were made. Roll back if something went wrong.
Rolled Back All changes have been undone. The site is back to how it was before. View what happened. This is a final state — the release cannot be changed.

  • "Roll back the last release"

Who Can Use Releases?

Releases are available to admins only. Editors and developers cannot create, publish, or roll back releases. This ensures that coordinated site updates are managed by people with full administrative access.

Frequently Asked Questions

Can I have multiple draft releases at the same time?

Yes. You can have as many draft releases open as you like. This is useful if you're preparing updates for different campaigns or events on different timelines.

What happens if I delete a draft release?

The release and all its recorded changes are deleted. No content on your site is affected — the changes were never published, so nothing was actually applied.

Do releases work with scheduled content?

Releases and scheduling are separate features. A release publishes its changes immediately when you click Publish. If you need time-based publishing, use the scheduling feature on individual collection items instead.

Is there a limit to how many changes a release can contain?

No hard limit. A release can contain as many changes as you need. However, very large releases (hundreds of changes) may take a moment longer to publish or roll back.
