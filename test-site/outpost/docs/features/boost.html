Boost

Comprehensive performance optimization for Outpost sites. Page caching, browser cache headers, GZIP compression, HTML minification, lazy loading, and database optimization — all configurable from the admin panel.

Overview

Boost is a built-in performance suite that makes Outpost sites fast without any external tools or plugins. It runs automatically once enabled, handling caching, compression, and optimization at every layer of the request lifecycle.

Configure everything from Settings → Boost in the admin panel.

Developer Mode — A single toggle at the top of the Boost settings page disables all caching and optimization. Use it during development to always see fresh content. When Developer Mode is on, Boost sends no-cache headers instead of caching headers.

Features

Page Caching

Boost caches the full rendered HTML output of every page for anonymous visitors. Subsequent requests for the same page are served directly from the cache file, bypassing all PHP processing, database queries, and template rendering.

  • Configurable TTL — how long cached pages live before they expire (default: 1 hour)
  • Path exclusions — glob patterns for pages that should never be cached (default: /outpost/* and /api/*)
  • Smart invalidation — cache is automatically cleared when Boost settings change
  • Cache preloading — one-click button crawls all page and collection item URLs to warm the cache

Pages are never cached for:

  • Logged-in users (session cookie present)
  • POST requests
  • Preview mode (?preview=TOKEN)
  • Requests with ?nocache
  • Paths matching exclusion patterns

Browser Cache Headers

Boost sets proper HTTP caching headers on static assets so browsers cache them locally. This dramatically reduces repeat-visit load times.

Asset TypeDefault TTLHeaders Set
CSS1 yearCache-Control, Expires, Last-Modified, ETag
JavaScript1 yearCache-Control, Expires, Last-Modified, ETag
Images30 daysCache-Control, Expires, Last-Modified, ETag
Fonts1 yearCache-Control, Expires, Last-Modified, ETag

Files with hashed filenames (e.g. app-abc123.js) are marked as immutable for maximum caching. Boost also handles If-None-Match and If-Modified-Since headers, returning 304 Not Modified when the file hasn't changed.

GZIP Compression

Boost compresses HTML responses using GZIP encoding, reducing transfer size by 60–80%. Compression is applied automatically based on the client's Accept-Encoding header.

Note — Requires the PHP zlib extension. If it's not available, compression is silently skipped. The Boost status panel shows whether GZIP is available.

HTML Minification

Strips unnecessary whitespace and comments from HTML output, reducing page size by 10–30%.

The following content is preserved exactly as-is:

  • <pre> blocks
  • <code> blocks
  • <script> blocks
  • <style> blocks
  • <textarea> blocks
  • Outpost block comment markers (<!-- outpost:hero -->)
  • IE conditional comments

Lazy Loading

Boost automatically adds loading="lazy" to <img> and <iframe> tags that don't already have a loading attribute. Images above the fold (the first N images on the page) are skipped to avoid delaying the initial paint.

  • Skip count — number of images to skip (default: 2). Set this to however many images appear in your hero/header area.

Database Optimization

One-click cleanup and compaction of the SQLite database.

The "Optimize Now" button:

  1. Removes expired rate limit entries
  2. Deletes activity log entries older than 90 days
  3. Deletes analytics data older than 1 year
  4. Clears expired preview tokens
  5. Runs VACUUM to reclaim disk space

API Endpoints

All endpoints require admin authentication.

GET boost/status

Returns full Boost status: configuration, page cache stats, template cache stats, database size, and compression availability.

GET boost/config

Returns the current Boost configuration.

PUT boost/config

Updates the Boost configuration. Automatically clears the page cache on any change.

PUT /outpost/api.php?action=boost/config
Content-Type: application/json

{
  "enabled": true,
  "developer_mode": false,
  "page_cache": true,
  "page_cache_ttl": 7200
}

POST boost/clear-cache

Clears all page cache and template cache files.

POST boost/preload

Warms the page cache by requesting all known page and collection item URLs. Returns the count of URLs warmed.

POST boost/optimize-db

Runs database cleanup and VACUUM. Returns details of what was cleaned.

How It Works

Request Lifecycle

  1. Static assets — Boost sets browser caching headers before the file is served. This happens in front-router.php, before any other PHP processing.
  2. Page cache check — For HTML page requests, Boost checks for a cached version. If found and not expired, it's served immediately with GZIP compression (if enabled). The request ends here.
  3. Output buffering — If no cache hit, Boost starts an output buffer. The normal Outpost rendering pipeline runs.
  4. Post-processing — When the buffer flushes, Boost applies lazy loading, HTML minification, and GZIP compression. The processed output is cached for future requests.

Developer Mode Priority

Developer Mode is the first check in every Boost function. When enabled:

  • Page cache is bypassed
  • Template cache is bypassed
  • Browser cache headers send no-cache, no-store, must-revalidate
  • HTML minification is skipped
  • GZIP compression is skipped
  • Lazy loading is skipped

Cache File Location

Page cache files are stored in outpost/cache/boost/ as MD5-hashed HTML files. Template cache files remain in outpost/cache/templates/.
