Sitemap

Outpost automatically generates an XML sitemap at /sitemap.xml and a /robots.txt file. No configuration or template changes needed — both update themselves as you publish content.

Auto-generation

A sitemap is available at /sitemap.xml from the moment your site is running. The first request generates and caches the sitemap; subsequent requests serve the cached version. The cache is invalidated automatically when pages or collection items are published or updated.

Note — Visit /sitemap.xml on your site to verify it is generating correctly before submitting it to search engines.

What is included

The sitemap contains all publicly accessible, published content:

  • All CMS pages with status = 'published'
  • All collection items with status = 'published' from collections that have sitemap_enabled turned on (the default)

Each entry in the sitemap includes:

  • <loc> — the full URL of the page (base URL auto-detected from the HTTP request host)
  • <lastmod> — the ISO 8601 timestamp of when the content was last updated
  • <changefreq> — a hint to crawlers about how often the content changes (see table below)
  • <priority> — a relative priority hint for crawlers (see table below)

Change frequency and priority

Outpost assigns <changefreq> and <priority> values automatically based on content type:

Content typechangefreqpriority
Homepage (/)daily1.0
Pagesweekly0.8
Collection itemsmonthly0.6
Tip — These values are hints, not directives. Search engines may crawl pages more or less frequently regardless of the values you set. Google has stated that it largely ignores <changefreq> and <priority>, but other crawlers (Bing, Yandex) may use them.

What is excluded

  • Draft pages and unpublished collection items
  • Admin paths (/outpost/)
  • Dynamically generated pages not managed in the CMS
  • Member-only and paid-only pages (visibility set to members or paid)
  • Collections with sitemap_enabled toggled off in collection settings

Per-collection control

Each collection has a sitemap_enabled setting that controls whether its items appear in the sitemap. This is enabled by default for all collections.

To exclude a collection from the sitemap:

  1. Go to Collections and open the collection you want to exclude.
  2. Open Collection Settings (the gear icon or settings tab).
  3. Toggle Include in sitemap off.
  4. Save. The sitemap cache is automatically invalidated.

This is useful for collections that hold internal data, draft-only content, or items that should not be indexed by search engines.

Warning — The sitemap base URL is auto-detected from the incoming HTTP request. If your site is behind a reverse proxy or load balancer that modifies the Host header, verify the URLs in your generated sitemap are correct.

robots.txt

Outpost automatically generates a /robots.txt file at your site root. The generated file:

  • Allows all user agents to crawl the site
  • Blocks crawling of the /outpost/ admin directory
  • Includes a Sitemap: directive pointing to your /sitemap.xml

The generated robots.txt looks like this:

User-agent: *
Allow: /
Disallow: /outpost/

Sitemap: https://yourdomain.com/sitemap.xml
Note — The robots.txt is generated dynamically by Outpost’s front router. No manual file creation is needed. The Sitemap URL uses the auto-detected base URL from the current request.

Submitting to Google Search Console

  1. Visit https://yourdomain.com/sitemap.xml to confirm it returns valid XML.
  2. Go to Google Search Console.
  3. Select your property.
  4. Navigate to Sitemaps in the left sidebar.
  5. Enter sitemap.xml in the “Add a new sitemap” field and click Submit.

Google will crawl the sitemap and report any errors. Resubmit after major content changes if you want to prompt a faster re-crawl, though Google will also recrawl automatically on its own schedule.

Submitting to Bing

  1. Go to Bing Webmaster Tools.
  2. Select your site.
  3. Navigate to Sitemaps.
  4. Enter your full sitemap URL: https://yourdomain.com/sitemap.xml and click Submit.

Excluding published pages from the sitemap

Draft pages are excluded automatically. If you need a page to be publicly accessible but absent from the sitemap — for example, a thank-you page or a login page — this requires a code modification to outpost/sitemap.php. Add the page’s slug or path to the exclusion list in that file.

Tip — For pages that should be completely hidden from search engines (not just the sitemap), add a <meta name="robots" content="noindex"> tag to that page’s <head> section in addition to excluding it from the sitemap.
