SEO

Outpost handles SEO automatically. The {% seo %} tag in your theme outputs title, description, Open Graph, Twitter Card, canonical URL, and JSON-LD — all pulled from the values you set in the admin. You just fill in the fields.

Where to update SEO in the admin

There are three places to manage SEO, from site-wide defaults down to individual pages and posts.

1. Site-wide defaults — Admin → Globals

These apply to every page on your site that doesn't have its own custom values set. Set them once and you're done.

Field nameWhat it doesExample
site_name Appended to every page title: Page Title — Site Name My Blog
site_tagline Fallback meta description for pages that have no description set Writing about design and code.
og_image Default social share image (1200×630px). Used when a page or post has no featured image. Upload via the image picker
twitter_handle Your Twitter/X handle for Twitter Card attribution. Include the @. @myhandle
Start here. Fill in site_name, site_tagline, and og_image in Globals first. This gives every page on your site a reasonable title, description, and social image without any per-page work.

2. Per-page SEO — Admin → Pages → click a page → SEO tab

Every page has a SEO tab in the page editor. Use this to override the site-wide defaults for that specific page.

FieldWhat it controls
Meta Title The <title> tag and og:title. If left blank, falls back to the page title + site name.
Meta Description The <meta name="description"> and og:description. If blank, falls back to the global tagline.
Visibility Set to Members or Paid to add a noindex robots tag and gate access to that page.

The SEO tab also shows a live search preview so you can see exactly how the page will appear in Google results before you save.

3. Per-post SEO — Admin → Collections → open a post → SEO tab

Individual posts and collection items work the same way. Open any item in the collection editor, find the SEO tab in the right sidebar, and set:

FieldWhat it controls
Meta Title Overrides the post title in <title> and OG tags
Meta Description Overrides the auto-generated excerpt in description tags

Even without setting these, posts get decent SEO automatically: the post title becomes the page title, the excerpt (or the first 40 words of body) becomes the description, and the featured_image becomes the social share image.

Tip — For blog posts, the most important thing to fill in is Excerpt (in the main post editor). That becomes your meta description automatically. You only need to touch the SEO tab if you want to write a different title or description than what's already in the post.

How values are resolved

Outpost walks down this chain and uses the first non-empty value it finds:

TagPriority (highest → lowest)
<title> Page SEO tab Meta Title → post Meta Title → post title field → site_name global
meta description Page SEO tab Meta Description → post Meta Description → post excerpt → first 40 words of body → site_tagline global
og:image / twitter:image Post featured_image field → og_image global
og:type article on individual posts; website on all other pages
twitter:site twitter_handle global — omitted if not set
canonical Built from your site's domain + current URL path, automatically

JSON-LD Article schema

On individual post pages, {% seo %} automatically outputs an Article structured data block. This is what Google uses to show article rich results — with date, author, and image in search.

You don't need to do anything for this to work. As long as a post has a title, published_at date, and optionally featured_image and author fields filled in, the JSON-LD is output automatically.

Protected pages

Pages with Visibility set to Members or Paid automatically get <meta name="robots" content="noindex">. They are excluded from search engine indexes.

For theme developers — the {% seo %} tag

If you're building a theme, all of the above is driven by a single tag in your head.html partial:

<!-- partials/head.html -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  {% seo %}

  <link rel="stylesheet" href="/outpost/themes/my-theme/assets/style.css">
</head>
<body>

No parameters. No manual OG tags. No {{ meta.title }} — that's all replaced by this one tag.

Note — {% seo %} replaces the legacy {{ meta.title }} and {{ meta.description }} tags. Don't use both — {% seo %} already outputs <title> and the description meta.

Canonical URL domain

Canonical tags use the current request's host by default. In production, set OUTPOST_SITE_URL in config.php to lock in the correct domain:

define('OUTPOST_SITE_URL', 'https://example.com'); // no trailing slash
