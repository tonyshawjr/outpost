RSS Feeds

Outpost auto-generates RSS 2.0 feeds for any collection. Visitors can subscribe in any feed reader — no configuration or template changes needed.

Overview

Outpost generates valid RSS 2.0 XML feeds for your site content. A site-wide feed aggregates items from all feed-enabled collections, and each collection with feeds enabled gets its own dedicated feed. Feeds update automatically whenever content is published or updated.

Feed URLs

Outpost responds to several URL patterns for feeds:

URLDescription
/feed.xmlSite-wide feed (all feed-enabled collections)
/feedAlias for /feed.xml
/rssAlias for /feed.xml
/rss.xmlAlias for /feed.xml
/{collection-slug}/feedPer-collection feed (e.g. /post/feed)
Tip — Use /feed.xml as the canonical feed URL when sharing or submitting to directories. The other aliases exist for convenience and compatibility.

Enabling feeds

Feeds are controlled per-collection with a feed_enabled toggle. By default, feeds are off for all collections.

To enable a feed for a collection:

  1. Go to Collections and open the collection.
  2. Open Collection Settings.
  3. Toggle Enable RSS feed on.
  4. Save.

The site-wide feed at /feed.xml only includes items from collections that have feeds enabled. If no collections have feeds enabled, the site-wide feed returns an empty channel.

Feed content

Each item in the feed includes:

  • <title> — the item’s title field
  • <link> — the full URL to the item on your site
  • <guid> — a permanent unique identifier (the item URL)
  • <pubDate> — the publication date in RFC 2822 format
  • <description> — the item’s excerpt if available, otherwise the first 500 characters of the body field with HTML stripped
  • <dc:creator> — the item’s author field, if present

Only published items are included. The feed returns the latest 50 items, sorted by publication date (newest first).

Auto-discovery

When feeds are enabled, Outpost automatically injects <link rel="alternate"> tags into the <head> of your pages via cms_seo() / <outpost-seo>:

<link rel="alternate" type="application/rss+xml"
      title="Site Name — Feed"
      href="https://yourdomain.com/feed.xml">

No manual template changes are needed. Feed readers and browsers that support RSS auto-discovery will detect your feeds automatically.

Note — Auto-discovery tags are only injected when at least one collection has feed_enabled turned on.

Feed metadata

The feed channel metadata is derived from your site settings:

  • Feed title — “Site Name — Collection Name” for per-collection feeds, or just “Site Name” for the site-wide feed
  • Feed description — your site tagline (from globals), or the collection name if no tagline is set
  • Feed link — your site’s base URL
  • Language — en (derived from site settings)

RSS readers

Outpost feeds are standard RSS 2.0 XML and work with any feed reader, including:

  • Feedly
  • NetNewsWire (macOS / iOS)
  • Inoreader
  • NewsBlur
  • Any other RSS 2.0 compatible reader

Customization

Feed output is generated by the CMS engine and does not use site templates. The content included in each feed item is determined automatically:

  • Description — uses the excerpt field if present, otherwise strips HTML from the body field and truncates to 500 characters
  • Author — uses the author field if present on the collection item
  • Title — uses the title field

To control what appears in your feed descriptions, add an excerpt field to your collection schema. Items with a dedicated excerpt will show that text instead of the auto-truncated body.

Tip — If you want to exclude a collection from the site-wide feed but still want its dedicated feed URL to work, that is not currently supported — the feed_enabled toggle controls both. If you need a per-collection feed, it will also contribute to the site-wide feed.
