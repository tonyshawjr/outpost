Ranger AI Assistant

Your built-in AI assistant that can do just about anything in Outpost — create pages, write content, build sites, manage your content — all by asking in plain English.

What is Ranger?

Ranger is like having a web developer on speed dial, right inside your admin panel. Instead of clicking through menus and filling out forms, you just tell Ranger what you want in your own words. It handles the rest.

Want a new blog collection with title, image, and body fields? Just ask. Need to update your site name, upload an image, or build an entire site from scratch? Ranger can do all of that — and more.

Ranger works with your choice of AI provider (Claude, OpenAI, or Google). You bring your own API key — no Outpost subscription or account needed.

Getting Started

Setting up Ranger takes about two minutes.

  1. Open Settings. Click Settings in the sidebar, then go to the Integrations tab.
  2. Pick a provider. Choose from Claude (Anthropic), OpenAI, or Google Gemini. We recommend Claude — it works best with Outpost (more on that below).
  3. Paste your API key. Get this from your provider's website. Outpost encrypts it before storing it — nobody can see it after you save.
  4. Choose a model. Each provider has different models. The default is a good starting point.
  5. Click Save. That's it for setup.
  6. Open Ranger. Click the sparkle icon in the top navigation bar, or click Ranger in the sidebar. Start chatting.

Ranger has access to every part of Outpost. Here are real examples you can type or paste directly into the chat.

Create and manage content

  • "Create a blog collection with title, excerpt, featured_image, and body fields"
  • "Add a new blog post titled 'Our Spring Sale' with some placeholder text"
  • "Update the About page title to 'About Our Team'"
  • "List all my collections"
  • "Delete the draft blog post called 'Test Post'"

Build and edit site files

  • "Build me a restaurant site with a homepage, menu page, and contact page"

Configure your site

Manage users and members

Use build tools

  • "Create a contact form with name, email, and message fields"

Get help and information

Tips for Getting the Best Results

Ranger is smart, but a little specificity goes a long way.

Be specific

Instead of "make a blog," try "create a blog collection with title, excerpt, featured_image, and body fields, using /blog/{slug} as the URL pattern." The more detail you give, the closer the result will be to what you want on the first try.

Paste screenshots for context

If something looks wrong on your site, paste a screenshot into the chat. Ranger can see images and use them to understand what you're working with. (Available with the Claude provider.)

Use the Stop button

If Ranger starts heading in the wrong direction, click Stop to halt the response. Then rephrase your request.

Edit previous messages

Made a typo or want to rephrase? You can edit a previous message and Ranger will re-process it with your updated wording.

Start fresh for new topics

If you switch from building site templates to managing users, start a new conversation. This keeps the context focused and prevents confusion.

Chain requests naturally

Ranger remembers your conversation. You can say "create a blog collection" and then follow up with "now add a featured_image field to it" without repeating the collection name.

Choosing a Provider

Ranger works with three AI providers. Each has trade-offs.

Provider Best for Notes
Claude (Anthropic) Everything Best tool use, largest output, prompt caching saves money after the first message. Recommended.
OpenAI (GPT) General use Solid alternative. Familiar if you already use ChatGPT. GPT-4o is fast and capable.
Google Gemini Budget-conscious use Cheapest per token. Good for simple tasks like listing content or updating settings.
Tip — If you're not sure which to pick, start with Claude. It handles complex multi-step tasks (like building an entire site) better than the alternatives.

Understanding Costs

Every message you send to Ranger uses "tokens" — think of them as units of text. Longer messages and more complex responses use more tokens, which cost money through your AI provider.

How billing works

  • You are billed directly by your AI provider (Anthropic, OpenAI, or Google) based on how many tokens you use.
  • Outpost itself is free — you only pay for the AI usage.
  • Costs vary by provider and model. A typical request costs a fraction of a cent.

The usage indicator

After every message, Ranger shows a small usage breakdown: how many tokens were used for input (your messages + system context) and output (Ranger's response), along with an estimated cost in USD.

Saving money with prompt caching

If you use Claude as your provider, Ranger automatically uses prompt caching. The first message in a conversation sends a large system prompt with all of Outpost's context. On every subsequent message, that context is reused from cache — reducing input token costs by up to 90%. This means the first message in a conversation is the most expensive, and follow-up messages are significantly cheaper.

Output Styles

You can customize how Ranger communicates. Go to Settings > Integrations and choose an output style:

Style What it does
Concise Short, direct answers. Just the facts, no extra explanation.
Detailed Thorough explanations with context and reasoning. Good for learning.
Casual Friendly and conversational. Less formal, more approachable.
Technical Precise, developer-oriented language. Includes code details and technical context.
Custom Write your own instructions for how Ranger should respond.

Conversation History

Ranger saves every conversation automatically. You can pick up where you left off, or review what you've done in the past.

  • Access history — click the history icon (clock) at the top of the Ranger panel to see all past conversations.
  • Load a conversation — click any conversation in the list to reload it and continue chatting.
  • Rename — click the title of a conversation to give it a more descriptive name.
  • Delete — remove conversations you no longer need. This is permanent.
  • Cost tracking — the history view shows total token usage and estimated cost for each conversation.

What Ranger Can Access

Different admin roles see different capabilities in Ranger. This is enforced by the server — Ranger cannot bypass your permissions.

Your role What Ranger can do
Editor Create and edit pages, collection items, global fields, and media. Cannot access settings, users, or site files.
Developer Everything an editor can do, plus site files, templates, and database queries.
Admin Full access to all 35 tools — including user management, site settings, backups, and email configuration.
Note — Ranger's permissions match what you can do manually in the admin panel. If you can't access the Settings page, Ranger can't change your settings either.

Security

Ranger is designed to be safe to use, even with powerful AI models.

  • Your API key is encrypted — stored using AES-256-GCM encryption. It cannot be read from the database.
  • File access is limited — Ranger can only read and write files inside your site root directory (excluding the outpost/ system directory). It cannot access system files.
  • Database queries are read-only — Ranger can look up data but cannot modify, delete, or create database tables directly.
  • External requests are filtered — when downloading images from URLs, Ranger blocks requests to private/internal networks.
  • All actions require authentication — every request goes through Outpost's normal session and CSRF protection.

Troubleshooting

Ranger says "No API key configured"

Go to Settings > Integrations, select a provider, and paste your API key. Make sure you click Save.

Responses are slow

Response speed depends on your AI provider. Complex requests (like building an entire site) take longer because Ranger is executing multiple steps. If basic responses are consistently slow, try switching to a faster model.

Ranger made a mistake

AI is not perfect. If Ranger creates something wrong, you can:

  • Ask Ranger to fix it: "That's not right, the URL pattern should be /blog/{slug} not /post/{slug}"
  • Undo it manually in the admin panel
  • Use the Stop button mid-response if you see it going off track

I hit my provider's rate limit

AI providers limit how many requests you can make per minute. Wait a moment and try again. If this happens frequently, check your provider's plan limits.
