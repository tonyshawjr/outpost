Ranger AI Assistant

An AI-powered assistant built into the admin panel — manage your entire CMS with natural language.

What is Ranger?

Ranger is an AI assistant that lives in your Outpost admin sidebar. It can do anything you can do in the admin panel — create pages, edit content, build themes, manage media, configure channels, set up forms, manage users, and more — all through a simple chat interface.

Ranger is powered by your choice of AI provider. Bring your own API key — no Outpost subscription or account required.

Getting Started

  1. Go to Settings > Integrations
  2. Choose an AI provider: Claude (Anthropic), OpenAI (GPT-4o/GPT-4), or Google (Gemini)
  3. Enter your API key — it is encrypted with AES-256-GCM before storage
  4. Optionally select a model and output style
  5. Click the Ranger icon in the admin sidebar to open the assistant

What Can Ranger Do?

Ranger has 35 tools covering every CMS operation:

Content Management

  • Create, update, delete, and list pages
  • Create and configure collections with schemas
  • Create, update, delete, and list collection items
  • Manage global field values

Theme & File Operations

  • Create, update, delete, and read theme files
  • List all files in the active theme
  • Debug templates for syntax errors

Media

  • Upload files to the media library
  • Download images from URLs and add to media

Configuration

  • Update site settings (name, tagline, email, etc.)
  • Configure SMTP email
  • Manage navigation menus and items
  • Manage folders and labels

Users & Members

  • List, create, update, or delete admin users
  • List, create, update, or delete site members

Build Tools

  • Create and manage forms, view submissions
  • Configure and sync external data channels
  • Set up webhooks
  • Create and manage backups

Developer Tools

  • Run read-only SQL queries against the database
  • Debug template files for errors
  • Search the developer documentation
  • Get system information (CMS version, PHP version, etc.)

Frontend Actions

  • Toggle dark mode
  • Open/close navigation

Features

Streaming Responses

Ranger streams responses in real time using Server-Sent Events (SSE). As the AI thinks and executes tools, you see the results appear immediately — no waiting for the full response to complete.

Conversation History

Conversations are automatically saved and can be continued later. The history panel shows all past conversations with preview text and timestamps. You can rename or delete conversations.

Token Usage & Cost Tracking

Every message shows input, output, and cache token counts with an estimated cost in USD. This helps you monitor usage and understand costs.

Prompt Caching (Claude)

When using Claude as your provider, Ranger uses prompt caching on the system prompt and tool definitions. After the first message in a session, subsequent messages reuse the cached prefix — reducing input token costs by up to 90%.

Dynamic Tool Loading

Ranger classifies each message by intent and sends only relevant tools to the AI provider. A content question sends ~13 tools instead of all 35, reducing token usage and improving response quality.

Screenshot Support

Paste a screenshot directly into the chat input for visual context. Available with the Claude provider.

Output Styles

Choose how Ranger communicates:

  • Concise — short, direct answers
  • Detailed — thorough explanations with context
  • Casual — friendly, conversational tone
  • Technical — precise, developer-oriented language
  • Custom — provide your own instructions

Role-Based Access

Ranger respects Outpost's role system. The tools available to each user depend on their role:

  • Editors — content tools (pages, items, globals, media)
  • Developers — content + code tools (themes, files, templates, database queries)
  • Admins — all 35 tools including users, settings, backups, and email configuration

Capability enforcement happens server-side on every tool execution — the AI cannot bypass role restrictions.

Security

  • API key encryption — stored using AES-256-GCM with a site-specific key
  • SSRF protection — outbound requests block private IP ranges and internal hostnames
  • Path traversal prevention — file operations are confined to the active theme directory
  • CSRF enforcement — all mutation endpoints require a valid token
  • SQL restrictions — database queries limited to SELECT statements only
  • XSS prevention — all AI output is sanitized before rendering
  • Error sanitization — internal errors are logged, never exposed to the client
