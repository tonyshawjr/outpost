Compass

Smart filtering, search, and faceted navigation for any collection. No PHP needed — just template tags. Drop in a search bar, category dropdown, or full faceted directory page with a few HTML elements.

What is Compass?

Compass is Outpost’s built-in filtering and search system. It turns any collection into a searchable, filterable directory — business listings, event calendars, product catalogs, job boards, member directories — without writing PHP or JavaScript.

Every Compass filter is a custom HTML element (<outpost-compass>) that you place in your template. Compass handles:

  • Text search across multiple fields
  • Dropdown, checkbox, radio, and toggle filters from folder labels or field values
  • Range sliders for numeric fields (price, rating, distance)
  • A–Z alphabetical filtering
  • Proximity / geolocation filtering (find businesses near me)
  • Nested category hierarchies
  • Relative date filtering (posted today, this week, this month)
  • Pagination and sorting
  • URL state sync — every filter selection is reflected in the URL for sharing and bookmarking
  • Automatic mobile flyout — filters collapse into a slide-out panel on small screens

Use cases:

  • Business directories with category, city, and keyword search
  • Event calendars filtered by date, type, and location
  • Job boards with role type, salary range, and location filters
  • Product catalogs with price range, brand, and attribute filters
  • Real estate listings with price, bedrooms, and proximity search
  • Member directories with alphabetical browsing
  • Resource libraries with tag and date filtering

Quick Start

The simplest Compass setup is a search bar, a dropdown, and a results container. This gives you a fully functional filtered directory in three lines of template code:

<!-- Search + filter bar -->
<outpost-compass type="search" collection="business" fields="title,description" placeholder="Search businesses..."></outpost-compass>
<outpost-compass type="dropdown" source="folder:categories" collection="business" label="All Categories"></outpost-compass>

<!-- Results -->
<outpost-compass-results collection="business" layout="grid" columns="3">
  <article>
    <h3 data-outpost="title">Business Name</h3>
    <p data-outpost="description">Description</p>
    <a data-bind="href:url">View Details</a>
  </article>
</outpost-compass-results>

That’s it. Compass automatically wires the search input and dropdown to filter the results. No JavaScript, no PHP, no API calls to write.

Compass Filter Types

Every Compass filter uses the same base element with a type attribute. The collection attribute tells Compass which collection to filter. The source attribute tells it where to get the filter options from.

dropdown

A single-select dropdown menu. Ideal for category filters, city selectors, or any field with a manageable number of distinct values.

<!-- From folder labels -->
<outpost-compass type="dropdown" source="folder:categories" collection="business" label="All Categories"></outpost-compass>

<!-- From field values (auto-populated from existing data) -->
<outpost-compass type="dropdown" source="field:city" collection="business" label="All Cities"></outpost-compass>
AttributeRequiredDescription
sourceYesData source: folder:slug for folder labels, field:fieldname for distinct field values, or label:slug for a specific label group.
collectionYesCollection slug to filter.
labelNoPlaceholder text shown when nothing is selected (e.g., “All Categories”).
sortNoSort options: alpha (default), count, custom.

checkbox

Multi-select checkboxes. Visitors can select multiple values and results match any of the selected options (OR logic).

<outpost-compass type="checkbox" source="folder:amenities" collection="business" label="Amenities"></outpost-compass>

<!-- With show-count to display how many items match each option -->
<outpost-compass type="checkbox" source="folder:tags" collection="event" label="Tags" show-count></outpost-compass>
AttributeRequiredDescription
sourceYesData source for checkbox options.
collectionYesCollection slug.
labelNoGroup heading above the checkboxes.
show-countNoBoolean attribute. Show match counts next to each option.
maxNoMaximum visible options before “Show more” toggle. Default: 8.

search

Full-text search input. Searches across the specified fields using an indexed lookup. Supports debounced live-search or submit-on-enter behavior.

<outpost-compass type="search" collection="business" fields="title,description,address" placeholder="Search businesses, events, services..."></outpost-compass>
AttributeRequiredDescription
collectionYesCollection slug.
fieldsYesComma-separated list of field slugs to search across.
placeholderNoInput placeholder text.
liveNoBoolean attribute. Enable live-search with debounce (300ms). Without this, search triggers on Enter or form submit.
min-charsNoMinimum characters before search triggers. Default: 2.

radio

Single-select radio buttons. Like dropdown but all options are visible at once. Good for small option sets (3–6 values).

<outpost-compass type="radio" source="field:type" collection="event" label="Event Type"></outpost-compass>
AttributeRequiredDescription
sourceYesData source for radio options.
collectionYesCollection slug.
labelNoGroup heading above the radio buttons.
show-allNoBoolean attribute. Include an “All” option that clears the filter. Default: true.

range

Numeric range slider for filtering by price, rating, distance, or any numeric field. Renders a dual-handle slider with min/max inputs.

<outpost-compass type="range" source="field:price" collection="product" label="Price Range" min="0" max="500" step="10" prefix="$"></outpost-compass>

<!-- Star rating -->
<outpost-compass type="range" source="field:rating" collection="business" label="Rating" min="1" max="5" step="0.5"></outpost-compass>
AttributeRequiredDescription
sourceYesNumeric field to filter on (use field:fieldname).
collectionYesCollection slug.
labelNoLabel above the slider.
minNoMinimum value. Auto-detected from data if not set.
maxNoMaximum value. Auto-detected from data if not set.
stepNoStep increment. Default: 1.
prefixNoValue prefix (e.g., $).
suffixNoValue suffix (e.g., mi, km).

az

A–Z alphabetical index. Renders a horizontal letter bar. Clicking a letter filters results to items starting with that letter.

<outpost-compass type="az" collection="business" field="title"></outpost-compass>
AttributeRequiredDescription
collectionYesCollection slug.
fieldNoField to index by. Default: title.
show-countsNoBoolean attribute. Show item count per letter.
disable-emptyNoBoolean attribute. Grey out letters with no matching items.

toggle

A single on/off toggle or label-based boolean filter. Useful for “Open now”, “Featured”, “Free”, or any boolean/label flag.

<!-- Toggle a boolean field -->
<outpost-compass type="toggle" source="field:featured" collection="business" label="Featured Only"></outpost-compass>

<!-- Toggle a folder label -->
<outpost-compass type="toggle" source="label:open-now" collection="business" label="Open Now"></outpost-compass>
AttributeRequiredDescription
sourceYesField or label to toggle on.
collectionYesCollection slug.
labelNoLabel text next to the toggle switch.

proximity

Geolocation-based proximity filter. Uses the browser’s Geolocation API to find items near the visitor. Items must have latitude/longitude fields (or an address field that Compass geocodes on index).

<outpost-compass type="proximity" collection="business" lat-field="latitude" lng-field="longitude" label="Near Me" radius="25" unit="mi"></outpost-compass>

<!-- With address field (auto-geocoded) -->
<outpost-compass type="proximity" collection="business" address-field="address" label="Near Me" radius="10" unit="mi"></outpost-compass>
AttributeRequiredDescription
collectionYesCollection slug.
lat-fieldConditionalField slug for latitude. Required unless address-field is set.
lng-fieldConditionalField slug for longitude. Required unless address-field is set.
address-fieldConditionalField slug for address (auto-geocoded on index). Alternative to lat/lng fields.
labelNoButton/label text. Default: “Near Me”.
radiusNoDefault search radius. Default: 25.
unitNoDistance unit: mi (miles, default) or km (kilometers).
show-distanceNoBoolean attribute. Show distance to each result item.

hierarchy

Nested category browser for hierarchical taxonomies. Renders a collapsible tree or breadcrumb-style drill-down.

<outpost-compass type="hierarchy" source="folder:categories" collection="business" label="Categories" display="tree"></outpost-compass>

<!-- Breadcrumb-style drill-down -->
<outpost-compass type="hierarchy" source="folder:categories" collection="product" label="Browse" display="breadcrumb"></outpost-compass>
AttributeRequiredDescription
sourceYesFolder with nested labels (folder:slug).
collectionYesCollection slug.
labelNoHeading text.
displayNoDisplay mode: tree (default, collapsible) or breadcrumb (drill-down).
show-countNoBoolean attribute. Show item counts per category.

time-since

Relative date filter. Shows options like “Today”, “This week”, “This month”, “This year” based on a date field.

<outpost-compass type="time-since" source="field:created_at" collection="event" label="Posted"></outpost-compass>

<!-- Custom intervals -->
<outpost-compass type="time-since" source="field:start_date" collection="event" label="When" intervals="today,this-week,this-month,next-month"></outpost-compass>
AttributeRequiredDescription
sourceYesDate field to filter on (field:fieldname).
collectionYesCollection slug.
labelNoHeading text.
intervalsNoComma-separated list of intervals. Options: today, this-week, this-month, this-year, next-week, next-month. Default: all past intervals.

pager

Pagination control for Compass results. Renders page numbers, previous/next buttons, and item count.

<outpost-compass type="pager" collection="business" per-page="12"></outpost-compass>
AttributeRequiredDescription
collectionYesCollection slug (must match the <outpost-compass-results> collection).
per-pageNoItems per page. Default: 12.
max-pagesNoMaximum page buttons to show. Default: 7.
show-countNoBoolean attribute. Show “Showing 1–12 of 48” text.

sort

Sort control for Compass results. Renders a dropdown with sort options.

<outpost-compass type="sort" collection="business" options="title:asc=A-Z,title:desc=Z-A,created_at:desc=Newest,rating:desc=Top Rated"></outpost-compass>
AttributeRequiredDescription
collectionYesCollection slug.
optionsYesComma-separated sort options in field:direction=Label format.
defaultNoDefault sort option (e.g., title:asc). First option is used if not set.

Results Container

The <outpost-compass-results> element defines where filtered results render. It works like <outpost-each> — the inner HTML is the template for each item.

<outpost-compass-results collection="business" layout="grid" columns="3">
  <article class="card">
    <img data-outpost="featured_image" data-type="image" src="placeholder.jpg">
    <h3 data-outpost="title">Business Name</h3>
    <p data-outpost="description">Description text</p>
    <span data-outpost="city">City</span>
    <a data-bind="href:url">View Details</a>
  </article>
</outpost-compass-results>
AttributeRequiredDescription
collectionYesCollection slug. Must match the Compass filter elements on the page.
layoutNoLayout mode: grid (default), list, or table.
columnsNoGrid columns (2, 3, or 4). Default: 3. Only applies to grid layout.
empty-messageNoMessage shown when no results match. Default: “No results found.”
loadingNoLoading indicator: skeleton (default), spinner, or none.

You can also use a partial for the item template:

<outpost-compass-results collection="business" layout="grid" columns="3">
  <outpost-include partial="business-card"></outpost-include>
</outpost-compass-results>

Helper Tags

Result count

<outpost-compass-count collection="business">0 results</outpost-compass-count>

Displays the current number of filtered results. Updates live as filters change.

Reset all filters

<outpost-compass-reset collection="business">Clear all filters</outpost-compass-reset>

Renders a button/link that clears all active Compass filters for the collection.

Active selections

<outpost-compass-selections collection="business"></outpost-compass-selections>

Renders pill badges showing the currently active filters. Each pill has a remove button to clear that individual filter.

Source Types

The source attribute determines where filter options come from. There are three source types:

SourceSyntaxDescriptionExample
Folder labels folder:slug Options come from the labels in a collection folder. Labels are managed in the admin under the collection’s folder settings. source="folder:categories"
Field values field:fieldname Options are auto-populated from distinct values of a field across all items. Ideal for fields like “city” or “type” where the values aren’t predefined. source="field:city"
Label group label:slug Options from a specific label group within a folder. Useful when a folder has multiple label groups (e.g., “amenities” within a “features” folder). source="label:amenities"

API Endpoints

Compass has four API endpoints. You typically don’t call these directly — the template engine handles them — but they are available for headless usage or custom JavaScript integrations.

GET compass/values

GET api.php?action=compass/values&collection=slug&source=folder:categories

Returns the available filter options for a given source. Used to populate dropdown, checkbox, and radio filters.

Response:

{
  "values": [
    { "slug": "restaurants", "name": "Restaurants", "count": 24 },
    { "slug": "retail", "name": "Retail", "count": 18 },
    { "slug": "services", "name": "Services", "count": 31 }
  ]
}

POST compass/filter

POST api.php?action=compass/filter

Executes a filter query and returns matching items. Accepts all filter parameters as a JSON body.

Request body:

{
  "collection": "business",
  "filters": {
    "search": { "query": "pizza", "fields": ["title", "description"] },
    "folder:categories": ["restaurants"],
    "field:city": ["jacksonville"]
  },
  "sort": "title:asc",
  "page": 1,
  "per_page": 12
}

Response:

{
  "items": [ ... ],
  "total": 48,
  "page": 1,
  "pages": 4,
  "facets": {
    "folder:categories": [
      { "slug": "restaurants", "name": "Restaurants", "count": 8 }
    ]
  }
}

GET compass/proximity

GET api.php?action=compass/proximity&collection=slug&lat=34.75&lng=-77.43&radius=25&unit=mi

Returns items within a geographic radius, sorted by distance. Used by the proximity filter type.

Response:

{
  "items": [
    { "id": 1, "title": "Joe's Pizza", "distance": 1.2, "unit": "mi", ... },
    { "id": 2, "title": "Main Street Deli", "distance": 3.8, "unit": "mi", ... }
  ],
  "total": 15
}

POST compass/reindex

POST api.php?action=compass/reindex&collection=slug

Rebuilds the Compass search index for a collection. Requires admin authentication. The index rebuilds automatically on item create/update/delete, but this endpoint forces a full rebuild.

Response:

{
  "success": true,
  "indexed": 156,
  "collection": "business"
}

URL State

Every active filter is reflected in the URL query string. This means filtered views are shareable and bookmarkable.

/explore?q=pizza&category=restaurants&city=jacksonville&sort=rating:desc

Compass reads URL parameters on page load and pre-populates filters accordingly. When a visitor changes a filter, the URL updates without a full page reload (using history.pushState).

Mobile Behavior

On screens below 768px, Compass automatically collapses all filter elements (except search and sort) into a mobile flyout panel. A “Filters” button appears, and tapping it slides out a panel from the left or bottom with all filter controls.

You can control this behavior with attributes on the results container:

<outpost-compass-results collection="business" mobile-breakpoint="640" mobile-position="bottom">
  ...
</outpost-compass-results>
AttributeDefaultDescription
mobile-breakpoint768Screen width (in px) below which filters collapse into the flyout.
mobile-positionleftFlyout position: left, right, or bottom.

Indexing

Compass maintains a lightweight search index for each collection that has Compass filters in any rendered template. The index stores:

  • Tokenized text for full-text search fields
  • Distinct values for dropdown/checkbox/radio sources
  • Numeric ranges for slider fields
  • Geocoordinates for proximity search

The index is stored in the SQLite database (compass_index table) and rebuilds automatically when:

  • A collection item is created, updated, or deleted
  • A folder label is added, renamed, or removed
  • The admin triggers a manual reindex via compass/reindex

The first page load with Compass tags triggers an initial index build if one doesn’t exist. Subsequent updates are incremental (only re-indexing the changed item).

Styling

Compass elements render with semantic HTML and CSS class names. Override the default styles using CSS custom properties on the :root or scoped to a container.

:root {
  --compass-accent: #3b82f6;          /* Active filter highlight color */
  --compass-bg: #ffffff;               /* Filter background */
  --compass-border: #e5e7eb;           /* Border color */
  --compass-radius: 8px;               /* Border radius */
  --compass-font-size: 14px;           /* Base font size */
  --compass-gap: 12px;                 /* Spacing between filter elements */
  --compass-input-height: 40px;        /* Input/select height */
  --compass-pill-bg: #eff6ff;          /* Active filter pill background */
  --compass-pill-color: #1d4ed8;       /* Active filter pill text color */
  --compass-mobile-overlay: rgba(0,0,0,0.5); /* Mobile flyout overlay */
}

All Compass elements use these class names for targeting:

  • .compass-filter — every filter element
  • .compass-search — search input wrapper
  • .compass-dropdown — dropdown wrapper
  • .compass-checkbox-group — checkbox group
  • .compass-radio-group — radio group
  • .compass-range — range slider wrapper
  • .compass-az — A–Z letter bar
  • .compass-toggle — toggle switch
  • .compass-results — results container
  • .compass-pager — pagination
  • .compass-sort — sort dropdown
  • .compass-selections — active filter pills
  • .compass-count — result count display
  • .compass-reset — reset button
  • .compass-mobile-trigger — mobile “Filters” button
  • .compass-mobile-panel — mobile flyout panel

Full Example

Here is a complete directory page with search, category dropdown, city dropdown, checkbox amenities, sort, results grid, and pagination:

<!-- Search bar -->
<section class="search-bar">
  <outpost-compass type="search" collection="business" fields="title,description,address" placeholder="Search businesses..."></outpost-compass>
  <outpost-compass type="dropdown" source="folder:categories" collection="business" label="All Categories"></outpost-compass>
  <outpost-compass type="dropdown" source="field:city" collection="business" label="All Cities"></outpost-compass>
</section>

<!-- Active filters + sort -->
<section class="filter-bar">
  <outpost-compass-count collection="business">0 results</outpost-compass-count>
  <outpost-compass-selections collection="business"></outpost-compass-selections>
  <outpost-compass-reset collection="business">Clear all</outpost-compass-reset>
  <outpost-compass type="sort" collection="business" options="title:asc=A-Z,title:desc=Z-A,created_at:desc=Newest,rating:desc=Top Rated"></outpost-compass>
</section>

<div class="directory-layout">
  <!-- Sidebar filters -->
  <aside class="filters">
    <outpost-compass type="checkbox" source="folder:amenities" collection="business" label="Amenities" show-count></outpost-compass>
    <outpost-compass type="range" source="field:rating" collection="business" label="Rating" min="1" max="5" step="0.5"></outpost-compass>
    <outpost-compass type="toggle" source="field:featured" collection="business" label="Featured Only"></outpost-compass>
    <outpost-compass type="proximity" collection="business" address-field="address" label="Near Me" radius="25" unit="mi"></outpost-compass>
  </aside>

  <!-- Results -->
  <main>
    <outpost-compass-results collection="business" layout="grid" columns="3">
      <article class="card">
        <img data-outpost="featured_image" data-type="image" src="placeholder.jpg">
        <h3 data-outpost="title">Business Name</h3>
        <p data-outpost="description">Description</p>
        <span data-outpost="city">City</span>
        <a data-bind="href:url">View Details</a>
      </article>
    </outpost-compass-results>

    <outpost-compass type="pager" collection="business" per-page="12" show-count></outpost-compass>
  </main>
</div>
Tip — Compass filters on the same page automatically wire together. All <outpost-compass> elements with the same collection attribute filter the same <outpost-compass-results> container. No wiring code needed.
Performance — Compass uses an indexed SQLite table for all filter queries, keeping response times under 50ms even with thousands of items. The search index is compact (typically under 1MB for 10,000 items) and updates incrementally.
