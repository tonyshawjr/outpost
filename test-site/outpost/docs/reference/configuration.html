Configuration

All configuration constants defined in outpost/config.php — database paths, upload limits, caching, sessions, SMTP, and more.

Location: outpost/config.php. This file is created during initial setup. Constants are defined with PHP's define() and are available throughout the entire CMS. Never commit this file to version control if it contains credentials.

How Configuration Works

Outpost reads configuration exclusively through PHP constants. There is no .env parsing, no YAML, no JSON config — just define() in a single PHP file. This keeps the setup minimal and dependency-free.

Constants are read once per request. To change a setting, edit config.php and the next request picks it up automatically — no restart needed on PHP-FPM or Apache.

Minimal config.php

A fresh install only needs a few constants to work:

<?php
define('OUTPOST_SITE_URL', 'https://mysite.com');
define('OUTPOST_SYNC_KEY', 'your-random-secret-key');

Everything else has sensible defaults and can be added as needed.

Full Reference

Core

ConstantDefaultDescription
OUTPOST_DB_PATH content/data/cms.db Path to the SQLite database file, inside the content/ directory. Change this to store the database outside the web root for additional security.
OUTPOST_SITE_URL (auto-detected) Canonical site URL including scheme and domain, no trailing slash (e.g. https://mysite.com). Used for sitemap generation, password reset emails, and absolute URL construction. Auto-detects from $_SERVER if not set — always set this explicitly in production.
OUTPOST_DEBUG false When true, PHP errors and warnings are displayed inline. Never enable in production — error output can expose file paths, credentials, and SQL queries.

File Uploads

ConstantDefaultDescription
OUTPOST_MAX_UPLOAD_SIZE 10485760 (10 MB) Maximum allowed file upload size in bytes. Must also be within PHP's upload_max_filesize and post_max_size ini values — whichever is smaller wins.
OUTPOST_UPLOADS_DIR content/uploads/ Directory where uploaded media files are stored, inside the content/ directory. A symlink at outpost/uploads ensures existing URLs continue to work. Must be writable by the web server.

Image Processing

ConstantDefaultDescription
OUTPOST_WEBP_AUTO_CONVERT true When true, uploaded JPEG and PNG images are automatically converted to WebP format. GIF, SVG, WebP, and other formats are not converted. Requires the GD extension with WebP support.
OUTPOST_WEBP_QUALITY 85 WebP compression quality (0–100). Higher values produce better quality but larger files. Only applies when OUTPOST_WEBP_AUTO_CONVERT is enabled.

Caching

ConstantDefaultDescription
OUTPOST_CACHE_DIR cache/ Directory for all cache files (both compiled templates and HTML page cache), relative to outpost/. Must be writable by the web server.
OUTPOST_CACHE_TTL 3600 (1 hour) HTML page cache time-to-live in seconds. After this duration, the next request re-renders the page and writes a fresh cache entry.
OUTPOST_TEMPLATE_CACHE true Enable or disable the compiled template cache. Set to false during template development to always recompile templates on each request. Leave true in production.

Sessions

ConstantDefaultDescription
OUTPOST_SESSION_NAME outpost_session Cookie name for the admin session. Change this if you run multiple Outpost installations on the same domain to avoid session conflicts.
OUTPOST_MEMBER_SESSION outpost_member Cookie name for the front-end member session.
OUTPOST_SESSION_LIFETIME 86400 (24 hours) How long an admin session stays valid in seconds. After this, the admin must log in again.
OUTPOST_MEMBER_LIFETIME 2592000 (30 days) How long a member's "remember me" session stays valid in seconds.

Builder Sync

ConstantDefaultDescription
OUTPOST_SYNC_KEY (none) Secret key for the Builder sync API. Must match the key configured in the desktop Builder app. Generate a long random string (32+ characters). Requests without the correct key are rejected with HTTP 403.

Email (SMTP)

All SMTP constants are optional. If none are set, Outpost uses PHP's native mail() function as a fallback — which requires your server to have a working MTA configured. For reliable delivery, configure SMTP.

ConstantDefaultDescription
OUTPOST_SMTP_HOST (none) SMTP server hostname (e.g. smtp.mailgun.org, smtp.sendgrid.net).
OUTPOST_SMTP_PORT 587 SMTP port. Use 587 for STARTTLS (recommended), 465 for SSL/TLS, or 25 for plain SMTP (not recommended).
OUTPOST_SMTP_USER (none) SMTP authentication username.
OUTPOST_SMTP_PASS (none) SMTP authentication password.
OUTPOST_SMTP_FROM (none) The From: email address for all outgoing mail (e.g. noreply@mysite.com).
OUTPOST_SMTP_FROM_NAME (none) The display name shown in the From: header (e.g. My Site).

reCAPTCHA

Outpost supports Google reCAPTCHA v3 for form spam protection. Both keys are required for reCAPTCHA to work.

ConstantDefaultDescription
OUTPOST_RECAPTCHA_SITE_KEY (none) Your reCAPTCHA v3 site key, embedded in the front-end JavaScript.
OUTPOST_RECAPTCHA_SECRET (none) Your reCAPTCHA v3 secret key, used server-side to verify tokens. Keep this private.

Members

ConstantDefaultDescription
OUTPOST_LOGIN_REDIRECT / Where members are redirected after a successful login. Override with a custom URL like /dashboard or /members/feed.
OUTPOST_REGISTRATION_ENABLED true When false, the member registration endpoint returns an error and the sign-up form can be hidden. Use this to create an invite-only site where only admins can add members.

Complete Example config.php

A production configuration with all sections filled in:

<?php
// ── Core ─────────────────────────────────────────────────
define('OUTPOST_SITE_URL',       'https://mysite.com');
define('OUTPOST_DEBUG',          false);

// ── Uploads ──────────────────────────────────────────────
define('OUTPOST_MAX_UPLOAD_SIZE', 20971520); // 20 MB

// ── Image Processing ─────────────────────────────────────
define('OUTPOST_WEBP_AUTO_CONVERT', true);
define('OUTPOST_WEBP_QUALITY',      85);

// ── Caching ──────────────────────────────────────────────
define('OUTPOST_CACHE_TTL',      1800);      // 30 minutes
define('OUTPOST_TEMPLATE_CACHE',  true);

// ── Builder Sync ─────────────────────────────────────────
define('OUTPOST_SYNC_KEY',        'sk_prod_aB3xQz9mLp2...');

// ── SMTP ─────────────────────────────────────────────────
define('OUTPOST_SMTP_HOST',      'smtp.mailgun.org');
define('OUTPOST_SMTP_PORT',      587);
define('OUTPOST_SMTP_USER',      'postmaster@mg.mysite.com');
define('OUTPOST_SMTP_PASS',      'your-smtp-password');
define('OUTPOST_SMTP_FROM',      'noreply@mysite.com');
define('OUTPOST_SMTP_FROM_NAME',  'My Site');

// ── reCAPTCHA ────────────────────────────────────────────
define('OUTPOST_RECAPTCHA_SITE_KEY', '6Lc...');
define('OUTPOST_RECAPTCHA_SECRET',   '6Lc...');

// ── Members ──────────────────────────────────────────────
define('OUTPOST_LOGIN_REDIRECT',         '/dashboard');
define('OUTPOST_REGISTRATION_ENABLED',   true);

Security Checklist

Production hardening. Before going live, verify these settings:
  • Set OUTPOST_DEBUG to false — never leave PHP error display on in production.
  • Set OUTPOST_SITE_URL explicitly — never rely on auto-detection.
  • Use a strong, random OUTPOST_SYNC_KEY if you use the Builder sync API.
  • Ensure config.php is not publicly accessible — it should be inside outpost/ which is served through the admin SPA, not directly browsable.
  • Add outpost/config.php to your .gitignore to prevent credential leaks.

Database Outside the Web Root

For maximum security, store the SQLite database outside the web-accessible directory:

// Store db one level above the public directory
define('OUTPOST_DB_PATH', dirname(__DIR__) . '/../db/cms.db');

The web server process must have read/write permission on the file and its containing directory.

Disable Registration for Invite-Only Sites

define('OUTPOST_REGISTRATION_ENABLED', false);

When disabled, the /outpost/member-pages/register.php endpoint returns a 403 error and the admin-visible "Members" registration toggle reflects this state.
