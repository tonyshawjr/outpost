Collection Loops

Iterate over published items in any collection. The most-used tag in every theme — list posts, products, events, portfolio pieces, or any structured content you define.

Basic Syntax

The {% for %} tag iterates over all published items in a collection. The collection is identified by its slug — the same slug you set when creating the collection in Admin → Collections.

{% for post in collection.post %}
  {{ post.title }}
  {{ post.url }}
  {{ post.slug }}
  {{ post.id }}
  {{ post.body | raw }}
  {{ post.excerpt }}
  {{ post.excerpt | or_body }}
  {{ post.featured_image }}
  {{ post.published_at }}
  {{ post.created_at }}
  {{ post.any_custom_field }}
{% endfor %}
Note — Only published items appear in collection loops. Items with draft status or scheduled status with a future publish date are automatically excluded. You never need to filter for this yourself.

Available Fields

Every loop item exposes these fields, regardless of the collection's custom schema.

Field Type Description
title string Item title
slug string URL slug — unique within the collection
url string Full URL built from the collection's url_pattern (e.g., /post/hello-world)
id integer Database row ID
status string Item status: published, draft, or scheduled
body html Rich text body content — always use the | raw filter to render HTML
excerpt string Short description set by the editor. Empty string if not set.
featured_image string Image URL. Empty string if no image is set — check with {% if post.featured_image %}.
published_at string Formatted publish date (e.g. "March 4, 2026")
created_at string Row creation timestamp
updated_at string Last-modified timestamp
{custom_field} varies Any field defined in the collection's schema — accessed by its field name

Empty Loop — the else Branch

Add an {% else %} clause to show a fallback when the collection has no published items (or no items match the current filter).

{% for post in collection.post %}
  <a href="{{ post.url }}">{{ post.title }}</a>
{% else %}
  <p>No posts published yet.</p>
{% endfor %}
Tip — Always include an {% else %} on filtered loops. When a visitor clicks a category that has no posts, they should see a friendly message, not an empty container.

Loop Options

Options are added after the collection name, space-separated. They can be combined freely (with one exception noted below).

limit:N — Show a Fixed Number of Items

Returns at most N items. Use this for "recent posts" sidebars, homepage highlights, and any widget that should never show more than a set count.

{% for post in collection.post limit:3 %}
  <a href="{{ post.url }}">{{ post.title }}</a>
{% endfor %}

orderby:field — Sort by a Field

Sort results by any item field. The sort is always descending (newest / highest first). The default order when no orderby is specified is by creation date, newest first.

{% for post in collection.post orderby:published_at %}
{% for event in collection.event orderby:date %}
{% for product in collection.product orderby:price %}
Note — orderby sorts descending. For published_at this gives newest-first, which is almost always what you want. For numeric fields like price, it gives highest-first.

paginate:N — Paginate Results

Shows N items per page and reads ?page= from the URL to determine the current page. Pair it with {% pagination %} immediately after the loop to render page navigation.

{% for post in collection.post paginate:10 %}
  <a href="{{ post.url }}">{{ post.title }}</a>
{% endfor %}
{% pagination %}

See Pagination for full styling and query-string details.

filteredby:param — Filter by Folder Label

Filters results to items tagged with the folder label whose slug matches the ?param= query string value. When no query parameter is present, all items are returned.

{% for post in collection.post filteredby:category %}
  <a href="{{ post.url }}">{{ post.title }}</a>
{% endfor %}

A visitor at /blog?category=technology sees only posts tagged with the "technology" category. A visitor at /blog sees all posts. The parameter name must match the folder slug configured in Admin → Folders.

related:var — Related Items

Returns items that share folder labels with the item bound to var. Used inside a {% single %} block to build a "You might also like" section.

{% single post from collection.post %}
  <h1>{{ post.title }}</h1>

  {% for related in collection.post related:post limit:3 %}
    <a href="{{ related.url }}">{{ related.title }}</a>
  {% endfor %}
{% endsingle %}

The current item is automatically excluded from the related results so it never appears in its own sidebar.

Combining Options

Options can be combined on the same tag in any order.

{% for post in collection.post limit:6 orderby:published_at %}

{% for post in collection.post paginate:10 orderby:published_at filteredby:category %}

{% for post in collection.post related:post limit:3 orderby:published_at %}
Warning — paginate:N and limit:N are mutually exclusive. Using both on the same loop is undefined — pick one. Use limit for fixed-count widgets; use paginate for browsable archive pages.

Accessing Custom Fields

Any field you define in the collection schema is available on every loop item using its field name.

{# Collection schema has fields: author, read_time, difficulty #}

{% for post in collection.post %}
  <article>
    <h2>{{ post.title }}</h2>
    <p class="meta">By {{ post.author }} &middot; {{ post.read_time }} min read</p>
    <span class="badge">{{ post.difficulty }}</span>
    <p>{{ post.excerpt }}</p>
    <a href="{{ post.url }}">Read more</a>
  </article>
{% endfor %}

Using Conditionals Inside Loops

Use {% if %} inside a loop to handle optional fields — fields the editor may or may not have filled in.

{% for post in collection.post %}
  <article>
    {% if post.featured_image %}
      <img src="{{ post.featured_image }}" alt="{{ post.title }}">
    {% endif %}
    <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
    {% if post.excerpt %}
      <p>{{ post.excerpt }}</p>
    {% else %}
      <p>{{ post.body | raw }}</p>
    {% endif %}
  </article>
{% endfor %}

The | or_body filter is a shorthand for this excerpt-or-body pattern:

{{ post.excerpt | or_body }}

If excerpt is empty, it falls back to rendering the body field as plain text (with HTML stripped). Useful for generating preview text without a separate {% if %} block.

Full Blog List Example

A complete archive page combining pagination, sorting, category filtering, featured images, and an empty state.

{% include 'head' %}
{% include 'nav' %}

<main class="blog-main">
  <header class="blog-header">
    <h1>Blog</h1>
  </header>

  <div class="post-grid">
    {% for post in collection.post paginate:12 orderby:published_at filteredby:category %}
      <article class="post-card">
        {% if post.featured_image %}
          <a href="{{ post.url }}" class="post-card-image">
            <img src="{{ post.featured_image }}" alt="{{ post.title }}">
          </a>
        {% endif %}
        <div class="post-card-body">
          <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
          <p>{{ post.excerpt | or_body }}</p>
          <time datetime="{{ post.published_at }}">{{ post.published_at }}</time>
        </div>
      </article>
    {% else %}
      <p class="empty-state">No posts found in this category.</p>
    {% endfor %}
  </div>

  {% pagination %}
</main>

{% include 'footer' %}

Sidebar "Recent Posts" Widget

A classic pattern: a full paginated list in the main column, plus a compact "recent" list in the sidebar using limit.

<div class="layout">
  <section class="main-col">
    {% for post in collection.post paginate:10 orderby:published_at %}
      <article>
        <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
        <p>{{ post.excerpt | or_body }}</p>
      </article>
    {% endfor %}
    {% pagination %}
  </section>

  <aside class="sidebar">
    <h3>Recent Posts</h3>
    <ul>
      {% for post in collection.post limit:5 orderby:published_at %}
        <li><a href="{{ post.url }}">{{ post.title }}</a></li>
      {% endfor %}
    </ul>
  </aside>
</div>
Note — Two separate {% for %} loops on the same page are perfectly valid. Each loop is independent. The paginated loop reads ?page=; the limit:5 loop always shows the 5 most recent posts regardless of the current page.

Multiple Collections on One Page

A homepage can pull from several collections at once.

{% include 'head' %}
{% include 'nav' %}

<section class="featured-posts">
  <h2>Latest Articles</h2>
  {% for post in collection.post limit:3 orderby:published_at %}
    <a href="{{ post.url }}">{{ post.title }}</a>
  {% endfor %}
</section>

<section class="upcoming-events">
  <h2>Upcoming Events</h2>
  {% for event in collection.event limit:4 orderby:date %}
    <div>
      <strong>{{ event.title }}</strong>
      <span>{{ event.date }}</span>
      <a href="{{ event.url }}">Details</a>
    </div>
  {% endfor %}
</section>

<section class="portfolio-highlights">
  <h2>Work</h2>
  {% for project in collection.portfolio limit:6 %}
    {% if project.featured_image %}
      <a href="{{ project.url }}">
        <img src="{{ project.featured_image }}" alt="{{ project.title }}">
      </a>
    {% endif %}
  {% endfor %}
</section>

{% include 'footer' %}
