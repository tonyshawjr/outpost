Taxonomy Loops

Iterate over the terms in a taxonomy — categories, tags, topics, or any classification system you define. Use taxonomy loops to build filter navigation, tag clouds, and category index pages.

Basic Syntax

The taxonomy slug (e.g., categories, tags) is set when you create the taxonomy in Admin → Taxonomies. Use it after taxonomy..

{% for term in taxonomy.categories %}
  {{ term.name }}
  {{ term.slug }}
  {{ term.count }}
  {{ term.description }}
{% endfor %}
Slug must match exactly The slug you use in taxonomy.slug must be an exact, case-sensitive match to the slug defined in Admin → Taxonomies. There is no error if the slug does not exist — the loop simply produces no output. For example, if your taxonomy was created with slug categories, then taxonomy.categories works and taxonomy.category (singular) silently returns nothing.

Available Term Fields

Field Description
term.name Display name — e.g., "Technology", "Design"
term.slug URL-safe identifier — e.g., "technology", "design"
term.count Number of published collection items assigned this term
term.description Optional description text set in the admin. Empty string if not provided.
term.parent_id Parent term's database ID for hierarchical taxonomies. 0 or null if this is a root-level term.
Note — term.count reflects only published items. Terms that exist in the admin but have no published items assigned will show a count of 0. You can hide zero-count terms using a conditional.

Category Filter Navigation

The most common use: a row of links that filter a collection loop. Each link passes a ?category=slug parameter, which filteredby:category on the loop reads.

<nav class="category-nav">
  <a href="/blog" class="category-link">All</a>
  {% for term in taxonomy.categories %}
    <a href="/blog?category={{ term.slug }}" class="category-link">
      {{ term.name }}
      <span class="count">{{ term.count }}</span>
    </a>
  {% endfor %}
</nav>

Hiding Empty Categories

Use a conditional to skip terms that have no published items.

<nav class="category-nav">
  {% for term in taxonomy.categories %}
    {% if term.count %}
      <a href="/blog?category={{ term.slug }}">
        {{ term.name }} ({{ term.count }})
      </a>
    {% endif %}
  {% endfor %}
</nav>

Tag Cloud

Render all tags as linked labels. Use data-count to drive CSS font-size scaling if desired.

<div class="tag-cloud">
  {% for tag in taxonomy.tags %}
    <a href="/blog?tag={{ tag.slug }}" class="tag" data-count="{{ tag.count }}">
      {{ tag.name }}
    </a>
  {% endfor %}
</div>
/* CSS: scale tags by count using a data attribute */
.tag[data-count="1"] { font-size: 13px; }
.tag[data-count="2"] { font-size: 14px; }
.tag[data-count="3"] { font-size: 16px; }
.tag { font-size: 18px; }

Sidebar Taxonomy Widget

A compact list of categories with counts, ideal for a sidebar or aside panel.

<aside class="sidebar-widget">
  <h3>Categories</h3>
  <ul class="category-list">
    {% for term in taxonomy.categories %}
      <li>
        <a href="/blog?category={{ term.slug }}">
          {{ term.name }}
        </a>
        <span class="count">{{ term.count }}</span>
      </li>
    {% else %}
      <li>No categories yet.</li>
    {% endfor %}
  </ul>
</aside>

Hierarchical Taxonomies

When a taxonomy is configured to allow parent–child relationships, top-level terms have term.parent_id equal to 0 or null. Child terms have the ID of their parent. Use a conditional to separate root terms from children.

<!-- Flat list showing only root terms -->
{% for term in taxonomy.categories %}
  {% if term.parent_id %}
    {# child term — skip at the top level #}
  {% else %}
    <li class="parent-term">
      <a href="/blog?category={{ term.slug }}">{{ term.name }}</a>
    </li>
  {% endif %}
{% endfor %}
Tip — Outpost does not currently support automatic nested rendering of hierarchical taxonomy trees in a single loop pass. To render a parent-with-children structure, use two passes: one for root terms and a filtered inner loop, or build the hierarchy with JavaScript from a taxonomy API endpoint.

Combining Taxonomy Nav with a Filtered Loop

This is the standard two-column blog layout: taxonomy terms on the left for navigation, filtered posts on the right.

{% include 'head' %}
{% include 'nav' %}

<div class="blog-layout">

  <aside class="sidebar">
    <h3>Categories</h3>
    {% for term in taxonomy.categories %}
      <a href="/blog?category={{ term.slug }}">
        {{ term.name }} ({{ term.count }})
      </a>
    {% endfor %}

    <h3>Tags</h3>
    {% for tag in taxonomy.tags %}
      <a href="/blog?tag={{ tag.slug }}" class="tag">
        {{ tag.name }}
      </a>
    {% endfor %}
  </aside>

  <main class="main-col">
    {% for post in collection.post paginate:10 orderby:published_at filteredby:category %}
      <article>
        <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
        <p>{{ post.excerpt | or_body }}</p>
      </article>
    {% else %}
      <p>No posts in this category.</p>
    {% endfor %}
    {% pagination %}
  </main>

</div>

{% include 'footer' %}

Multiple Taxonomies on One Collection

A collection can have more than one taxonomy attached. Filter by any of them independently using different query parameter names.

<!-- Category filter -->
<div class="filter-row">
  <strong>Category:</strong>
  <a href="/blog">All</a>
  {% for term in taxonomy.categories %}
    <a href="/blog?category={{ term.slug }}">{{ term.name }}</a>
  {% endfor %}
</div>

<!-- Tag filter -->
<div class="filter-row">
  <strong>Tag:</strong>
  {% for tag in taxonomy.tags %}
    <a href="/blog?tag={{ tag.slug }}">{{ tag.name }}</a>
  {% endfor %}
</div>

<!-- Posts filtered by category -->
{% for post in collection.post paginate:10 filteredby:category %}
  <a href="{{ post.url }}">{{ post.title }}</a>
{% endfor %}
{% pagination %}
Note — You can only filter a single loop by one taxonomy parameter at a time. To filter by both category and tag simultaneously, you would need a custom approach. The filteredby option reads a single query string key.

Using Term Description

If editors have added descriptions to taxonomy terms, display them on category index pages or in tooltip-style hover cards.

<div class="category-grid">
  {% for term in taxonomy.categories %}
    <div class="category-card">
      <h3><a href="/blog?category={{ term.slug }}">{{ term.name }}</a></h3>
      {% if term.description %}
        <p>{{ term.description }}</p>
      {% endif %}
      <span class="count">{{ term.count }} posts</span>
    </div>
  {% endfor %}
</div>

Taxonomy Terms in the Head Partial

Taxonomy loops work anywhere in a template, including inside partials. Here is an example using a category name as a keyword meta tag — useful if a filtered page is the active view.

{# In head.html — list all category names as keywords #}
<meta name="keywords" content="{% for term in taxonomy.categories %}{{ term.name }}, {% endfor %}">
