Single Items

Fetch and render one specific collection item by its URL slug. This is the template tag that powers every individual post, project, product, or detail page.

Basic Syntax

{% single post from collection.post %}
  {{ post.title }}
  {{ post.body | raw }}
  {{ post.published_at }}
{% else %}
  <p>Post not found.</p>
{% endsingle %}

How It Works

When a visitor hits a collection URL like /post/hello-world, the router extracts the slug (hello-world) and sets it as an internal parameter. The {% single %} tag reads that slug, queries the collection for a published item matching it, and renders the main block. If no published item with that slug exists, the {% else %} branch renders instead.

Note — Like collection loops, {% single %} only finds published items. A draft or future-scheduled item returns "not found" to the visitor, even if it exists in the database.

Choosing the Variable Name

The identifier between {% single and from is the variable name you use to access fields inside the block. It can be any valid name — choose one that reads naturally for your content type.

{% single article from collection.post %}
  {{ article.title }}
{% endsingle %}

{% single project from collection.portfolio %}
  {{ project.title }}
  {{ project.description | raw }}
{% endsingle %}

{% single product from collection.shop %}
  {{ product.title }}
  <span class="price">{{ product.price }}</span>
{% endsingle %}

Available Fields

Single items expose the same fields as collection loop items.

Field Type Description
title string Item title
slug string URL slug
url string Full URL (useful for canonical tags and share links)
id integer Database ID
body html Rich text body — always use | raw
excerpt string Short description
featured_image string Image URL — empty string if not set
published_at string Publish timestamp
created_at string Creation timestamp
updated_at string Last-modified timestamp
{custom_field} varies Any field from the collection schema

Meta Tag Integration

On single-item pages, {{ meta.title }} and {{ meta.description }} automatically pull from the item's SEO fields set in the admin. If the editor hasn't filled in SEO fields, they fall back to the item's title and a plain-text excerpt of the body.

This is why {% include 'head' %} must appear on the same template file that contains {% single %} — the head partial reads the meta values that the single tag establishes.

{# In partials/head.html #}
<title>{{ meta.title }}My Site{{ /meta.title }}</title>
<meta name="description" content="{{ meta.description }}Welcome to my site{{ /meta.description }}">

{# In post.html — {% include 'head' %} renders AFTER {% single %} sets the meta #}
{% single post from collection.post %}
  ...
{% endsingle %}
Tip — The template engine resolves {% single %} before rendering partials, so meta values set by the item are always available inside {% include 'head' %} even though the include appears earlier in the file.

Using Conditionals Inside a Single

Fields that editors might leave blank should be guarded with {% if %}.

{% single post from collection.post %}
  <article>
    <h1>{{ post.title }}</h1>

    {% if post.featured_image %}
      <figure>
        <img src="{{ post.featured_image }}" alt="{{ post.title }}">
      </figure>
    {% endif %}

    {% if post.excerpt %}
      <p class="lead">{{ post.excerpt }}</p>
    {% endif %}

    <div class="prose">
      {{ post.body | raw }}
    </div>

    {% if post.author %}
      <footer class="byline">By {{ post.author }}</footer>
    {% endif %}
  </article>
{% else %}
  <p>Not found.</p>
{% endsingle %}

Related Items Inside a Single

Nest a {% for %} loop with related:var inside a {% single %} block to fetch items that share folder labels with the current item. The current item is automatically excluded.

{% single post from collection.post %}
  <h1>{{ post.title }}</h1>
  {{ post.body | raw }}

  <aside class="related">
    <h3>Related Posts</h3>
    {% for related in collection.post related:post limit:3 %}
      <article>
        <a href="{{ related.url }}">{{ related.title }}</a>
        <p>{{ related.excerpt | or_body }}</p>
      </article>
    {% else %}
      <p>No related posts.</p>
    {% endfor %}
  </aside>
{% else %}
  <h1>Not found</h1>
{% endsingle %}

Gallery Fields on Single Items

If the collection schema includes a gallery field, use {% for img in gallery.field_name %} inside the single block to iterate its images.

{% single project from collection.portfolio %}
  <h1>{{ project.title }}</h1>
  {{ project.description | raw }}

  <div class="project-gallery">
    {% for img in gallery.screenshots %}
      <img src="{{ img.url }}" alt="{{ project.title }}">
    {% endfor %}
  </div>
{% endsingle %}

Complete Single Post Page

A full post.html template with header, hero image, body, related posts, and a 404 fallback.

{% include 'head' %}
{% include 'nav' %}

<main>
  {% single post from collection.post %}

    <article class="post">
      <header class="post-header">
        <h1>{{ post.title }}</h1>
        <div class="post-meta">
          <time datetime="{{ post.published_at }}">{{ post.published_at }}</time>
          {% if post.author %}
            <span class="separator">&middot;</span>
            <span>{{ post.author }}</span>
          {% endif %}
        </div>
      </header>

      {% if post.featured_image %}
        <figure class="post-hero">
          <img src="{{ post.featured_image }}" alt="{{ post.title }}">
        </figure>
      {% endif %}

      <div class="prose">
        {{ post.body | raw }}
      </div>
    </article>

    <aside class="related-posts">
      <h3>Related Posts</h3>
      <div class="related-grid">
        {% for related in collection.post related:post limit:3 %}
          <article>
            {% if related.featured_image %}
              <a href="{{ related.url }}">
                <img src="{{ related.featured_image }}" alt="{{ related.title }}">
              </a>
            {% endif %}
            <h4><a href="{{ related.url }}">{{ related.title }}</a></h4>
            <p>{{ related.excerpt | or_body }}</p>
          </article>
        {% else %}
          <p>No related posts.</p>
        {% endfor %}
      </div>
    </aside>

  {% else %}
    <div class="not-found">
      <h1>Post not found</h1>
      <p><a href="/blog">&larr; Back to blog</a></p>
    </div>
  {% endsingle %}
</main>

{% include 'footer' %}

Edge Cases

Draft Items Return "Not Found"

If an editor saves a post as draft and a visitor visits its URL directly, the {% else %} branch renders. This is intentional — draft content is never visible to the public regardless of how the URL is accessed.

Scheduled Items Before Their Publish Date

An item with scheduled status and a future published_at date is also treated as not found. Once the publish date passes and the item is promoted to published status, it becomes available.

Missing slug Parameter

If the slug parameter is not set (for example, if the template is rendered outside a collection URL context), {% single %} returns not found and the {% else %} branch fires.

Canonical URLs

Use {{ post.url }} to build a canonical link tag in your head partial, ensuring search engines index the correct URL for each item.

{# In partials/head.html, after {% single %} has run #}
<link rel="canonical" href="{{ meta.canonical }}">
