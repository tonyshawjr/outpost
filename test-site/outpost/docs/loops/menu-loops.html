Menu Loops

Render navigation menus defined in Admin → Navigation. Menu loops give editors full control over link labels, URLs, order, and nesting — without touching any template code.

Basic Syntax

Reference a menu by its slug — the identifier set when the menu was created in the admin. The menu slug main is the conventional name for the primary site navigation.

Available Item Fields

Field Description
data-outpost="label" Display text for the link — set by the editor. Use on a <span> or similar text element.
data-outpost="url" Link destination — use on an <a> element to set its href. Can be a relative path, absolute URL, or anchor.
target _self (default, same tab) or _blank (new tab). Set automatically from admin configuration.
children Array of child menu items — same structure as top-level items. Empty array if no children. Nested menus require custom PHP handling.

Basic Horizontal Nav

The simplest possible navigation — a flat list of links with no dropdown support.

Nav with Dropdown Menus

Dropdown menus with child items require custom template logic. Here is a basic example:

CSS for Hover Dropdowns

.nav-item { position: relative; }
.dropdown {
 display: none;
 position: absolute;
 top: 100%;
 left: 0;
 background: #fff;
 border: 1px solid #e4e4e7;
 border-radius: 8px;
 padding: 8px 0;
 min-width: 180px;
 list-style: none;
 box-shadow: 0 4px 16px rgba(0,0,0,.08);
}
.nav-item.has-dropdown:hover .dropdown,
.nav-item.has-dropdown:focus-within .dropdown {
 display: block;
}
.dropdown a {
 display: block;
 padding: 8px 16px;
 text-decoration: none;
 color: #374151;
}
.dropdown a:hover { background: #f4f4f5; }
