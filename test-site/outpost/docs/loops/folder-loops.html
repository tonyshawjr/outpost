Folder Loops

Iterate over the labels in a folder — categories, tags, topics, or any classification system you define. Use folder loops to build filter navigation, tag clouds, and category index pages.

Renamed in v1.0.0-beta.7 — "Taxonomies" are now called Folders and "Terms" are now called Labels. The old syntax {% for term in taxonomy.slug %} still works for backward compatibility, but {% for label in folder.slug %} is the recommended form going forward.

Basic Syntax

The folder slug (e.g., categories, tags) is set when you create the folder in Admin → Folders. Use it after folder..

{% for label in folder.categories %}
  {{ label.name }}
  {{ label.slug }}
  {{ label.count }}
  {{ label.description }}
{% endfor %}
Slug must match exactly The slug you use in folder.slug must be an exact, case-sensitive match to the slug defined in Admin → Folders. There is no error if the slug does not exist — the loop simply produces no output. For example, if your folder was created with slug categories, then folder.categories works and folder.category (singular) silently returns nothing.

Available Label Fields

Field Description
label.name Display name — e.g., "Technology", "Design"
label.slug URL-safe identifier — e.g., "technology", "design"
label.count Number of published collection items assigned this label
label.description Optional description text set in the admin. Empty string if not provided.
label.parent_id Parent label's database ID for hierarchical folders. 0 or null if this is a root-level label.
Note — label.count reflects only published items. Labels that exist in the admin but have no published items assigned will show a count of 0. You can hide zero-count labels using a conditional.

Category Filter Navigation

The most common use: a row of links that filter a collection loop. Each link passes a ?category=slug parameter, which filteredby:category on the loop reads.

<nav class="category-nav">
  <a href="/blog" class="category-link">All</a>
  {% for label in folder.categories %}
    <a href="/blog?category={{ label.slug }}" class="category-link">
      {{ label.name }}
      <span class="count">{{ label.count }}</span>
    </a>
  {% endfor %}
</nav>

Hiding Empty Categories

Use a conditional to skip labels that have no published items.

<nav class="category-nav">
  {% for label in folder.categories %}
    {% if label.count %}
      <a href="/blog?category={{ label.slug }}">
        {{ label.name }} ({{ label.count }})
      </a>
    {% endif %}
  {% endfor %}
</nav>

Tag Cloud

Render all tags as linked labels. Use data-count to drive CSS font-size scaling if desired.

<div class="tag-cloud">
  {% for tag in folder.tags %}
    <a href="/blog?tag={{ tag.slug }}" class="tag" data-count="{{ tag.count }}">
      {{ tag.name }}
    </a>
  {% endfor %}
</div>
/* CSS: scale tags by count using a data attribute */
.tag[data-count="1"] { font-size: 13px; }
.tag[data-count="2"] { font-size: 14px; }
.tag[data-count="3"] { font-size: 16px; }
.tag { font-size: 18px; }

Sidebar Folder Widget

A compact list of categories with counts, ideal for a sidebar or aside panel.

<aside class="sidebar-widget">
  <h3>Categories</h3>
  <ul class="category-list">
    {% for label in folder.categories %}
      <li>
        <a href="/blog?category={{ label.slug }}">
          {{ label.name }}
        </a>
        <span class="count">{{ label.count }}</span>
      </li>
    {% else %}
      <li>No categories yet.</li>
    {% endfor %}
  </ul>
</aside>

Hierarchical Folders

When a folder is configured to allow parent–child relationships, top-level labels have label.parent_id equal to 0 or null. Child labels have the ID of their parent. Use a conditional to separate root labels from children.

<!-- Flat list showing only root labels -->
{% for label in folder.categories %}
  {% if label.parent_id %}
    {# child label — skip at the top level #}
  {% else %}
    <li class="parent-label">
      <a href="/blog?category={{ label.slug }}">{{ label.name }}</a>
    </li>
  {% endif %}
{% endfor %}
Tip — Outpost does not currently support automatic nested rendering of hierarchical folder trees in a single loop pass. To render a parent-with-children structure, use two passes: one for root labels and a filtered inner loop, or build the hierarchy with JavaScript from a folder API endpoint.

Combining Folder Nav with a Filtered Loop

This is the standard two-column blog layout: folder labels on the left for navigation, filtered posts on the right.

{% include 'head' %}
{% include 'nav' %}

<div class="blog-layout">

  <aside class="sidebar">
    <h3>Categories</h3>
    {% for label in folder.categories %}
      <a href="/blog?category={{ label.slug }}">
        {{ label.name }} ({{ label.count }})
      </a>
    {% endfor %}

    <h3>Tags</h3>
    {% for tag in folder.tags %}
      <a href="/blog?tag={{ tag.slug }}" class="tag">
        {{ tag.name }}
      </a>
    {% endfor %}
  </aside>

  <main class="main-col">
    {% for post in collection.post paginate:10 orderby:published_at filteredby:category %}
      <article>
        <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
        <p>{{ post.excerpt | or_body }}</p>
      </article>
    {% else %}
      <p>No posts in this category.</p>
    {% endfor %}
    {% pagination %}
  </main>

</div>

{% include 'footer' %}

Multiple Folders on One Collection

A collection can have more than one folder attached. Filter by any of them independently using different query parameter names.

<!-- Category filter -->
<div class="filter-row">
  <strong>Category:</strong>
  <a href="/blog">All</a>
  {% for label in folder.categories %}
    <a href="/blog?category={{ label.slug }}">{{ label.name }}</a>
  {% endfor %}
</div>

<!-- Tag filter -->
<div class="filter-row">
  <strong>Tag:</strong>
  {% for tag in folder.tags %}
    <a href="/blog?tag={{ tag.slug }}">{{ tag.name }}</a>
  {% endfor %}
</div>

<!-- Posts filtered by category -->
{% for post in collection.post paginate:10 filteredby:category %}
  <a href="{{ post.url }}">{{ post.title }}</a>
{% endfor %}
{% pagination %}
Note — You can only filter a single loop by one folder parameter at a time. To filter by both category and tag simultaneously, you would need a custom approach. The filteredby option reads a single query string key.

Using Label Description

If editors have added descriptions to folder labels, display them on category index pages or in tooltip-style hover cards.

<div class="category-grid">
  {% for label in folder.categories %}
    <div class="category-card">
      <h3><a href="/blog?category={{ label.slug }}">{{ label.name }}</a></h3>
      {% if label.description %}
        <p>{{ label.description }}</p>
      {% endif %}
      <span class="count">{{ label.count }} posts</span>
    </div>
  {% endfor %}
</div>

Folder Labels in the Head Partial

Folder loops work anywhere in a template, including inside partials. Here is an example using a category name as a keyword meta tag — useful if a filtered page is the active view.

{# In head.html — list all category names as keywords #}
<meta name="keywords" content="{% for label in folder.categories %}{{ label.name }}, {% endfor %}">
