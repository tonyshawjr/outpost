Gallery Loops

Loop over multiple images stored in a gallery field. Galleries let editors upload and reorder an ordered set of images per collection item — perfect for portfolio projects, property listings, product shots, and event albums.

What Is a Gallery Field?

A gallery is a schema field type that stores an ordered list of uploaded images. Unlike a single image field, a gallery holds as many images as the editor uploads, in the order they choose. Drag-and-drop reordering is supported in the admin.

Gallery fields must be added to a collection's schema before they can be used in templates. Go to Admin → Collections → select a collection → Edit Schema → Add Field → choose type Gallery.

Note — The gallery field type is distinct from the single image field type. A single image field stores one URL; a gallery field stores an ordered array of URLs. You loop over gallery fields; you output image fields directly.

Basic Syntax

Use a standard {% for %} loop, referencing the gallery field by the name you gave it in the schema. Each iteration variable is a plain image URL — apply the | image filter to resolve it to a full URL.

{% for img in item.photos %}
  <img src="{{ img | image }}" alt="">
{% endfor %}

Where:

  • item is the loop variable from an outer collection loop or {% single %} block
  • photos is the name of the gallery field in the schema
  • img is the loop variable — a relative or absolute image path
  • | image resolves the path to a full public URL

Available Data Per Image

Expression Description
{{ img | image }} Full public URL to the image file. Always use this to output the src.
{{ img }} The raw stored path (e.g., /outpost/uploads/photo.jpg). Rarely needed directly.
Tip — Gallery images do not have individual alt text fields unless you build a custom structure using a repeater. For decorative portfolio images, an empty alt="" is acceptable. For accessible galleries where alt text matters, consider a repeater field (image + alt text) instead of a bare gallery.

Gallery Inside a Collection Loop

Nest the gallery loop inside an outer collection loop to show a thumbnail strip for each item in a listing page.

{% for project in collection.portfolio %}
  <article class="project-card">
    <h2><a href="{{ project.url }}">{{ project.title }}</a></h2>

    <div class="thumbnail-strip">
      {% for img in project.photos %}
        <a href="{{ project.url }}">
          <img src="{{ img | image }}" alt="{{ project.title }}">
        </a>
      {% endfor %}
    </div>

    {% if project.excerpt %}
      <p>{{ project.excerpt }}</p>
    {% endif %}
  </article>
{% endfor %}

Gallery Inside a Single Item

On a detail page, loop the gallery to render a full photo grid or a lightbox-ready strip.

{% single project from collection.portfolio %}
  <article>
    <h1>{{ project.title }}</h1>

    {% if project.featured_image %}
      <figure class="hero-image">
        <img src="{{ project.featured_image | image }}" alt="{{ project.title }}">
      </figure>
    {% endif %}

    <div class="prose">
      {{ project.description | raw }}
    </div>

    {% if project.photos %}
      <section class="photo-gallery">
        <h2>Photos</h2>
        <div class="gallery-grid">
          {% for img in project.photos %}
            <figure>
              <img src="{{ img | image }}" alt="{{ project.title }}">
            </figure>
          {% endfor %}
        </div>
      </section>
    {% endif %}
  </article>
{% else %}
  <p>Project not found.</p>
{% endsingle %}

Image Order

Images are returned in the order the editor arranged them in the admin. The admin gallery field supports drag-and-drop reordering — the template always reflects the saved order. You do not need to specify a sort order in the template tag.

Empty Gallery

If an editor has not uploaded any images to a gallery field, the loop body is silently skipped — no error is thrown and no empty markup is produced. Guard with an {% if item.gallery_field %} check if you need to show or hide a section heading along with the gallery.

{% if project.photos %}
  <section class="gallery-section">
    <h2>Gallery</h2>
    <div class="gallery-grid">
      {% for img in project.photos %}
        <img src="{{ img | image }}" alt="{{ project.title }}">
      {% endfor %}
    </div>
  </section>
{% endif %}

CSS Grid Gallery Layout

A three-column responsive grid is the most common layout for gallery images. Combine it with object-fit: cover to keep all thumbnails uniform height regardless of the original image dimensions.

.gallery-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 12px;
}

.gallery-grid img {
  width: 100%;
  aspect-ratio: 4 / 3;
  object-fit: cover;
  border-radius: 6px;
  display: block;
}

@media (max-width: 640px) {
  .gallery-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}
Tip — Use aspect-ratio instead of a fixed height so the grid scales proportionally on different screen sizes without JavaScript.

Masonry-Style Layout

For a Pinterest-style layout that preserves each image's natural proportions, use CSS columns instead of grid.

.gallery-masonry {
  column-count: 3;
  column-gap: 12px;
}

.gallery-masonry img {
  display: block;
  width: 100%;
  margin-bottom: 12px;
  border-radius: 6px;
  break-inside: avoid;
}

@media (max-width: 640px) {
  .gallery-masonry { column-count: 2; }
}
<div class="gallery-masonry">
  {% for img in project.photos %}
    <img src="{{ img | image }}" alt="{{ project.title }}">
  {% endfor %}
</div>

Lightbox-Ready Gallery

Wrap each image in an anchor tag pointing to the full-size URL. A JavaScript lightbox library (Photoswipe, GLightbox, SimpleLightbox, etc.) can pick these up via a data-lightbox attribute or a shared class. The template itself needs no library-specific logic.

<div class="gallery-grid" id="gallery">
  {% for img in project.photos %}
    <a href="{{ img | image }}" data-lightbox="project-{{ project.id }}"
       data-title="{{ project.title }}">
      <img src="{{ img | image }}" alt="{{ project.title }}">
    </a>
  {% endfor %}
</div>

<!-- Add your lightbox library here -->
<link rel="stylesheet" href="/assets/lightbox.css">
<script src="/assets/lightbox.js"></script>

First Image as Hero, Rest as Thumbnails

A common pattern for property listings or event pages: display the first gallery image large, then the remaining images as a smaller strip below. Because Outpost does not expose a loop index variable directly, use a CSS-based approach with :first-child selectors to handle this without additional template logic.

<div class="gallery-layout">
  {% for img in project.photos %}
    <img src="{{ img | image }}" alt="{{ project.title }}">
  {% endfor %}
</div>
/* CSS: first image spans full width, rest are thumbnails */
.gallery-layout {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  gap: 8px;
}

.gallery-layout img:first-child {
  grid-column: 1 / -1; /* full width */
  aspect-ratio: 16 / 9;
  object-fit: cover;
  border-radius: 8px;
}

.gallery-layout img:not(:first-child) {
  aspect-ratio: 1;
  object-fit: cover;
  border-radius: 6px;
}

Showing Only the First N Images

The {% for %} gallery loop does not support a limit option directly. To show only the first few images on a listing page (to avoid loading all photos), use CSS to hide images beyond a count, or handle it in JavaScript. Alternatively, use featured_image on listing pages and reserve the full gallery for the detail page.

{# Listing page: show featured_image only #}
{% for project in collection.portfolio %}
  {% if project.featured_image %}
    <img src="{{ project.featured_image | image }}" alt="{{ project.title }}">
  {% endif %}
{% endfor %}

{# Detail page: show full gallery #}
{% single project from collection.portfolio %}
  {% for img in project.photos %}
    <img src="{{ img | image }}" alt="{{ project.title }}">
  {% endfor %}
{% endsingle %}

Multiple Gallery Fields on One Item

A collection schema can have more than one gallery field. For example, a real estate listing might have a photos gallery and a floor_plans gallery. Loop over each separately using their field names.

{% single listing from collection.property %}
  <h1>{{ listing.title }}</h1>

  <section>
    <h2>Photos</h2>
    <div class="gallery-grid">
      {% for img in listing.photos %}
        <img src="{{ img | image }}" alt="{{ listing.title }}">
      {% endfor %}
    </div>
  </section>

  {% if listing.floor_plans %}
    <section>
      <h2>Floor Plans</h2>
      <div class="gallery-grid">
        {% for img in listing.floor_plans %}
          <img src="{{ img | image }}" alt="Floor plan">
        {% endfor %}
      </div>
    </section>
  {% endif %}
{% endsingle %}

Performance: Lazy Loading

For galleries with many images, add loading="lazy" to every image except the first one (which should be eagerly loaded as it is likely above the fold). This is a standard HTML attribute and requires no JavaScript.

<div class="gallery-grid">
  {% for img in project.photos %}
    <img src="{{ img | image }}" alt="{{ project.title }}" loading="lazy">
  {% endfor %}
</div>
Note — All modern browsers support loading="lazy". It defers offscreen image loads until the user scrolls near them, significantly improving initial page load time for image-heavy pages.
