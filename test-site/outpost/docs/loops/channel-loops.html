Channel Loops

Iterate over cached items from external APIs. Channels bring external data into your templates with the same for-loop syntax you already use for collections.

Basic Syntax

The {% for %} tag iterates over all cached items in a channel. The channel is identified by its slug — the same slug you set when creating the channel in Admin → Channels.

{% for item in channel.listings %}
  <div class="listing">
    <h2>{{ item.title }}</h2>
    <p>{{ item.description }}</p>
    <p class="price">{{ item.price }}</p>
    <img src="{{ item.image_url }}" alt="{{ item.title }}">
  </div>
{% endfor %}
Note — Only items cached from the last sync appear in channel loops. If the cache TTL has expired, Outpost auto-syncs before rendering so your templates always show fresh data.

Available Fields

Each item exposes the fields mapped during channel setup. Unlike collections, there are no fixed fields — everything comes from the external API. Common patterns:

  • {{ item.field_name }} — auto-escaped text output
  • {{ item.field_name | raw }} — raw HTML output (use for HTML content from the API)
  • Field names match what you mapped in the Schema step of the channel builder

In addition to your mapped data fields, every channel item has built-in properties:

Property Description
item._id Internal database ID (auto-generated by Outpost)
item._external_id The unique ID from the source API (mapped via the ID field setting)
item._slug The URL slug for this item (mapped via the slug field setting)
item.url Full URL path for this item (only available if the channel has a URL pattern)
item.{field} Any data field from the API that was selected during channel setup

Empty Loop — the else Branch

Add an {% else %} clause to show a fallback when the channel has no cached items (or the external API returned no data).

{% for item in channel.listings %}
  <div class="listing-card">
    <h2>{{ item.title }}</h2>
    <p>{{ item.description }}</p>
  </div>
{% else %}
  <p>No listings available.</p>
{% endfor %}
Tip — Always include an {% else %} on channel loops. If the external API is temporarily unreachable and the cache is empty, visitors should see a friendly message rather than a blank page.

Loop Options

Options are added after the channel name, space-separated.

Option Example Description
limit:N limit:5 Return at most N items
orderby:field orderby:price Sort by field (descending)

limit:N — Show a Fixed Number of Items

Returns at most N items. Use this for homepage highlights, sidebar widgets, and any section that should cap the number of results.

{% for item in channel.listings limit:6 %}
  <div class="listing-card">
    <h2>{{ item.title }}</h2>
    <p class="price">{{ item.price }}</p>
  </div>
{% endfor %}

orderby:field — Sort by a Field

Sort results by any mapped field. The sort is always descending (newest / highest first). The default order when no orderby is specified is the order items were synced.

{% for item in channel.listings orderby:price %}
  <p>{{ item.title }} — ${{ item.price }}</p>
{% endfor %}

Combining Options

Options can be combined on the same tag in any order.

{% for item in channel.listings limit:3 orderby:updated_at %}
  <article>
    <h2>{{ item.title }}</h2>
    <time>{{ item.updated_at }}</time>
  </article>
{% endfor %}

Single Channel Items

Fetch one item by slug using the {% single %} tag. This is used on detail pages where the URL contains the item slug.

{% single listing from channel.listings %}
  <h1>{{ listing.title }}</h1>
  <p>{{ listing.description }}</p>
  <img src="{{ listing.image_url }}" alt="{{ listing.title }}">
{% else %}
  <p>Listing not found.</p>
{% endsingle %}

The {% else %} block renders when no item matches the current slug — use it for a 404-style fallback.

Channel URL Patterns

URL patterns are set in Admin → Channels → Configure step. They control how channel items get their own pages.

  • Pattern format: /listing/{slug}
  • Router behavior: The front router matches incoming URLs against the pattern, extracts the slug, and sets $_GET['slug']
  • Template resolution: The template filename must match the first segment of the URL pattern — a pattern of /listing/{slug} requires a listing.html template in your theme
  • Fallback: If no slug-named template exists, the router falls back to channel.html

Example flow:

  1. A visitor navigates to /listing/downtown-loft
  2. The router matches the URL against the channel's pattern /listing/{slug}
  3. The slug (downtown-loft) is extracted and set as $_GET['slug']
  4. The matching template file (listing.html) is rendered
  5. Inside the template, {% single listing from channel.listings %} looks up the item by that slug
Note — This is the same convention used for collection URL patterns. If you already know how collection routing works, channel routing is identical.

Multiple Channels on One Page

You can use multiple channel loops on the same page. Each loop is independent.

{% for listing in channel.properties limit:3 %}
  <div class="property-card">
    <h3>{{ listing.address }}</h3>
    <p>{{ listing.price }}</p>
  </div>
{% endfor %}

{% for review in channel.google_reviews limit:5 %}
  <div class="review">
    <p><strong>{{ review.author }}</strong> — {{ review.rating }} stars</p>
    <p>{{ review.text }}</p>
  </div>
{% endfor %}

Mixing Channels and Collections

Channel loops and collection loops work side by side on the same page. This is useful for pages that combine your own content with external data.

<section class="our-posts">
  <h2>Latest Articles</h2>
  {% for post in collection.post limit:3 orderby:published_at %}
    <a href="{{ post.url }}">{{ post.title }}</a>
  {% endfor %}
</section>

<section class="external-listings">
  <h2>Featured Listings</h2>
  {% for listing in channel.mls limit:6 %}
    <div class="listing-card">
      <h3>{{ listing.address }}</h3>
      <p>{{ listing.price }}</p>
    </div>
  {% endfor %}
</section>

Full Page Example (Real Estate Listings)

A complete template showing a channel loop with conditionals, images, and links to single item pages. The channel slug is properties with URL pattern /property/{slug}.

Listing page (properties.html)

{% include 'head' %}
{% include 'nav' %}

<main>
  <h1>Available Properties</h1>

  <div class="property-grid">
    {% for property in channel.properties limit:20 %}
      <a href="{{ property.url }}" class="property-card">
        {% if property.image %}
          <img src="{{ property.image }}" alt="{{ property.title }}">
        {% endif %}
        <h2>{{ property.title }}</h2>
        <p class="price">{{ property.price }}</p>
        <p class="location">{{ property.city }}, {{ property.state }}</p>
        {% if property.bedrooms %}
          <span>{{ property.bedrooms }} beds</span>
        {% endif %}
        {% if property.bathrooms %}
          <span>{{ property.bathrooms }} baths</span>
        {% endif %}
      </a>
    {% else %}
      <p>No properties available at this time.</p>
    {% endfor %}
  </div>
</main>

{% include 'footer' %}

Detail page (property.html)

{% include 'head' %}
{% include 'nav' %}

{% single property from channel.properties %}
<main class="property-detail">
  <h1>{{ property.title }}</h1>
  <p class="price">{{ property.price }}</p>

  {% if property.image %}
    <img src="{{ property.image }}" alt="{{ property.title }}" class="hero-image">
  {% endif %}

  <div class="property-info">
    {% if property.bedrooms %}
      <span>{{ property.bedrooms }} beds</span>
    {% endif %}
    {% if property.bathrooms %}
      <span>{{ property.bathrooms }} baths</span>
    {% endif %}
    {% if property.sqft %}
      <span>{{ property.sqft }} sqft</span>
    {% endif %}
  </div>

  <div class="description">{{ property.description | raw }}</div>

  <p class="location">{{ property.city }}, {{ property.state }}</p>
</main>
{% else %}
<main>
  <h1>Property Not Found</h1>
  <p>This property listing is no longer available.</p>
</main>
{% endsingle %}

{% include 'footer' %}
Caching Behavior — Channel data is cached in SQLite. Auto-sync fires when the TTL expires on the next template render. Manual sync is available from the admin dashboard. If the external API is down, the last cached data is served — your site never goes blank because of an upstream outage.
Comparison with Collection Loops — Collection loops use collection.slug, channel loops use channel.slug. Everything else — loop options, the {% else %} branch, conditionals inside loops — works identically. You can prototype with a collection and later switch to a channel (or vice versa) by simply changing the prefix.

For full details on setting up channels, authentication, caching configuration, and the sync dashboard, see Channels.
