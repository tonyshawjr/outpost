Personal Blog

A complete walkthrough for building a personal blog — posts, folders, author bio, paginated archive, and single post pages. This is the most thorough recipe and covers all core Outpost concepts.

Collections Folders Globals Pagination

1. Collection setup

Go to Admin → Collections → New Collection and create a collection with the following settings:

SettingValue
NamePosts
Slugpost
URL pattern/post/{slug}

Add the following fields to the schema:

Field nameTypeNotes
titleTextRequired. Used as the post heading and SEO title.
excerptTextareaShort summary shown in listing pages and meta description.
featured_imageImageHero image for the post and the listing card.
bodyRichtextFull post content — TipTap editor.
authorTextAuthor display name.

2. Folder setup

Go to Admin → Folders → New Folder and create two folders, both attached to the post collection:

NameSlugHierarchical
CategorycategoryYes — allows parent/child categories
TagtagNo — flat list
Slug matching is exact The slug you use in folder.slug in your templates must match the slug you entered when creating the folder in the admin — exactly, including singular vs. plural. If the slug is category, use folder.category. A mismatch produces no output and no error. See Folder Loops for details.

2b. Work / experience collection (optional)

A personal blog often includes a work history or resume section. Add a second collection for this:

SettingValue
NameWork
Slugwork
URL pattern(none — not publicly routed)

Fields to add:

Field nameType
companyText
roleText

end_dateText

Then loop over it in your homepage sidebar:

{% for job in collection.work limit:4 orderby:start_date %}
  <div class="work-item">
    <span class="company">{{ job.company }}</span>
    <span class="role">{{ job.role }}</span>
    <span class="dates">{{ job.start_date }} — {{ job.end_date }}</span>
  </div>
{% endfor %}

{# Optional CV download link #}
{% if @cv_url %}
  <a href="{{ @cv_url | link }}" target="_blank">Download CV</a>
{% endif %}

Add cv_url as a Link global field in Admin → Global Fields so users can enter the CV download URL.

4. Theme file structure

themes/blog/
  index.html         ← homepage — recent posts + bio
  blog.html          ← all posts / paginated archive
  post.html          ← single post
  about.html         ← about page (add to theme.json routes as /about)
  contact.html       ← contact page (optional)
  404.html           ← not found page
  partials/
    head.html        ← DOCTYPE through <body>
    nav.html         ← site navigation
    footer.html      ← </body></html>
  assets/
    style.css
  theme.json
Add front matter to every template It is good practice to open each .html template with a {#--- ---#} front matter block that documents its route, fields, and loops. It is stripped at compile time so there is zero runtime cost. See Front Matter Convention.

5. partials/head.html

This partial contains everything from <!DOCTYPE html> through the opening <body> tag. It is included at the top of every template.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>{{ meta.title }}{{ @site_name }}{{ /meta.title }}</title>
  <meta name="description" content="{{ meta.description }}{{ @bio_short }}{{ /meta.description }}">
  <link rel="stylesheet" href="/outpost/themes/blog/assets/style.css">
  {# Google Analytics — only output if ga4_id global is set #}
  {% if @ga4_id %}
  <script async src="https://www.googletagmanager.com/gtag/js?id={{ @ga4_id }}"></script>
  <script>
    window.dataLayer = window.dataLayer || [];
    function gtag(){dataLayer.push(arguments);}
    gtag('js', new Date());
    gtag('config', '{{ @ga4_id }}');
  </script>
  {% endif %}
</head>
<body>
Note {{ meta.title }}{{ @site_name }}{{ /meta.title }} uses the per-item SEO title when set, and falls back to the global site_name field. The @ prefix means "look this up as a global field." See Meta Tags.

8. blog.html — Paginated archive

The archive page lists all posts with pagination and category filter links from the folder loop.

{% include 'head' %}
{% include 'nav' %}

<main>
  <header class="archive-header">
    <h1>All Posts</h1>
  </header>

  {# Category filter links — reads ?category= from the URL #}
  <nav class="filter-nav">
    <a href="/blog" class="filter-link" data-label="">All</a>
    {% for label in folder.category %}
      <a href="/blog?category={{ label.slug }}"
         class="filter-link"
         data-label="{{ label.slug }}">
        {{ label.name }} ({{ label.count }})
      </a>
    {% endfor %}
  </nav>

  {# Paginated, filterable post list #}
  <div class="posts-list">
    {% for post in collection.post filteredby:category paginate:10 %}
      <article class="post-row">
        <div class="post-row-meta">
          <span class="post-date">{{ post.published_at }}</span>
        </div>
        <div class="post-row-body">
          <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
          {% if post.excerpt %}
            <p>{{ post.excerpt }}</p>
          {% endif %}
          {% if post.author %}
            <span class="post-author">by {{ post.author }}</span>
          {% endif %}
        </div>
      </article>
    {% endfor %}
  </div>

  {# Pagination controls — automatically preserves ?category= param #}
  {% pagination %}
</main>

{# Highlight the active filter using JS #}
<script>
  const params = new URLSearchParams(location.search);
  const active = params.get('category') || '';
  document.querySelectorAll('.filter-link').forEach(a => {
    if (a.dataset.label === active) a.classList.add('active');
  });
</script>

{% include 'footer' %}

9. post.html — Single post

The single post template renders all fields, folder labels, and a related posts section.

{% include 'head' %}
{% include 'nav' %}

<main>
  {% single post from collection.post %}

    <article class="post-single">
      {# Featured image #}
      {% if post.featured_image %}
        <div class="post-hero-image">
          <img src="{{ post.featured_image | image }}" alt="{{ post.title }}">
        </div>
      {% endif %}

      <header class="post-header">
        <h1>{{ post.title }}</h1>
        <div class="post-meta">
          {% if post.author %}
            <span>by {{ post.author }}</span>
          {% endif %}
          <span>{{ post.published_at }}</span>
        </div>
      </header>

      {# Richtext body — use | raw to render HTML #}
      <div class="post-body">
        {{ post.body | raw }}
      </div>

      {# Folder labels — category and tag #}
      <footer class="post-footer">
        {% for label in post.labels.category %}
          <a href="/blog?category={{ label.slug }}" class="post-tag">{{ label.name }}</a>
        {% endfor %}
        {% for label in post.labels.tag %}
          <a href="/blog?tag={{ label.slug }}" class="post-tag post-tag-secondary">{{ label.name }}</a>
        {% endfor %}
      </footer>
    </article>

    {# Related posts #}
    <section class="related-posts">
      <h2>Related Posts</h2>
      <div class="posts-grid">
        {% for related in collection.post related:post limit:3 %}
          <article class="post-card">
            <h3><a href="{{ related.url }}">{{ related.title }}</a></h3>
            {% if related.excerpt %}
              <p>{{ related.excerpt }}</p>
            {% endif %}
          </article>
        {% endfor %}
      </div>
    </section>

  {% else %}
    <div class="not-found">
      <h1>Post not found</h1>
      <a href="/blog">Back to all posts</a>
    </div>
  {% endsingle %}
</main>

11. RSS feed

Outpost does not auto-generate an RSS feed, but you can build one using the public Content API. Fetch from /outpost/content/items?collection=post&limit=20&orderby=published_at&order=desc and transform the JSON response into an RSS XML document on your frontend or in a small standalone PHP script.

Tip For a quick RSS feed, create a file called feed.php in your site root, set Content-Type: application/rss+xml, fetch from the Content API using file_get_contents or cURL, and template the XML manually. No Outpost dependencies needed.

12. SEO checklist

Outpost handles most SEO fundamentals automatically — canonical URLs, sitemaps, and clean slug-based routing. For a blog specifically:

  • Set the excerpt field on every post — it is used as the meta description fallback
  • Use the {{ meta.title }}Default{{ /meta.title }} and {{ meta.description }}Default{{ /meta.description }} tags in head.html — they output per-item overrides when set in the editor, and fall back to the defaults you provide
  • Add featured_image to every post — used for Open Graph previews when you add <meta property="og:image"> to head.html
  • Add a sitemap.xml — Outpost generates one automatically; link to it in robots.txt

Open Graph tags in partials/head.html:

{# Open Graph — for link previews in social and messaging apps #}
<meta property="og:title" content="{{ meta.title }}{{ @site_name }}{{ /meta.title }}">
<meta property="og:description" content="{{ meta.description }}{{ @bio_short }}{{ /meta.description }}">
<meta property="og:type" content="website">
{% if featured_image %}
  <meta property="og:image" content="{{ featured_image | image }}">
{% endif %}

13. Scheduled publishing

Outpost supports scheduled publishing — set a future Publish At date on a post and it will not appear in the frontend (or the Content API) until that date and time. This is useful for preparing posts in advance and publishing them on a schedule.

Scheduled posts appear as "Scheduled" in the admin post list. The status switches to "Published" automatically when the publish date is reached on the next request to the site.

Note Outpost does not run a background cron job. Scheduled posts become visible on the first page request after the publish time. For high-traffic sites this is effectively instant. For low-traffic sites, the post may appear slightly late if nobody visits the site exactly at the scheduled time.

14. theme.json

Every theme needs a theme.json at its root. For the blog theme:

{
  "name": "Blog",
  "version": "1.0.0",
  "description": "A minimal personal blog theme",
  "author": "Your Name",
  "screenshot": "assets/screenshot.png",
  "routes": {
    "/": "index.html",
    "/blog": "blog.html",
    "/post/{slug}": "post.html"
  }
}
Tip The routes object in theme.json maps URL patterns to template files. The router uses this to determine which template to render for each URL. Collection URL patterns (like /post/{slug}) automatically set $_GET['slug'] for the {% single %} tag.
