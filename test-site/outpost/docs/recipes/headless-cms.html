Headless / API

Use Outpost as a content backend. Manage everything in the admin, then pull your content via the public REST API into any frontend — React, Next.js, Vue, Astro, a mobile app, or a static site generator.

REST API No Auth Required CORS Open JSON

1. The public Content API

Outpost includes a read-only public Content API that requires no authentication. It returns only published content and is open to cross-origin requests — no CORS configuration needed.

PropertyValue
Base URLhttps://yoursite.com/outpost/content/
AuthenticationNone — fully public
CORSOpen (Access-Control-Allow-Origin: *)
Content filterPublished items only — drafts are never exposed
Response formatJSON
Note The public Content API is separate from the admin API at /outpost/api.php. The admin API requires a session cookie and CSRF token. Use the Content API for all frontend reads.

2. Key endpoints

EndpointDescription
GET /outpost/content/items?collection=post List items from a collection
GET /outpost/content/item?collection=post&slug=my-slug Single item by slug
GET /outpost/content/globals All global fields as a flat object
GET /outpost/content/folders?collection=post All folder labels attached to a collection
GET /outpost/content/pages All public pages
GET /outpost/content/menus?slug=main A navigation menu with its items and children

3. Query parameters

ParameterTypeDescription
collectionstringCollection slug (required for item endpoints)
slugstringItem slug (for single item fetch)
limitintegerMax items to return (default: 20, max: 100)
offsetintegerSkip N items — use for manual pagination
orderbystringField to sort by, e.g. published_at, title
orderasc / descSort direction (default: desc)
filteredbystringFolder slug to filter by, e.g. category
labelstringFolder label slug to match (used with filteredby)

4. Response shape

All list endpoints return the same envelope:

{
  "ok": true,
  "data": {
    "items": [
      {
        "id": 42,
        "slug": "my-first-post",
        "title": "My First Post",
        "status": "published",
        "published_at": "2025-01-15 09:00:00",
        "url": "/post/my-first-post",
        "data": {
          "excerpt": "A short summary of the post.",
          "featured_image": "/outpost/uploads/2025/01/hero.jpg",
          "body": "<p>Full richtext HTML...</p>",
          "author": "Jane Smith"
        },
        "labels": {
          "category": [
            { "id": 3, "name": "Engineering", "slug": "engineering" }
          ],
          "tag": [
            { "id": 7, "name": "PHP", "slug": "php" }
          ]
        }
      }
    ],
    "total": 48,
    "limit": 10,
    "offset": 0
  }
}
Tip All custom field values are nested under data. The top-level properties (id, slug, title, status, published_at, url) are always present.

5. Basic JavaScript fetch

Fetch and render a list of posts with plain JavaScript:

const BASE = 'https://yoursite.com/outpost/content';

async function getPosts(limit = 10) {
  const res = await fetch(`${BASE}/items?collection=post&limit=${limit}`);
  if (!res.ok) throw new Error(`API error: ${res.status}`);
  const { data } = await res.json();
  return data.items;
}

async function renderPosts() {
  const posts = await getPosts(6);
  const container = document.querySelector('#posts');

  container.innerHTML = posts.map(post => `
    <article>
      ${post.data.featured_image
        ? `<img src="${post.data.featured_image}" alt="${post.title}">`
        : ''
      }
      <h2><a href="${post.url}">${post.title}</a></h2>
      <p>${post.data.excerpt || ''}</p>
      <time>${post.published_at}</time>
    </article>
  `).join('');
}

renderPosts();

6. Next.js — static generation

Use getStaticProps (Pages Router) or fetch in a Server Component (App Router) to build statically at deploy time.

// pages/blog/index.js — Next.js Pages Router
const BASE = process.env.OUTPOST_URL + '/outpost/content';

export async function getStaticProps() {
  const res = await fetch(`${BASE}/items?collection=post&limit=20`);
  const { data } = await res.json();

  return {
    props: { posts: data.items },
    revalidate: 60, // ISR: revalidate every 60 seconds
  };
}

export default function BlogIndex({ posts }) {
  return (
    <main>
      <h1>Blog</h1>
      {posts.map(post => (
        <article key={post.slug}>
          <h2><a href={post.url}>{post.title}</a></h2>
          <p>{post.data.excerpt}</p>
        </article>
      ))}
    </main>
  );
}
// pages/blog/[slug].js — dynamic single post page
export async function getStaticPaths() {
  const res = await fetch(`${BASE}/items?collection=post&limit=100`);
  const { data } = await res.json();

  return {
    paths: data.items.map(p => ({ params: { slug: p.slug } })),
    fallback: 'blocking',
  };
}

export async function getStaticProps({ params }) {
  const res = await fetch(
    `${BASE}/item?collection=post&slug=${params.slug}`
  );
  const { data } = await res.json();

  if (!data.item) return { notFound: true };

  return {
    props: { post: data.item },
    revalidate: 60,
  };
}

export default function Post({ post }) {
  return (
    <article>
      <h1>{post.title}</h1>
      <div dangerouslySetInnerHTML={{ __html: post.data.body }} />
    </article>
  );
}

7. Fetching global fields

Global fields are useful for site-wide data like the site name, logo, and contact details. Fetch them once and pass them to a layout component.

const globalsRes = await fetch(`${BASE}/globals`);
const { data } = await globalsRes.json();

// data.globals is a flat object:

8. Filtering by folder

Use filteredby and label to fetch only items that belong to a specific folder label:

// Posts in the "engineering" category
const res = await fetch(
  `${BASE}/items?collection=post&filteredby=category&label=engineering&limit=10`
);
const { data } = await res.json();
console.log(data.items);

9. When to use the Content API vs. the Admin API

SituationUse
Reading published content in a frontend app Content API — no auth, CORS open
Creating, updating, or deleting content programmatically Admin API — requires session auth + CSRF token
Reading draft or unpublished content Admin API
Server-side rendering at build time (Next.js, Astro) Content API — fast, no credentials needed
Mobile app reading published articles Content API
Syncing theme files from the Builder desktop app Sync API

10. Using Outpost as a mobile app backend

The Content API works for any HTTP client. In a React Native or Flutter app, point your fetch calls to your Outpost instance and render the content natively. Image URLs in the data object are absolute paths — prefix them with your Outpost base URL if needed.

// React Native example
import React, { useEffect, useState } from 'react';
import { FlatList, Text, Image, View } from 'react-native';

const BASE = 'https://yoursite.com/outpost/content';

export default function ArticleList() {
  const [articles, setArticles] = useState([]);

  useEffect(() => {
    fetch(`${BASE}/items?collection=article&limit=20`)
      .then(r => r.json())
      .then(({ data }) => setArticles(data.items));
  }, []);

  return (
    <FlatList
      data={articles}
      keyExtractor={item => item.slug}
      renderItem={({ item }) => (
        <View>
          <Text>{item.title}</Text>
          <Text>{item.data.excerpt}</Text>
        </View>
      )}
    />
  );
}
Tip Set OUTPOST_URL as an environment variable in your frontend project rather than hardcoding it. This makes it easy to point at a staging Outpost instance during development and production at deploy time.

11. Vue 3 / Nuxt pattern

Fetching content in a Nuxt 3 server component or a Vue composable follows the same pattern as Next.js. Use useFetch in Nuxt for automatic SSR + caching:

// composables/usePosts.js — Nuxt 3
export function usePosts(limit = 10) {
  const config = useRuntimeConfig();
  return useFetch(
    `${config.public.outpostUrl}/outpost/content/items`,
    {
      query: { collection: 'post', limit },
      transform: ({ data }) => data.items,
    }
  );
}

// pages/blog.vue
<script setup>
const { data: posts } = await usePosts(12);
</script>

<template>
  <article v-for="post in posts" :key="post.slug">
    <h2>
      <a :href="post.url">{{ post.title }}</a>
    </h2>
    <p>{{ post.data.excerpt }}</p>
  </article>
</template>

12. Astro static site generation

Astro fetches at build time by default. Use fetch inside --- frontmatter:

---
// src/pages/blog/index.astro
const BASE = import.meta.env.OUTPOST_URL + '/outpost/content';
const res = await fetch(`${BASE}/items?collection=post&limit=50`);
const { data } = await res.json();
const posts = data.items;
---

<html>
  <body>
    <main>
      {posts.map(post => (
        <article>
          <h2><a href={post.url}>{post.title}</a></h2>
          <p>{post.data.excerpt}</p>
        </article>
      ))}
    </main>
  </body>
</html>
Tip When using Astro with output: 'static', all API calls happen at build time. Set up a webhook from your Outpost deploy pipeline (or a cron job) to trigger a rebuild whenever content changes.

13. Image URLs in API responses

Image fields in the data object contain the path as stored — typically /outpost/uploads/2025/01/image.jpg. When using Outpost as a headless backend for a separate domain, prepend your Outpost base URL:

const BASE_URL = 'https://cms.yoursite.com';

data.items.forEach(post => {
  const image = post.data.featured_image
    ? BASE_URL + post.data.featured_image
    : null;

  console.log(image);
  // https://cms.yoursite.com/outpost/uploads/2025/01/hero.jpg
});

14. Error handling

The API returns a consistent JSON envelope for both success and error responses. Always check ok before accessing data:

async function fetchItems(collection, params = {}) {
  const url = new URL('/outpost/content/items', BASE_URL);
  url.searchParams.set('collection', collection);
  Object.entries(params).forEach(([k, v]) => url.searchParams.set(k, v));

  const res = await fetch(url.toString());
  const json = await res.json();

  if (!json.ok) {
    throw new Error(json.error || 'API request failed');
  }

  return json.data;
}

// Usage
try {
  const { items, total } = await fetchItems('post', { limit: 10 });
  console.log(`Loaded ${items.length} of ${total} posts`);
} catch (err) {
  console.error('Failed to load posts:', err.message);
}
