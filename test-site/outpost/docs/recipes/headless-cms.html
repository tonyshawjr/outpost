Headless / API

Use Outpost as a content backend. Manage everything in the admin, then pull your content via the public REST API into any frontend — React, Next.js, Vue, Astro, a mobile app, or a static site generator.

REST API No Auth Required CORS Open JSON

Quickstart

Get content from Outpost into a frontend app in three steps:

  1. Create content in the Outpost admin — collections, pages, globals, menus
  2. Fetch it from the public Content API — no auth, no API key needed
  3. Render it in your frontend framework of choice
// Fetch all published posts
const res = await fetch(
  'https://yoursite.com/outpost/api.php?action=content/items&collection=post'
);
const { data, meta } = await res.json();
console.log(`${meta.total} posts`, data);

1. The public Content API

Outpost includes a read-only public Content API that requires no authentication. It returns only published content and is open to cross-origin requests.

PropertyValue
Base URLhttps://yoursite.com/outpost/api.php?action=content/...
AuthenticationNone — fully public
CORSOpen (Access-Control-Allow-Origin: *)
Content filterPublished items only — drafts and members-only pages are never exposed
Rate limit120 requests per 60 seconds per IP
Response formatJSON
Note All endpoints go through api.php?action=content/.... The admin API uses the same api.php but requires a session cookie and CSRF token. The Content API endpoints are public and read-only.

2. Endpoints

EndpointDescription
GET api.php?action=content/items&collection=post List published items from a collection
GET api.php?action=content/items&collection=post&slug=my-post Single item by slug
GET api.php?action=content/pages All public pages with their fields
GET api.php?action=content/pages&path=/about Single page by path
GET api.php?action=content/globals All global fields as a flat object
GET api.php?action=content/menus&slug=main Navigation menu with nested items
GET api.php?action=content/menus List all menus (slug, name, item count)
GET api.php?action=content/collections List all collections with schema info
GET api.php?action=content/folders All folders (optionally filtered by collection)
GET api.php?action=content/labels&folder=categories Labels within a folder
GET api.php?action=content/media Media library listing
GET api.php?action=content/schema Full site schema — collections, pages, globals, folders
Pages vs. Items — don't mix them up

content/pages returns standalone pages (Home, About, Contact) identified by their URL path (e.g. /about). content/items returns entries from a collection (blog posts, products) identified by collection slug + item slug. If you're fetching a specific page like "About Us", use content/pages&path=/about. If you're fetching a blog post, use content/items&collection=blog&slug=my-post.

3. Query parameters

Items endpoint

ParameterTypeDescription
collectionstringCollection slug (required)
slugstringFetch a single item by slug
limitintegerMax items to return (default: 10, max: 100)
offsetintegerSkip N items for pagination
orderbystringSort field: created_at, updated_at, published_at, slug, sort_order
orderstringasc or desc (default: desc)
{folder_slug}stringFilter by folder label — e.g. &categories=engineering
Folder filtering To filter items by folder labels, pass the folder slug as the parameter name and the label slug as the value. For example, if you have a folder called categories with a label engineering, use &categories=engineering. You can combine multiple folder filters in one request.

Folders endpoint

ParameterTypeDescription
collectionstringFilter folders to a specific collection slug (optional)

5. JavaScript helper

A reusable fetch wrapper for any endpoint:

const OUTPOST = 'https://yoursite.com/outpost/api.php';

async function outpost(endpoint, params = {}) {
  const url = new URL(OUTPOST);
  url.searchParams.set('action', endpoint);
  Object.entries(params).forEach(([k, v]) => url.searchParams.set(k, v));

  const res = await fetch(url.toString());
  if (!res.ok) {
    const err = await res.json().catch(() => ({}));
    throw new Error(err.error || `HTTP ${res.status}`);
  }
  return res.json();
}

// Usage
const { data, meta } = await outpost('content/items', {
  collection: 'post',
  limit: '10',
});
console.log(`${meta.total} posts`, data);

6. Next.js

Use getStaticProps (Pages Router) or fetch in a Server Component (App Router).

// pages/blog/index.js — Next.js Pages Router
const API = process.env.OUTPOST_URL + '/outpost/api.php';

export async function getStaticProps() {
  const res = await fetch(
    `${API}?action=content/items&collection=post&limit=20`
  );
  const { data } = await res.json();

  return {
    props: { posts: data },
    revalidate: 60, // ISR: revalidate every 60 seconds
  };
}

export default function BlogIndex({ posts }) {
  return (
    <main>
      <h1>Blog</h1>
      {posts.map(post => (
        <article key={post.slug}>
          <h2><a href={post.url}>{post.title}</a></h2>
          <p>{post.fields.excerpt}</p>
        </article>
      ))}
    </main>
  );
}
// pages/blog/[slug].js — single post page
export async function getStaticPaths() {
  const res = await fetch(
    `${API}?action=content/items&collection=post&limit=100`
  );
  const { data } = await res.json();

  return {
    paths: data.map(p => ({ params: { slug: p.slug } })),
    fallback: 'blocking',
  };
}

export async function getStaticProps({ params }) {
  const res = await fetch(
    `${API}?action=content/items&collection=post&slug=${params.slug}`
  );
  const { data } = await res.json();

  if (!data) return { notFound: true };

  return {
    props: { post: data },
    revalidate: 60,
  };
}

export default function Post({ post }) {
  return (
    <article>
      <h1>{post.title}</h1>
      <div dangerouslySetInnerHTML={{ __html: post.fields.body }} />
    </article>
  );
}

// src/pages/blog/[slug].astro
const { slug } = Astro.params;

);

<article>

8. Vue / Nuxt

Use useFetch in Nuxt 3 for automatic SSR + caching:

// composables/useOutpost.js — Nuxt 3
export function usePosts(limit = 10) {
  const config = useRuntimeConfig();
  const api = config.public.outpostUrl + '/outpost/api.php';
  return useFetch(api, {
    query: { action: 'content/items', collection: 'post', limit },
    transform: ({ data }) => data,
  });
}

// pages/blog.vue
<script setup>
const { data: posts } = await usePosts(12);
</script>

<template>
  <article v-for="post in posts" :key="post.slug">
    <h2>
      <a :href="post.url">{{ post.title }}</a>
    </h2>
    <p>{{ post.fields.excerpt }}</p>
  </article>
</template>

9. Fetching globals and menus

Globals and menus are typically fetched once and shared across your layout.

// Fetch globals
const globalsRes = await fetch(
  `${API}?action=content/globals`
);
const { data: globals } = await globalsRes.json();
console.log(globals.site_name); // "My Blog"

// Fetch a navigation menu
const menuRes = await fetch(
  `${API}?action=content/menus&slug=main`
);
const { data: menu } = await menuRes.json();

menu.items.forEach(item => {
  console.log(item.label, item.url);
  item.children.forEach(child => {
    console.log('  ', child.label, child.url);
  });
});

10. Filtering by folder

Pass the folder slug as the parameter name and the label slug as the value:

// Posts in the "engineering" category
const res = await fetch(
  `${API}?action=content/items&collection=post&categories=engineering&limit=10`
);
const { data: posts } = await res.json();

// Combine multiple folder filters
const res2 = await fetch(
  `${API}?action=content/items&collection=post&categories=engineering&tags=php`
);

11. Image URLs

Image fields contain paths like /outpost/uploads/2025/01/image.jpg. When fetching from a separate domain, prepend your Outpost base URL:

const BASE_URL = 'https://cms.yoursite.com';

posts.forEach(post => {
  const image = post.fields.featured_image
    ? BASE_URL + post.fields.featured_image
    : null;
  console.log(image);
});

12. Mobile apps

The Content API works for any HTTP client — React Native, Flutter, Swift, Kotlin:

// React Native
import React, { useEffect, useState } from 'react';
import { FlatList, Text, View } from 'react-native';

const API = 'https://yoursite.com/outpost/api.php';

export default function ArticleList() {
  const [articles, setArticles] = useState([]);

  useEffect(() => {
    fetch(`${API}?action=content/items&collection=article&limit=20`)
      .then(r => r.json())
      .then(({ data }) => setArticles(data));
  }, []);

  return (
    <FlatList
      data={articles}
      keyExtractor={item => item.slug}
      renderItem={({ item }) => (
        <View>
          <Text>{item.title}</Text>
          <Text>{item.fields.excerpt}</Text>
        </View>
      )}
    />
  );
}

13. Content API vs. Admin API

SituationUse
Reading published content in a frontend app Content API — no auth, CORS open
Creating, updating, or deleting content Admin API — requires session auth + CSRF token (or API key)
Reading draft or unpublished content Admin API
Server-side rendering at build time Content API
Mobile app reading published articles Content API
Syncing theme files from the Builder Sync API
Tip Set OUTPOST_URL as an environment variable in your frontend project rather than hardcoding the URL. This makes it easy to switch between staging and production.

14. Error handling

Always check res.ok before parsing. The API returns JSON error bodies with an error key.

StatusMeaningExample cause
200Success—
400Bad RequestMissing collection param on items endpoint
404Not FoundCollection, page, item, or menu slug doesn't exist
405Method Not AllowedSent a POST instead of GET
429Rate LimitedMore than 120 requests in 60 seconds from same IP
 
// Robust fetch wrapper with error handling
async function outpost(endpoint, params = {}) {
  const url = new URL(OUTPOST);
  url.searchParams.set('action', endpoint);
  Object.entries(params).forEach(([k, v]) => url.searchParams.set(k, v));

  const res = await fetch(url.toString());

  if (!res.ok) {
    const body = await res.json().catch(() => ({}));

    if (res.status === 404) return null; // not found
    if (res.status === 429) {
      const retry = res.headers.get('Retry-After');
      throw new Error(`Rate limited — retry after ${retry}s`);
    }
    throw new Error(body.error || `HTTP ${res.status}`);
  }

  return res.json();
}

// Usage: returns null if page doesn't exist
const page = await outpost('content/pages', { path: '/about' });
if (!page) {
  console.log('Page not found');
} else {
  console.log(page.data.title);
}

15. Edge cases

  • Empty collection — returns { "data": [], "meta": { "total": 0 } }, not an error
  • Non-existent folder label — returns empty results, not an error
  • Members-only pages — silently excluded from content/pages
  • Draft/scheduled items — silently excluded from content/items
  • limit over 100 — silently capped at 100
  • offset past total — returns empty data: [] with correct meta.total
  • Globals page (__global__) — never returned by content/pages; use content/globals
