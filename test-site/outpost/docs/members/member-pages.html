Member Pages

Outpost ships a complete set of pre-built authentication pages for front-end members — login, registration, profile management, password reset, and logout. All live under /outpost/ and are handled by the engine without requiring any theme templates.

Built-in member page URLs

The following pages are provided and handled directly by the Outpost engine. You do not need to create theme templates for them.

URL Purpose
/outpost/login Sign-in form — accepts username or email address
/outpost/register New account registration form (if registration is enabled)
/outpost/forgot-password Request a password reset email
/outpost/reset-password Set a new password using a token from the reset email
/outpost/verify-email Email verification landing page — validates a token from the verification email
/outpost/profile Member profile — update name, email, password, avatar (requires login)
/outpost/logout Destroys member session and redirects to home

Linking to member pages from templates

Hard-code the paths in your Liquid theme partials — typically in nav.html or footer.html:

<nav class="member-nav">
  <a href="/outpost/login">Sign In</a>
  <a href="/outpost/register">Create Account</a>
  <a href="/outpost/profile">My Profile</a>
  <a href="/outpost/logout">Sign Out</a>
</nav>

To make the login and register URLs configurable by the admin (without editing template files), define them as global fields and use the | link filter:

<a href="{{ @login_url | link }}">Sign In</a>
<a href="{{ @register_url | link }}">Join</a>

Login page

The login page at /outpost/login provides:

  • Username or email address input — either works
  • Password input
  • "Remember me" checkbox — sets a 30-day persistent cookie when checked
  • "Forgot password?" link to the password reset flow
  • Redirect support: if the member was sent to login from a protected page, they are returned there after successful login
  • Custom redirect: use /outpost/login?redirect=/dashboard to redirect to a specific page after login
  • Rate limiting: 5 failed attempts per IP per 60 seconds before the IP is temporarily blocked

Registration page

The registration page at /outpost/register can be enabled or disabled in Admin → Settings → Members. When enabled, it provides:

  • Fields: username, email address, password
  • Password minimum length: 8 characters
  • Validates uniqueness of both username and email before creating the account
  • On success: account is created with role free_member and status active
  • If email verification is disabled: member is automatically logged in and redirected
  • If email verification is enabled: member sees a "Check your email" message with a link to the sign-in page. No auto-login occurs.
Note — When registration is disabled, visiting /outpost/register redirects to the login page. New members can only be created by an admin from Admin → Members → New Member.

Email verification page

When Require Email Verification is enabled in Settings, new members receive an email with a verification link pointing to /outpost/verify-email?token=.... This page:

  • Validates the token against the database (must be unexpired and belong to a member account)
  • On success: sets email_verified to the current timestamp, clears the token, and displays a "Verified! Sign in" message with a link to the login page
  • On error: shows "This verification link is invalid or has expired" with links to sign in or register

Tokens expire after 24 hours. If a member's token expires, they can request a new one from the login page using the Resend verification email button that appears when an unverified member tries to sign in.

Member profile page

The profile page at /outpost/profile requires the member to be logged in — non-authenticated visitors are redirected to the login page. The profile page lets members:

  • View their current username, email address, and display name
  • Update their display name and email address
  • Upload or change their avatar image
  • Change their password (requires entering the current password for confirmation)

Admins can also view and edit member accounts from Admin → Members, including resetting passwords without needing the current password.

Logout

/outpost/logout destroys the member session cookie and the remember-me cookie (if set), then redirects to the site home (/) by default. Override the destination with a ?redirect= parameter:

<a href="/outpost/logout?redirect=/goodbye">Sign Out</a>

Member roles

Front-end member accounts have one of two roles:

RoleAssigned byAccess
free_member Automatically on registration Access to pages with visibility set to Members
paid_member Manually by an admin, or via payment integration Access to pages with visibility set to Members or Paid

Member roles are completely separate from admin roles. A free_member cannot access the admin panel, and an admin user is not automatically a front-end member.

Session lifetime

SettingDefault
Session duration (no "remember me")Until browser is closed (session cookie)
Session duration ("remember me" checked)30 days (persistent cookie)
Session namespaceoutpost_member (separate from admin sessions)

Custom login redirect

By default, after login, members are redirected to the page they were trying to access before being sent to the login form, or to / if they navigated directly to /outpost/login. To override the default post-login destination for all members, set the constant in config.php:

// config.php — send all members to /dashboard after login
define('OUTPOST_LOGIN_REDIRECT', '/dashboard');
Note — OUTPOST_LOGIN_REDIRECT only applies when there is no ?redirect= parameter in the login URL. The per-request ?redirect= parameter always takes precedence.

Checking login state in templates

Liquid templates have two built-in variables for checking member status:

{# Check if any member is logged in #}
{% if member %}
  <p>Welcome back!</p>
  <a href="/outpost/profile">My Profile</a>
  <a href="/outpost/logout">Sign Out</a>
{% else %}
  <a href="/outpost/login">Sign In</a>
  <a href="/outpost/register">Create Account</a>
{% endif %}

{# Check specifically for paid members #}
{% if paid_member %}
  <div class="premium-content">
    {{ premium_body | raw }}
  </div>
{% else %}
  <p>This content is for paid members. <a href="/upgrade">Upgrade your account.</a></p>
{% endif %}
Tip — {% if member %} is true for both free_member and paid_member roles. {% if paid_member %} is only true for paid_member. To show content to free members but not to guests, use {% if member %}. To show content only to paid members, use {% if paid_member %}.

Protecting pages from the admin

Individual pages can be restricted to members or paid members via the Visibility selector in the page editor's SEO section:

Visibility settingWho can see the page
Public (default)Everyone — logged-in or not
MembersAny logged-in member (free_member or paid_member)
PaidOnly paid_member accounts

Non-authenticated visitors accessing a restricted page are redirected to the login form with a ?redirect= parameter so they are returned to the page after login. See Protecting Pages for the full reference.

Managing members in the admin

Admin users with the admin or super_admin role can view and manage all member accounts from Admin → Members. From there you can:

  • View all registered members with their role, status, and last active date
  • Create new members directly (bypassing the registration form)
  • Change a member's role between free_member and paid_member
  • Reset any member's password (no email required — sets a new password immediately)
  • Suspend or delete member accounts

Enabling or disabling registration

You can enable or disable public registration from Admin → Settings → Members. When disabled:

  • /outpost/register redirects to the login page
  • New members can only be created by admins from Admin → Members
  • Existing member accounts and login still work normally
