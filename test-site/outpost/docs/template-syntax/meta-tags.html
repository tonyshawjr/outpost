Meta Tags

Two dedicated helpers — {{ meta.title }} and {{ meta.description }} — output SEO-ready values with automatic fallback chains for both regular pages and collection single items.

{{ meta.title }}Default{{ /meta.title }}

Outputs the page title as raw text. You wrap it in the HTML tags yourself — this lets you use the same value in multiple places like <title> and Open Graph tags without duplicating logic.

<title>{{ meta.title }}My Blog{{ /meta.title }}</title>
<meta property="og:title" content="{{ meta.title }}My Blog{{ /meta.title }}">
Note — The inline syntax {{ meta.title "Default" }} is also supported for backwards compatibility.

Fallback Chain for meta.title

The helper walks through the following sources in order, using the first non-empty value it finds:

  1. Item's meta_title field — on single collection item pages (e.g. a blog post detail page), the custom SEO title set per-item in the admin.
  2. Item's title field — automatic fallback for single item pages when no custom meta title is set.
  3. Page's meta_title field — the SEO title set on the current CMS page.
  4. The provided default string — the quoted value you write in the template tag.

This means a blog post with a custom meta title set in the admin will use that title. A post without one automatically uses the post's main title. A static page uses its page-level meta title. The default string is the ultimate fallback when nothing else has been set.

{{ meta.description }}Default{{ /meta.description }}

Outputs the meta description as raw text. Use it in <meta name="description"> and Open Graph tags.

<meta name="description" content="{{ meta.description }}{{ /meta.description }}">
<meta property="og:description" content="{{ meta.description }}{{ /meta.description }}">
Note — An empty wrapping tag {{ meta.description }}{{ /meta.description }} means no hardcoded fallback — the tag outputs whatever the CMS provides, or empty if nothing is set. The inline syntax {{ meta.description "" }} is also supported.

Fallback Chain for meta.description

  1. Item's meta_description field — custom description set per-item in the admin SEO sidebar.
  2. Auto-excerpt from item's excerpt field — first 30 words of the item's excerpt, on single item pages.
  3. Auto-excerpt from item's body field — first 30 words of the richtext body, on single item pages when no excerpt exists.
  4. Page's meta_description field — the description set on the current CMS page.
  5. The provided default string — the quoted value in the template tag.

Per-item SEO in the Admin

When a content editor opens a collection item (e.g. a blog post) in the admin, the right sidebar contains a SEO tab with:

  • Meta Title input — 60 character limit displayed with a live counter and warning when exceeded.
  • Meta Description textarea — 160 character limit with the same live feedback.
  • Google search preview — a live preview showing how the result will appear in search results, updating as the editor types.

These fields are stored in the item's JSON data alongside all other field values. No database migration is needed to add SEO fields to an existing collection.

Character Limits

Field Recommended Length Admin Warning at
Meta title 50–60 characters 60 characters
Meta description 120–160 characters 160 characters

Titles longer than 60 characters may be truncated by Google in search results. Descriptions longer than 160 characters are typically truncated. These are guidelines, not hard limits — the CMS does not prevent saving longer values.

Complete head.html Example

This is the recommended pattern for a theme's partials/head.html partial, showing both meta helpers used alongside Open Graph and Twitter Card tags.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">

  <title>{{ meta.title }}{{ @site_name }}{{ /meta.title }}</title>
  <meta name="description" content="{{ meta.description }}{{ /meta.description }}">

  <!-- Open Graph -->
  <meta property="og:title" content="{{ meta.title }}{{ @site_name }}{{ /meta.title }}">
  <meta property="og:description" content="{{ meta.description }}{{ /meta.description }}">
  <meta property="og:type" content="website">
  <meta property="og:site_name" content="{{ @site_name }}">

  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="{{ meta.title }}{{ @site_name }}{{ /meta.title }}">
  <meta name="twitter:description" content="{{ meta.description }}{{ /meta.description }}">

  {% if @favicon %}
    <link rel="icon" href="{{ @favicon | image }}">
  {% endif %}

  <link rel="stylesheet" href="/themes/my-theme/assets/style.css">

  {% if @ga4_id %}
  <script async src="https://www.googletagmanager.com/gtag/js?id={{ @ga4_id }}"></script>
  <script>
    window.dataLayer = window.dataLayer || [];
    function gtag(){dataLayer.push(arguments);}
    gtag('js', new Date());
    gtag('config', '{{ @ga4_id }}');
  </script>
  {% endif %}
</head>
<body>

Blog Post Page Example

On a single post page, the fallback chain for meta.title means the post's main title is used automatically even if no custom SEO title has been set. A content editor only needs to fill in the SEO fields when they want to customize what appears in search results.

<!-- post.html -->
{% include 'head' %}
{% include 'nav' %}

{% single post from collection.post %}
<article>
  <h1>{{ post.title }}</h1>
  <!-- The <title> tag in head.html will use post.meta_title if set,
       or fall back to post.title automatically -->
  <div class="prose">{{ post.body | raw }}</div>
</article>
{% else %}
<p>Post not found.</p>
{% endsingle %}

{% include 'footer' %}
Tip — Because the fallback chain includes the item's title field, you do not need to tell content editors to fill in the SEO title for every post. Only fill it in when the post title alone isn't sufficient for search — for example, when you want to include the site name or rephrase for search intent.
