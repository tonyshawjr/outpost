Output Tags & Filters

Output tags render field values into your HTML. Every field you reference in a template is automatically registered in the admin editor — no configuration required.

Basic Text Output

The simplest output tag renders a field's value, HTML-escaped, directly into your markup.

HTML-escaping means characters like <, >, and & are converted to safe entities automatically, protecting against XSS. The field is registered in the admin page editor the first time the template is rendered.

With a Default Value

Wrap a default value between the opening and closing tag. The default is shown as a placeholder in the admin editor, and output on the front end when the field is empty. The closing tag uses the field name prefixed with /.

Note — The inline syntax data-outpost="field" (default: "Default") is also supported for backwards compatibility, but wrapping tags are recommended because they support multiline content and HTML in defaults.

Filters

Append a filter with the pipe character | to change how a field is rendered and what input type appears in the admin editor.

| raw — Rich Text / HTML

Outputs the field value without HTML escaping. Use this for fields that contain richtext editor content (TipTap). Content is sanitized before it is stored, so the output is safe.

Warning — Only use | raw for content managed through the CMS richtext editor. Never use it for fields where an editor could enter arbitrary HTML from an untrusted source.

| image — Image URL

Opens the media picker in the admin editor. Outputs the file path or URL of the selected image. Use it in src attributes and CSS background properties.

| link — URL / Link Input

Renders a URL input field in the admin editor. Outputs the URL as a plain string — use it in href attributes.

| textarea — Multi-line Text

Provides a multi-line textarea in the admin editor. Newlines in the stored value are converted to <br> tags on output. Ideal for bios, addresses, and short descriptions where paragraph breaks matter but full richtext is unnecessary.

| select — Dropdown Selector

The admin editor shows a dropdown with options defined in the collection schema. Outputs the selected option's string value.

| toggle — Boolean / Checkbox

Renders a toggle switch in the admin editor. Returns true or false — do not use as direct output. Use it inside an <outpost-if> conditional instead.

Note — The filter declaration data-outpost="field" data-type="toggle" only needs to appear once in the template to register the field as a toggle type. In practice you will always reference the field inside <outpost-if> — the declaration is implicit.

| color — Color Picker

Opens a color picker in the admin editor. Outputs a hex color string like #14b8a6. Use it in inline CSS or pass it to CSS custom properties.

| number — Numeric Input

Renders a number input in the admin editor. Outputs the value as a string.

| date — Date Input

Renders a date picker in the admin editor. Outputs the stored date in ISO 8601 format: 2025-03-04. Format or localize it with JavaScript on the front end.

| focal — Focal Point Position

Outputs the image's focal point as a CSS-ready X% Y% string. Use with object-position or background-position to ensure the important part of the image stays visible when cropped. Returns 50% 50% (center) if no custom focal point has been set.

Note — Set the focal point by clicking on an image preview in the media library detail panel. The | focal filter looks up the stored coordinates for the media file and outputs them as percentages.

| or_body — Excerpt with Body Fallback

If the field (typically excerpt) is empty, this filter automatically generates a 40-word excerpt from the body field of the same item. This is the cleanest way to handle list pages where some items have manual excerpts and some do not.

data-edit — On-Page Editing (Opt-In)

Append data-edit to any field tag to enable on-page editing for that field. When a logged-in admin views the front-end site, fields marked with data-edit become click-to-edit. Fields without data-edit are only editable in the admin panel.

The data-edit modifier works with all field types that support on-page editing: text, textarea, richtext (| raw), and image. It also works with global fields (@ prefix) and collection item fields inside <outpost-single> blocks:

Note — The data-edit modifier only controls front-end editing. All fields remain fully editable in the admin panel regardless of this modifier. The field scanner strips data-edit before detecting types, so data-outpost="hero" data-type="image" data-edit correctly registers as an image field.

Loop Variable Fields

Inside a <outpost-each> or <outpost-single> block, access item fields using dot notation: data-outpost="field". The same filters apply.

Tag Description
data-outpost="field" Escaped text output for any custom field.
data-outpost="field" data-type="raw" Raw HTML output — for richtext fields.
data-outpost="url" Auto-built URL from the collection's url_pattern (e.g. /post/my-slug).
data-outpost="slug" The item's URL slug.
data-outpost="id" Database integer ID.
data-outpost="published_at" Formatted publish date (e.g. "March 4, 2026").
data-outpost="status" Item status: published, draft, or scheduled.
data-outpost="created_at" Creation timestamp.
data-outpost="updated_at" Last-modified timestamp.
data-outpost="featured_image" Direct access to the featured image field (escaped URL).

The templates engine uses HTML data attributes instead of data-outpost tags. The data-outpost attribute specifies the field name. The data-type attribute specifies the field type (defaults to text if omitted). The element's inner content serves as the default value.

Text Output

Add data-outpost="field_name" to any HTML element. The element's text content is used as the default value.

<h1 data-outpost="page_title"></h1>
<h1 data-outpost="hero_heading">Welcome to my site</h1>

Rich Text (data-type="richtext")

<div class="prose" data-outpost="post_body" data-type="richtext"></div>

Image (data-type="image")

<img data-outpost="hero_image" data-type="image" src="placeholder.jpg" alt="">

Link (data-type="link")

<a data-outpost="cta_url" data-type="link" href="#" class="btn">
 <span data-outpost="cta_label">Learn more</span>
</a>

Textarea (data-type="textarea")

<p data-outpost="author_bio" data-type="textarea"></p>

Select (data-type="select")

<span class="category" data-outpost="post_category" data-type="select"></span>

Color (data-type="color")

<section>
 <span data-outpost="section_bg" data-type="color" hidden>#ffffff</span>
</section>

Number (data-type="number")

<span data-outpost="max_results" data-type="number">10</span>

Date (data-type="date")

<time data-outpost="event_date" data-type="date">2026-01-01</time>

Focal Point (data-type="focal")

<img data-outpost="hero_image" data-type="image" data-focal src="placeholder.jpg">

Excerpt with Body Fallback (data-type="or_body")

<p data-outpost="excerpt" data-type="or_body"></p>

Example: Hero Section

<section class="hero">
 <h1 data-outpost="hero_heading">We build things people love.</h1>
 <div data-outpost="hero_body" data-type="richtext"></div>
 <outpost-if field="cta_url" exists>
 <a data-outpost="cta_url" data-type="link" href="#" class="btn">
 <span data-outpost="cta_label">Get started</span>
 </a>
 </outpost-if>
</section>
