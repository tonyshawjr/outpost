Conditionals

Show or hide template sections based on whether fields have values, what those values are, and whether the visitor is a logged-in admin.

Simple Field Check

The most common conditional — show an element only when a field has been filled in. True when the field is non-empty (not an empty string, not 0, not null, not an empty array).

{% if field_name %}
  <!-- rendered when field_name has a value -->
{% endif %}
{% if field_name %}
  <!-- rendered when field_name has a value -->
{% else %}
  <!-- rendered when field_name is empty -->
{% endif %}

A typical use case: show a featured image only when one has been uploaded, with a CSS placeholder as the fallback.

<article class="post-card">
  {% if featured_image %}
    <img src="{{ featured_image | image }}" alt="{{ page_title }}" class="post-thumb">
  {% else %}
    <div class="post-thumb post-thumb--placeholder"></div>
  {% endif %}
  <h2>{{ page_title }}</h2>
  <p>{{ page_excerpt | or_body }}</p>
</article>

Global Field Check

Use the @ prefix to check whether a global field has a value. This is especially important for optional global fields like social media links — without the check, an empty <a href=""> tag would be rendered for editors who have not filled in that field.

{% if @global_name %}
  <!-- rendered when the global field has a value -->
{% endif %}
<div class="social-links">
  {% if @twitter_url %}
    <a href="{{ @twitter_url | link }}" target="_blank" rel="noopener">Twitter</a>
  {% endif %}
  {% if @github_url %}
    <a href="{{ @github_url | link }}" target="_blank" rel="noopener">GitHub</a>
  {% endif %}
  {% if @linkedin_url %}
    <a href="{{ @linkedin_url | link }}" target="_blank" rel="noopener">LinkedIn</a>
  {% endif %}
  {% if @instagram_url %}
    <a href="{{ @instagram_url | link }}" target="_blank" rel="noopener">Instagram</a>
  {% endif %}
</div>

Loop Variable Check

Inside a {% for %} or {% single %} block, check whether an item's field has a value using dot notation.

{% if item.field %}
  <!-- rendered when the item field has a value -->
{% endif %}
{% for post in collection.post limit:10 %}
<article class="post-card">
  {% if post.featured_image %}
    <img src="{{ post.featured_image }}" alt="{{ post.title }}">
  {% endif %}
  <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
  {% if post.excerpt %}
    <p>{{ post.excerpt }}</p>
  {% endif %}
</article>
{% else %}
<p class="empty">No posts published yet.</p>
{% endfor %}

String Comparisons

Compare a field's value against a literal string using == (equality) or != (inequality). Values must be wrapped in double quotes. Comparisons use strict string equality.

{% if item.field == "value" %}
  <!-- rendered when item.field exactly equals "value" -->
{% endif %}

{% if item.field != "value" %}
  <!-- rendered when item.field does not equal "value" -->
{% endif %}

Equality: Highlighting Featured Posts

{% for post in collection.post limit:10 %}
<article class="post-card{% if post.category == "featured" %} post-card--featured{% endif %}">
  {% if post.category == "featured" %}
    <span class="badge">Featured</span>
  {% endif %}
  <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
</article>
{% endfor %}

Inequality: Excluding Archived Items

{% for project in collection.portfolio %}
  {% if project.status != "archived" %}
    <a href="{{ project.url }}" class="project-thumb">
      <img src="{{ project.thumbnail | image }}" alt="{{ project.title }}">
      <span>{{ project.title }}</span>
    </a>
  {% endif %}
{% endfor %}

Multi-branch Status Display

{% single post from collection.post %}
<div class="post-status">
  {% if post.status == "published" %}
    <span class="status status--live">Live</span>
  {% else %}
    {% if post.status == "draft" %}
      <span class="status status--draft">Draft</span>
    {% else %}
      <span class="status">{{ post.status }}</span>
    {% endif %}
  {% endif %}
</div>
{% endsingle %}

Admin Check

The special admin variable is true only when a logged-in admin is previewing the page through the admin panel. Use it to show edit hints, debug information, or admin-only UI elements that should never be visible to public visitors.

{% if admin %}
  <!-- only visible to logged-in admins in preview mode -->
{% endif %}
<section class="hero">
  <h1>{{ hero_heading "Click to edit this heading" }}</h1>
  <p>{{ hero_subheading "Add your tagline here" }}</p>

  {% if admin %}
    <div class="admin-hint">
      <strong>Admin:</strong> Edit the hero content in the page editor on the right.
    </div>
  {% endif %}
</section>
<!-- Show collection item count for admin debugging -->
{% if admin %}
  <p style="font-size:12px;color:#999;">
    Debug: rendering collection.post
  </p>
{% endif %}
{% for post in collection.post %}
  ...
{% endfor %}

Nested Conditionals

Conditionals can be nested to any depth. The indentation is optional but improves readability.

<!-- Portfolio card with nested checks -->
{% for project in collection.portfolio limit:6 %}
<div class="project-card">
  {% if project.hero_image %}
    <a href="{{ project.url }}">
      <img src="{{ project.hero_image }}" alt="{{ project.title }}">
    </a>
  {% else %}
    {% if project.thumbnail %}
      <a href="{{ project.url }}">
        <img src="{{ project.thumbnail }}" alt="{{ project.title }}">
      </a>
    {% endif %}
  {% endif %}
  <h3><a href="{{ project.url }}">{{ project.title }}</a></h3>
  {% if project.client %}
    <p class="client">{{ project.client }}</p>
  {% endif %}
</div>
{% endfor %}

Conditionals Inside Single Items

{% single post from collection.post %}
<article>
  <header>
    {% if post.featured_image %}
      <img src="{{ post.featured_image }}" alt="{{ post.title }}" class="post-hero">
    {% endif %}
    <h1>{{ post.title }}</h1>
    {% if post.author %}
      <p class="byline">By {{ post.author }}</p>
    {% endif %}
    {% if post.published_at %}
      <time datetime="{{ post.published_at }}">{{ post.published_at }}</time>
    {% endif %}
  </header>

  <div class="prose">{{ post.body | raw }}</div>

  {% if post.cta_url %}
    <div class="post-cta">
      <a href="{{ post.cta_url | link }}" class="btn">
        {{ post.cta_label "Read more" }}
      </a>
    </div>
  {% endif %}
</article>
{% else %}
<p>Post not found.</p>
{% endsingle %}

Limitations

Note — The template language is intentionally simple. Keep complex logic in PHP helpers or JavaScript rather than template conditionals.
  • No {% elsif %} / {% elseif %} — use nested {% if %}...{% else %}...{% if %} blocks.
  • No and / or operators — achieve compound checks with nested {% if %} tags.
  • No negation shorthand ({% if !field %}) — use {% if field %}{% else %}...{% endif %} or {% if field != "value" %}.
  • String comparisons only — no numeric or date comparisons with < / >.
