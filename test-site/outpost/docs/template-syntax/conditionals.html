Conditionals

Show or hide template sections based on whether fields have values, what those values are, and whether the visitor is a logged-in admin.

Simple Field Check

The most common conditional — show an element only when a field has been filled in. True when the field is non-empty (not an empty string, not 0, not null, not an empty array).

A typical use case: show a featured image only when one has been uploaded, with a CSS placeholder as the fallback.

Global Field Check

Use the @ prefix to check whether a global field has a value. This is especially important for optional global fields like social media links — without the check, an empty <a href=""> tag would be rendered for editors who have not filled in that field.

Loop Variable Check

Inside a <outpost-each> or <outpost-single> block, check whether an item's field has a value using dot notation.

String Comparisons

Compare a field's value against a literal string using == (equality) or != (inequality). Values must be wrapped in double quotes. Comparisons use strict string equality.

Admin Check

The special admin variable is true only when a logged-in admin is previewing the page through the admin panel. Use it to show edit hints, debug information, or admin-only UI elements that should never be visible to public visitors.

Nested Conditionals

Conditionals can be nested to any depth. The indentation is optional but improves readability.

Conditionals Inside Single Items

In the templates engine, conditionals use the <outpost-if> custom element. The content inside is rendered only when the condition is true. Use a second <outpost-if> with the empty attribute to handle the false case.

Field Exists

<outpost-if field="featured_image" exists>
 <img data-outpost="featured_image" data-type="image" src="" alt="">
</outpost-if>

Empty Check (Negation)

<outpost-if field="featured_image" exists>
 <img data-outpost="featured_image" data-type="image" src="" alt="">
</outpost-if>
<outpost-if field="featured_image" empty>
 <div class="placeholder"></div>
</outpost-if>
Note -- There is no <outpost-else> element. Use a second <outpost-if> with the empty attribute to achieve if/else behavior.

Global Field Check

<outpost-if field="twitter_url" scope="global" exists>
 <a data-outpost="twitter_url" data-scope="global" data-type="link" href="#" target="_blank">Twitter</a>
</outpost-if>

String Equality

<outpost-if field="category" equals="featured">
 <span class="badge">Featured</span>
</outpost-if>

String Inequality

<outpost-if field="status" not="archived">
 <a href="" data-outpost="url"><span data-outpost="title"></span></a>
</outpost-if>

Member Auth Check

<outpost-if member="logged-in">
 <p>Welcome back, member!</p>
</outpost-if>

<outpost-if member="logged-out">
 <a href="/login">Sign in</a>
</outpost-if>

Example: Post Detail Page

<outpost-single collection="post">
 <article>
 <outpost-if field="featured_image" exists>
 <img data-outpost="featured_image" data-type="image" src="" alt="" class="post-hero">
 </outpost-if>
 <h1 data-outpost="title"></h1>
 <outpost-if field="author" exists>
 <p class="byline">By <span data-outpost="author"></span></p>
 </outpost-if>
 <div class="prose" data-outpost="body" data-type="richtext"></div>
 </article>
</outpost-single>

<outpost-single collection="post" else>
 <p>Post not found.</p>
</outpost-single>

Limitations

Note — The template language is intentionally simple. Keep complex logic in PHP helpers or JavaScript rather than template conditionals.
  • No elseif or <outpost-else> — use a pair of <outpost-if> blocks with exists and empty attributes.
  • No and / or operators — achieve compound checks with nested <outpost-if> elements.
  • For negation, use <outpost-if field="name" empty> or <outpost-if field="name" not="value">.
  • String comparisons only — no numeric or date comparisons with < / >.
