Media

Upload and manage images, documents, and video files from the admin. Files are stored on disk, auto-processed for web delivery, and referenced by URL in page fields, collection items, global fields, and templates.

Media library

The media library is at Admin → Media. It shows all uploaded files as a responsive grid of thumbnails. Click any file to open its detail panel, which shows the full path, pixel dimensions (for images), file size, and a Copy Path button you can use to paste the URL into any text or link field.

You can also upload files directly from within the page editor and collection item editor — clicking an image field opens the media library in an inline modal. Newly uploaded files are immediately available for selection without leaving the editor.

Supported file types

CategoryExtensions
Images (raster) jpg, jpeg, png, gif, webp, avif
Images (vector) svg
Documents pdf
Video mp4, webm
Audio mp3, wav

Upload limits and auto-processing

BehaviorDetail
Max file size 10 MB per file by default. Configure via OUTPOST_MAX_UPLOAD_SIZE in config.php.
Image resizing Images wider than 2400px are automatically scaled down to 2400px. Aspect ratio is always preserved. Smaller images are not upscaled.
Thumbnail generation A thumbnail is generated at max 400×300px (aspect ratio preserved) and stored alongside the original with a thumb_ prefix in the filename.
Dimensions recorded Pixel width and height are stored in the database for all uploaded images.
Filename sanitization Filenames are sanitized on upload — special characters are replaced with hyphens, and a Unix timestamp prefix is added to prevent naming collisions.
Note — Processing (resize + thumbnail) only applies to raster images (jpg, png, gif, webp, avif). SVG, PDF, video, and audio files are stored as-is without processing.

File storage paths

All uploaded files are stored under outpost/uploads/. Files are never put in subdirectories — everything is flat in the uploads folder.

FileStored path example
Original upload outpost/uploads/1700000000_hero-photo.jpg
Generated thumbnail outpost/uploads/thumb_1700000000_hero-photo.jpg
PDF document outpost/uploads/1700000001_company-brochure.pdf
SVG icon (sanitized) outpost/uploads/1700000002_logo.svg

The URL stored in the database (and output by the | image filter) is the web-root-relative path:

/outpost/uploads/1700000000_hero-photo.jpg
Warning — Never delete the outpost/uploads/ directory. It contains all user media and is not regenerated automatically. When re-deploying, always preserve this directory. See Deploy for the correct deploy sequence.

SVG sanitization

SVG files can contain embedded scripts that execute in a browser if served with image/svg+xml content-type. Outpost automatically sanitizes uploaded SVGs before saving them to disk:

  • <script> tags and their contents are removed entirely.
  • on* event handler attributes (onclick, onload, onerror, etc.) are stripped from all elements.
  • External resource references (xlink:href pointing to external URLs) are removed.

The sanitized SVG is written to disk in place of the raw upload. The result is safe to serve directly and embed inline in HTML. See Sanitization for implementation details.

Using media in templates — the | image filter

Image fields in page templates, collection items, and global fields use the | image filter. This filter both outputs the stored URL and registers the field as an image picker widget in the admin editor — so editors see a proper media browser, not a raw text box.

Page and collection image fields

{# Page template — hero image field #}
<img
  src="{{ hero_image | image }}"
  alt="{{ hero_title }}"
  width="1200"
  height="630"
  loading="lazy"
>

{# Collection item — featured image #}
{% if post.featured_image %}
  <img
    src="{{ post.featured_image | image }}"
    alt="{{ post.title }}"
    loading="lazy"
  >
{% endif %}

{# As a CSS background image #}
<div style="background-image: url('{{ banner_image | image }}')">
  ...
</div>

Global image fields

{# Site logo in the header partial #}
<img
  src="{{ @logo | image }}"
  alt="{{ @site_name }}"
  height="32"
>

{# Author profile photo — only render if the global is set #}
{% if @profile_photo %}
  <img
    src="{{ @profile_photo | image }}"
    alt="{{ @author_name }}"
    class="avatar"
  >
{% endif %}

Conditional rendering for optional images

Use {% if field_name %} to avoid rendering a broken <img> when the image field is empty:

{% if post.featured_image %}
  <figure class="post-cover">
    <img src="{{ post.featured_image | image }}" alt="{{ post.title }}">
  </figure>
{% endif %}

Documents and non-image files

For PDFs, audio, video, and other non-image files, use the | link filter in your template. This outputs the raw URL for use in an anchor tag or media element:

{# PDF download link #}
<a href="{{ brochure_pdf | link }}" download>
  Download Brochure (PDF)
</a>

{# Embedded video file #}
<video controls preload="metadata">
  <source src="{{ demo_video | link }}" type="video/mp4">
</video>

{# Audio player #}
<audio controls>
  <source src="{{ episode_audio | link }}" type="audio/mpeg">
</audio>

Gallery fields

For collection schema fields of type gallery, the field holds an ordered array of image URLs. Loop over them with {% for image in item.gallery_field %}:

<div class="gallery">
  {% for image in post.gallery %}
    <a href="{{ image | image }}">
      <img
        src="{{ image | image }}"
        alt=""
        loading="lazy"
      >
    </a>
  {% endfor %}
</div>

Gallery fields are reordered in the admin via drag-and-drop. The order in the template output matches the order set in the editor. See Gallery Loops for the full reference.

Responsive images

Outpost stores the original image at up to 2400px wide and generates a 400px thumbnail. The CMS does not create intermediate sizes (e.g., 640px, 960px). For responsive images, use CSS sizing or the HTML srcset attribute manually:

<img
  src="{{ hero_image | image }}"
  alt="{{ hero_alt }}"
  style="width: 100%; height: auto;"
  loading="lazy"
  decoding="async"
>
Tip — Add loading="lazy" and decoding="async" to all non-hero images. These native HTML attributes defer loading of off-screen images and decode images asynchronously, improving page performance with no JavaScript required.

Copying file paths

In the media library, click any file to open its detail panel. Click Copy Path to copy the web-root-relative path (e.g., /outpost/uploads/1700000000_photo.jpg) to the clipboard. Paste it directly into:

  • A URL field in a page or collection item editor
  • A richtext editor body (for inline images in content)
  • A template file in the code editor
  • CSS as a background-image URL

Editing images

Select an image in the media grid to open its detail panel. The panel includes tools for editing alt text, resizing, and cropping — all without leaving the media library.

Alt text

Every media file has an alt_text field stored in the database. Type a description in the Alt text input and press Enter or click Save. Alt text is returned in both the Admin API and Public Content API responses, so you can use it in templates for accessibility.

Resize

Enter new pixel dimensions in the Width and Height fields. A lock icon between them toggles aspect-ratio locking (on by default) — changing one dimension automatically calculates the other. Click Apply Resize to process. The original file is overwritten and the thumbnail is regenerated.

  • Maximum dimension: 4800px per side
  • You can provide just width or just height — the other is calculated automatically
  • Applies to raster images only (JPEG, PNG, GIF, WebP). SVGs cannot be resized.

Crop

Click Crop Image to reveal the crop controls. Enter the origin (X, Y) and the crop region size (W, H) in pixels. Click Apply Crop to process. Values are clamped to image bounds on the server.

Warning — Both resize and crop operations are destructive. The original file is overwritten on disk. There is no undo. If you need to preserve the original, download it first.
Note — Image editing requires the PHP GD extension. If GD is not installed, the resize and crop controls will still appear in the admin, but the server will return an error when you try to apply a transformation. Most PHP installations include GD by default.

Deleting files

Deleting a file from the media library removes the record from the database and deletes both the original file and its thumbnail from disk. Any templates or content fields that still reference the deleted path will output a broken image or link.

Note — There is no "find references" feature. If you need to replace an image, upload the new file first, update all fields and templates that reference the old path, and then delete the old file.

Public URL access

All files in outpost/uploads/ are publicly accessible at /outpost/uploads/filename. There is no authentication on upload URLs — any uploaded file can be accessed by anyone with the URL. Do not upload sensitive or private files through the media library.

Configuring the upload limit

The default maximum upload size is 10 MB. To change it, define the constant in config.php before Outpost initializes:

// config.php — set upload limit to 25 MB
define('OUTPOST_MAX_UPLOAD_SIZE', 25 * 1024 * 1024);
Note — PHP's own upload limits (upload_max_filesize and post_max_size in php.ini) also apply. OUTPOST_MAX_UPLOAD_SIZE can only set a limit lower than PHP's — it cannot exceed PHP's configured maximum.
