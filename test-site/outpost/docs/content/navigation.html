Navigation Menus

Create and manage menus in the admin, then render them in your theme templates with a simple loop. Menus support nested items for dropdown navigation and multiple independent menus per site.

Overview

Outpost lets you create any number of named menus — a main navigation, a footer link list, a social media menu, a legal links bar, and so on. Each menu is identified by a slug that you reference in templates. Items in each menu are ordered and nested via drag-and-drop in the admin.

Creating a menu

  1. Go to Admin → Navigation and click New Menu.
  2. Enter a Name (e.g., "Main Navigation") and a Slug (e.g., main).
  3. Save. The empty menu appears in the list.
  4. Click the menu to open it and add items with Add Item.

Common menu naming conventions:

NameSlugTypical use
Main NavigationmainPrimary header navigation
FooterfooterFooter links (Privacy, Terms, Contact, Sitemap)
SocialsocialSocial media profile links (Twitter, GitHub, LinkedIn)
LegallegalLegal/compliance links below the footer
UtilityutilityTop-of-header utility links (Login, Account, Cart)

Menu item properties

Each item in a menu has the following properties:

PropertyDescription
Label The visible link text shown to visitors.
URL The link destination. Accepts relative paths (/about), absolute URLs (https://example.com), anchor links (#section), and mailto: / tel: links.
Target _self (same tab, default) or _blank (new tab). Use _blank for external URLs.
Children Nested sub-items that form a dropdown. Up to one level of nesting is supported.
Note — Only two levels of nesting are supported: top-level items and their direct children. Grandchildren are not rendered by the standard template loop.

Reordering items

Drag menu items up and down to reorder them. To make an item a child of another, drag it beneath and slightly indented from the parent item. Child items appear visually indented in the admin editor. Saving the menu persists the new order and hierarchy.

Basic template usage

Loop over menu items using {% for item in menu.slug %}. Replace slug with the menu's slug:

<nav class="main-nav">
  {% for item in menu.main %}
    <a href="{{ item.url }}" target="{{ item.target }}">
      {{ item.label }}
    </a>
  {% endfor %}
</nav>

Available item fields

FieldTypeDescription
item.labelstringLink display text
item.urlstringLink destination URL
item.targetstring_self or _blank
item.childrenarrayArray of child items (same structure as parent items); empty if no children

External links and security

When item.target is _blank, Outpost automatically adds rel="noopener noreferrer" to the rendered link for security. This prevents the opened page from accessing the opener's window object. You can also add it manually in your template:

{% for item in menu.main %}
  <a
    href="{{ item.url }}"
    target="{{ item.target }}"
    {% if item.target == "_blank" %}
      rel="noopener noreferrer"
    {% endif %}
  >{{ item.label }}</a>
{% endfor %}

Dropdown menus

Check for item.children and loop over it to render a dropdown. If an item has no children, the check evaluates false and no dropdown is rendered:

<nav class="main-nav">
  {% for item in menu.main %}
    <div class="nav-item">
      <a href="{{ item.url }}" target="{{ item.target }}">
        {{ item.label }}
      </a>

      {% if item.children %}
        <ul class="dropdown">
          {% for child in item.children %}
            <li>
              <a href="{{ child.url }}" target="{{ child.target }}">
                {{ child.label }}
              </a>
            </li>
          {% endfor %}
        </ul>
      {% endif %}
    </div>
  {% endfor %}
</nav>

Dropdown CSS

A minimal CSS-only dropdown using :hover:

.nav-item {
  position: relative;
  display: inline-block;
}

.dropdown {
  display: none;
  position: absolute;
  top: 100%;
  left: 0;
  list-style: none;
  margin: 0;
  padding: 6px 0;
  background: #fff;
  border: 1px solid #e5e7eb;
  border-radius: 6px;
  min-width: 200px;
  box-shadow: 0 4px 16px rgba(0,0,0,.08);
  z-index: 100;
}

.nav-item:hover .dropdown,
.nav-item:focus-within .dropdown {
  display: block;
}

.dropdown li a {
  display: block;
  padding: 8px 16px;
  white-space: nowrap;
  text-decoration: none;
}

.dropdown li a:hover {
  background: #f9fafb;
}
Tip — Use :focus-within on the parent alongside :hover so keyboard users can open the dropdown by tabbing to the parent link and then pressing Tab again to move into the child links.

Mobile navigation pattern

A common pattern is a hamburger button that toggles a full-screen mobile menu panel. This works with a small script — no framework needed:

<button
  id="menu-toggle"
  aria-expanded="false"
  aria-controls="mobile-nav"
>
  Menu
</button>

<nav id="mobile-nav" class="mobile-nav hidden">
  {% for item in menu.main %}
    <a href="{{ item.url }}">{{ item.label }}</a>
    {% if item.children %}
      {% for child in item.children %}
        <a href="{{ child.url }}" class="sub-link">
          — {{ child.label }}
        </a>
      {% endfor %}
    {% endif %}
  {% endfor %}
</nav>

<script>
  const btn = document.getElementById('menu-toggle');
  const nav = document.getElementById('mobile-nav');
  btn.addEventListener('click', () => {
    const open = nav.classList.toggle('hidden') === false;
    btn.setAttribute('aria-expanded', String(open));
  });
</script>

Multiple menus in one template

Reference different menu slugs anywhere in your templates. A single page request can output multiple menus without any performance concern — menu data is loaded once per request and cached in memory.

{# partials/nav.html — main header nav #}
<header>
  <a href="/">{{ @site_name }}</a>
  <nav>
    {% for item in menu.main %}
      <a href="{{ item.url }}">{{ item.label }}</a>
    {% endfor %}
  </nav>
</header>

{# partials/footer.html — footer nav + social links #}
<footer>
  <nav class="footer-links">
    {% for item in menu.footer %}
      <a href="{{ item.url }}">{{ item.label }}</a>
    {% endfor %}
  </nav>
  <div class="social-links">
    {% for item in menu.social %}
      <a href="{{ item.url }}" target="{{ item.target }}">
        {{ item.label }}
      </a>
    {% endfor %}
  </div>
</footer>

Active link highlighting

The template engine does not automatically add an active class to the current page's nav item. The most reliable approach is a small JavaScript snippet that compares each link's href to window.location.pathname:

<script>
  const path = window.location.pathname;
  document.querySelectorAll('.main-nav a').forEach(a => {
    const href = a.getAttribute('href');
    if (href === path) {
      a.classList.add('active');
    }
  });
</script>

For section-level highlighting (e.g., all /blog/* pages highlight the Blog link), use startsWith with a guard against the root path:

<script>
  const path = window.location.pathname;
  document.querySelectorAll('.main-nav a').forEach(a => {
    const href = a.getAttribute('href');
    if (href !== '/' && path.startsWith(href)) {
      a.classList.add('active');
    } else if (href === path) {
      a.classList.add('active');
    }
  });
</script>

Menus as data vs. global fields

For simple single-link scenarios — like a Twitter URL in the footer — using a Global Field ({{ @twitter_url }}) is often simpler than creating a full menu. Use menus when you need ordered, labeled, structured link lists that content editors should be able to manage without touching template files.

ScenarioBetter approach
Site-wide header navigation with 5 linksMenu
Footer with 10 links in multiple groupsMultiple menus (one per group) or one menu with groups as parent items
Single social media profile URLGlobal field ({{ @twitter_url }})
Social media links — 4 platforms, all in a rowMenu (slug: social)
CTA button with an editable URLGlobal field ({{ @cta_url | link }})

Placing menus in partials

The standard pattern is to place all navigation in the partials/nav.html and partials/footer.html partials, which are included in every template. This means the menu is edited once and appears consistently everywhere.

{# index.html — standard page template #}
{% include 'head' %}
{% include 'nav' %}

<main>
  {{ hero_title }}
</main>

{% include 'footer' %}
