Taxonomies

Classify and group collection items using flexible category and tag systems. Taxonomies can be flat (like tags) or hierarchical (like categories with parent/child terms), and their terms are filterable from templates.

What are taxonomies?

A taxonomy is a labeling system for your content. Where a collection holds the items themselves, a taxonomy provides the classification layer that organizes them. You define a taxonomy (e.g., "Categories"), populate it with terms (e.g., "Technology", "Design", "Business"), and then assign those terms to individual collection items when editing them in the admin.

Once in place, templates can loop over all terms in a taxonomy to build filter navigation, and collection loops can be filtered by a term selected from the URL query string. This is the standard pattern for blog categories, project tags, portfolio types, and any other content classification need.

Taxonomy types

TypeStructureUse cases
Flat A simple list of peer terms — no hierarchy, no parent/child relationship. Tags, Topics, Skills, Services, Industries, Technologies
Hierarchical Terms can have a parent, forming a tree. Each term stores a parent_id pointing to its parent. Categories (Technology → JavaScript), Locations (Europe → France → Paris)

Creating a taxonomy

  1. Go to Admin → Taxonomies and click New Taxonomy.
  2. Enter a Name (e.g., Category), a Slug (e.g., category), and choose Flat or Hierarchical.
  3. Select the Collection this taxonomy applies to.
  4. Save. The taxonomy appears in the sidebar under Taxonomies.
  5. Click the taxonomy name to open its term list. Add terms with New Term.
Note — A taxonomy belongs to one collection. If you need the same labeling on two collections (e.g., both Posts and Tutorials share a Topics taxonomy), create two separate taxonomies with the same terms.

Term fields

Every term has the following built-in fields:

FieldTypeDescription
name string Display name shown to visitors (e.g., "JavaScript")
slug string URL-safe identifier auto-generated from name (e.g., "javascript") — used in filter URLs
description string Optional plain-text description of the term
parent_id integer ID of the parent term. 0 or empty means top-level. Hierarchical taxonomies only.
count integer Number of published collection items currently tagged with this term
url string The archive URL for this term (if a term archive route is configured)
data.* mixed Custom fields added to this taxonomy's term schema (see below)

Assigning terms to collection items

When you open any item in the admin collection editor, the sidebar shows a section for each taxonomy attached to that collection. Click terms to toggle them on or off. Multiple terms can be assigned from the same taxonomy — a blog post can be in both "Technology" and "Design" at once.

Assignments are saved as part of the item when you save. The count on each term is recalculated automatically.

Tip — The count field reflects only published items. Draft items with that term do not increment the count.

Looping over all terms in a taxonomy

Use {% for term in taxonomy.slug %} to iterate over every term. Replace slug with the taxonomy's slug. This is used to build filter navigation, tag clouds, and category lists.

{# Build a category filter bar #}
<nav class="category-filter">
  <a href="/blog">All</a>
  {% for term in taxonomy.category %}
    <a href="/blog?category={{ term.slug }}">
      {{ term.name }}
      <span class="count">{{ term.count }}</span>
    </a>
  {% endfor %}
</nav>

All built-in term variables are available in the loop body:

{% for term in taxonomy.tag %}
  <li>
    <a href="/blog?tag={{ term.slug }}">{{ term.name }}</a>
    {% if term.description %}
      <p class="term-desc">{{ term.description }}</p>
    {% endif %}
    <span>{{ term.count }} posts</span>
  </li>
{% endfor %}

Filtering a collection loop by taxonomy term

Add the filteredby: modifier to a collection loop to filter items by a URL query parameter. When a visitor loads /blog?category=design, the loop returns only items tagged with the "design" term. Without the query parameter, all published items are returned.

{# URL: /blog?category=design — shows only items tagged "design" #}
{% for item in collection.post filteredby:category %}
  <article>
    <h2><a href="{{ item.url }}">{{ item.title }}</a></h2>
    <p>{{ item.excerpt }}</p>
  </article>
{% endfor %}
Note — The value after filteredby: is the URL query string key, not the taxonomy slug. Use a short, readable key. It will appear in visitor-facing URLs like /blog?category=technology.

You can combine filteredby: with other loop modifiers like paginate: and orderby::

{% for item in collection.post
    filteredby:category
    orderby:published_at
    paginate:10 %}
  <h2>{{ item.title }}</h2>
{% endfor %}

{% pagination %}

Filtering works alongside pagination transparently — each page of results contains only items matching the active term.

Full filter + list pattern

The most common use of taxonomies is a filter bar paired with a filtered item list. Here is a complete blog.html template implementing this pattern:

{# blog.html — category filter + paginated post list #}
<nav class="filter-bar">
  <a href="/blog">All Posts</a>
  {% for term in taxonomy.category %}
    <a href="/blog?category={{ term.slug }}">
      {{ term.name }}
    </a>
  {% endfor %}
</nav>

<div class="post-list">
{% for post in collection.post filteredby:category orderby:published_at paginate:12 %}
  <article>
    {% if post.featured_image %}
      <img src="{{ post.featured_image | image }}" alt="{{ post.title }}">
    {% endif %}
    <h2><a href="{{ post.url }}">{{ post.title }}</a></h2>
    <p>{{ post.excerpt }}</p>
    <div class="post-tags">
      {% for term in post.terms.category %}
        <span class="tag">{{ term.name }}</span>
      {% endfor %}
    </div>
  </article>
{% endfor %}
</div>

{% pagination %}

Displaying an item's own terms

Inside a {% single %} block or a collection loop, you can access the terms assigned to that specific item using item.terms.taxonomy-slug:

{% single post from collection.post %}
  <h1>{{ post.title }}</h1>

  {# Show this post's categories as links #}
  {% if post.terms.category %}
    <div class="post-categories">
      {% for term in post.terms.category %}
        <a href="/blog?category={{ term.slug }}">{{ term.name }}</a>
      {% endfor %}
    </div>
  {% endif %}

  {{ post.body | raw }}
{% else %}
  <p>Post not found.</p>
{% endsingle %}

Hierarchical taxonomies

For hierarchical taxonomies, the loop returns all terms in a single flat list ordered by creation. Each term carries a parent_id — terms with parent_id == "0" are top-level; all others are children. You construct the visual hierarchy in your template:

<ul class="category-tree">
{% for term in taxonomy.category %}
  {% if term.parent_id == "0" %}
    <li class="parent">
      <strong>{{ term.name }}</strong>
    </li>
  {% else %}
    <li class="child">
      <a href="/blog?category={{ term.slug }}">— {{ term.name }}</a>
    </li>
  {% endif %}
{% endfor %}
</ul>
Tip — When creating hierarchical terms in the admin, create parent terms first. The loop returns terms in creation order, so parents will naturally appear before their children in the flat output list, making visual nesting easier without sorting logic.

Multiple taxonomies on one collection

A collection can have more than one taxonomy attached to it. For example, a blog might use both a hierarchical "Category" taxonomy and a flat "Tag" taxonomy. Each appears as a separate term selector in the item editor, and each can be looped and filtered independently in templates:

{# Category nav — hierarchical #}
<div class="category-nav">
  {% for term in taxonomy.category %}
    <a href="/blog?category={{ term.slug }}">{{ term.name }}</a>
  {% endfor %}
</div>

{# Tag cloud — flat #}
<div class="tag-cloud">
  {% for term in taxonomy.tag %}
    <a href="/blog?tag={{ term.slug }}" class="tag">{{ term.name }}</a>
  {% endfor %}
</div>

{# Items filtered by category only (tag filter could be added too) #}
{% for post in collection.post filteredby:category %}
  <h2>{{ post.title }}</h2>
{% endfor %}

Custom fields on terms

You can add custom fields to a taxonomy's terms. Open the taxonomy in Admin → Taxonomies → [Taxonomy] → Settings and add fields to the term schema. Each term then has those extra fields editable on its detail page.

Custom field values are accessed via term.data.field_name:

{# Category with a brand color and icon custom field #}
{% for term in taxonomy.category %}
  <a
    href="/blog?category={{ term.slug }}"
    {% if term.data.color %}
      style="border-left: 3px solid {{ term.data.color }}"
    {% endif %}
  >
    {% if term.data.icon %}
      <img src="{{ term.data.icon | image }}" alt="" width="20">
    {% endif %}
    {{ term.name }}
    <span>{{ term.count }}</span>
  </a>
{% endfor %}

Available custom field types for terms:

TypeUse case
textSingle-line string — labels, slugs, short descriptions
textareaMulti-line plain text — extended descriptions
imageMedia picker — icon, thumbnail, cover image per term
colorHex color picker — brand color per category
numberNumeric value — sort order, display weight

Database tables

Taxonomies use three tables in the SQLite database:

TablePurposeKey columns
taxonomies One row per taxonomy id, name, slug, type (flat/hierarchical), collection_id, schema (JSON)
terms One row per term id, taxonomy_id, name, slug, description, parent_id, count, data (JSON)
item_terms Junction table — maps items to terms item_id, term_id

Template tag quick reference

TagWhat it does
{% for term in taxonomy.slug %} Loop over all terms in the named taxonomy
{{ term.name }} Term display name
{{ term.slug }} Term URL slug
{{ term.count }} Published item count for this term
{{ term.description }} Optional description set on the term
{{ term.parent_id }} Parent term ID (hierarchical only); 0 = top-level
{{ term.data.field }} Custom field value on this term
{% for term in item.terms.slug %} Loop over terms assigned to a specific collection item
filteredby:key Collection loop modifier — filter by ?key=term-slug in URL
